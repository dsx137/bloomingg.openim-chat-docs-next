This documentation set follows the current OpenIM SDK capability model and the public methods exposed by iOS Core SDK v3.8.3. Each method page covers the platform call shape, parameters, return value, and usage boundaries.

The pages describe how the platform wrapper maps to OpenIM core capabilities. Platform method names and parameter wrappers can differ; when the wrapper differs from the core export, follow the platform SDK v3.8.3 signature shown on the method page.

## Integration Model

| Model | Description |
| ----- | ----------- |
| Initialization | Configure the API address, WebSocket address, local data directory, logging, and connection listener. |
| Login state | Register listeners before login when possible, then call the login API with a backend-issued token. |
| Async result | Platform methods return results through Promise, Future, or callback patterns depending on the SDK. |
| Message creation | Create local message objects with `Create*Message` methods before sending them. |
| operationID | Generate a traceable operationID for each business operation so client and server logs can be correlated. |

## Currently Unavailable Capabilities

iOS Core SDK v3.8.3 does not expose standalone methods for public group variants, polls, scheduled messages, message threads, translation, independent reports, group metadata, or metacounters. Add these pages only after the platform SDK exposes corresponding methods.
