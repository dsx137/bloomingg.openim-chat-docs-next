Use **Get history messages** when your application needs this OpenIM capability from iOS Core SDK v3.8.3. This page follows the platform SDK reference and keeps the platform call signature visible next to the underlying core binding.

## Overview

Call `GetAdvancedHistoryMessageList` through the platform wrapper to get history messages. The current user usually needs to be initialized and logged in before calling business APIs. Keep `operationID` traceable so client and server logs can be matched during troubleshooting.

## Core Binding Method

| Item             | Description                                                                                  |
| ---------------- | -------------------------------------------------------------------------------------------- |
| Method signature | `GetAdvancedHistoryMessageList(callback Base, operationID string, getMessageOptions string)` |
| Capability       | Conversations and messages                                                                   |
| Source location  | `open_im_sdk/conversation_msg.go`                                                            |

## iOS Call Reference

iOS Core SDK v3.8.3 exposes the following platform method signature. The core binding signature above explains the underlying capability and parameter mapping.

Source:`Framework/OpenIMCore.xcframework/.../Open_im_sdk.objc.h`.

```objective-c showLineNumbers
FOUNDATION_EXPORT void Open_im_sdkGetAdvancedHistoryMessageList(id<Open_im_sdk_callbackBase> _Nullable callback, NSString* _Nullable operationID, NSString* _Nullable getMessageOptions);
```

## Parameters

| Parameter         | Type                               | Required   | Description                                     |
| ----------------- | ---------------------------------- | ---------- | ----------------------------------------------- |
| callback          | `id<Open_im_sdk_callbackBase>` | Yes        | Callback or listener.                           |
| getMessageOptions | NSString\*                         | As defined | Message object, message content, or message ID. |

## Return Value

Synchronous creation methods return a value directly. Asynchronous methods report success or failure through the callback. Pass a traceable operationID for each call.
