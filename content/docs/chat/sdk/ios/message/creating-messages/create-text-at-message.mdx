Use **Create an @ text message** when your application needs this OpenIM capability from iOS Core SDK v3.8.3. This page follows the platform SDK reference and keeps the platform call signature visible next to the underlying core binding.

## Overview

Call `CreateTextAtMessage` through the platform wrapper to create an @ text message. The current user usually needs to be initialized and logged in before calling business APIs. Keep `operationID` traceable so client and server logs can be matched during troubleshooting.

## Core Binding Method

| Item             | Description                                                                                     |
| ---------------- | ----------------------------------------------------------------------------------------------- |
| Method signature | `CreateTextAtMessage(operationID string, text, atUserList, atUsersInfo, message string) string` |
| Capability       | Conversations and messages                                                                      |
| Source location  | `open_im_sdk/conversation_msg.go`                                                               |

## iOS Call Reference

iOS Core SDK v3.8.3 exposes the following platform method signature. The core binding signature above explains the underlying capability and parameter mapping.

Synchronous creation methods return a value directly. Asynchronous methods report success or failure through the callback. Pass a traceable operationID for each call.
