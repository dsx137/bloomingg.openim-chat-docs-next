Use **Log out** when your application needs this OpenIM capability from iOS Core SDK v3.8.3. This page follows the platform SDK reference and keeps the platform call signature visible next to the underlying core binding.

## Overview

Call `Logout` through the platform wrapper to log out. The current user usually needs to be initialized and logged in before calling business APIs. Keep `operationID` traceable so client and server logs can be matched during troubleshooting.

## Core Binding Method

| Item             | Description                                 |
| ---------------- | ------------------------------------------- |
| Method signature | `Logout(callback Base, operationID string)` |
| Capability       | SDK lifecycle                    |
| Source location  | `open_im_sdk/init_login.go`                 |

## iOS Call Reference

iOS Core SDK v3.8.3 exposes the following platform method signature. The core binding signature above explains the underlying capability and parameter mapping.

Source:`Framework/OpenIMCore.xcframework/.../Open_im_sdk.objc.h`.

```objective-c showLineNumbers
FOUNDATION_EXPORT void Open_im_sdkLogout(id<Open_im_sdk_callbackBase> _Nullable callback, NSString* _Nullable operationID);
```

## Parameters

| Parameter | Type                               | Required | Description           |
| --------- | ---------------------------------- | -------- | --------------------- |
| callback  | `id<Open_im_sdk_callbackBase>` | Yes      | Callback or listener. |

## Return Value

Synchronous creation methods return a value directly. Asynchronous methods report success or failure through the callback. Pass a traceable operationID for each call.

## Code Example

The following example shows the iOS Core SDK v3.8.3 call shape. Replace placeholder values with real business data and add production error handling.

```objective-c showLineNumbers
Open_im_sdkLogout(callback, @"operationID");
```
