Use **Find Message List** when your application needs this OpenIM capability from uni-app SDK v3.8.3. This page follows the platform SDK reference and keeps the platform call signature visible next to the underlying core binding.

## Overview

Call `FindMessageList` through the platform wrapper to find message list. The current user usually needs to be initialized and logged in before calling business APIs. Keep `operationID` traceable so client and server logs can be matched during troubleshooting.

## Core Binding Method

| Item             | Description                                                                     |
| ---------------- | ------------------------------------------------------------------------------- |
| Method signature | `FindMessageList(callback Base, operationID string, findMessageOptions string)` |
| Capability       | Conversations and messages                                                      |
| Source location  | `open_im_sdk/conversation_msg.go`                                               |

## uni-app Call Reference

uni-app SDK v3.8.3 exposes the following platform method signature. The core binding signature above explains the underlying capability and parameter mapping.

Source:`ios/OpenIMUniPlugin/OpenIMUniPlugin/Classes/OpenIMModule.m`.

```js showLineNumbers
openIM.findMessageList(operationID, findOptions, callback);
```

## Parameters

| Parameter   | Type     | Required   | Description                                         |
| ----------- | -------- | ---------- | --------------------------------------------------- |

## Return Value

Synchronous creation methods return a value directly. Asynchronous methods report success or failure through the callback. Pass a traceable operationID for each call.
