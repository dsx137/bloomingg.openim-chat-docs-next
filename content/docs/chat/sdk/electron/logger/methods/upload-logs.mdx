Use **Upload logs** when your application needs this OpenIM capability from Electron SDK v3.8.3. This page follows the platform SDK reference and keeps the platform call signature visible next to the underlying OpenIM core.

## Overview

Call `UploadLogs` through the platform wrapper to upload logs. The current user usually needs to be initialized and logged in before calling business APIs. Keep `operationID` traceable so client and server logs can be matched during troubleshooting.

## Underlying Capability

| Item             | Description                                                                                      |
| ---------------- | ------------------------------------------------------------------------------------------------ |
| Method signature | `UploadLogs(callback Base, operationID string, line int, ex string, progress UploadLogProgress)` |
| Capability       | Logger |
| Source location  | `open_im_sdk/third.go`                                                                           |

## Electron Call Reference

Electron SDK v3.8.3 exposes the following platform method signature. The underlying capability table above explains the OpenIM core capability and parameter mapping.

## Parameters

| Parameter | Type             | Required | Description                                                                          |
| --------- | ---------------- | -------- | ------------------------------------------------------------------------------------ |
| params    | UploadLogsParams | Yes      | Pass according to the platform type definition.                                      |
| opid      | As defined       | No       | Trace ID for correlating client and server logs. The SDK generates one when omitted. |

Returns a `Promise` or registers a callback result. If the call fails, the Promise rejects; handle failures with `try/catch` or `.catch()` in application code.
