SDK lifecycle covers initialization, login, logout, login status, foreground/background state, and network change notification. Register required listeners after initialization and before login whenever possible.

## Lifecycle Order

| Stage | Method | Description |
| ----- | ------ | ----------- |
| Version record | [Get SDK version](/docs/chat/sdk/v4/electron/application/methods/get-sdk-version) | Record the SDK version in startup logs, support tickets, and compatibility checks. |
| Initialization | [Initialize the SDK](/docs/chat/sdk/v4/electron/application/methods/init-sdk) | Create the SDK instance, logging, connection management, local data directory, and base resources. |
| Listeners | [Event handlers](/docs/chat/sdk/v4/electron/event-handler/overview) | Register connection, message, conversation, friend, group, and user listeners before login when possible. |
| Login | [Login](/docs/chat/sdk/v4/electron/application/methods/login) | Log in a user with a token issued by your backend service. |
| State query | [Get login status](/docs/chat/sdk/v4/electron/application/methods/get-login-status) / [Get login user ID](/docs/chat/sdk/v4/electron/application/methods/get-login-user-id) | Decide whether user-scoped conversation, message, group, and friend APIs can be called. |
| Runtime state | [Set foreground/background status](/docs/chat/sdk/v4/electron/application/methods/set-app-background-status) / [Network status changed](/docs/chat/sdk/v4/electron/application/methods/network-status-changed) | Notify the SDK when app or network state changes. |
| Logout | [Log out](/docs/chat/sdk/v4/electron/application/methods/logout) | End the current user session without destroying the SDK instance. |
| Release | [Uninitialize the SDK](/docs/chat/sdk/v4/electron/application/methods/un-init-sdk) | Release global SDK resources when the app leaves IM capability entirely or tests recreate the instance. |

## Boundaries

| Boundary | Description |
| -------- | ----------- |
| Initialization creates the runtime environment | Successful initialization does not mean the user is logged in. User, friend, group, conversation, and message APIs usually require login. |
| Login depends on a backend token | The client should not mint tokens. Refresh expired or invalid tokens through your backend, then log in again. |
| Logout is not SDK release | `Logout` ends the current user session; `UnInitSDK` releases the global SDK instance. |
| operationID should be traceable | Generate operationID values that can correlate client logs, server logs, and support tickets. |
