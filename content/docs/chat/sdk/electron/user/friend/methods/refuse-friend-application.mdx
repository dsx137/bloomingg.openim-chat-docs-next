Use **Refuse friend application** when your application needs this OpenIM capability from Electron SDK v3.8.3. This page follows the platform SDK reference and keeps the platform call signature visible next to the underlying OpenIM core.

## Overview

Call `RefuseFriendApplication` through the platform wrapper to refuse friend application. The current user usually needs to be initialized and logged in before calling business APIs. Keep `operationID` traceable so client and server logs can be matched during troubleshooting.

## Underlying Capability

| Item             | Description                                                                          |
| ---------------- | ------------------------------------------------------------------------------------ |
| Method signature | `RefuseFriendApplication(callback Base, operationID string, userIDHandleMsg string)` |
| Capability       | Friends                                                                              |
