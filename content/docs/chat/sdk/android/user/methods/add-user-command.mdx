Use `AddUserCommand` for add user command in the OpenIM Android SDK.

## Feature

AddUserCommand provides the Android SDK operation for add user command.

Initialize the SDK before calling user-scoped APIs, and pass a unique `operationID` whenever the core binding signature includes it so client and server logs can be correlated.

## Core Binding Method

| Item             | Description                                                                                |
| ---------------- | ------------------------------------------------------------------------------------------ |
| Method signature | `AddUserCommand(callback Base, operationID string, Type int32, uuid string, value string)` |
| Capability       | Users, friends, and blocklist                                                              |
| Source file      | `open_im_sdk/user.go`                                                                      |

## Input Parameters

| Parameter     | Type                                          | Description                                               |
| ------------- | --------------------------------------------- | --------------------------------------------------------- |
| `callback`    | [Base](/docs/chat/sdk/v4/android/models/base) | Async callback for success and failure results.           |
| `operationID` | `string`                                      | Trace ID for this operation.                              |
| `Type`        | `int32`                                       | Parameter passed according to the core binding signature. |
| `uuid`        | `string`                                      | Parameter passed according to the core binding signature. |
| `value`       | `string`                                      | Parameter passed according to the core binding signature. |



## Return Value

The result is returned through the callback. Success is delivered through `OnSuccess`, failure through `OnError`, and sending or upload methods may also report progress.
