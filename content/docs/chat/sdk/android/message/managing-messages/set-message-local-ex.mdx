Use `SetMessageLocalEx` for set message local ex in the OpenIM Android SDK.

## Feature

SetMessageLocalEx provides the Android SDK operation for set message local ex.

Initialize the SDK before calling user-scoped APIs, and pass a unique `operationID` whenever the core binding signature includes it so client and server logs can be correlated.

## Core Binding Method

| Item             | Description                                                                                         |
| ---------------- | --------------------------------------------------------------------------------------------------- |
| Method signature | `SetMessageLocalEx(callback Base, operationID string, conversationID, clientMsgID, localEx string)` |
| Capability       | Conversations and messages                                                                          |
| Source file      | `open_im_sdk/conversation_msg.go`                                                                   |

## Android Call Reference

The Java method prototype in Android SDK v3.8.3 is shown below. The core binding signature explains the underlying `open_im_sdk` capability and parameter mapping.

```java showLineNumbers
public void setMessageLocalEx(OnBase<String> callBack, String conversationID, String clientMsgID, String localEx)
```

## Input Parameters

| Parameter        | Type                                                          | Description                                               |
| ---------------- | ------------------------------------------------------------- | --------------------------------------------------------- |
| `callBack`       | [On Base](/docs/chat/sdk/v4/android/models/on-base)`<String>` | Async callback for success and failure results.           |
| `conversationID` | `String`                                                      | Parameter passed according to the core binding signature. |
| `clientMsgID`    | `String`                                                      | Parameter passed according to the core binding signature. |
| `localEx`        | `String`                                                      | Parameter passed according to the core binding signature. |



## Return Value

The result is returned through the callback. Success is delivered through `OnSuccess`, failure through `OnError`, and sending or upload methods may also report progress.
