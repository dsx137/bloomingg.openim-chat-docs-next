Use **Set Custom Business Listener** when your application needs this OpenIM capability from React Native SDK v3.8.3. This page follows the platform SDK reference and keeps the platform call signature visible next to the underlying core binding.

## Overview

Call `SetCustomBusinessListener` through the platform wrapper to set custom business listener. The current user usually needs to be initialized and logged in before calling business APIs. Keep `operationID` traceable so client and server logs can be matched during troubleshooting.

## Core Binding Method

| Item             | Description                                                    |
| ---------------- | -------------------------------------------------------------- |
| Method signature | `SetCustomBusinessListener(listener OnCustomBusinessListener)` |
| Capability       | Event handlers                                                 |
| Source location  | `open_im_sdk/listener.go`                                      |

## React Native Call Reference

React Native SDK v3.8.3 exposes the following platform method signature. The core binding signature above explains the underlying capability and parameter mapping.

```ts showLineNumbers
setCustomBusinessListener(/* ... */);
```

## Parameters

| Parameter | Type                 | Required   | Description                                 |
| --------- | -------------------- | ---------- | ------------------------------------------- |
| Parameter | See method signature | As defined | Pass according to the call signature above. |

## Return Value

Returns `Promise`. If the call fails, the Promise rejects and the error is wrapped as OpenIMApiError.
