Use **Upload file** when your application needs this OpenIM capability from React Native SDK v3.8.3. This page follows the platform SDK reference and keeps the platform call signature visible next to the underlying core binding.

## Overview

Call `UploadFile` through the platform wrapper to upload file. The current user usually needs to be initialized and logged in before calling business APIs. Keep `operationID` traceable so client and server logs can be matched during troubleshooting.

## Core Binding Method

| Item             | Description                                                                              |
| ---------------- | ---------------------------------------------------------------------------------------- |
| Method signature | `UploadFile(callback Base, operationID string, req string, progress UploadFileCallback)` |
| Capability       | File upload |
| Source location  | `open_im_sdk/third.go`                                                                   |

## React Native Call Reference

React Native SDK v3.8.3 exposes the following platform method signature. The core binding signature above explains the underlying capability and parameter mapping.

Source:`src/sdk.ts`.

```ts showLineNumbers
uploadFile(params: UploadFileParams, operationID: string = id())
```

## Parameters

| Parameter | Type             | Required | Description                                                  |
| --------- | ---------------- | -------- | ------------------------------------------------------------ |
| params    | UploadFileParams | Yes      | Request object. Use the fields defined by the platform type. |

## Return Value

Returns `Promise`. If the call fails, the Promise rejects and the error is wrapped as OpenIMApiError.

## Code Example

The following example shows the React Native SDK v3.8.3 call shape. Replace placeholder values with real business data and add production error handling.

```ts showLineNumbers
import OpenIMSDK from '@openim/rn-client-sdk';

const result = await OpenIMSDK.uploadFile({});
```
