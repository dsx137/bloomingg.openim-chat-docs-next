Use **Dismiss group** when your application needs this OpenIM capability from React Native SDK v3.8.3. This page follows the platform SDK reference and keeps the platform call signature visible next to the underlying core binding.

## Overview

Call `DismissGroup` through the platform wrapper to dismiss group. The current user usually needs to be initialized and logged in before calling business APIs. Keep `operationID` traceable so client and server logs can be matched during troubleshooting.

## Core Binding Method

| Item             | Description                                                       |
| ---------------- | ----------------------------------------------------------------- |
| Method signature | `DismissGroup(callback Base, operationID string, groupID string)` |
| Capability       | Group                                                             |
| Source location  | `open_im_sdk/group.go`                                            |

## React Native Call Reference

React Native SDK v3.8.3 exposes the following platform method signature. The core binding signature above explains the underlying capability and parameter mapping.

Source:`src/sdk.ts`.

```ts showLineNumbers
dismissGroup(groupID: string, operationID: string = id())
```

## Parameters

| Parameter | Type   | Required | Description                    |
| --------- | ------ | -------- | ------------------------------ |
| groupID   | string | Yes      | Group ID or list of group IDs. |

## Return Value

Returns `Promise`. If the call fails, the Promise rejects and the error is wrapped as OpenIMApiError.

## Code Example

The following example shows the React Native SDK v3.8.3 call shape. Replace placeholder values with real business data and add production error handling.

```ts showLineNumbers
import OpenIMSDK from '@openim/rn-client-sdk';

const result = await OpenIMSDK.dismissGroup('groupID');
```
