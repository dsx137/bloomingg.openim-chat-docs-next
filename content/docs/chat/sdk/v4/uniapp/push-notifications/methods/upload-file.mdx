Use **Upload file** when your application needs this OpenIM capability from uni-app SDK v3.8.3. This page follows the platform SDK reference and keeps the platform call signature visible next to the underlying core binding.

## Overview

Call `UploadFile` through the platform wrapper to upload file. The current user usually needs to be initialized and logged in before calling business APIs. Keep `operationID` traceable so client and server logs can be matched during troubleshooting.

## Core Binding Method

| Item             | Description                                                                              |
| ---------------- | ---------------------------------------------------------------------------------------- |
| Method signature | `UploadFile(callback Base, operationID string, req string, progress UploadFileCallback)` |
| Capability       | Push, logs, and files                                                                    |
| Source location  | `open_im_sdk/third.go`                                                                   |

## uni-app Call Reference

uni-app SDK v3.8.3 exposes the following platform method signature. The core binding signature above explains the underlying capability and parameter mapping.

Source:`ios/OpenIMUniPlugin/OpenIMUniPlugin/Classes/OpenIMModule.m`.

```js showLineNumbers
openIM.uploadFile(operationID, reqData, callback);
```

## Parameters

| Parameter | Type     | Required   | Description                                         |
| --------- | -------- | ---------- | --------------------------------------------------- |
| reqData   | string   | As defined | Pass according to the uni-app native plugin export. |
| callback  | Function | Yes        | Pass according to the uni-app native plugin export. |

## Return Value

Synchronous creation methods return a value directly. Asynchronous methods report success or failure through the callback. Pass a traceable operationID for each call.
