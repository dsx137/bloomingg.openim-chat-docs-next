Use **Uninitialize the SDK** when your application needs this OpenIM capability from React Native SDK v3.8.3-patch.12.4. This page follows the platform SDK reference and keeps the platform call signature visible next to the underlying core binding.

## Overview

Call `UnInitSDK` through the platform wrapper to uninitialize the sdk. The current user usually needs to be initialized and logged in before calling business APIs. Keep `operationID` traceable so client and server logs can be matched during troubleshooting.

## Core Binding Method

| Item             | Description                     |
| ---------------- | ------------------------------- |
| Method signature | `UnInitSDK(operationID string)` |
| Capability       | Initialization and login        |
| Source location  | `open_im_sdk/init_login.go`     |

## React Native Call Reference

React Native SDK v3.8.3-patch.12.4 exposes the following platform method signature. The core binding signature above explains the underlying capability and parameter mapping.

Source:`src/sdk.ts`.

```ts showLineNumbers
unInitSDK(operationID: string = id())
```

## Return Value

Returns `Promise`. If the call fails, the Promise rejects and the error is wrapped as OpenIMApiError.
