Use **Get multiple conversations** when your application needs this OpenIM capability from React Native SDK v3.8.3-patch.12.4. This page follows the platform SDK reference and keeps the platform call signature visible next to the underlying core binding.

## Overview

Call `GetMultipleConversation` through the platform wrapper to get multiple conversations. The current user usually needs to be initialized and logged in before calling business APIs. Keep `operationID` traceable so client and server logs can be matched during troubleshooting.

## Core Binding Method

| Item             | Description                                                                             |
| ---------------- | --------------------------------------------------------------------------------------- |
| Method signature | `GetMultipleConversation(callback Base, operationID string, conversationIDList string)` |
| Capability       | Conversations and messages                                                              |
| Source location  | `open_im_sdk/conversation_msg.go`                                                       |

## React Native Call Reference

React Native SDK v3.8.3-patch.12.4 exposes the following platform method signature. The core binding signature above explains the underlying capability and parameter mapping.

Source:`src/sdk.ts`.

```ts showLineNumbers
getMultipleConversation(conversationIDList: string[], operationID: string = id())
```

## Parameters

| Parameter          | Type     | Required | Description                                  |
| ------------------ | -------- | -------- | -------------------------------------------- |
| conversationIDList | string[] | Yes      | Conversation ID or list of conversation IDs. |

## Return Value

Returns `Promise`. If the call fails, the Promise rejects and the error is wrapped as OpenIMApiError.
