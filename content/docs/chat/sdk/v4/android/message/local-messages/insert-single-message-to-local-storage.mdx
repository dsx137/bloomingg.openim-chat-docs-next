Use `InsertSingleMessageToLocalStorage` for insert single message to local storage in the OpenIM Android SDK.

## Feature

InsertSingleMessageToLocalStorage provides the Android SDK operation for insert single message to local storage.

Initialize the SDK before calling user-scoped APIs, and pass a unique `operationID` whenever the core binding signature includes it so client and server logs can be correlated.

## Core Binding Method

| Item             | Description                                                                                                          |
| ---------------- | -------------------------------------------------------------------------------------------------------------------- |
| Method signature | `InsertSingleMessageToLocalStorage(callback Base, operationID string, message string, recvID string, sendID string)` |
| Capability       | Conversations and messages                                                                                           |
| Source file      | `open_im_sdk/conversation_msg.go`                                                                                    |

## Android Call Reference

The Java method prototype in Android SDK v3.8.3 is shown below. The core binding signature explains the underlying `open_im_sdk` capability and parameter mapping.

```java showLineNumbers
public void insertSingleMessageToLocalStorage(OnBase<String> base, Message message, String receiver, String sender)
```

## Input Parameters

| Parameter    | Type                                                          | Description                                               |
| ------------ | ------------------------------------------------------------- | --------------------------------------------------------- |
| `base`       | [On Base](/docs/chat/sdk/v4/android/models/on-base)`<String>` | Async callback for success and failure results.           |
| `message`    | [Message](/docs/chat/sdk/v4/android/models/message)           | Message payload or message JSON.                          |
| `receiverID` | String                                                        | Parameter passed according to the core binding signature. |
| `senderID`   | String                                                        | Parameter passed according to the core binding signature. |



## Return Value

The result is returned through the callback. Success is delivered through `OnSuccess`, failure through `OnError`, and sending or upload methods may also report progress.
