## Overview

Receive incoming messages through `OnAdvanceMsgListener`. The listener provides a `Message` for every new received message; use its conversation and group data to route it in your UI.

## How this maps to OpenIM

| Concept | OpenIM API | Notes |
| --- | --- | --- |
| Incoming message | `onRecvNewMessage(Message)` | Add it to the appropriate conversation UI. |
| Revocation | `onRecvMessageRevokedV2(RevokedInfo)` | Update the affected displayed message. |

## Prerequisites

- An initialized client.
- The listener registered before login.

## Install and configure

Follow the [Android overview](/docs/chat/sdk/v4/android/overview).

## Initialize the SDK

Initialize the client and log in after registering the listener.

## Implement the task

### 1. Register the listener

```java
OpenIMClient.getInstance().messageManager.setAdvancedMsgListener(new OnAdvanceMsgListener() {
    @Override public void onRecvNewMessage(Message message) {
        // Route and render the message using its conversation data.
    }

    @Override public void onRecvMessageRevokedV2(RevokedInfo info) {
        // Replace or refresh the revoked message in the UI.
    }
});
```

### 2. Log in

Call `login` after registering the listener.

### 3. Handle the result

The SDK also reports C2C read receipts, group read receipts, deleted messages, offline messages, and online-only messages through this listener.

## Verify the result

- Send a message to the logged-in user from another client.
- Confirm `onRecvNewMessage` receives the `Message`.

## Troubleshooting

| Case | Cause | Handling |
| --- | --- | --- |
| Message not displayed | The callback does not update UI state. | Add the received message to the correct conversation and refresh the UI. |
