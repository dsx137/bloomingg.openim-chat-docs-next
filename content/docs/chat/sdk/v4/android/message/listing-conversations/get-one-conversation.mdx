Use `GetOneConversation` for get one conversation in the OpenIM Android SDK.

## Feature

Reads get one conversation from the local SDK state or OpenIM service according to the method contract.

Initialize the SDK before calling user-scoped APIs, and pass a unique `operationID` whenever the core binding signature includes it so client and server logs can be correlated.

## Core Binding Method

| Item             | Description                                                                                 |
| ---------------- | ------------------------------------------------------------------------------------------- |
| Method signature | `GetOneConversation(callback Base, operationID string, sessionType int32, sourceID string)` |
| Capability       | Conversations and messages                                                                  |
| Source file      | `open_im_sdk/conversation_msg.go`                                                           |

## Android Call Reference

The Java method prototype in Android SDK v3.8.3 is shown below. The core binding signature explains the underlying `open_im_sdk` capability and parameter mapping.

```java showLineNumbers
public void getOneConversation(OnBase<ConversationInfo> base, String sourceId, int sessionType)
```

## Input Parameters

| Parameter     | Type                                                                                                                             | Description                                               |
| ------------- | -------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------- |
| `base`        | [On Base](/docs/chat/sdk/v4/android/models/on-base)`<`[Conversation Info](/docs/chat/sdk/v4/android/models/conversation-info)`>` | Async callback for success and failure results.           |
| `sourceId`    | String                                                                                                                           | Parameter passed according to the core binding signature. |
| `sessionType` | int                                                                                                                              | Parameter passed according to the core binding signature. |



## Return Value

The result is returned through the callback. Success is delivered through `OnSuccess`, failure through `OnError`, and sending or upload methods may also report progress.

## Code Example

The following example uses the Android SDK v3.8.3 call style.

```java showLineNumbers

    new OnBase<ConversationInfo>() {
    @Override
    public void onError(int code, String error) {
        // TODO: handle the callback result
    }

    @Override
    public void onSuccess(ConversationInfo data) {
        // TODO: handle the callback result
    }
},
    sourceId,
    sessionType

```
