## Overview

Create a text `Message` and send it to a recipient user or group after the user is logged in.

## How this maps to OpenIM

| Concept | OpenIM object | Notes |
| --- | --- | --- |
| Text content | `Message` | Created by `createTextMessage`. |
| Direct recipient | `recvUid` | Supply a user ID. |
| Group recipient | `recvGid` | Supply a group ID. |

## Prerequisites

- An initialized and logged-in `OpenIMClient`.
- The recipient user ID or group ID.

## Install and configure

Install and initialize the SDK as described in the [Android overview](/docs/chat/sdk/v4/android/overview).

## Initialize the SDK

Register an `OnAdvanceMsgListener` before login when the app needs to receive messages.

## Implement the task

### 1. Prepare the message

```java
Message message = OpenIMClient.getInstance()
    .messageManager
    .createTextMessage("Hello, OpenIM");
```

### 2. Send to a user

```java
OpenIMClient.getInstance().messageManager.sendMessage(new OnMsgSendCallback() {
    @Override public void onProgress(long progress) {}
    @Override public void onSuccess(Message sentMessage) {}
    @Override public void onError(int code, String error) {}
}, message, recipientUserId, "", null);
```

### 3. Handle the result

`onSuccess` receives the sent `Message`. `onProgress` reports upload progress for messages that upload content, and `onError` reports a failed send.

## Verify the result

- Confirm `onSuccess` receives a message.
- Confirm the recipient receives `OnAdvanceMsgListener.onRecvNewMessage`.

## Troubleshooting

| Case | Cause | Handling |
| --- | --- | --- |
| Send callback fails | The request could not be completed. | Handle `onError` and retain enough UI state to retry. |
| No receive event | The recipient has not registered a message listener. | Register `OnAdvanceMsgListener` before login. |
