Use `CheckFriend` for check friend in the OpenIM Android SDK.

## Feature

CheckFriend provides the Android SDK operation for check friend.

Initialize the SDK before calling user-scoped APIs, and pass a unique `operationID` whenever the core binding signature includes it so client and server logs can be correlated.

## Core Binding Method

| Item             | Description                                                         |
| ---------------- | ------------------------------------------------------------------- |
| Method signature | `CheckFriend(callback Base, operationID string, userIDList string)` |
| Capability       | Users, friends, and blocklist                                       |
| Source file      | `open_im_sdk/relation.go`                                           |

## Android Call Reference

The Java method prototype in Android SDK v3.8.3 is shown below. The core binding signature explains the underlying `open_im_sdk` capability and parameter mapping.

```java showLineNumbers
public void checkFriend(OnBase<List<FriendshipInfo>> base, List<String> uidList)
```

## Input Parameters

| Parameter  | Type                           | Description                                     |
| ---------- | ------------------------------ | ----------------------------------------------- |
| `callBack` | `OnBase<List<FriendshipInfo>>` | Async callback for success and failure results. |
| `uidList`  | String                         | List of user IDs, usually encoded as JSON.      |

### FriendInfo

| Field    | Type   | Description                                 |
| -------- | ------ | ------------------------------------------- |
| `userID` | String | User ID.                                    |
| `result` | int    | SDK field returned or submitted by the SDK. |

## Return Value

The result is returned through the callback. Success is delivered through `OnSuccess`, failure through `OnError`, and sending or upload methods may also report progress.
