Use `SearchFriends` for search friends in the OpenIM Android SDK.

## Feature

Reads search friends from the local SDK state or OpenIM service according to the method contract.

Initialize the SDK before calling user-scoped APIs, and pass a unique `operationID` whenever the core binding signature includes it so client and server logs can be correlated.

## Core Binding Method

| Item             | Description                                                            |
| ---------------- | ---------------------------------------------------------------------- |
| Method signature | `SearchFriends(callback Base, operationID string, searchParam string)` |
| Capability       | Users, friends, and blocklist                                          |
| Source file      | `open_im_sdk/relation.go`                                              |

## Android Call Reference

The Java method prototype in Android SDK v3.8.3 is shown below. The core binding signature explains the underlying `open_im_sdk` capability and parameter mapping.

```java showLineNumbers
public void searchFriends(OnBase<List<FriendInfo>> base, List<String> keywordList, boolean isSearchUserID, boolean isSearchNickname, boolean isSearchRemark)
```

## Input Parameters

| Parameter          | Type                       | Description                                               |
| ------------------ | -------------------------- | --------------------------------------------------------- |
| `callBack`         | `OnBase<List<FriendInfo>>` | Async callback for success and failure results.           |
| `keywordList`      | `List<String>`             | Parameter passed according to the core binding signature. |
| `isSearchUserID`   | Boolean                    | Parameter passed according to the core binding signature. |
| `isSearchNickname` | Boolean                    | Parameter passed according to the core binding signature. |
| `isSearchRemark`   | Boolean                    | Parameter passed according to the core binding signature. |



## Return Value

The result is returned through the callback. Success is delivered through `OnSuccess`, failure through `OnError`, and sending or upload methods may also report progress.
