Use `TransferGroupOwner` for transfer group owner in the OpenIM Android SDK.

## Feature

TransferGroupOwner provides the Android SDK operation for transfer group owner.

Initialize the SDK before calling user-scoped APIs, and pass a unique `operationID` whenever the core binding signature includes it so client and server logs can be correlated.

## Core Binding Method

| Item             | Description                                                                                    |
| ---------------- | ---------------------------------------------------------------------------------------------- |
| Method signature | `TransferGroupOwner(callback Base, operationID string, groupID string, newOwnerUserID string)` |
| Capability       | Groups                                                                                         |
| Source file      | `open_im_sdk/group.go`                                                                         |

## Android Call Reference

The Java method prototype in Android SDK v3.8.3 is shown below. The core binding signature explains the underlying `open_im_sdk` capability and parameter mapping.

```java showLineNumbers
public void transferGroupOwner(OnBase<String> base, String gid, String uid)
```

## Input Parameters

| Parameter  | Type                                                | Description                                               |
| ---------- | --------------------------------------------------- | --------------------------------------------------------- |
| `callBack` | [On Base](/docs/chat/sdk/v4/android/models/on-base) | Async callback for success and failure results.           |
| `gid`      | String                                              | Parameter passed according to the core binding signature. |
| `uid`      | String                                              | User ID.                                                  |



## Return Value

The result is returned through the callback. Success is delivered through `OnSuccess`, failure through `OnError`, and sending or upload methods may also report progress.

The following example uses the Android SDK v3.8.3 call style.

        // TODO: handle the callback result
