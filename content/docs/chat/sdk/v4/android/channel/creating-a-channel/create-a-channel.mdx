## Overview

OpenIM groups provide the closest equivalent to a group channel. Create a group with member IDs, administrator IDs, group information, and an owner ID.

## How this maps to OpenIM

| Concept | OpenIM object | Notes |
| --- | --- | --- |
| Group channel | `GroupInfo` | Contains group information. |
| Members | `memberUserIDs` | User IDs added as members. |
| Owner | `ownerUserID` | User ID of the group owner. |

## Prerequisites

- An initialized, logged-in client.
- A `GroupInfo` instance and the required user IDs.

## Install and configure

Follow the [Android overview](/docs/chat/sdk/v4/android/overview).

## Initialize the SDK

Initialize and log in before calling group APIs.

## Implement the task

### 1. Prepare group information

Create the `GroupInfo` and lists of member and administrator user IDs required by your app.

### 2. Create the group

```java
OpenIMClient.getInstance().groupManager.createGroup(
    memberUserIDs,
    adminUserIDs,
    groupInfo,
    ownerUserID,
    new OnBase<GroupInfo>() {
        @Override public void onSuccess(GroupInfo createdGroup) {}
        @Override public void onError(int code, String error) {}
    }
);
```

### 3. Handle the result

The success callback receives the created `GroupInfo`.

## Verify the result

- Confirm the callback returns a group.
- Confirm the group appears in the member's joined group list.

## Troubleshooting

| Case | Cause | Handling |
| --- | --- | --- |
| Create fails | The group request was rejected. | Handle `onError` and correct the supplied IDs or group information. |
