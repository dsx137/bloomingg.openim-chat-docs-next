Use **Send message without OSS** when your application needs this OpenIM capability from Flutter SDK 3.8.3+hotfix.12. This page follows the platform SDK reference and keeps the platform call signature visible next to the underlying core binding.

## Overview

Call `SendMessageNotOss` through the platform wrapper to send message without oss. The current user usually needs to be initialized and logged in before calling business APIs. Keep `operationID` traceable so client and server logs can be matched during troubleshooting.

## Flutter Call Reference

Flutter SDK 3.8.3+hotfix.12 exposes the following platform method signature. The core binding signature above explains the underlying capability and parameter mapping.

Source:`lib/src/manager/im_message_manager.dart`.

```dart showLineNumbers
Future<Message> sendMessageNotOss({ required Message message, required OfflinePushInfo offlinePushInfo, String? userID, String? groupID, bool isOnlineOnly = false, String? operationID, })
```

| isOnlineOnly    | bool                                                                  | No       | Pass as defined.                                             |

## Return Value

Returns `Future<Message>`. If the call fails, the Future throws an exception. Catch it in the application layer and keep the operationID in logs.
