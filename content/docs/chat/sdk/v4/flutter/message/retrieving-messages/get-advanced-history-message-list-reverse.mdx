Use **Get history messages in reverse** when your application needs this OpenIM capability from Flutter SDK 3.8.3+hotfix.12. This page follows the platform SDK reference and keeps the platform call signature visible next to the underlying core binding.

## Overview

Call `GetAdvancedHistoryMessageListReverse` through the platform wrapper to get history messages in reverse. The current user usually needs to be initialized and logged in before calling business APIs. Keep `operationID` traceable so client and server logs can be matched during troubleshooting.

## Flutter Call Reference

Flutter SDK 3.8.3+hotfix.12 exposes the following platform method signature. The core binding signature above explains the underlying capability and parameter mapping.

Source:`lib/src/manager/im_message_manager.dart`.

```dart showLineNumbers
Future<AdvancedMessage> getAdvancedHistoryMessageListReverse({ String? conversationID, Message? startMsg, GetHistoryViewType viewType = GetHistoryViewType.history, int? count, String? operationID, })
```

## Parameters

| Parameter      | Type                                                 | Required | Description                                     |
| -------------- | ---------------------------------------------------- | -------- | ----------------------------------------------- |
| conversationID | String?                                              | No       | Conversation ID or list of conversation IDs.    |
| startMsg       | [Message](/docs/chat/sdk/v4/flutter/models/message)? | No       | Message object, message content, or message ID. |
| viewType       | GetHistoryViewType                                   | No       | Pass as defined.                                |
| count          | int?                                                 | No       | Pagination parameter.                           |

## Return Value

Returns `Future<AdvancedMessage>`. If the call fails, the Future throws an exception. Catch it in the application layer and keep the operationID in logs.
