Use **Invite users to group** when your application needs this OpenIM capability from Flutter SDK v3.8.3. This page follows the platform SDK reference and keeps the platform call signature visible next to the underlying core binding.

## Overview

Call `InviteUserToGroup` through the platform wrapper to invite users to group. The current user usually needs to be initialized and logged in before calling business APIs. Keep `operationID` traceable so client and server logs can be matched during troubleshooting.

## Core Binding Method

| Item             | Description                                                                                              |
| ---------------- | -------------------------------------------------------------------------------------------------------- |
| Method signature | `InviteUserToGroup(callback Base, operationID string, groupID string, reason string, userIDList string)` |
| Capability       | Group                                                                                                    |
| Source location  | `open_im_sdk/group.go`                                                                                   |

## Flutter Call Reference

Flutter SDK v3.8.3 exposes the following platform method signature. The core binding signature above explains the underlying capability and parameter mapping.

Source:`lib/src/manager/im_group_manager.dart`.

```dart showLineNumbers
Future inviteUserToGroup({ required String groupID, required List<String> userIDList, String? reason, String? operationID, })
```

## Parameters

| Parameter  | Type               | Required | Description                    |
| ---------- | ------------------ | -------- | ------------------------------ |
| groupID    | String             | Yes      | Group ID or list of group IDs. |
| userIDList | `List<String>` | Yes      | User ID or list of user IDs.   |
| reason     | String?            | No       | Pass as defined.               |

## Return Value

Returns `Future`. If the call fails, the Future throws an exception. Catch it in the application layer and keep the operationID in logs.
