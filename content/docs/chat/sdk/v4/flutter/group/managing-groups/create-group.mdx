Use **Create a group** when your application needs this OpenIM capability from Flutter SDK v3.8.3. This page follows the platform SDK reference and keeps the platform call signature visible next to the underlying core binding.

## Overview

Call `CreateGroup` through the platform wrapper to create a group. The current user usually needs to be initialized and logged in before calling business APIs. Keep `operationID` traceable so client and server logs can be matched during troubleshooting.

## Core Binding Method

| Item             | Description                                                           |
| ---------------- | --------------------------------------------------------------------- |
| Method signature | `CreateGroup(callback Base, operationID string, groupReqInfo string)` |
| Capability       | Group                                                                 |
| Source location  | `open_im_sdk/group.go`                                                |

## Flutter Call Reference

Flutter SDK v3.8.3 exposes the following platform method signature. The core binding signature above explains the underlying capability and parameter mapping.

Source:`lib/src/manager/im_group_manager.dart`.

```dart showLineNumbers
Future<GroupInfo> createGroup({ required GroupInfo groupInfo, List<String> memberUserIDs = const [], List<String> adminUserIDs = const [], String? ownerUserID, String? operationID, })
```

## Parameters

| Parameter     | Type                                                     | Required | Description                                                  |
| ------------- | -------------------------------------------------------- | -------- | ------------------------------------------------------------ |
| groupInfo     | [GroupInfo](/docs/chat/sdk/v4/flutter/models/group-info) | Yes      | Request object. Use the fields defined by the platform type. |
| memberUserIDs | `List<String>`                                       | No       | User ID or list of user IDs.                                 |
| adminUserIDs  | `List<String>`                                       | No       | User ID or list of user IDs.                                 |
| ownerUserID   | String?                                                  | No       | User ID or list of user IDs.                                 |

## Return Value

Returns `Future<GroupInfo>`. If the call fails, the Future throws an exception. Catch it in the application layer and keep the operationID in logs.
