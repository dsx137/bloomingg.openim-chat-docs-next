Use **Get group info** when your application needs this OpenIM capability from Flutter SDK v3.8.3. This page follows the platform SDK reference and keeps the platform call signature visible next to the underlying core binding.

## Overview

Call `GetSpecifiedGroupsInfo` through the platform wrapper to get group info. The current user usually needs to be initialized and logged in before calling business APIs. Keep `operationID` traceable so client and server logs can be matched during troubleshooting.

## Core Binding Method

| Item             | Description                                                                     |
| ---------------- | ------------------------------------------------------------------------------- |
| Method signature | `GetSpecifiedGroupsInfo(callback Base, operationID string, groupIDList string)` |
| Capability       | Group                                                                           |
| Source location  | `open_im_sdk/group.go`                                                          |

## Flutter Call Reference

Flutter SDK v3.8.3 exposes the following platform method signature. The core binding signature above explains the underlying capability and parameter mapping.

Source:`lib/src/manager/im_group_manager.dart`.

```dart showLineNumbers
Future<List<GroupInfo>> getGroupsInfo({ required List<String> groupIDList, String? operationID, })
```

## Parameters

| Parameter   | Type               | Required | Description                    |
| ----------- | ------------------ | -------- | ------------------------------ |
| groupIDList | `List<String>` | Yes      | Group ID or list of group IDs. |

## Return Value

Returns `Future<List<GroupInfo>>`. If the call fails, the Future throws an exception. Catch it in the application layer and keep the operationID in logs.
