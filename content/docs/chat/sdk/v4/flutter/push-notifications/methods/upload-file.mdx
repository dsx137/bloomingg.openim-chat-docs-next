Use **Upload file** when your application needs this OpenIM capability from Flutter SDK 3.8.3+hotfix.12. This page follows the platform SDK reference and keeps the platform call signature visible next to the underlying core binding.

## Overview

Call `UploadFile` through the platform wrapper to upload file. The current user usually needs to be initialized and logged in before calling business APIs. Keep `operationID` traceable so client and server logs can be matched during troubleshooting.

## Core Binding Method

| Item             | Description                                                                              |
| ---------------- | ---------------------------------------------------------------------------------------- |
| Method signature | `UploadFile(callback Base, operationID string, req string, progress UploadFileCallback)` |
| Capability       | Push, logs, and files                                                                    |
| Source location  | `open_im_sdk/third.go`                                                                   |

## Flutter Call Reference

Flutter SDK 3.8.3+hotfix.12 exposes the following platform method signature. The core binding signature above explains the underlying capability and parameter mapping.

Source:`lib/src/manager/im_manager.dart`.

```dart showLineNumbers
Future uploadFile({ required String id, required String filePath, required String fileName, String? contentType, String? cause, String? operationID, })
```

## Parameters

| Parameter   | Type    | Required | Description      |
| ----------- | ------- | -------- | ---------------- |
| id          | String  | Yes      | Pass as defined. |
| filePath    | String  | Yes      | Pass as defined. |
| fileName    | String  | Yes      | Pass as defined. |
| contentType | String? | No       | Pass as defined. |

## Return Value

Returns `Future`. If the call fails, the Future throws an exception. Catch it in the application layer and keep the operationID in logs.
