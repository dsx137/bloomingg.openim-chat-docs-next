Use **Upload logs** when your application needs this OpenIM capability from Flutter SDK v3.8.3. This page follows the platform SDK reference and keeps the platform call signature visible next to the underlying core binding.

## Overview

Call `UploadLogs` through the platform wrapper to upload logs. The current user usually needs to be initialized and logged in before calling business APIs. Keep `operationID` traceable so client and server logs can be matched during troubleshooting.

## Core Binding Method

| Item             | Description                                                                                      |
| ---------------- | ------------------------------------------------------------------------------------------------ |
| Method signature | `UploadLogs(callback Base, operationID string, line int, ex string, progress UploadLogProgress)` |
| Capability       | Logger |
| Source location  | `open_im_sdk/third.go`                                                                           |

## Flutter Call Reference

Flutter SDK v3.8.3 exposes the following platform method signature. The core binding signature above explains the underlying capability and parameter mapping.

Source:`lib/src/manager/im_manager.dart`.

```dart showLineNumbers
Future uploadLogs({ String? ex, int line = 0, String? operationID, })
```

## Parameters

| Parameter | Type    | Required | Description      |
| --------- | ------- | -------- | ---------------- |
| ex        | String? | No       | Pass as defined. |
| line      | int     | No       | Pass as defined. |

## Return Value

Returns `Future`. If the call fails, the Future throws an exception. Catch it in the application layer and keep the operationID in logs.
