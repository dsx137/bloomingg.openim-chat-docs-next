Use **Write client logs** when your application needs this OpenIM capability from Flutter SDK v3.8.3. This page follows the platform SDK reference and keeps the platform call signature visible next to the underlying core binding.

## Overview

Call `Logs` through the platform wrapper to write client logs. The current user usually needs to be initialized and logged in before calling business APIs. Keep `operationID` traceable so client and server logs can be matched during troubleshooting.

## Core Binding Method

| Item             | Description                                                                                                                 |
| ---------------- | --------------------------------------------------------------------------------------------------------------------------- |
| Method signature | `Logs(callback Base, operationID string, logLevel int, file string, line int, msgs string, err string, keyAndValue string)` |
| Capability       | Logger |
| Source location  | `open_im_sdk/third.go`                                                                                                      |

## Flutter Call Reference

Flutter SDK v3.8.3 exposes the following platform method signature. The core binding signature above explains the underlying capability and parameter mapping.

Source:`lib/src/manager/im_manager.dart`.

```dart showLineNumbers
Future logs({ int logLevel = 5, String? file, int line = 0, String? msgs, String? err, List<dynamic>? keyAndValues, String? operationID, })
```

## Parameters

| Parameter    | Type                 | Required | Description                                     |
| ------------ | -------------------- | -------- | ----------------------------------------------- |
| logLevel     | int                  | No       | Pass as defined.                                |
| file         | String?              | No       | Pass as defined.                                |
| line         | int                  | No       | Pass as defined.                                |
| msgs         | String?              | No       | Message object, message content, or message ID. |
| err          | String?              | No       | Pass as defined.                                |
| keyAndValues | `List<dynamic>?` | No       | Pass as defined.                                |

## Return Value

Returns `Future`. If the call fails, the Future throws an exception. Catch it in the application layer and keep the operationID in logs.
