## Overview

Use local insertion only for product-controlled system rows, migration, or client-side hints that should not be delivered as normal chat messages.

## OpenIM model

`insertSingleMessageToLocalStorage()` and `insertGroupMessageToLocalStorage()` write to local storage. They do not send a message to other users.

## Prerequisites

- Create a valid `MessageItem`.
- Know whether the row belongs to a single chat or a group chat.

## API signatures

```ts
insertSingleMessageToLocalStorage: (
  data: InsertSingleMsgParams,
  operationID?: string,
) => Promise<WsResponse<unknown>>;
insertGroupMessageToLocalStorage: (
  data: InsertGroupMsgParams,
  operationID?: string,
) => Promise<WsResponse<unknown>>;
```

## Implementation

```ts
const { data: message } = await OpenIM.createTextMessage('Local note');

await OpenIM.insertSingleMessageToLocalStorage({
  message,
  recvID: 'user_b',
  sendID: currentUserID,
});
```

## Returned data and events

Reload the related conversation from local storage after insertion.

## Verify the result

- Confirm no remote receiver gets a new message event.
- Confirm the local conversation renders the inserted row.

## Troubleshooting

| Case | Cause | Handling |
| --- | --- | --- |
| Message appears in the wrong conversation | Wrong `recvID`, `groupID`, or `sendID` | Keep single-chat and group-chat insert paths separate. |
