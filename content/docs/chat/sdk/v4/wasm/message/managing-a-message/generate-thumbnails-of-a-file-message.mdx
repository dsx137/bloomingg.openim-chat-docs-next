## Overview

OpenIM image and video message APIs accept snapshot metadata, but thumbnail generation itself is handled by your app or media pipeline.

This guide separates public OpenIM SDK support from product behavior that needs application code.

## OpenIM model

Provide accurate `PicBaseInfo` or video snapshot fields when creating media messages. Do not expect the JS SDK to synthesize thumbnails for every format.

Support status: the WASM SDK covers part of this workflow; the remaining behavior belongs in your application or backend.

Key concepts to keep straight:

- `userID` identifies an OpenIM user.
- `conversationID` identifies a local conversation row.
- `recvID` sends a direct message to one user.
- `groupID` sends to or operates on a group conversation.
- `operationID` is optional on most methods and is useful for tracing client/server logs.

## Prerequisites

- Install an OpenIM WASM SDK version that is compatible with OpenIM Server.
- Serve `openIM.wasm`, `sql-wasm.wasm`, and `wasm_exec.js` from the browser app public path.
- Create one SDK instance with `getSDK()` and call `login()` with `userID`, token, `platformID: 5`, `apiAddr`, and `wsAddr`.

## API signatures

Relevant public signatures:

```ts
function getSDK(config?: WasmPathConfig): SDK;
createImageMessageByFile: (params: ImageMsgParamsByURL & { file: File }, operationID?: string) =>
  Promise<WsResponse<MessageItem>>;
createVideoMessageByFile: (
  data: VideoMsgParamsByURL & { videoFile: File; snapshotFile: File },
  operationID?: string,
) => Promise<WsResponse<MessageItem>>;
createImageMessageByURL: (params: ImageMsgParamsByURL, operationID?: string) =>
  Promise<WsResponse<MessageItem>>;
createVideoMessageByURL: (data: VideoMsgParamsByURL, operationID?: string) =>
  Promise<WsResponse<MessageItem>>;
sendMessage: (params: SendMsgParams, operationID?: string) => Promise<WsResponse<MessageItem>>;
```

Relevant events:

```ts
on<E extends CbEvents>(event: E, fn: Cbfn<E>): this;
off<E extends CbEvents>(event: E, fn: Cbfn<E>): this | undefined;
CbEvents.OnProgress
CbEvents.UploadComplete
```

Response and event payloads use these common shapes:

```ts
type WsResponse<T> = {
  event: string;
  errCode: number;
  errMsg: string;
  data: T;
  operationID: string;
};
type WSEvent<T = unknown> = {
  event: CbEvents;
  data: T;
  errCode: number;
  errMsg: string;
  operationID: string;
};
```

## Implementation

Generate media metadata before creating the message.

```ts
const snapshot = await createVideoSnapshot(videoFile);

const { data: message } = await OpenIM.createVideoMessageByFile({
  videoFile,
  snapshotFile: snapshot.file,
  videoPath: videoFile.name,
  duration: snapshot.duration,
  videoType: videoFile.type,
  snapshotPath: snapshot.file.name,
  videoUUID: crypto.randomUUID(),
  videoUrl: '',
  videoSize: videoFile.size,
  snapshotUUID: crypto.randomUUID(),
  snapshotSize: snapshot.file.size,
  snapshotUrl: '',
  snapshotWidth: snapshot.width,
  snapshotHeight: snapshot.height,
});
```

## Returned data and events

When the page describes a partial, server-side, or unsupported Sendbird feature, treat the listed OpenIM calls as building blocks. Keep the authoritative product state in your backend when the SDK does not own it.

Watch these callbacks while testing: `CbEvents.OnProgress`, `CbEvents.UploadComplete`.

Always log `operationID` with failures. It is the fastest way to match browser behavior with OpenIM server logs.

## Verify the result

- Confirm `login()` succeeds and `CbEvents.OnConnectSuccess` fires before calling the task API.
- Confirm the SDK response has `errCode === 0` or follows your deployment's success convention.
- Confirm the expected SDK event fires in another signed-in browser session when the action should be visible to other users.
- Confirm the application-owned part of the feature survives refresh, reconnect, and a second device session.

## Troubleshooting

| Case                              | Cause                                                                 | Handling                                                                                               |
| --------------------------------- | --------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------ |
| WASM asset requests return 404    | The SDK assets were not copied to the app public path.                | Copy `openIM.wasm`, `sql-wasm.wasm`, and `wasm_exec.js` from the npm package assets directory.         |
| The client never connects         | `apiAddr`, `wsAddr`, token, platform ID, or network routing is wrong. | Register connection events before `login()` and test the API and WebSocket URLs from the same browser. |
| The response has an error code    | The server rejected the request or the current user lacks permission. | Log `errCode`, `errMsg`, and `operationID`; verify group role, token, and target IDs.                  |
| A Sendbird method name is missing | The feature is not part of the OpenIM WASM SDK public API.            | Use the OpenIM alternative on this page or implement the feature in a trusted backend.                 |
