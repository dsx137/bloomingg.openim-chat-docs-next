## Overview

Every message is created first, then passed to `sendMessage()`.

## OpenIM model

The methods on this page only create `MessageItem` objects. Delivery still depends on `sendMessage()`.

## Prerequisites

- Upload or prepare media URLs when using URL-based factories.
- Prepare the target `recvID` or `groupID`.

## API signatures

```ts
createSoundMessageByURL: (data: SoundMsgParamsByURL, operationID?: string) => Promise<WsResponse<MessageItem>>;
createFileMessageByURL: (data: FileMsgParamsByURL, operationID?: string) => Promise<WsResponse<MessageItem>>;
createFaceMessage: (data: FaceMessageParams, operationID?: string) => Promise<WsResponse<MessageItem>>;
createLocationMessage: (data: LocationMsgParams, operationID?: string) => Promise<WsResponse<MessageItem>>;
createCardMessage: (data: CardElem, operationID?: string) => Promise<WsResponse<MessageItem>>;
sendMessage: (params: SendMsgParams, operationID?: string) => Promise<WsResponse<MessageItem>>;
```

## Implementation

```ts
const { data: cardMessage } = await OpenIM.createCardMessage({
  userID: 'user_b',
  nickname: 'Alex',
  faceURL: 'https://example.com/avatar.png',
  ex: '',
});

await OpenIM.sendMessage({
  recvID: 'user_b',
  groupID: '',
  message: cardMessage,
});
```

## Returned data and events

All factories return `MessageItem`. Listen to message events after sending.

## Verify the result

- Confirm the created message has the expected `contentType`.
- Confirm the receiving client can render the message payload.

## Troubleshooting

| Case | Cause | Handling |
| --- | --- | --- |
| Media does not load | URL or metadata is invalid | Verify URL, size, duration, and MIME fields before sending. |
