## Overview

Use `getUsersInfo()` to retrieve public profile data for application users by `userID`. This can support flows such as searching by user ID before sending a friend request or opening a public profile card.

Do not use `getUsersInfo()` to fill a direct conversation title. OpenIM conversation objects already expose `showName`; conversation lists and chat headers should prefer the conversation data.

## API signatures

```ts
getUsersInfo: (data: string[], operationID?: string) => Promise<WsResponse<PublicUserItem[]>>;
```

## Implementation

```ts
try {
  const response = await OpenIM.getUsersInfo(['user_a', 'user_b']);
  const users = response.data ?? [];
  renderUsers(users);
} catch (error) {
  console.error('getUsersInfo failed', { error });
  throw error;
}
```

## Returned data

The response contains `PublicUserItem[]` with `userID`, `nickname`, `faceURL`, and `ex`.
