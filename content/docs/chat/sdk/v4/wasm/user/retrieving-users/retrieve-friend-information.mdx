## Overview

Use this page when you need relationship data for specified friends, not the whole friend list.

## OpenIM model

Friend information is scoped to the current logged-in user and returns `FriendUserItem`.

## Prerequisites

- Log in before calling relationship APIs.
- Prepare the target friend user IDs.

## API signatures

```ts
getSpecifiedFriendsInfo: (
  data: GetSpecifiedFriendsParams,
  operationID?: string,
) => Promise<WsResponse<FriendUserItem[]>>;
checkFriend: (
  data: string[],
  operationID?: string,
) => Promise<WsResponse<FriendshipInfo[]>>;
```

## Implementation

Fetch relationship details by specified friend user IDs, and use `checkFriend()` when you only need to know whether a relationship exists.

```ts
const { data: friends } = await OpenIM.getSpecifiedFriendsInfo({
  friendUserIDList: ['user_a', 'user_b'],
  filterBlack: true,
});

const { data: checks } = await OpenIM.checkFriend(['user_a', 'user_b']);
```

## Returned data and events

`getSpecifiedFriendsInfo()` returns `FriendUserItem[]`. `checkFriend()` returns `FriendshipInfo[]`.

## Verify the result

- Confirm each requested `friendUserID` exists.
- Confirm blacklisted users are filtered only when `filterBlack` is enabled.

## Troubleshooting

| Case | Cause | Handling |
| --- | --- | --- |
| Missing users | The target is not a friend or was filtered | Check the friendship state and blacklist setting. |
