## Overview

Friend requests are the workflow for creating a friendship when your product requires approval.

## OpenIM model

Sent and received friend applications are represented by `FriendApplicationItem`.

## Prerequisites

- Log in as the current user.
- Prepare the target `toUserID` for a new request.

## API signatures

```ts
addFriend: (data: AddFriendParams, operationID?: string) => Promise<WsResponse<unknown>>;
getFriendApplicationListAsRecipient: (
  data?: GetFriendApplicationListAsRecipientParams,
  operationID?: string,
) => Promise<WsResponse<FriendApplicationItem[]>>;
getFriendApplicationListAsApplicant: (
  data?: GetFriendApplicationListAsApplicationParams,
  operationID?: string,
) => Promise<WsResponse<FriendApplicationItem[]>>;
getFriendApplicationUnhandledCount: (
  data: GetSelfUnhandledApplyCountParams,
  operationID?: string,
) => Promise<WsResponse<number>>;
acceptFriendApplication: (
  data: AccessFriendApplicationParams,
  operationID?: string,
) => Promise<WsResponse<unknown>>;
refuseFriendApplication: (
  data: AccessFriendApplicationParams,
  operationID?: string,
) => Promise<WsResponse<unknown>>;
```

## Implementation

```ts
await OpenIM.addFriend({ toUserID: 'user_b', reqMsg: 'Hi' });

const { data: received } = await OpenIM.getFriendApplicationListAsRecipient({
  handleResults: [0],
  offset: 0,
  count: 20,
});

await OpenIM.acceptFriendApplication({
  toUserID: 'user_a',
  handleMsg: 'Approved',
});
```

## Returned data and events

Watch `CbEvents.OnFriendApplicationAdded`, `CbEvents.OnFriendApplicationAccepted`, `CbEvents.OnFriendApplicationRejected`, and `CbEvents.OnFriendAdded`.

## Verify the result

- Confirm the recipient receives a pending application.
- Confirm accepting the request creates a friend relationship for both users.

## Troubleshooting

| Case | Cause | Handling |
| --- | --- | --- |
| No pending request | The wrong list direction was queried | Use recipient list for requests sent to the current user. |
