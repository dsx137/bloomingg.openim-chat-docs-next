## Overview

Authenticate a WASM user by registering the global callback, initializing the module, calling `login()`, and handling token lifecycle events.

This guide uses public OpenIM WASM SDK signatures and assumes the SDK version is compatible with OpenIM Server.

## OpenIM model

Browser clients should receive `userID` and token from a trusted backend. `login()` starts the session; events sent to `commonEventFunc` tell the UI when the WebSocket is ready.

Support status: public WASM SDK APIs cover this workflow.

Key concepts to keep straight:

- `userID` identifies an OpenIM user.
- `conversationID` identifies a local conversation row.
- `recvID` sends a direct message to one user.
- `groupID` sends to or operates on a group conversation.
- `operationID` is optional on most methods and is useful for tracing client/server logs.

## Prerequisites

- Install an OpenIM WASM SDK version that is compatible with OpenIM Server.
- Serve `openIM.wasm` and the Go `wasm_exec.js` runtime from the browser app public path.
- Register `commonEventFunc`, call `initSDK()` with configuration JSON, then call `login()` with an operation ID, user ID, and token.

## API signatures

Relevant public signatures:

```ts
initSDK(operationID: string, config: string): unknown;
login(operationID: string, userID: string, token: string): Promise<string>;
logout(operationID: string): Promise<string>;
getLoginStatus(operationID: string): Promise<number>;
```

Relevant events:

```ts
commonEventFunc(callback: (event: string) => void): boolean;
// Event names: OnConnecting, OnConnectSuccess, OnConnectFailed,
// OnKickedOffline, OnUserTokenExpired, and OnUserTokenInvalid.
```

Response and event payloads use these common shapes:

```ts
type CallbackEvent = {
  event: string;
  errCode: number;
  errMsg: string;
  data?: unknown;
  operationID: string;
};
```

## Implementation

Register the callback and initialize the module before logging in.

```ts
globalThis.commonEventFunc((serializedEvent: string) => {
  const event = JSON.parse(serializedEvent);
  if (event.event === 'OnConnectSuccess') console.info('OpenIM connected');
});
globalThis.initSDK(operationID, JSON.stringify({ platformID: 5, apiAddr, wsAddr }));
await globalThis.login(operationID, userID, token);
```

Register token and kick-offline handlers before calling `login()`.

```ts
globalThis.commonEventFunc((serializedEvent: string) => {
  const event = JSON.parse(serializedEvent);
  if (event.event === 'OnUserTokenExpired') requestFreshToken();
  if (event.event === 'OnUserTokenInvalid') forceSignOut();
  if (event.event === 'OnKickedOffline') showSignedInElsewhereDialog();
});

const loginStatus = await globalThis.getLoginStatus(operationID);
```

Call `logout()` before switching accounts in the same browser session.

## Returned data and events

Promise calls resolve their raw result, commonly a JSON string. Persist IDs such as `conversationID`, `groupID`, and `clientMsgID` after parsing the returned result.

Watch these callback event names while testing: `OnConnecting`, `OnConnectSuccess`, `OnConnectFailed`, `OnKickedOffline`, `OnUserTokenExpired`, and `OnUserTokenInvalid`.

Always log `operationID` with failures. It is the fastest way to match browser behavior with OpenIM server logs.

## Verify the result

- Confirm `login()` succeeds and `CbEvents.OnConnectSuccess` fires before calling the task API.
- Confirm the OpenIM call resolves and the returned data or follow-up SDK event matches the expected state.
- Confirm the expected SDK event fires in another signed-in browser session when the action should be visible to other users.

## Troubleshooting

| Case                           | Cause                                                                 | Handling                                                                                               |
| ------------------------------ | --------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------ |
| WASM asset requests return 404 | The SDK assets were not copied to the app public path.                | Copy `openIM.wasm` and the Go `wasm_exec.js` runtime to the browser public path.         |
| The client never connects      | `apiAddr`, `wsAddr`, token, platform ID, or network routing is wrong. | Register connection events before `login()` and test the API and WebSocket URLs from the same browser. |
| The SDK call rejects | The server rejected the request or the current user lacks permission. | Handle the error in `catch`, log the rejected error with `operationID`, and verify group role, token, and target IDs. |
