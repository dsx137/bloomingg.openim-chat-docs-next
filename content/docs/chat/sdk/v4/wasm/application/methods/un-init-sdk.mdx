Use **Uninitialize the SDK** when your application needs this OpenIM capability from WASM SDK v3.8.3. This page follows the platform SDK reference and keeps the platform call signature visible next to the underlying OpenIM core.

## Overview

Call `UnInitSDK` through the platform wrapper to uninitialize the sdk. The current user usually needs to be initialized and logged in before calling business APIs. Keep `operationID` traceable so client and server logs can be matched during troubleshooting.

## Underlying Capability

| Item             | Description                     |
| ---------------- | ------------------------------- |
| Method signature | `UnInitSDK(operationID string)` |
| Capability       | SDK lifecycle        |
| Source location  | `open_im_sdk/init_login.go`     |

## WASM Call Reference

WASM SDK v3.8.3 does not expose an independent `unInitSDK` platform method. This page keeps the underlying capability notes without presenting another platform wrapper as the current WASM API.
