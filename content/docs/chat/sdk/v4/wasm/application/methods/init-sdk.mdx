Use **Initialize the SDK** when your application needs this OpenIM capability from WASM SDK v3.8.3. This page follows the platform SDK reference and keeps the platform call signature visible next to the underlying OpenIM core.

## Overview

Call `InitSDK` through the platform wrapper to initialize the sdk. The current user usually needs to be initialized and logged in before calling business APIs. Keep `operationID` traceable so client and server logs can be matched during troubleshooting.

## Underlying Capability

| Item             | Description                                                                |
| ---------------- | -------------------------------------------------------------------------- |
| Method signature | `InitSDK(listener OnConnListener, operationID string, config string) bool` |
| Capability       | SDK lifecycle                                                   |
| Source location  | `open_im_sdk/init_login.go`                                                |

## WASM Call Reference

WASM SDK v3.8.3 initializes the browser SDK instance with `getSDK(config)`. This creates or returns the singleton SDK instance and configures the WASM asset paths used by the browser runtime.

Source: `src/sdk/index.ts`.

```ts showLineNumbers
getSDK(config?: WasmPathConfig)
```

## Parameters

| Parameter | Type           | Required | Description                                                                                                              |
| --------- | -------------- | -------- | ------------------------------------------------------------------------------------------------------------------------ |
| config    | WasmPathConfig | No       | WASM asset path and debug options. Pass `coreWasmPath`, `sqlWasmPath`, and `debug` according to the runtime environment. |

## Return Value

Returns the SDK instance. Repeated calls return the same instance in the current browser context.

## Code Example

The following example shows the WASM SDK v3.8.3 initialization shape. Publish the WASM assets in your web app before calling `getSDK()`.

```ts showLineNumbers
import { getSDK } from '@openim/wasm-client-sdk';

const openIM = getSDK({
  coreWasmPath: '/openIM.wasm',
  sqlWasmPath: '/sql-wasm.wasm',
  debug: true,
});
```
