Use **Set Custom Business Listener** when your application needs this OpenIM capability from iOS Core SDK v3.8.3. This page follows the platform SDK reference and keeps the platform call signature visible next to the underlying core binding.

## Overview

Call `SetCustomBusinessListener` through the platform wrapper to set custom business listener. The current user usually needs to be initialized and logged in before calling business APIs. Keep `operationID` traceable so client and server logs can be matched during troubleshooting.

## Core Binding Method

| Item             | Description                                                    |
| ---------------- | -------------------------------------------------------------- |
| Method signature | `SetCustomBusinessListener(listener OnCustomBusinessListener)` |
| Capability       | Event handlers                                                 |
| Source location  | `open_im_sdk/listener.go`                                      |

## iOS Call Reference

iOS Core SDK v3.8.3 exposes the following platform method signature. The core binding signature above explains the underlying capability and parameter mapping.

Source:`Framework/OpenIMCore.xcframework/.../Open_im_sdk.objc.h`.

```objective-c showLineNumbers
FOUNDATION_EXPORT void Open_im_sdkSetCustomBusinessListener(id<Open_im_sdk_callbackOnCustomBusinessListener> _Nullable listener);
```

## Parameters

| Parameter | Type                                                   | Required   | Description           |
| --------- | ------------------------------------------------------ | ---------- | --------------------- |
| listener  | `id<Open_im_sdk_callbackOnCustomBusinessListener>` | As defined | Callback or listener. |

## Return Value

Synchronous creation methods return a value directly. Asynchronous methods report success or failure through the callback. Pass a traceable operationID for each call.

## Code Example

The following example shows the iOS Core SDK v3.8.3 call shape. Replace placeholder values with real business data and add production error handling.
