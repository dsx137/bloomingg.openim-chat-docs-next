## What you can build

OpenIMSDK adds instant messaging to an iOS application connected to a self-hosted OpenIM Server. The SDK provides Objective-C APIs for initializing the client, authenticating a user, and creating and sending messages. Its core uses SQLite for local data storage.

## Install

Add the CocoaPods dependency to your `Podfile`:

```ruby
pod 'OpenIMSDK', '3.8.3-hotfix.12'
```

Run CocoaPods installation for the app, then import the SDK where it is used:

```objc
@import OpenIMSDK;
```

## Initialize

Configure the OpenIM Server API and WebSocket addresses before logging in. Register connection callbacks during initialization so the app can reflect connection, token, and forced-offline states.

```objc
@import OpenIMSDK;

OIMInitConfig *config = [OIMInitConfig new];
config.apiAddr = @"<OpenIM Server API address>";
config.wsAddr = @"<OpenIM Server WebSocket address>";

BOOL initialized = [OIMManager.manager initSDKWithConfig:config
    onConnecting:^{
        // The SDK is connecting to the IM server.
    }
    onConnectFailure:^(NSInteger code, NSString * _Nullable message) {
        // Handle the connection failure.
    }
    onConnectSuccess:^{
        // The SDK connected to the IM server.
    }
    onKickedOffline:^{
        // Notify the user that this session was forced offline.
    }
    onUserTokenExpired:^{
        // Obtain a new token and call login again.
    }
    onUserTokenInvalid:^(NSString * _Nullable message) {
        // Handle an invalid token.
    }];

if (!initialized) {
    // Do not attempt to log in until initialization succeeds.
}
```

## Core workflows

- Initialize the SDK with the API and WebSocket addresses of your OpenIM Server.
- Log in with a user ID and token issued through your application server.
- Create a message with `OIMMessageInfo` and send it to a user or group.
- Handle connection callbacks, including token expiry and forced offline events.

## Compatibility

- SDK package: `OpenIMSDK` `3.8.3-hotfix.12`
- Minimum iOS deployment target: iOS 11.0
- Server: a deployed OpenIM Server with reachable API and WebSocket addresses
