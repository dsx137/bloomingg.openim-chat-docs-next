Use **Create a card message** when your application needs this OpenIM capability from Flutter SDK v3.8.3. This page follows the platform SDK reference and keeps the platform call signature visible next to the underlying core binding.

## Overview

Call `CreateCardMessage` through the platform wrapper to create a card message. The current user usually needs to be initialized and logged in before calling business APIs. Keep `operationID` traceable so client and server logs can be matched during troubleshooting.

## Core Binding Method

| Item             | Description                                                     |
| ---------------- | --------------------------------------------------------------- |
| Method signature | `CreateCardMessage(operationID string, cardInfo string) string` |
| Capability       | Conversations and messages                                      |
| Source location  | `open_im_sdk/conversation_msg.go`                               |

## Flutter Call Reference

Flutter SDK v3.8.3 exposes the following platform method signature. The core binding signature above explains the underlying capability and parameter mapping.

Source:`lib/src/manager/im_message_manager.dart`.

```dart showLineNumbers
Future<Message> createCardMessage({ required String userID, required String nickname, String? faceURL, String? ex, String? operationID, })
```

## Parameters

| Parameter | Type    | Required | Description                  |
| --------- | ------- | -------- | ---------------------------- |
| userID    | String  | Yes      | User ID or list of user IDs. |
| nickname  | String  | Yes      | Pass as defined.             |
| faceURL   | String? | No       | Pass as defined.             |
| ex        | String? | No       | Pass as defined.             |

## Return Value

Returns `Future<Message>`. If the call fails, the Future throws an exception. Catch it in the application layer and keep the operationID in logs.

## Code Example

The following example shows the Flutter SDK v3.8.3 call shape. Replace placeholder values with real business data and add production error handling.

```dart showLineNumbers
final result = await OpenIM.iMManager.messageManager.createCardMessage(
  userID: 'userID',
  nickname: 'nickname',
  faceURL: 'faceURL',
  ex: 'ex',
);
```
