Use **Create a forward message** when your application needs this OpenIM capability from Flutter SDK v3.8.3. This page follows the platform SDK reference and keeps the platform call signature visible next to the underlying core binding.

## Overview

Call `CreateForwardMessage` through the platform wrapper to create a forward message. The current user usually needs to be initialized and logged in before calling business APIs. Keep `operationID` traceable so client and server logs can be matched during troubleshooting.

## Core Binding Method

| Item             | Description                                                 |
| ---------------- | ----------------------------------------------------------- |
| Method signature | `CreateForwardMessage(operationID string, m string) string` |
| Capability       | Conversations and messages                                  |
| Source location  | `open_im_sdk/conversation_msg.go`                           |

## Flutter Call Reference

Flutter SDK v3.8.3 exposes the following platform method signature. The core binding signature above explains the underlying capability and parameter mapping.

Source:`lib/src/manager/im_message_manager.dart`.

```dart showLineNumbers
Future<Message> createForwardMessage({ required Message message, String? operationID, })
```

## Parameters

| Parameter | Type                                                | Required | Description                                     |
| --------- | --------------------------------------------------- | -------- | ----------------------------------------------- |
| message   | [Message](/docs/chat/sdk/v4/flutter/models/message) | Yes      | Message object, message content, or message ID. |

## Return Value

Returns `Future<Message>`. If the call fails, the Future throws an exception. Catch it in the application layer and keep the operationID in logs.
