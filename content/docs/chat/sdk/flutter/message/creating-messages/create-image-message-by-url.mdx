Use **Create an image message by URL** when your application needs this OpenIM capability from Flutter SDK v3.8.3. This page follows the platform SDK reference and keeps the platform call signature visible next to the underlying core binding.

## Overview

Call `CreateImageMessageByURL` through the platform wrapper to create an image message by url. The current user usually needs to be initialized and logged in before calling business APIs. Keep `operationID` traceable so client and server logs can be matched during troubleshooting.

## Core Binding Method

| Item             | Description                                                                                                                |
| ---------------- | -------------------------------------------------------------------------------------------------------------------------- |
| Method signature | `CreateImageMessageByURL(operationID string, sourcePath string, sourcePicture, bigPicture, snapshotPicture string) string` |
| Capability       | Conversations and messages                                                                                                 |
| Source location  | `open_im_sdk/conversation_msg.go`                                                                                          |

## Flutter Call Reference

Flutter SDK v3.8.3 exposes the following platform method signature. The core binding signature above explains the underlying capability and parameter mapping.

Source:`lib/src/manager/im_message_manager.dart`.

```dart showLineNumbers
Future<Message> createImageMessageByURL({ required PictureInfo sourcePicture, required PictureInfo bigPicture, required PictureInfo snapshotPicture, String? sourcePath, String? operationID, })
```

## Parameters

| Parameter       | Type                                                         | Required | Description      |
| --------------- | ------------------------------------------------------------ | -------- | ---------------- |
| sourcePicture   | [PictureInfo](/docs/chat/sdk/v4/flutter/models/picture-info) | Yes      | Pass as defined. |
| bigPicture      | [PictureInfo](/docs/chat/sdk/v4/flutter/models/picture-info) | Yes      | Pass as defined. |
| snapshotPicture | [PictureInfo](/docs/chat/sdk/v4/flutter/models/picture-info) | Yes      | Pass as defined. |
| sourcePath      | String?                                                      | No       | Pass as defined. |

## Return Value

Returns `Future<Message>`. If the call fails, the Future throws an exception. Catch it in the application layer and keep the operationID in logs.
