Use **Hide conversation** when your application needs this OpenIM capability from Flutter SDK v3.8.3. This page follows the platform SDK reference and keeps the platform call signature visible next to the underlying core binding.

## Overview

Call `HideConversation` through the platform wrapper to hide conversation. The current user usually needs to be initialized and logged in before calling business APIs. Keep `operationID` traceable so client and server logs can be matched during troubleshooting.

## Core Binding Method

| Item             | Description                                                                  |
| ---------------- | ---------------------------------------------------------------------------- |
| Method signature | `HideConversation(callback Base, operationID string, conversationID string)` |
| Capability       | Conversations and messages                                                   |
| Source location  | `open_im_sdk/conversation_msg.go`                                            |

## Flutter Call Reference

Flutter SDK v3.8.3 exposes the following platform method signature. The core binding signature above explains the underlying capability and parameter mapping.

Source:`lib/src/manager/im_conversation_manager.dart`.

```dart showLineNumbers
Future hideConversation({ required String conversationID, String? operationID, })
```

## Parameters

| Parameter      | Type   | Required | Description                                  |
| -------------- | ------ | -------- | -------------------------------------------- |
| conversationID | String | Yes      | Conversation ID or list of conversation IDs. |

## Return Value

Returns `Future`. If the call fails, the Future throws an exception. Catch it in the application layer and keep the operationID in logs.
