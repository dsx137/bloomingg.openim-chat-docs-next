Use **Search local messages** when your application needs this OpenIM capability from Flutter SDK v3.8.3. This page follows the platform SDK reference and keeps the platform call signature visible next to the underlying core binding.

## Overview

Call `SearchLocalMessages` through the platform wrapper to search local messages. The current user usually needs to be initialized and logged in before calling business APIs. Keep `operationID` traceable so client and server logs can be matched during troubleshooting.

## Core Binding Method

| Item             | Description                                                                  |
| ---------------- | ---------------------------------------------------------------------------- |
| Method signature | `SearchLocalMessages(callback Base, operationID string, searchParam string)` |
| Capability       | Conversations and messages                                                   |
| Source location  | `open_im_sdk/conversation_msg.go`                                            |

## Flutter Call Reference

Flutter SDK v3.8.3 exposes the following platform method signature. The core binding signature above explains the underlying capability and parameter mapping.

Source:`lib/src/manager/im_message_manager.dart`.

```dart showLineNumbers
Future<SearchResult> searchLocalMessages({ String? conversationID, List<String> keywordList = const [], int keywordListMatchType = 0, List<String> senderUserIDList = const [], List<int> messageTypeList = const [], int searchTimePosition = 0, int searchTimePeriod = 0, int pageIndex = 1, int count = 40, String? operationID, })
```

## Parameters

| Parameter            | Type               | Required | Description                                     |
| -------------------- | ------------------ | -------- | ----------------------------------------------- |
| conversationID       | String?            | No       | Conversation ID or list of conversation IDs.    |
| keywordList          | `List<String>` | No       | Pass as defined.                                |
| keywordListMatchType | int                | No       | Pass as defined.                                |
| senderUserIDList     | `List<String>` | No       | User ID or list of user IDs.                    |
| messageTypeList      | `List<int>`    | No       | Message object, message content, or message ID. |
| searchTimePosition   | int                | No       | Pass as defined.                                |
| searchTimePeriod     | int                | No       | Pass as defined.                                |
| pageIndex            | int                | No       | Pagination parameter.                           |
| count                | int                | No       | Pagination parameter.                           |

## Return Value

Returns `Future<SearchResult>`. If the call fails, the Future throws an exception. Catch it in the application layer and keep the operationID in logs.
