Use **Initialize the SDK** when your application needs this OpenIM capability from Flutter SDK v3.8.3. This page follows the platform SDK reference and keeps the platform call signature visible next to the underlying core binding.

## Overview

Call `InitSDK` through the platform wrapper to initialize the sdk. The current user usually needs to be initialized and logged in before calling business APIs. Keep `operationID` traceable so client and server logs can be matched during troubleshooting.

## Core Binding Method

| Item             | Description                                                                |
| ---------------- | -------------------------------------------------------------------------- |
| Method signature | `InitSDK(listener OnConnListener, operationID string, config string) bool` |
| Capability       | SDK lifecycle                                                   |
| Source location  | `open_im_sdk/init_login.go`                                                |

## Flutter Call Reference

Flutter SDK v3.8.3 exposes the following platform method signature. The core binding signature above explains the underlying capability and parameter mapping.

Source:`lib/src/manager/im_manager.dart`.

```dart showLineNumbers
Future<dynamic> initSDK({ required int platformID, required String apiAddr, required String wsAddr, required String dataDir, required OnConnectListener listener, int logLevel = 6, bool isNeedEncryption = false, bool isCompression = false, bool isLogStandardOutput = true, String? logFilePath, String? operationID, })
```

## Parameters

| Parameter           | Type              | Required | Description                                                  |
| ------------------- | ----------------- | -------- | ------------------------------------------------------------ |
| platformID          | int               | Yes      | Pass as defined.                                             |
| apiAddr             | String            | Yes      | Pass as defined.                                             |
| wsAddr              | String            | Yes      | Pass as defined.                                             |
| dataDir             | String            | Yes      | Request object. Use the fields defined by the platform type. |
| listener            | OnConnectListener | Yes      | Callback or listener.                                        |
| logLevel            | int               | No       | Pass as defined.                                             |
| isNeedEncryption    | bool              | No       | Pass as defined.                                             |
| isCompression       | bool              | No       | Pass as defined.                                             |
| isLogStandardOutput | bool              | No       | Pass as defined.                                             |
| logFilePath         | String?           | No       | Pass as defined.                                             |

## Return Value

Returns `Future<dynamic>`. If the call fails, the Future throws an exception. Catch it in the application layer and keep the operationID in logs.
