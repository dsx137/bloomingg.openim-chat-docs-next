Use **Update self info** when your application needs this OpenIM capability from Flutter SDK v3.8.3. This page follows the platform SDK reference and keeps the platform call signature visible next to the underlying core binding.

## Overview

Call `SetSelfInfo` through the platform wrapper to update self info. The current user usually needs to be initialized and logged in before calling business APIs. Keep `operationID` traceable so client and server logs can be matched during troubleshooting.

## Core Binding Method

| Item             | Description                                                       |
| ---------------- | ----------------------------------------------------------------- |
| Method signature | `SetSelfInfo(callback Base, operationID string, userInfo string)` |
| Capability       | User                                                              |
| Source location  | `open_im_sdk/user.go`                                             |

## Flutter Call Reference

Flutter SDK v3.8.3 exposes the following platform method signature. The core binding signature above explains the underlying capability and parameter mapping.

Source:`lib/src/manager/im_user_manager.dart`.

```dart showLineNumbers
Future<String?> setSelfInfo({ String? nickname, String? faceURL, int? globalRecvMsgOpt, String? ex, String? operationID, })
```

## Parameters

| Parameter        | Type    | Required | Description                                     |
| ---------------- | ------- | -------- | ----------------------------------------------- |
| nickname         | String? | No       | Pass as defined.                                |
| faceURL          | String? | No       | Pass as defined.                                |

## Return Value

Returns `Future<String?>`. If the call fails, the Future throws an exception. Catch it in the application layer and keep the operationID in logs.
