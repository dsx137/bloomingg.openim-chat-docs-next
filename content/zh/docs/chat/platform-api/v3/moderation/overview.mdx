内容审核模块用于由可信后端执行用户屏蔽、屏蔽关系查询和群成员禁言等管理动作。它适合配合风控、举报处理、运营后台和社群治理流程使用。

## 能力范围

| 能力 | 说明 |
| ---- | ---- |
| 屏蔽用户 | 将目标用户加入黑名单，限制后续互动。 |
| 取消屏蔽 | 按业务处理结果解除用户屏蔽关系。 |
| 查询屏蔽关系 | 查询用户的屏蔽与被屏蔽关系，用于后台排查和状态展示。 |
| 群成员禁言 | 在指定群组内禁言或解除禁言成员。 |

## 接入建议

审核动作应由后端根据举报、风控命中、管理员操作或自动化策略触发。执行前记录操作者、目标用户、目标群组、原因和有效期；执行后同步更新业务后台的审核记录。

屏蔽和群内禁言是不同范围的能力。屏蔽关系通常影响用户之间的互动；群成员禁言只影响指定群组内的发言权限。产品侧应在 UI 中清晰区分这两类状态。
