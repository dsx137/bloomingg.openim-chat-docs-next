初始化 SDK 全局实例、日志、长连接管理、本地数据目录和基础资源。应用进程内通常只调用一次。

## 功能介绍

初始化 SDK 全局实例、日志、长连接管理、本地数据目录和基础资源。应用进程内通常只调用一次。

初始化类方法应在应用进程内按需调用；用户态方法通常要求 `InitSDK` 已成功执行，并通过 `operationID` 串联日志。

## 底层能力

| 项目     | 说明                                                                       |
| -------- | -------------------------------------------------------------------------- |
| 方法签名 | `InitSDK(listener OnConnListener, operationID string, config string) bool` |
| 所属能力 | SDK 生命周期                                                               |
| 源码位置 | `open_im_sdk/init_login.go`                                                |

## Electron 调用参考

Electron SDK v3.8.3 暴露以下平台方法原型。底层能力表用于说明 OpenIM core 能力和参数映射。

源码位置：`src/core/index.ts`。

```ts showLineNumbers
initSDK(param: InitConfig, opid = uuidV4())
```

## 输入参数

| 参数名称 | 参数类型   | 是否必填 | 描述                               |
| -------- | ---------- | -------- | ---------------------------------- |
| param    | InitConfig | 是       | 按平台类型定义传入。               |
| opid     | As defined | 否       | 链路追踪 ID；不传时 SDK 自动生成。 |

## 返回结果

返回 `Promise` 或注册回调结果。调用失败时 Promise reject；建议在业务层使用 `try/catch` 或 `.catch()` 处理错误。

## 代码示例

以下示例展示 Electron SDK v3.8.3 的调用骨架，请按业务上下文补充真实参数、错误处理和 UI 状态更新。

```ts showLineNumbers
import { getWithRenderProcess } from '@openim/electron-client-sdk/lib/render';

const { instance: openIM } = getWithRenderProcess();

const result = await openIM.initSDK({});
```
