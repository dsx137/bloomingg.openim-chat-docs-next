按成员类型筛选并分页获取群成员列表。

## 功能介绍

按成员类型筛选并分页获取群成员列表。

群组方法通常要求当前用户已登录，且群主、管理员、成员权限由服务端校验；群组变化通过 `SetGroupListener` 接收。

## 底层能力

| 项目     | 说明                                                                                                             |
| -------- | ---------------------------------------------------------------------------------------------------------------- |
| 方法签名 | `GetGroupMemberList(callback Base, operationID string, groupID string, filter int32, offset int32, count int32)` |
| 所属能力 | 群组                                                                                                             |
| 源码位置 | `open_im_sdk/group.go`                                                                                           |

## Electron 调用参考

Electron SDK v3.8.3 暴露以下平台方法原型。底层能力表用于说明 OpenIM core 能力和参数映射。

源码位置：`src/core/modules/group.ts`。

```ts showLineNumbers
getGroupMemberList(params: GetGroupMemberParams, opid = uuidV4())
```

## 输入参数

| 参数名称 | 参数类型             | 是否必填 | 描述                               |
| -------- | -------------------- | -------- | ---------------------------------- |
| params   | GetGroupMemberParams | 是       | 按平台类型定义传入。               |
| opid     | As defined           | 否       | 链路追踪 ID；不传时 SDK 自动生成。 |

## 返回结果

返回 `Promise` 或注册回调结果。调用失败时 Promise reject；建议在业务层使用 `try/catch` 或 `.catch()` 处理错误。

## 代码示例

以下示例展示 Electron SDK v3.8.3 的调用骨架，请按业务上下文补充真实参数、错误处理和 UI 状态更新。

```ts showLineNumbers
import { getWithRenderProcess } from '@openim/electron-client-sdk/lib/render';

const { instance: openIM } = getWithRenderProcess();

const result = await openIM.getGroupMemberList({});
```
