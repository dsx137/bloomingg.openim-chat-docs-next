按反向方向获取历史消息，适合围绕游标向另一方向加载。

## 功能介绍

按反向方向获取历史消息，适合围绕游标向另一方向加载。

创建消息类方法同步返回消息 JSON；发送、查询、删除、已读和本地写入方法通过回调返回。消息与会话事件分别通过消息监听器和会话监听器接收。

## 底层能力

| 项目     | 说明                                                                                                |
| -------- | --------------------------------------------------------------------------------------------------- |
| 方法签名 | `GetAdvancedHistoryMessageListReverse(callback Base, operationID string, getMessageOptions string)` |
| 所属能力 | 会话与消息                                                                                          |
| 源码位置 | `open_im_sdk/conversation_msg.go`                                                                   |

## Electron 调用参考

Electron SDK v3.8.3 暴露以下平台方法原型。底层能力表用于说明 OpenIM core 能力和参数映射。

源码位置：`src/core/modules/message.ts`。

```ts showLineNumbers
getAdvancedHistoryMessageListReverse(params: GetAdvancedHistoryMsgParams, opid = uuidV4())
```

## 输入参数

| 参数名称 | 参数类型                    | 是否必填 | 描述                               |
| -------- | --------------------------- | -------- | ---------------------------------- |
| params   | GetAdvancedHistoryMsgParams | 是       | 按平台类型定义传入。               |
| opid     | As defined                  | 否       | 链路追踪 ID；不传时 SDK 自动生成。 |

## 返回结果

返回 `Promise` 或注册回调结果。调用失败时 Promise reject；建议在业务层使用 `try/catch` 或 `.catch()` 处理错误。

## 代码示例

以下示例展示 Electron SDK v3.8.3 的调用骨架，请按业务上下文补充真实参数、错误处理和 UI 状态更新。

```ts showLineNumbers
import { getWithRenderProcess } from '@openim/electron-client-sdk/lib/render';

const { instance: openIM } = getWithRenderProcess();

const result = await openIM.getAdvancedHistoryMessageListReverse({});
```
