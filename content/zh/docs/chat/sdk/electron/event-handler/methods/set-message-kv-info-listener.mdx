接收消息扩展键值的新增、更新和删除事件。

## 功能介绍

消息 KV 事件应更新消息扩展状态，例如表态、标注或业务自定义元数据。建议同步本地消息缓存和当前可见消息单元。

## 底层能力

| 项目     | 说明                                                         |
| -------- | ------------------------------------------------------------ |
| 方法签名 | `SetMessageKvInfoListener(listener OnMessageKvInfoListener)` |
| 所属能力 | 监听器                                                       |
| 源码位置 | `open_im_sdk/listener.go`                                    |

## Electron 调用参考

Electron SDK v3.8.3 暴露以下平台方法原型。底层能力表用于说明 OpenIM core 能力和参数映射。

源码位置：`src/render.ts`。

```ts showLineNumbers
on(event: CbEvents, listener: (event) => void)
```

## 输入参数

| 参数名称 | 参数类型 | 是否必填 | 描述                   |
| -------- | -------- | -------- | ---------------------- |
| event    | CbEvents | 是       | 需要订阅的事件名称。   |
| listener | Function | 是       | 事件触发时执行的回调。 |

## 返回结果

返回 `Promise` 或注册回调结果。调用失败时 Promise reject；建议在业务层使用 `try/catch` 或 `.catch()` 处理错误。

## 代码示例

Electron 使用事件订阅方式接收回调。按需要替换事件名和处理函数。

```ts showLineNumbers
import { getWithRenderProcess } from '@openim/electron-client-sdk/lib/render';
import { CbEvents } from '@openim/wasm-client-sdk';

const { instance: openIM } = getWithRenderProcess();

openIM.on(CbEvents.OnRecvNewMessages, (event) => {
  console.log(event);
});
```
