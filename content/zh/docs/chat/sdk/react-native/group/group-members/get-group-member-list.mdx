按成员类型筛选并分页获取群成员列表。

## 功能介绍

按成员类型筛选并分页获取群成员列表。

群组方法通常要求当前用户已登录，且群主、管理员、成员权限由服务端校验；群组变化通过 `SetGroupListener` 接收。

## Core Binding 方法

| 项目     | 说明                                                                                                             |
| -------- | ---------------------------------------------------------------------------------------------------------------- |
| 方法签名 | `GetGroupMemberList(callback Base, operationID string, groupID string, filter int32, offset int32, count int32)` |
| 所属能力 | 群组                                                                                                             |
| 源码位置 | `open_im_sdk/group.go`                                                                                           |

## React Native 调用参考

React Native SDK v3.8.3 的 TypeScript 方法原型如下。Core binding 方法签名用于说明底层能力和参数映射。

源码位置：`src/sdk.ts`。

```ts showLineNumbers
getGroupMemberList(params: GetGroupMemberParams, operationID: string = id())
```

## 输入参数

| 参数名称 | 参数类型             | 是否必填 | 描述                                   |
| -------- | -------------------- | -------- | -------------------------------------- |
| params   | GetGroupMemberParams | 是       | 请求对象，字段以对应平台类型定义为准。 |

## 返回结果

返回 `Promise`。调用失败时 Promise reject，错误会被封装为 OpenIMApiError。

## 代码示例

以下示例展示 React Native SDK v3.8.3 的调用骨架，请按业务上下文补充真实参数、错误处理和 UI 状态更新。

```ts showLineNumbers
import OpenIMSDK from '@openim/rn-client-sdk';

const result = await OpenIMSDK.getGroupMemberList({});
```
