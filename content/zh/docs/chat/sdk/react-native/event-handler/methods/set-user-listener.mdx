设置用户事件监听器，用于接收当前用户资料更新、用户在线状态变化和用户命令增删改事件。

## 功能介绍

设置用户事件监听器，用于接收当前用户资料更新、用户在线状态变化和用户命令增删改事件。

`Set*Listener` 会替换当前 SDK 保存的同类监听器；如果多个模块都要消费事件，应在业务层封装统一分发。

## Core Binding 方法

| 项目     | 说明                                       |
| -------- | ------------------------------------------ |
| 方法签名 | `SetUserListener(listener OnUserListener)` |
| 所属能力 | 监听器                                     |
| 源码位置 | `open_im_sdk/listener.go`                  |

## React Native 调用参考

React Native SDK v3.8.3 的 TypeScript 方法原型如下。Core binding 方法签名用于说明底层能力和参数映射。

```ts showLineNumbers
setUserListener(/* ... */);
```

## 输入参数

| 参数名称 | 参数类型   | 是否必填   | 描述                 |
| -------- | ---------- | ---------- | -------------------- |
| 参数     | 见方法原型 | 按方法原型 | 按上方调用原型传入。 |

## 返回结果

返回 `Promise`。调用失败时 Promise reject，错误会被封装为 OpenIMApiError。

以下示例展示 React Native SDK v3.8.3 的调用骨架，请按业务上下文补充真实参数、错误处理和 UI 状态更新。

const result = await OpenIMSDK.setUserListener();
