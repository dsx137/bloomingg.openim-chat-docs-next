OpenIM WASM SDK 用于在浏览器应用中接入 OpenIMSDK。浏览器包由 `@openim/wasm-client-sdk` 提供，运行时需要 `openIM.wasm`、`sql-wasm.wasm` 和 `wasm_exec.js`，并通过 `getSDK()` 创建供当前浏览器会话复用的 SDK 实例。

本文档以 `@openim/wasm-client-sdk@3.8.5-hotfix.0` 的公开声明为 API 基线。正文只介绍当前可用的业务 API 和事件：已废弃方法不单独成文，其能力由替代 API 说明；好友和会话列表只介绍分页接口。WASM 包仅用于核对方法、请求、返回和事件，不作为本站运行依赖。

## 接入流程

1. 根据部署环境发布 WASM 资源并初始化 SDK。
2. 从可信后端取得当前用户的 `userID`、token 和 OpenIMServer 地址。
3. 在 `login()` 前注册连接和 token 事件，再建立登录会话。
4. 使用用户、好友、会话、群组和消息 API 构建业务界面。
5. 将事件携带的增量数据幂等地合并到应用状态，并在退出或切换账号时清理处理器。

```ts
import { getSDK } from '@openim/wasm-client-sdk';

const openimsdk = getSDK({
  coreWasmPath: '/openIM.wasm',
  sqlWasmPath: '/sql-wasm.wasm',
});
```

## 关键数据边界

- 单聊发送时填写 `recvID`，群聊发送时填写 `groupID`；两种消息最终都进入会话列表。
- `ConversationItem` 保存当前账号的会话展示信息、未读数、草稿和接收设置；`GroupItem` 与 `GroupMemberItem` 保存群组及成员关系。
- SDK 事件是增量信号，应用应在首次进入页面时主动查询所需数据；重新登录后的业务变化由对应事件同步。
- Token 和房间凭据都属于敏感数据，不应写入普通日志或持久化到不受控存储。

纯查询 API 的 Promise 返回页面所需快照，不应描述为触发共享事件。对于会改变状态的 API，应分别处理 Promise 结果和事件增量；各事件的完整 `on()`、`off()` 与合并代码只出现在对应能力归属页，概览不重复注册。
