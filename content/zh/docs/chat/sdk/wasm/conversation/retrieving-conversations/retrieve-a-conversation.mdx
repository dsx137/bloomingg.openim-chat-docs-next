OpenIMSDK 支持按聊天目标读取一个会话，也支持按 `conversationID` 批量读取会话。调用前应完成 SDK 登录；连接状态和 `OnConnectSuccess` 的处理见[认证与管理登录会话](/sdk/wasm/getting-started/authenticate-and-manage-session)。

## 常见使用场景

是否需要先获取会话，取决于用户从哪里进入聊天页：

| 入口 | 推荐处理方式 |
| ---- | ------------ |
| 从通讯录点击某个用户 | 使用对方 `userID` 和 `SessionType.Single` 调用 `getOneConversation()`，获取用于打开聊天页的单聊会话。 |
| 从群资料或已加入群组列表进入聊天 | 使用 `groupID` 和 `SessionType.WorkingGroup` 调用 `getOneConversation()`。 |
| 从会话列表进入聊天 | 直接使用列表中的 `ConversationItem`，通常不需要再次查询。需要刷新时，再按其 `conversationID` 查询。 |
| 从通知或业务跳转携带 `conversationID` 进入 | 使用 `getMultipleConversation([conversationID])` 读取对应会话，并检查返回列表是否包含目标记录。 |

例如，用户在通讯录中点击好友时，直接获取单聊会话，再把完整的 `ConversationItem` 交给聊天页：

```ts
import { SessionType } from '@openim/wasm-client-sdk';

async function openChatFromContact(targetUserID: string) {
  const { data: conversation } = await openimsdk.getOneConversation({
    sourceID: targetUserID,
    sessionType: SessionType.Single,
  });

  openChatPage({
    targetUserID,
    conversation,
  });
}
```

双方是否已经产生过消息，不影响这里的调用方式。从通讯录进入单聊时都应调用 `getOneConversation()`：SDK 会根据目标 `userID` 和会话类型返回 `ConversationItem`，首次聊天和已有聊天可以共用同一套打开页面流程。发送消息时仍把目标用户 ID 传给 `sendMessage()` 的 `recvID`。

## 按聊天目标获取会话

调用 `getOneConversation()` 时，单聊把对方 `userID` 作为 `sourceID` 并使用 `SessionType.Single`；群聊把 `groupID` 作为 `sourceID` 并使用 `SessionType.WorkingGroup`。

```ts
import { SessionType } from '@openim/wasm-client-sdk';

const { data: conversation } = await openimsdk.getOneConversation({
  sourceID: groupID,
  sessionType: SessionType.WorkingGroup,
});

openConversation(conversation.conversationID);
```

响应中的 `data` 是 `ConversationItem`。即使双方还没有产生过消息，也应通过这个方法获取会话，不需要客户端自行拼接 `conversationID` 或构造 `ConversationItem`。调用失败时按 SDK 返回的错误处理，重点检查登录状态、目标 ID、会话类型和连接状态。

## 获取 conversationID

只需要会话 ID 时，使用相同的目标参数调用 `getConversationIDBySessionType()`：

```ts
const { data: conversationID } = await openimsdk.getConversationIDBySessionType({
  sourceID: targetUserID,
  sessionType: SessionType.Single,
});
```

这个方法返回字符串，不返回完整 `ConversationItem`。会话设置、已读、隐藏和删除等方法通常直接接收该 `conversationID`。

## 按 conversationID 批量获取

已经保存一组会话 ID 时，调用 `getMultipleConversation()`：

```ts
const { data: conversations } =
  await openimsdk.getMultipleConversation(conversationIDs);

const byID = new Map(
  conversations.map((conversation) => [conversation.conversationID, conversation])
);
```

返回值是 `ConversationItem[]`。调用成功但某个 ID 没有结果时，应按缺失记录处理，不要依赖数组位置与请求数组完全一一对应。失败时记录错误对象和目标标识。

## 调用结果与后续变化

本页三个方法都应直接使用 Promise 返回值完成当前导航或查询：`getOneConversation()` 返回单个 `ConversationItem`，`getConversationIDBySessionType()` 返回会话 ID，`getMultipleConversation()` 返回会话数组。不要等待会话事件后才打开聊天页，也不要把查询成功描述为触发某个会话事件。

页面打开后的会话新增和属性变化由[获取会话列表](/sdk/wasm/conversation/retrieving-conversations/retrieve-conversation-list)中的事件处理器负责；需要校准指定会话时，再调用本页方法重新查询。
