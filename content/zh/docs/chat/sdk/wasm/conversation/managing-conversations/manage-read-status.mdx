OpenIM 的普通会话未读数保存在 `ConversationItem.unreadCount` 中。群聊 @ 状态保存在 `groupAtType`，两者是不同状态。

## 标记会话已读

打开会话并阅读完当前可见消息后，调用 `markConversationMessageAsRead()`：

```ts
await OpenIM.markConversationMessageAsRead(conversationID);
```

不要在只预览通知或后台收到消息时提前标记已读。调用成功后仍应通过 `CbEvents.OnConversationChanged` 合并该会话的最新未读数。

## 获取总未读数

调用 `getTotalUnreadMsgCount()` 获取当前用户所有会话的总未读数：

```ts
const { data: totalUnread } = await OpenIM.getTotalUnreadMsgCount();
updateApplicationBadge(totalUnread);
```

前台运行期间监听 `CbEvents.OnTotalUnreadMessageCountChanged` 更新应用角标。总未读数不是把当前已加载各页会话的未读数简单相加得到的值，应以 SDK 返回和事件为准。

```ts
import { CbEvents } from '@openim/wasm-client-sdk';

const handleTotalUnreadChanged = ({ data }) => {
  updateApplicationBadge(data);
};

OpenIM.on(CbEvents.OnTotalUnreadMessageCountChanged, handleTotalUnreadChanged);

function removeTotalUnreadListener() {
  OpenIM.off(CbEvents.OnTotalUnreadMessageCountChanged, handleTotalUnreadChanged);
}
```

组件卸载、退出登录或切换账号时调用 `removeTotalUnreadListener()`。

## 清理群聊 @ 状态

用户处理完群聊中的 @ 消息后，使用 `setConversation()` 把 `groupAtType` 更新为普通状态：

```ts
import { GroupAtType } from '@openim/wasm-client-sdk';

await OpenIM.setConversation({
  conversationID,
  groupAtType: GroupAtType.AtNormal,
});
```

消息已读和清理群聊 @ 提醒状态可能在同一界面发生，但不要把两者当作同一个 API 状态。

## 调用后的状态变化

- `markConversationMessageAsRead()` 成功表示当前会话的标记已读请求已完成。会话未读数通过 `CbEvents.OnConversationChanged` 更新；事件的 `data` 是 `ConversationItem[]`，按 `conversationID` 合并。完整监听方式见[获取会话列表](/sdk/wasm/conversation/retrieving-conversations/retrieve-conversation-list)。
- 使用 `setConversation()` 清理 `groupAtType` 后，同样通过 `CbEvents.OnConversationChanged` 合并该会话的 @ 状态，不需要注册新的事件。
- `getTotalUnreadMsgCount()` 只返回调用时的总未读快照，不触发总未读事件。后续总数变化由本页唯一归属的 `CbEvents.OnTotalUnreadMessageCountChanged` 持续更新。

单个会话未读数和账号总未读数必须分别维护，不能用已加载会话的 `unreadCount` 相加替代 SDK 返回的总数。
