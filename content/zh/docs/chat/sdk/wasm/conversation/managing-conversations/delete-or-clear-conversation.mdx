删除会话前应先明确是否保留会话行、是否删除该会话下的消息，以及操作是否只针对本地会话数据库。

| 方法                                      | 用途                                                         |
| ----------------------------------------- | ------------------------------------------------------------ |
| `deleteConversation()`                    | 删除指定会话记录。                                           |
| `deleteConversationAndDeleteAllMsg()`     | 删除指定会话记录及该会话下的消息。                           |
| `clearConversationAndDeleteAllMsg()`      | 保留会话入口，清空该会话下的消息和消息摘要状态。             |
| `deleteAllConversationFromLocal()`        | 清理当前账号本地数据库中的全部会话记录。                     |

## 删除一个会话

```ts
await OpenIM.deleteConversation(
  conversationID
);
```

只删除会话记录不等于删除好友关系、退出群组或解散群组。相关聊天目标仍存在时，后续新消息或同步可以重新生成会话。

## 同时处理消息

需要连同指定会话消息一起处理时，根据产品语义选择以下方法：

```ts
await OpenIM.deleteConversationAndDeleteAllMsg(conversationID);

await OpenIM.clearConversationAndDeleteAllMsg(conversationID);
```

`deleteConversationAndDeleteAllMsg()` 适合“删除此聊天”，会话在后续产生新消息时仍可能重新生成。`clearConversationAndDeleteAllMsg()` 适合“清空聊天记录”，会话入口继续保留。两种操作都只处理当前账号的会话数据，不会删除好友关系、退出群组或影响其他用户。

## 清理全部本地会话

`deleteAllConversationFromLocal()` 是账号级本地清理操作，影响范围大，不应放在单条会话菜单中：

```ts
await OpenIM.deleteAllConversationFromLocal();
```

执行前清理当前账号的页面状态和缓存引用，并确认产品确实需要删除全部本地会话。它不等于删除服务端聊天关系或让其他用户的数据消失。

## 调用后的状态变化

- `deleteConversation()` 成功后，从当前列表移除对应 `conversationID`。聊天关系仍存在时，新消息或后续同步可能重新生成该会话。
- `deleteConversationAndDeleteAllMsg()` 和 `clearConversationAndDeleteAllMsg()` 成功后，立即清理当前消息列表，再通过 `CbEvents.OnConversationChanged` 或重新查询校准会话摘要、最后一条消息和未读状态。`OnConversationChanged` 的 `data` 是 `ConversationItem[]`，应按 `conversationID` 幂等合并。
- `deleteAllConversationFromLocal()` 只清理当前账号的本地会话数据库。调用成功后清空本地列表和分页状态；该本地操作不应描述为向其他用户广播会话事件。

`OnConversationChanged` 的完整监听方式见[获取会话列表](/sdk/wasm/conversation/retrieving-conversations/retrieve-conversation-list)。涉及消息列表清理时，还应同步更新当前聊天页，不能只处理会话行。
