会话列表中的置顶、消息接收选项、私聊、阅后即焚、消息销毁、标记、备注和扩展字段等状态，都可以通过 `setConversation()` 设置。调用时传入目标 `conversationID` 和本次需要修改的字段；未传入的字段保持原有状态。

## 参数说明

| 字段              | 类型                    | 说明                       |
| ----------------- | ----------------------- | -------------------------- |
| `conversationID`  | `string`                | 必填，目标会话 ID。        |
| `recvMsgOpt`      | `MessageReceiveOptType` | 消息接收选项。             |
| `groupAtType`     | `GroupAtType`           | 群聊 @ 提醒状态。          |
| `burnDuration`    | `number`                | 阅后即焚时长，单位为秒。   |
| `msgDestructTime` | `number`                | 消息销毁时长，单位为秒。   |
| `isPinned`        | `boolean`               | 是否置顶会话。             |
| `isPrivateChat`   | `boolean`               | 是否启用私聊状态。         |
| `isMsgDestruct`   | `boolean`               | 是否启用消息销毁。         |
| `isMarked`        | `boolean`               | 是否标记该会话。           |
| `remark`          | `string`                | 当前用户为会话设置的备注。 |
| `ex`              | `string`                | 应用约定的会话扩展字符串。 |

```ts
import { MessageReceiveOptType } from '@openim/wasm-client-sdk';

await openimsdk.setConversation(
  {
    conversationID,
    isPinned: true,
    isMarked: true,
    remark: '项目讨论',
    recvMsgOpt: MessageReceiveOptType.Normal,
    ex: JSON.stringify({ category: 'work' }),
  }
);
```

`ex` 是完整的字符串值，不会自动合并 JSON。更新前应先解析并保留已有业务字段。

## 设置消息销毁

开启消息销毁时，可以在同一次调用中设置开关和销毁时长：

```ts
await openimsdk.setConversation({
  conversationID,
  isMsgDestruct: true,
  msgDestructTime: 3600,
});
```

`msgDestructTime` 的单位为秒。关闭消息销毁时传入 `isMsgDestruct: false`；是否同时保留原销毁时长由产品交互决定。

## 调用后的状态变化

`setConversation()` 的 Promise 成功表示服务端已完成本次会话设置请求。SDK 通过 `CbEvents.OnConversationChanged` 同步变化后的会话状态；事件的 `data` 是 `ConversationItem[]`，应按 `conversationID` 合并本次设置涉及的字段。

上述字段共用同一个会话变更事件，不需要为每个字段分别注册处理器。完整的事件注册、清理和列表合并方式见[获取会话列表](/sdk/wasm/conversation/retrieving-conversations/retrieve-conversation-list)。

需要显示当前会话快照时，再调用[获取会话](/sdk/wasm/conversation/retrieving-conversations/retrieve-a-conversation)中的查询方法。重新登录后的变化由会话事件同步。
