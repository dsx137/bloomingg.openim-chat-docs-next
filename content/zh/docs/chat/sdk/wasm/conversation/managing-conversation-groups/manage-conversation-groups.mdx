会话分组用于整理当前登录用户的会话列表，例如“工作”“未读”或“稍后处理”。它管理的是 `conversationID` 与分组的关系，不会创建群组、修改群成员，也不会改变消息的发送目标。

## 分组类型

`ConversationGroupType` 公开三种类型：

| 枚举值                        | 数值 | 用途                                 |
| ----------------------------- | ---: | ------------------------------------ |
| `ConversationGroupTypeNormal` |  `0` | 普通自定义分组，可由用户创建和维护。 |
| `ConversationGroupTypeFilter` |  `1` | 按 SDK 或服务端规则维护的筛选分组。  |
| `ConversationGroupTypeAll`    |  `2` | 查询全部类型时使用。                 |

普通用户自定义分类应使用 `ConversationGroupTypeNormal`。筛选分组的具体生成规则由当前 OpenIM 部署决定，不要仅凭名称在客户端伪造系统分组。

## 创建和查询分组

### 创建分组

创建普通分组时至少传入名称。需要在创建时放入一个会话，可以同时传入 `conversationID`：

#### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `name` | `string` | 必填 | 分组名称。 |
| `order` | `number` | 可选 | 分组顺序值，由产品统一约定排序方向。 |
| `ex` | `string` | 可选 | 应用约定的分组扩展字符串。 |
| `conversationID` | `string` | 可选 | 创建后立即加入该分组的会话 ID。 |
| `conversationGroupType` | `ConversationGroupType` | 可选 | 分组类型；普通自定义分组建议明确传入 `ConversationGroupTypeNormal`。 |

```ts
import { ConversationGroupType } from '@openim/wasm-client-sdk';

const { data: group } = await OpenIM.createConversationGroup({
  name: '工作',
  order: 100,
  conversationID,
  conversationGroupType: ConversationGroupType.ConversationGroupTypeNormal,
});
```

`createConversationGroup()` 返回 `ConversationGroup`。常用字段包括 `conversationGroupID`、`name`、`conversationGroupType`、`hidden`、`unreadCount` 和 `conversationIDs`。

### 按类型查询分组

按类型查询分组：

```ts
const { data: groups } = await OpenIM.getConversationGroups(
  ConversationGroupType.ConversationGroupTypeNormal
);
```

Promise 成功后，`data` 是匹配该类型的 `ConversationGroup[]`。查询全部类型时使用 `ConversationGroupTypeAll`。

### 分页查询分组内的会话

需要同时分页读取某个分组内的会话详情时，使用 `getConversationGroupInfoWithConversations()`：

#### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `conversationGroupID` | `string` | 必填 | 要查询的会话分组 ID。 |
| `pagination.pageNumber` | `number` | 必填 | 页码，从 `1` 开始。 |
| `pagination.showNumber` | `number` | 必填 | 每页读取的会话数量。 |

```ts
const { data } = await OpenIM.getConversationGroupInfoWithConversations({
  conversationGroupID: group.conversationGroupID,
  pagination: {
    pageNumber: 1,
    showNumber: 20,
  },
});

renderConversationGroup(data.group, data.conversationElems, data.conversationTotal);
```

响应中的 `conversationTotal` 是分组内会话总数，`conversationElems` 是当前页的 `ConversationItem[]`。

## 添加或移除会话

### 添加会话到分组

调用 `addConversationsToGroups()` 时，`conversationIDs` 是要添加的会话 ID 数组，`conversationGroupIDs` 是目标分组 ID 数组：

```ts
await OpenIM.addConversationsToGroups({
  conversationIDs: [conversationID],
  conversationGroupIDs: [group.conversationGroupID],
});
```

### 从分组移除会话

调用 `removeConversationsFromGroups()` 时，分别传入要移除的会话 ID 数组和目标分组 ID 数组：

```ts
await OpenIM.removeConversationsFromGroups({
  conversationIDs: [conversationID],
  conversationGroupIDs: [group.conversationGroupID],
});
```

### 查询会话所属分组

同一个会话可以关联多个分组。需要反向查询某个会话所属的全部分组时，调用：

```ts
const { data: groupIDs } = await OpenIM.getConversationGroupIDsByConversationID(conversationID);
```

添加或移除关联不会删除会话，也不会清空消息。

## 更新、排序和删除

### 更新分组

`updateConversationGroup()` 可以修改名称、扩展字段或隐藏状态：

```ts
await OpenIM.updateConversationGroup({
  conversationGroupID: group.conversationGroupID,
  name: '重点工作',
  hidden: false,
});
```

`conversationGroupID` 指定目标分组；只传本次需要更新的字段。Promise 成功后，`data` 是更新后的 `ConversationGroup`。

### 调整分组顺序

批量调整顺序时，为每个分组提供 `conversationGroupID` 和新的 `order`：

```ts
await OpenIM.setConversationGroupOrder([
  { conversationGroupID: firstGroupID, order: 100 },
  { conversationGroupID: secondGroupID, order: 200 },
]);
```

### 删除分组

删除分组时传入会话分组 ID：

```ts
await OpenIM.deleteConversationGroup(group.conversationGroupID);
```

参数在 SDK 声明中命名为 `groupID`，这里实际表示 `conversationGroupID`，不要误传 OpenIM 群组 ID。该操作删除的是会话分组及其关联关系，不会删除分组中的会话或消息。

## 调用后的状态变化

| API | Promise 成功后 | 相关事件 |
| --- | --- | --- |
| `createConversationGroup()` | 可以先使用返回的 `ConversationGroup` 展示新分组。 | `OnConversationGroupAdded` 同步新增分组。 |
| `updateConversationGroup()`、`setConversationGroupOrder()` | 分组资料或顺序更新已经完成。 | `OnConversationGroupChanged` 同步变化后的分组。 |
| `deleteConversationGroup()` | 分组及其会话关联已经删除。 | `OnConversationGroupDeleted` 同步被删除的分组。 |
| `addConversationsToGroups()` | 会话与分组的关联已经添加。 | `OnConversationGroupMemberAdded` 同步关联和会话详情。 |
| `removeConversationsFromGroups()` | 会话与分组的关联已经移除。 | `OnConversationGroupMemberDeleted` 同步被移除的关联。 |

三个查询方法只建立调用时的分组快照，不应描述为触发分组事件。主动调用返回的数据和后续事件可能覆盖同一分组，应始终按 `conversationGroupID` 幂等合并。

## 监听分组变化

分组本身和成员关系使用不同事件：

| 事件                               | `data` 类型                                  | 处理方式                            |
| ---------------------------------- | -------------------------------------------- | ----------------------------------- |
| `OnConversationGroupAdded`         | `ConversationGroup[]`                        | 合并新增分组。                      |
| `OnConversationGroupChanged`       | `ConversationGroup[]`                        | 按 `conversationGroupID` 更新分组。 |
| `OnConversationGroupDeleted`       | `ConversationGroup[]`                        | 移除已删除分组。                    |
| `OnConversationGroupMemberAdded`   | `ConversationGroupMemberChangedCallbackData` | 合并新增的会话关联和会话详情。      |
| `OnConversationGroupMemberDeleted` | `ConversationGroupMemberChangedCallbackData` | 移除对应的会话关联。                |

```ts
import { CbEvents } from '@openim/wasm-client-sdk';

function handleConversationGroupAdded({ data }) {
  mergeConversationGroups(data);
}

function handleConversationGroupChanged({ data }) {
  mergeConversationGroups(data);
}

function handleConversationGroupDeleted({ data }) {
  for (const group of data) {
    removeConversationGroup(group.conversationGroupID);
  }
}

function handleConversationGroupMemberAdded({ data }) {
  mergeConversationGroupMembers(
    data.group.conversationGroupID,
    data.conversationIDs,
    data.conversations
  );
}

function handleConversationGroupMemberDeleted({ data }) {
  removeConversationGroupMembers(
    data.group.conversationGroupID,
    data.conversationIDs
  );
}

OpenIM.on(CbEvents.OnConversationGroupAdded, handleConversationGroupAdded);
OpenIM.on(CbEvents.OnConversationGroupChanged, handleConversationGroupChanged);
OpenIM.on(CbEvents.OnConversationGroupDeleted, handleConversationGroupDeleted);
OpenIM.on(CbEvents.OnConversationGroupMemberAdded, handleConversationGroupMemberAdded);
OpenIM.on(CbEvents.OnConversationGroupMemberDeleted, handleConversationGroupMemberDeleted);

function disposeConversationGroupEvents() {
  OpenIM.off(CbEvents.OnConversationGroupAdded, handleConversationGroupAdded);
  OpenIM.off(CbEvents.OnConversationGroupChanged, handleConversationGroupChanged);
  OpenIM.off(CbEvents.OnConversationGroupDeleted, handleConversationGroupDeleted);
  OpenIM.off(CbEvents.OnConversationGroupMemberAdded, handleConversationGroupMemberAdded);
  OpenIM.off(CbEvents.OnConversationGroupMemberDeleted, handleConversationGroupMemberDeleted);
}
```

事件可能在主动调用完成后再次到达，状态层应按 ID 幂等合并，不能简单追加数组。
