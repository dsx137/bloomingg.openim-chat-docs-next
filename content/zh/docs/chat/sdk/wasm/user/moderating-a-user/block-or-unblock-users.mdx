OpenIM 黑名单是当前用户主动维护的单向关系。调用 `addBlack()` 将目标用户加入黑名单，调用 `removeBlack()` 将其移出黑名单。该能力可用于屏蔽用户、限制聊天入口，以及展示资料卡中的关系状态。

## 关系边界

黑名单只表示当前用户与目标用户之间的屏蔽关系。群管理员禁言或移除成员，以及平台封禁账号，属于不同的管理能力，应分别通过群成员 API 或可信后端实现。

黑名单对消息发送的限制是非对称的：拉黑后，对方不能向当前用户发送消息，但当前用户仍可向对方发送消息。如果产品要求双方都不能继续发送消息，还需要在界面和业务层限制当前用户的发送入口。

| 需求                   | OpenIM 对应能力                                                                                                                     |
| ---------------------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| 当前用户屏蔽另一个用户 | `addBlack({ toUserID })`                                                                                                            |
| 当前用户解除屏蔽       | `removeBlack(toUserID)`                                                                                                             |
| 查看当前用户黑名单     | `getBlackList()`                                                                                                                    |
| 禁言某个群成员         | 使用 `changeGroupMemberMute()`，见 [禁言群组或群成员](/sdk/wasm/group/moderating-groups/mute-a-group-or-member) |
| 管理群角色或管理员     | 使用群成员 API，见 [更新群成员资料和角色](/sdk/wasm/group/managing-group-members/update-group-member-info)             |

全局封禁、风控处罚和管理后台策略应由可信后端维护，客户端只负责展示后端返回的结果。

## 拉黑用户

拉黑用户时传入目标 `userID`。接口调用成功后，根据黑名单事件更新本地关系状态；需要重新校准完整列表时，使用[获取黑名单列表](/sdk/wasm/user/moderating-a-user/retrieve-a-list-of-blocked-users)中的查询流程。

```ts
async function blockUser(targetUserID: string) {
  try {
    await OpenIM.addBlack({
      toUserID: targetUserID,
      ex: JSON.stringify({ source: 'profile-menu' }),
    });
  } catch (error) {
    console.error('addBlack failed', { error, targetUserID });
    throw error;
  }
}
```

取消拉黑时，将目标用户 ID 作为 `removeBlack()` 的第一个参数传入。

```ts
async function unblockUser(targetUserID: string) {
  try {
    await OpenIM.removeBlack(targetUserID);
  } catch (error) {
    console.error('removeBlack failed', { error, targetUserID });
    throw error;
  }
}
```

## 调用后的状态变化

- `addBlack()` 成功表示目标用户已经加入当前账号的黑名单；`CbEvents.OnBlackAdded` 的 `data` 是对应的 `BlackUserItem`，应按 `userID` 写入或覆盖本地黑名单记录。
- `removeBlack()` 成功表示拉黑关系已经解除；`CbEvents.OnBlackDeleted` 携带被移出的黑名单记录，应按 `userID` 从本地集合移除。

调用成功后，可结束当前按钮的提交状态；黑名单列表和其他登录端的最终状态仍通过事件增量更新，或重新查询完整列表进行校准。

## 监听黑名单事件

`CbEvents.OnBlackAdded` 和 `CbEvents.OnBlackDeleted` 用于同步拉黑关系变化。列表快照由“获取黑名单列表”页面负责，本页只处理写操作和增量事件。

```ts
function handleBlackAdded({ data }) {
  upsertBlockedUser(data);
}

function handleBlackDeleted({ data }) {
  removeBlockedUser(data.userID);
}

OpenIM.on(CbEvents.OnBlackAdded, handleBlackAdded);
OpenIM.on(CbEvents.OnBlackDeleted, handleBlackDeleted);

function disposeBlackEvents() {
  OpenIM.off(CbEvents.OnBlackAdded, handleBlackAdded);
  OpenIM.off(CbEvents.OnBlackDeleted, handleBlackDeleted);
}
```

黑名单列表返回 `BlackUserItem[]`。判断关系状态时应以 `userID` 为准，昵称、头像和扩展字段只用于展示。完整查询用法见[获取黑名单列表](/sdk/wasm/user/moderating-a-user/retrieve-a-list-of-blocked-users)。
