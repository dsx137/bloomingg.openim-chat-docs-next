OpenIM SDK 通过订阅机制同步用户在线状态。建议只订阅当前界面可见的用户，例如会话列表、联系人当前页或正在查看的资料卡中的用户。

在线状态只反映用户是否连接到 OpenIM。好友关系、群成员状态、禁言状态和应用自身记录的活跃时间，应分别使用对应的数据源。

## 订阅可见用户的状态

调用方应先收集当前可见用户的 `userID[]`，再传给 `subscribeUsersStatus()`。订阅成功后，返回结果包含这些用户当前的在线状态。

```ts
function uniqueUserIDs(userIDs: string[]) {
  return [...new Set(userIDs.filter(Boolean))];
}

async function subscribeVisibleUserStatus(userIDs: string[]) {
  const targetUserIDs = uniqueUserIDs(userIDs);

  if (targetUserIDs.length === 0) {
    return [];
  }

  try {
    const { data } = await OpenIM.subscribeUsersStatus(targetUserIDs);
    return data;
  } catch (error) {
    console.error('subscribeVisibleUserStatus failed', { error, targetUserIDs });
    throw error;
  }
}

const states = await subscribeVisibleUserStatus(visibleUserIDs);
mergePresence(states);
```

每个登录用户最多可订阅 3000 名用户的在线状态。分页、搜索结果或虚拟滚动窗口变化时，只订阅新进入可见范围的用户，并取消订阅已经离开可见范围的用户。不要一次性订阅整个应用的用户目录。

## 调用后的状态变化

- `subscribeUsersStatus()` 成功后，先使用返回的 `UserOnlineState[]` 建立当前状态快照；此后已订阅用户的变化通过 `CbEvents.OnUserStatusChanged` 到达。
- `getSubscribeUsersStatus()` 只返回调用时的状态数组，不应描述为触发在线状态事件。
- `unsubscribeUsersStatus()` 成功后，从本地订阅集合和状态缓存中移除对应 `userID`，后续不再依赖这些用户的状态事件更新界面。

返回数组和事件数据都使用 `userID` 合并。不要按列表位置覆盖状态，因为可见用户集合会随分页、搜索和虚拟滚动变化。

## 监听状态变化

已订阅用户上线或离线时，SDK 会触发 `CbEvents.OnUserStatusChanged`。应按 `userID` 将事件中的 `UserOnlineState` 写入本地状态表，再更新会话列表、成员列表或资料卡中的在线标识。

```ts
const presenceByUserID = new Map<string, UserOnlineState>();

function handleUserStatusChanged({ data }) {
  presenceByUserID.set(data.userID, data);
  renderPresenceDot(data.userID, data.status === OnlineState.Online);
}

OpenIM.on(CbEvents.OnUserStatusChanged, handleUserStatusChanged);

function disposeUserStatusEvents() {
  OpenIM.off(CbEvents.OnUserStatusChanged, handleUserStatusChanged);
}
```

如果多个页面都要展示在线状态，建议在统一的用户状态层注册事件。各页面只订阅和读取所需用户的状态，避免多个组件重复处理同一个事件。

## 查询已订阅用户状态

需要局部刷新、断线重连或重新进入列表页时，可以调用 `getSubscribeUsersStatus()`，读取当前账号已订阅用户的状态。目标用户尚未订阅时，应先调用 `subscribeUsersStatus()`。

```ts
async function refreshSubscribedStatus() {
  try {
    const { data: subscribedStates } = await OpenIM.getSubscribeUsersStatus();
    mergePresence(subscribedStates);
    return subscribedStates;
  } catch (error) {
    console.error('getSubscribeUsersStatus failed', { error });
    throw error;
  }
}
```

需要指定一组用户并持续接收状态变化时，使用 `subscribeUsersStatus(userIDs)`；需要刷新当前账号已经订阅的用户状态时，使用 `getSubscribeUsersStatus()`。

## 取消订阅

用户离开页面、列表翻页、搜索结果变化或组件卸载时，应取消对已离开可见范围用户的订阅。这样可以减少无效推送，也能避免已经卸载的页面继续更新状态。

```ts
async function unsubscribeUserStatus(userIDs: string[]) {
  const targetUserIDs = uniqueUserIDs(userIDs);

  if (targetUserIDs.length === 0) {
    return [];
  }

  try {
    const { data } = await OpenIM.unsubscribeUsersStatus(targetUserIDs);

    for (const userID of targetUserIDs) {
      presenceByUserID.delete(userID);
    }

    return data;
  } catch (error) {
    console.error('unsubscribeUsersStatus failed', { error, targetUserIDs });
    throw error;
  }
}

await unsubscribeUserStatus(['user-a', 'user-b']);
```

可将 `UserOnlineState.status` 与 `OnlineState.Online`、`OnlineState.Offline` 比较。`platformIDs` 可能为空，即使没有具体平台信息，界面也应能正常展示在线状态。退出登录或销毁统一用户状态层时，调用本页示例中的 `disposeUserStatusEvents()` 清理事件。
