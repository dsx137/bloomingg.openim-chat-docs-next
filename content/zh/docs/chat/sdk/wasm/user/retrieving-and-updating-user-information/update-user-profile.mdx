本页只处理当前登录用户的资料。调用 `getSelfUserInfo()` 获取 `SelfUserInfo`，调用 `setSelfInfo()` 更新当前用户可自行维护的昵称、头像、扩展字段和全局消息接收选项。其他用户的账号级公开资料见[获取指定用户资料](/sdk/wasm/user/retrieving-users/retrieve-users)。

由管理员修改用户资料、维护组织通讯录或同步应用用户目录时，应在可信后端完成权限校验，再调用服务端能力。

## 读取当前用户资料

完成 SDK 初始化并调用 `login()` 后，使用 `getSelfUserInfo()` 获取当前账号资料。这些数据可用于初始化个人资料页、设置页中的头像和昵称，以及账号切换后的本地状态。

```ts
async function loadCurrentUser() {
  try {
    const { data } = await OpenIM.getSelfUserInfo();
    setCurrentUser(data);
    return data;
  } catch (error) {
    console.error('getSelfUserInfo failed', { error });
    throw error;
  }
}

const me = await loadCurrentUser();
```

## 更新当前用户资料

`setSelfInfo()` 只能更新当前登录用户。用户通常可以自行更新 `nickname`、`faceURL` 和 `ex`。提交时只传发生变化的字段；头像地址应使用可信的上传结果，结构化扩展数据应先序列化为 JSON 字符串。

```ts
type EditableSelfProfile = {
  nickname?: string;
  faceURL?: string;
  ex?: string;
};

async function updateMyProfile(input: EditableSelfProfile) {
  const payload: EditableSelfProfile = {};

  if (input.nickname !== undefined) {
    payload.nickname = input.nickname.trim();
  }

  if (input.faceURL !== undefined) {
    payload.faceURL = input.faceURL.trim();
  }

  if (input.ex !== undefined) {
    payload.ex = input.ex;
  }

  if (Object.keys(payload).length === 0) {
    return;
  }

  try {
    await OpenIM.setSelfInfo(payload);
  } catch (error) {
    console.error('setSelfInfo failed', { error });
    throw error;
  }

  return loadCurrentUser();
}
```

`EditableSelfProfile` 只包含昵称、头像和扩展字段，避免把 `userID`、创建时间、全局消息接收设置等其他 `SelfUserInfo` 字段误传给普通资料编辑表单。

示例在资料写入成功后，才调用 `loadCurrentUser()` 重新读取资料，并为两个步骤分别处理错误。这样，重新读取失败时不会被误报为写入失败。写入成功但读取失败时，可以保留提交成功状态，并提示用户重新刷新。

### 更新扩展字段

`SelfUserInfo.ex` 是完整字符串，调用 `setSelfInfo({ ex })` 会整体覆盖旧值，不会自动合并 JSON 字段。多个业务模块共享该字段时，应先读取当前值，处理空值和解析失败，再合并各自命名空间并写回完整结果。

```ts
function parseProfileExtra(ex?: string): Record<string, unknown> {
  if (!ex) return {};

  try {
    return JSON.parse(ex);
  } catch {
    return {};
  }
}

async function updateProfileExtra(patch: Record<string, unknown>) {
  const { data: currentUser } = await OpenIM.getSelfUserInfo();
  const currentExtra = parseProfileExtra(currentUser.ex);

  await OpenIM.setSelfInfo({
    ex: JSON.stringify({ ...currentExtra, ...patch }),
  });
}
```

`ex` 只适合保存轻量、敏感度低且与聊天展示直接相关的数据。组织、权限、风控、审计、搜索索引和后台管理字段应由可信后端维护；多端并发写入需要由业务后端提供版本或冲突控制。

### 设置全局消息接收选项

全局消息接收选项也是当前用户资料的一部分，使用 `setSelfInfo()` 写入 `globalRecvMsgOpt`。资料编辑表单和消息通知设置可以共用同一个 API，但应分别构造参数，避免保存昵称时意外覆盖接收选项。

```ts
import { MessageReceiveOptType } from '@openim/wasm-client-sdk';

await OpenIM.setSelfInfo({
  globalRecvMsgOpt: MessageReceiveOptType.NotNotify,
});
```

管理员编辑用户资料时，应由可信后端完成权限校验和审计。客户端设置页只处理当前登录用户自己的资料；修改其他账号资料时，应调用后端管理接口。

## 调用后的状态变化

- `getSelfUserInfo()` 成功后，以返回的 `SelfUserInfo` 建立当前账号资料快照。该查询不会触发资料更新事件。
- `setSelfInfo()` 成功表示本次资料设置已经完成。SDK 通过 `CbEvents.OnSelfInfoUpdated` 同步当前账号的最终 `SelfUserInfo`，资料设置页、用户菜单和全局账号状态应使用事件数据覆盖旧快照。
- 如果提交页面需要立即确认服务端最终值，可以在 `setSelfInfo()` 成功后重新调用 `getSelfUserInfo()`；重新查询失败不应被误报为资料写入失败。

当前账号只有一份 `SelfUserInfo`，切换账号时必须先清理旧账号资料，不能把上一个账号的事件数据写入当前状态。

## 监听资料变化

当前账号资料发生变化时，SDK 会触发 `CbEvents.OnSelfInfoUpdated`。资料设置页、用户菜单和全局用户状态层可以监听该事件，并用新的 `SelfUserInfo` 更新本地状态。

```ts
function handleSelfInfoUpdated({ data }) {
  setCurrentUser(data);
}

OpenIM.on(CbEvents.OnSelfInfoUpdated, handleSelfInfoUpdated);

function disposeSelfInfoEvents() {
  OpenIM.off(CbEvents.OnSelfInfoUpdated, handleSelfInfoUpdated);
}
```

事件也可能来自当前账号的其他登录端。页面卸载、退出登录或切换账号时调用 `disposeSelfInfoEvents()`；事件未到达时，再通过 `getSelfUserInfo()` 校准当前账号资料。

其他用户公开资料的读取方式见[获取指定用户资料](/sdk/wasm/user/retrieving-users/retrieve-users)，在线状态见[获取用户在线状态](/sdk/wasm/user/retrieving-and-updating-user-information/retrieve-the-online-status-of-a-user)。
