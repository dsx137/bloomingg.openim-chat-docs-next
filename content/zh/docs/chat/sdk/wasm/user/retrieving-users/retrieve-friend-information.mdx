已经知道一个或多个好友的 `userID`，并且只需要刷新这些好友的关系资料时，使用 `getSpecifiedFriendsInfo()`。该接口适合好友资料卡、联系人详情和好友设置等按需查询场景，无需为少量用户重新获取整页好友列表。

## 获取指定好友资料

调用 `getSpecifiedFriendsInfo()`，在 `friendUserIDList` 中传入要查询的好友用户 ID。`filterBlack` 决定是否过滤当前用户黑名单中的用户，普通好友资料页通常传 `true`。

```ts
const { data: friends } = await OpenIM.getSpecifiedFriendsInfo({
  friendUserIDList: ['user_a', 'user_b'],
  filterBlack: true,
});

const friendsByID = new Map(friends.map((friend) => [friend.userID, friend]));
```

Promise 成功后，`data` 是匹配到的 `FriendUserItem[]`。页面常用字段包括 `userID`、`nickname`、`faceURL`、`remark`、`isPinned`、`ex` 和 `attachedInfo`。

如果返回数据为空，通常表示目标用户不是当前用户的好友，或已被 `filterBlack` 过滤。展示好友候选人、陌生人资料卡或业务搜索结果时，应使用 [按用户 ID 查询公开资料](/sdk/wasm/user/retrieving-users/retrieve-users) 中的 `getUsersInfo()`。

## 检查好友关系

调用 `checkFriend()`，可以判断当前用户是否已与一组目标用户建立好友关系。该接口适合更新资料卡中的关系标识、加好友按钮状态，以及发送申请前的校验。需要展示好友备注、头像或置顶状态时，仍应使用 `getSpecifiedFriendsInfo()`。

```ts
const { data: checks } = await OpenIM.checkFriend(['user_a', 'user_b']);

const friendStateByID = new Map(
  checks.map((item) => [item.userID, item.result])
);
```

`result` 为 `1` 表示目标用户是当前用户的好友，为 `0` 表示不是好友。目标用户位于当前用户的黑名单中时也会返回非好友；如果还需要区分黑名单状态，应另行查询黑名单。

## 调用结果与关系变化

`getSpecifiedFriendsInfo()` 和 `checkFriend()` 成功后，以返回数组更新本次查询结果。这两个查询方法只读取当前关系快照，不应描述为触发好友事件。建立、删除好友关系或修改关系资料后的增量事件统一由[更新或删除好友](/sdk/wasm/user/managing-friends/update-or-delete-friends)处理。页面首次打开或用户主动刷新时，再调用 `getSpecifiedFriendsInfo()` 获取指定好友的当前资料；重新登录后的变化由事件同步。
