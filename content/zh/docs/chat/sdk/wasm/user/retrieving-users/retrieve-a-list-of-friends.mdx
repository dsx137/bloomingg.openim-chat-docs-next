好友列表可用于构建通讯录、联系人选择器和好友关系缓存。所有场景都必须调用 `getFriendListPage()` 分页读取 `FriendUserItem`，不要一次加载完整好友列表。

## 分页读取好友列表

调用 `getFriendListPage()`，按 `offset` 和 `count` 分页读取当前用户的好友记录。即使好友数量暂时不多，也必须保留分页状态，避免好友增长后出现长时间阻塞和过量内存占用。

### 参数说明

| 参数名        | 类型      | 是否必填 | 说明                                                      |
| ------------- | --------- | -------- | --------------------------------------------------------- |
| `offset`      | `number`  | 必填     | 起始偏移量。第一页传 `0`。                                |
| `count`       | `number`  | 必填     | 本次读取数量。                                            |
| `filterBlack` | `boolean` | 可选     | 是否过滤当前用户黑名单中的用户。普通通讯录通常传 `true`。 |

```ts
const pageSize = 50;

const { data: friends } = await OpenIM.getFriendListPage(
  {
    offset: 0,
    count: pageSize,
    filterBlack: true,
  }
);

const rows = friends.map((friend) => ({
  userID: friend.userID,
  displayName: friend.remark || friend.nickname || friend.userID,
  avatar: friend.faceURL,
  pinned: friend.isPinned,
}));
```

### 返回结果

Promise 成功后，`data` 是当前页的 `FriendUserItem[]`。页面常用字段如下：

| 字段           | 类型      | 说明                         |
| -------------- | --------- | ---------------------------- |
| `userID`       | `string`  | 好友用户 ID。                |
| `nickname`     | `string`  | 好友账号级公开昵称。         |
| `faceURL`      | `string`  | 好友账号级公开头像。         |
| `remark`       | `string`  | 当前用户为该好友设置的备注。 |
| `isPinned`     | `boolean` | 当前用户是否置顶该好友。     |
| `attachedInfo` | `string`  | SDK 返回的附加信息。         |
| `ex`           | `string`  | 好友关系扩展字段。           |

## 分页状态

加载下一页时，把 `offset` 设为当前已加载的好友数量。若本次返回数量小于 `count`，通常表示已经没有更多好友。

```ts
function getNextFriendOffset(currentFriends: FriendUserItem[]) {
  return currentFriends.length;
}

function hasMoreFriends(lastPage: FriendUserItem[], pageSize: number) {
  return lastPage.length === pageSize;
}
```

## 搜索好友

在通讯录、转发选择器或单聊联系人选择器中，可以调用 `searchFriends()` 搜索当前用户的好友。返回结果仍是好友关系数据，展示名、头像和关系字段的处理方式应与好友列表保持一致。

### 参数说明

| 参数名             | 类型       | 是否必填 | 说明                                                               |
| ------------------ | ---------- | -------- | ------------------------------------------------------------------ |
| `keywordList`      | `string[]` | 必填     | 搜索关键词。当前只支持一个非空关键词，即传入只包含一个元素的数组。 |
| `isSearchUserID`   | `boolean`  | 必填     | 是否匹配好友的 `userID`。                                          |
| `isSearchNickname` | `boolean`  | 必填     | 是否匹配好友账号级昵称。                                           |
| `isSearchRemark`   | `boolean`  | 必填     | 是否匹配当前用户设置的好友备注。                                   |

```ts
async function searchFriends(keyword: string) {
  const normalizedKeyword = keyword.trim();

  if (!normalizedKeyword) {
    return [];
  }

  const { data: friends } = await OpenIM.searchFriends({
    keywordList: [normalizedKeyword],
    isSearchUserID: true,
    isSearchNickname: true,
    isSearchRemark: true,
  });

  return friends.map((friend) => ({
    userID: friend.userID,
    displayName: friend.remark || friend.nickname || friend.userID,
    avatar: friend.faceURL,
  }));
}
```

### 返回结果

Promise 成功后，`data` 是匹配当前关键词的 `SearchedFriendsInfo[]`。每项包含 `FriendUserItem` 的字段，并额外提供 `relationship` 表示当前用户与结果用户的关系类型。展示联系人名称时，优先使用 `remark`，再依次回退到 `nickname` 和 `userID`。

如果要搜索并添加新好友，通常先通过业务后端或精确的 `userID` 找到候选用户，再通过 [获取指定用户资料](/sdk/wasm/user/retrieving-users/retrieve-users) 展示用户信息，最后进入好友申请流程。

## 列表如何更新

首次进入好友列表时，从 `offset: 0` 读取第一页，并用返回结果初始化列表。加载更多时，把新一页按 `userID` 追加到已有列表中。

`searchFriends()` 返回的是当前关键词的搜索结果，只用于展示搜索界面，不要用它覆盖完整的好友列表。

添加、删除好友或好友资料发生变化时，通过[更新或删除好友](/sdk/wasm/user/managing-friends/update-or-delete-friends)页面介绍的事件更新本地好友数据。如果列表正在分页，好友数量变化可能使原来的 `offset` 不再准确，此时应清空已加载的数据，并从第一页重新读取。

重新登录后，好友关系变化会通过好友事件同步到本地列表；首次进入页面时再调用分页查询建立列表快照。
