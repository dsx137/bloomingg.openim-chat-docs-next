好友备注、置顶状态和好友关系扩展字段都属于当前用户维护的好友关系数据。调用 `updateFriends()` 更新这些字段，调用 `deleteFriend()` 删除当前用户与目标用户的好友关系。

## 更新好友关系资料

调用 `updateFriends()` 可以更新好友备注、置顶状态和关系扩展字段。一次调用会把相同的字段值应用到 `friendUserIDs` 指定的所有好友；如果要为不同好友设置不同备注，应分别调用。

### 参数说明

| 参数名          | 类型       | 是否必填 | 说明                                         |
| --------------- | ---------- | -------- | -------------------------------------------- |
| `friendUserIDs` | `string[]` | 必填     | 要更新的好友用户 ID 列表。                   |
| `remark`        | `string`   | 可选     | 好友备注。                                   |
| `isPinned`      | `boolean`  | 可选     | 是否置顶该好友。                             |
| `ex`            | `string`   | 可选     | 好友关系扩展字段；本次传入值会整体覆盖原值。 |

```ts
async function updateFriendRemark(friendUserID: string, remark: string) {
  try {
    await openimsdk.updateFriends({
      friendUserIDs: [friendUserID],
      remark: remark.trim(),
    });
  } catch (error) {
    console.error('updateFriends failed', { error, friendUserID });
    throw error;
  }
}
```

### 返回结果

Promise 成功表示好友资料更新请求已经完成。好友列表和联系人详情再通过 `OnFriendInfoChanged` 合并增量，必要时重新查询相关好友资料校准状态。

`ex` 是一个完整字符串，不会按 JSON 字段增量合并。多个业务模块共用该字段时，应先读取原值，合并各自命名空间后再写回完整结果。还要处理并发写入造成的覆盖；复杂场景建议由业务后端统一维护。

### 备注、置顶和扩展字段

`updateFriends()` 统一处理三类好友关系设置：

- 传入 `remark` 修改好友备注；不同好友需要不同备注时应分别调用。
- 传入 `isPinned` 设置好友是否置顶，可以一次更新多个好友。
- 传入 `ex` 更新好友关系扩展字符串；该字段按完整值覆盖，不会自动合并 JSON。

一次批量调用会把相同字段值应用到全部 `friendUserIDs`。提交前应确认目标用户范围，避免把同一备注或扩展数据误写给多个好友。

## 删除好友关系

调用 `deleteFriend()` 删除当前用户与目标用户的好友关系。删除后，应同步更新好友列表、好友资料卡和相关操作入口。

```ts
async function deleteFriend(friendUserID: string) {
  try {
    await openimsdk.deleteFriend(friendUserID);
  } catch (error) {
    console.error('deleteFriend failed', { error, friendUserID });
    throw error;
  }
}
```

删除成功后，可以先从本地好友集合中移除目标 `userID`，再通过事件或重新查询第一页来校准状态。资料卡中的“发消息”“修改备注”等好友操作也应同步刷新。

## 调用后的状态变化

- `updateFriends()` 成功表示更新请求已经完成；最终好友资料通过 `CbEvents.OnFriendInfoChanged` 同步，事件的 `data` 是变化后的 `FriendUserItem`，按 `userID` 覆盖本地记录。
- `deleteFriend()` 成功表示好友关系已经删除；通过 `CbEvents.OnFriendDeleted` 从好友列表和关系缓存中移除对应 `userID`。
- `CbEvents.OnFriendAdded` 用于同步新建立的好友关系，通常来自好友申请被同意等流程，不应写成由 `updateFriends()` 或 `deleteFriend()` 触发。

## 监听好友事件

好友关系发生变化后，SDK 会通知当前账号的各个登录端：

| 事件                           | 适用场景                           |
| ------------------------------ | ---------------------------------- |
| `CbEvents.OnFriendAdded`       | 好友关系建立。                     |
| `CbEvents.OnFriendInfoChanged` | 好友备注、置顶状态或扩展字段变化。 |
| `CbEvents.OnFriendDeleted`     | 好友关系被删除。                   |

```ts
function handleFriendInfoChanged({ data }) {
  mergeFriend(data);
}

function handleFriendAdded({ data }) {
  mergeFriend(data);
}

function handleFriendDeleted({ data }) {
  removeFriend(data.userID);
  resetContactDetailIfNeeded(data.userID);
}

openimsdk.on(CbEvents.OnFriendAdded, handleFriendAdded);
openimsdk.on(CbEvents.OnFriendInfoChanged, handleFriendInfoChanged);
openimsdk.on(CbEvents.OnFriendDeleted, handleFriendDeleted);

function disposeFriendEvents() {
  openimsdk.off(CbEvents.OnFriendAdded, handleFriendAdded);
  openimsdk.off(CbEvents.OnFriendInfoChanged, handleFriendInfoChanged);
  openimsdk.off(CbEvents.OnFriendDeleted, handleFriendDeleted);
}
```

如果更新或删除后没有收到预期事件，可以调用 `getSpecifiedFriendsInfo()` 或 `checkFriend()` 重新确认目标用户的关系状态。
