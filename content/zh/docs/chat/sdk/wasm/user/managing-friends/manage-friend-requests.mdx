好友申请用于建立需要对方确认的好友关系。当前用户既可以向其他用户发送申请，也可以查询收到和发出的申请，并处理自己收到的申请。

## 发送好友申请

调用 `addFriend()` 向目标用户发送好友申请。`toUserID` 是申请接收方的 OpenIM 用户 ID；`reqMsg` 是申请附言，没有附言时也必须传空字符串。产品层还应限制附言长度并校验内容。

```ts
async function sendFriendRequest(toUserID: string, reqMsg: string) {
  try {
    await OpenIM.addFriend({
      toUserID,
      reqMsg,
    });
  } catch (error) {
    console.error('addFriend failed', { error, toUserID });
    throw error;
  }
}

await sendFriendRequest('user_b', '你好，我是 Alex');
```

Promise 成功表示服务端已收到申请，不表示对方已经同意。申请状态通过申请列表查询和好友申请事件同步。

## 查询申请列表

### 查询收到的申请

当前用户作为接收方时，调用 `getFriendApplicationListAsRecipient()` 获取其他用户发来的申请。查询待处理列表时，通常传入 `handleResults: [ApplicationHandleResult.Unprocessed]`；查询历史记录时，可以按产品需要在 `handleResults` 中加入 `ApplicationHandleResult.Agree` 或 `ApplicationHandleResult.Reject`。

#### 参数说明

| 参数名 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `handleResults` | `number[]` | 可选 | 要读取的处理状态。待处理申请传 `[ApplicationHandleResult.Unprocessed]`；历史记录可加入 `Agree` 或 `Reject`。 |
| `offset` | `number` | 可选 | 起始偏移量，第一页传 `0`。 |
| `count` | `number` | 可选 | 本次读取数量。 |

```ts
import { ApplicationHandleResult } from '@openim/wasm-client-sdk';

async function loadReceivedFriendRequests(offset = 0, count = 20) {
  try {
    const { data } = await OpenIM.getFriendApplicationListAsRecipient({
      handleResults: [ApplicationHandleResult.Unprocessed],
      offset,
      count,
    });

    return data ?? [];
  } catch (error) {
    console.error('getFriendApplicationListAsRecipient failed', { error, offset, count });
    throw error;
  }
}
```

Promise 成功后，`data` 是当前页的 `FriendApplicationItem[]`，可用于展示收到的待处理申请、历史记录和申请详情。

### 查询发出的申请

当前用户作为申请方时，调用 `getFriendApplicationListAsApplicant()` 获取自己发出的申请。第一页传 `offset: 0`，`count` 表示本次读取数量。

```ts
async function loadSentFriendRequests(offset = 0, count = 20) {
  try {
    const { data } = await OpenIM.getFriendApplicationListAsApplicant({
      offset,
      count,
    });

    return data ?? [];
  } catch (error) {
    console.error('getFriendApplicationListAsApplicant failed', { error, offset, count });
    throw error;
  }
}
```

Promise 成功后，`data` 是当前页已发出的 `FriendApplicationItem[]`。

申请列表的分页方式与好友列表相同：第一页传 `offset: 0`，后续请求把 `offset` 设为当前已加载的申请数量。返回数量小于 `count` 时，表示该列表已没有更多记录。收到申请新增、处理或删除事件后，如果列表仍在继续分页，应重置分页状态并从第一页重新加载，避免旧 `offset` 跳过或重复读取记录。

## 未处理申请数量

调用 `getFriendApplicationUnhandledCount()` 获取未处理申请数。`time` 通常传入上次查看申请列表的时间，用于统计该时间点之后的未处理申请。

```ts
async function loadUnhandledFriendRequestCount(lastViewedAt: number) {
  try {
    const { data } = await OpenIM.getFriendApplicationUnhandledCount({
      time: lastViewedAt,
    });

    return data ?? 0;
  } catch (error) {
    console.error('getFriendApplicationUnhandledCount failed', { error, lastViewedAt });
    throw error;
  }
}
```

Promise 成功后，`data` 是符合统计条件的未处理申请数量。

## 处理好友申请

调用 `acceptFriendApplication()` 同意申请，调用 `refuseFriendApplication()` 拒绝申请。两个方法都接收申请方的 `toUserID` 和处理附言 `handleMsg`；没有附言时，`handleMsg` 仍需传空字符串。

```ts
async function acceptFriendRequest(applicantUserID: string) {
  try {
    await OpenIM.acceptFriendApplication({
      toUserID: applicantUserID,
      handleMsg: '已通过',
    });
  } catch (error) {
    console.error('acceptFriendApplication failed', { error, applicantUserID });
    throw error;
  }
}

async function refuseFriendRequest(applicantUserID: string) {
  try {
    await OpenIM.refuseFriendApplication({
      toUserID: applicantUserID,
      handleMsg: '暂不添加',
    });
  } catch (error) {
    console.error('refuseFriendApplication failed', { error, applicantUserID });
    throw error;
  }
}
```

Promise 成功表示对应的处理请求已经完成。同意申请后，新的好友关系还会通过好友事件同步。

## 删除好友申请记录

`deleteFriendRequests()` 按申请双方的用户 ID 批量删除申请记录：

```ts
await OpenIM.deleteFriendRequests([
  {
    fromUserID: application.fromUserID,
    toUserID: application.toUserID,
  },
]);
```

数组中的每一项都要同时提供原申请的 `fromUserID` 和 `toUserID`。Promise 成功表示这些申请记录的删除请求已经完成。

删除记录不等于拒绝申请，也不会删除已经建立的好友关系。待处理申请应先使用 `acceptFriendApplication()` 或 `refuseFriendApplication()` 表达处理结果；已建立好友关系应使用 `deleteFriend()` 解除。

## 调用后的状态变化

| API | Promise 成功后 | 相关事件和处理方式 |
| --- | --- | --- |
| `addFriend()` | 服务端已接收好友申请，不代表双方已经成为好友。 | 通过 `OnFriendApplicationAdded` 把申请写入收到或发出的申请列表。 |
| `acceptFriendApplication()` | 申请处理请求已成功。 | 通过 `OnFriendApplicationAccepted` 更新申请状态；新好友关系由 `OnFriendAdded` 同步。 |
| `refuseFriendApplication()` | 拒绝处理请求已成功。 | 通过 `OnFriendApplicationRejected` 更新申请状态并移出待处理列表。 |
| `deleteFriendRequests()` | 指定申请记录的删除请求已完成。 | 通过 `OnFriendApplicationDeleted` 移除对应记录。 |

申请事件中的 `data` 是 `FriendApplicationItem`。客户端应使用申请双方的用户 ID 定位记录，不能只按展示名称或当前列表位置更新。查询申请列表和未处理数量的方法只返回调用时的快照，不会触发申请事件。

## 监听申请事件

建议通过以下事件同步好友申请列表：

- `CbEvents.OnFriendApplicationAdded`
- `CbEvents.OnFriendApplicationAccepted`
- `CbEvents.OnFriendApplicationRejected`
- `CbEvents.OnFriendApplicationDeleted`

事件回调通过 `data` 携带本次变化。申请发起者和接收者都会收到 `OnFriendApplicationAdded`，因此不能把每个新增事件都计入“收到的申请”和未处理数量。应先获取当前登录用户 ID，再通过 `data.toUserID === currentUserID` 判断当前用户是否为接收方；否则，应把该申请写入已发送列表。

事件处理器应直接根据事件数据更新本地申请列表和未处理数量。好友关系建立后的列表同步由 `OnFriendAdded` 的主归属页[更新或删除好友](/sdk/wasm/user/managing-friends/update-or-delete-friends)处理。首次进入页面或用户主动刷新时，再重新查询申请列表建立完整快照；重新登录后的变化由事件同步。

```ts
const currentUserID = getCurrentUserID();

function handleFriendApplicationAdded({ data }) {
  if (data.toUserID === currentUserID) {
    upsertReceivedFriendApplication(data);
    increaseUnhandledFriendApplicationCount(data);
    return;
  }

  upsertSentFriendApplication(data);
}

function handleFriendApplicationAccepted({ data }) {
  markFriendApplicationAccepted(data);
  removePendingFriendApplication(data);
}

function handleFriendApplicationRejected({ data }) {
  markFriendApplicationRejected(data);
  removePendingFriendApplication(data);
}

function handleFriendApplicationDeleted({ data }) {
  removeFriendApplication(data);
}

OpenIM.on(CbEvents.OnFriendApplicationAdded, handleFriendApplicationAdded);
OpenIM.on(CbEvents.OnFriendApplicationAccepted, handleFriendApplicationAccepted);
OpenIM.on(CbEvents.OnFriendApplicationRejected, handleFriendApplicationRejected);
OpenIM.on(CbEvents.OnFriendApplicationDeleted, handleFriendApplicationDeleted);

function disposeFriendApplicationEvents() {
  OpenIM.off(CbEvents.OnFriendApplicationAdded, handleFriendApplicationAdded);
  OpenIM.off(CbEvents.OnFriendApplicationAccepted, handleFriendApplicationAccepted);
  OpenIM.off(CbEvents.OnFriendApplicationRejected, handleFriendApplicationRejected);
  OpenIM.off(CbEvents.OnFriendApplicationDeleted, handleFriendApplicationDeleted);
}
```
