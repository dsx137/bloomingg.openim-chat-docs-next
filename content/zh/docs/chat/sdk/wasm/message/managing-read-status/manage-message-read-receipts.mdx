OpenIMSDK 的单聊和群聊采用不同的已读回执模型。单聊只需要判断对方是否已读某些消息；群聊还可以按单条消息查询已读和未读成员，并统计成员级阅读情况。

## 上报单聊已读回执

用户打开单聊并阅读完可见消息后，阅读端调用 `markConversationMessageAsRead()`：

```ts
await openimsdk.markConversationMessageAsRead(conversationID);
```

该方法不接收消息 ID。SDK 会根据该单聊会话中由对方发送的未读消息生成已读上报，同时清理当前用户的会话未读数。Promise 成功表示本次已读处理已经完成，不表示消息发送方的界面已经更新；发送方通过 `OnRecvC2CReadReceipt` 接收回执，再把 `msgIDList` 对应的消息更新为对端已读。

不要在只预览通知、消息仍未展示或应用处于后台时提前调用。会话未读数的维护方式见[管理会话已读状态](/sdk/wasm/conversation/managing-conversations/manage-read-status)。

## 查询群聊已读成员

在群聊中，消息已读状态用于判断某条消息已经被哪些成员阅读。OpenIMClientSDK 以单条群消息为维度，通过 `getGroupMessageReaderList()` 查询已读或未读成员列表。

查询前需要准备消息所在的 `conversationID` 和目标消息的 `clientMsgID`。`GroupMessageReaderFilter.Read` 查询已读成员，`GroupMessageReaderFilter.UnRead` 查询未读成员。

```ts
import { GroupMessageReaderFilter } from '@openim/wasm-client-sdk';

const { data: readMembers } = await openimsdk.getGroupMessageReaderList({
  conversationID,
  clientMsgID,
  filter: GroupMessageReaderFilter.Read,
  offset: 0,
  count: 50,
});

renderReadMembers(readMembers);
```

### 参数说明

| 参数             | 类型                       | 是否必填 | 说明                                 |
| ---------------- | -------------------------- | -------- | ------------------------------------ |
| `conversationID` | `string`                   | 必填     | 群消息所在的会话 ID。                |
| `clientMsgID`    | `string`                   | 必填     | 要查询读状态的消息 ID。              |
| `filter`         | `GroupMessageReaderFilter` | 必填     | `Read` 查询已读成员，`UnRead` 查询未读成员。 |
| `offset`         | `number`                   | 必填     | 分页起始位置，第一页传 `0`。         |
| `count`          | `number`                   | 必填     | 本次查询数量。                       |

### 查询未读成员

同一个 API 可以查询仍未读该消息的群成员。未读列表适合消息详情页、发送后查看成员阅读状态，或管理员查看群内消息阅读情况。

```ts
const { data: unreadMembers } = await openimsdk.getGroupMessageReaderList({
  conversationID,
  clientMsgID,
  filter: GroupMessageReaderFilter.UnRead,
  offset: 0,
  count: 50,
});

renderUnreadMembers(unreadMembers);
```

返回值是当前页的 `GroupMemberItem[]`。如果需要准确的总数，继续分页直到返回数量小于 `count`；如果界面只需要当前页列表或入口处显示的计数，可以按页面大小增量加载。

两个查询示例的 Promise 成功后直接使用 `data` 建立已读或未读成员快照。读取成员列表不会触发已读回执事件，列表项按 `groupID:userID` 去重。

## 上报群聊已读回执

成员阅读群消息后，阅读端调用 `sendGroupMessageReadReceipt()`，一次上报一条或多条群消息的成员级已读回执：

```ts
await openimsdk.sendGroupMessageReadReceipt({
  conversationID,
  clientMsgIDList: visibleUnreadGroupMessageIDs,
});
```

Promise 成功表示服务端已接受本次回执上报，不等于其他客户端的已读界面已经更新。群聊中调用 `markConversationMessageAsRead()` 只负责清理当前用户的会话未读数；需要记录某条群消息由哪些成员读过时，还要调用 `sendGroupMessageReadReceipt()`。

## 接收单聊和群聊回执

消息发送方或其他在线端通过回执事件更新消息状态。单聊监听 `OnRecvC2CReadReceipt`，群聊监听 `OnRecvGroupReadReceipt`：

```ts
import { CbEvents } from '@openim/wasm-client-sdk';

function handleGroupReadReceipt({ data }) {
  for (const item of data.groupMessageReadInfo) {
    updateMessageReadState(item.clientMsgID, {
      readCount: item.hasReadCount,
      unreadCount: item.unreadCount,
      readMembers: item.readMembers,
    });
  }
}

function handleC2CReadReceipt({ data }) {
  for (const receipt of data) {
    const conversationID = resolveC2CConversationID(receipt);
    for (const clientMsgID of receipt.msgIDList) {
      markPeerRead(conversationID, clientMsgID, receipt.readTime);
    }
  }
}

openimsdk.on(CbEvents.OnRecvGroupReadReceipt, handleGroupReadReceipt);
openimsdk.on(CbEvents.OnRecvC2CReadReceipt, handleC2CReadReceipt);

function removeReadReceiptListeners() {
  openimsdk.off(CbEvents.OnRecvGroupReadReceipt, handleGroupReadReceipt);
  openimsdk.off(CbEvents.OnRecvC2CReadReceipt, handleC2CReadReceipt);
}
```

`OnRecvGroupReadReceipt` 的 `data` 是 `GroupMessageReceiptInfo`，先按 `conversationID` 定位会话，再按每项的 `clientMsgID` 更新消息。`OnRecvC2CReadReceipt` 的 `data` 是 `ReceiptInfo[]`；按 `userID` 和 `sessionType` 定位单聊会话，再将 `msgIDList` 中的消息更新为对端已读。单聊回执不包含群成员列表，也没有已读人数或未读人数，不要套用群消息的成员级统计结构。

本页是两个已读回执事件的完整监听示例归属页。组件卸载、退出登录或切换账号时调用 `removeReadReceiptListeners()`；需要显示当前读状态时，再查询群消息成员列表或结合本地单聊消息状态获取快照。

群消息读状态和会话未读数不是同一个概念。成员列表查询建立某条群消息当前的已读或未读快照，后续回执事件提供增量变化；需要重新展示完整成员列表时，可以再次从第一页查询。单聊没有成员列表查询，直接根据 `OnRecvC2CReadReceipt` 更新消息的对端已读状态。
