消息置顶用于固定展示会话中的重要消息。它与会话列表置顶不是同一个状态：消息置顶使用 `setConversationPinnedMsg()`，会话列表置顶使用 `setConversation({ isPinned: true })`。

## 获取置顶消息

`getConversationPinnedMsg()` 接收 `conversationID`，返回 `MessageItem[]`：

```ts
const { data: pinnedMessages } = await OpenIM.getConversationPinnedMsg(conversationID);

renderPinnedMessages(pinnedMessages);
```

Promise 成功后直接使用 `data` 建立该会话的置顶消息快照；查询本身不会触发置顶事件。列表中的消息仍使用 `clientMsgID` 去重。消息被删除、撤回或修改后，应结合对应消息事件更新展示，不要永久缓存初次查询结果。

## 置顶或取消置顶

把 `pinned` 设为 `true` 表示置顶，设为 `false` 表示取消置顶：

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `conversationID` | `string` | 必填 | 消息所属的会话 ID。 |
| `clientMsgID` | `string` | 必填 | 要置顶或取消置顶的消息 ID。 |
| `pinned` | `boolean` | 必填 | `true` 表示置顶，`false` 表示取消置顶。 |

```ts
await OpenIM.setConversationPinnedMsg({
  conversationID,
  clientMsgID: message.clientMsgID,
  pinned: true,
});

await OpenIM.setConversationPinnedMsg({
  conversationID,
  clientMsgID: message.clientMsgID,
  pinned: false,
});
```

可置顶的消息类型、数量上限和操作权限由当前 OpenIM 服务端配置决定。客户端可以隐藏无权限入口，但不能用本地状态绕过服务端校验。

## 监听变化

`CbEvents.OnChangedPinnedMsg` 的 `data` 是 `PinnedMessageChangeData`，包含 `conversationID` 和 `msgs`。其中每个 `PinnedMessageDetail` 都包含置顶时间、操作用户 ID 和完整消息：

```ts
function handlePinnedMessagesChanged({ data }) {
  replacePinnedMessages(
    data.conversationID,
    data.msgs.map((item) => ({
      message: item.msg,
      pinnedTime: item.pinnedTime,
      operatorUserID: item.userID,
    }))
  );
}

OpenIM.on(CbEvents.OnChangedPinnedMsg, handlePinnedMessagesChanged);

function disposePinnedMessageEvents() {
  OpenIM.off(CbEvents.OnChangedPinnedMsg, handlePinnedMessagesChanged);
}
```

`setConversationPinnedMsg()` 的 Promise 成功表示服务端已完成本次置顶设置。`OnChangedPinnedMsg` 提供变化后的置顶消息详情快照，不是 Promise 的返回值；按 `conversationID` 替换对应会话的置顶集合，再按消息的 `clientMsgID` 渲染去重。

事件也可能来自其他客户端或其他有权限的成员。组件卸载、退出登录或切换账号时调用 `disposePinnedMessageEvents()`；需要显示当前会话的置顶消息时，再调用 `getConversationPinnedMsg()` 获取快照，重新登录后的变化由置顶消息事件同步。
