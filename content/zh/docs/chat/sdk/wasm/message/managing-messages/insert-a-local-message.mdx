本地插入消息用于迁移、系统提示、本地占位消息，或产品自定义的只读记录。它只写入当前客户端的本地存储，不会把消息发送给对端，也不会替代 `sendMessage()`。

## 单聊本地插入

`insertSingleMessageToLocalStorage()` 需要 `message`、`recvID` 和 `sendID`。

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `message` | `MessageItem` | 必填 | 要写入本地单聊记录的完整消息对象。 |
| `recvID` | `string` | 必填 | 单聊接收方用户 ID。 |
| `sendID` | `string` | 必填 | 单聊发送方用户 ID。 |

```ts
const { data: message } = await OpenIM.createTextMessage('本地提示消息');

await OpenIM.insertSingleMessageToLocalStorage({
  message,
  recvID: 'user_b',
  sendID: currentUserID,
});
```

## 群聊本地插入

`insertGroupMessageToLocalStorage()` 需要 `message`、`groupID` 和 `sendID`。

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `message` | `MessageItem` | 必填 | 要写入本地群聊记录的完整消息对象。 |
| `groupID` | `string` | 必填 | 目标群组 ID。 |
| `sendID` | `string` | 必填 | 消息发送方用户 ID。 |

```ts
const { data: message } = await OpenIM.createTextMessage('群内本地提示');

await OpenIM.insertGroupMessageToLocalStorage({
  message,
  groupID,
  sendID: currentUserID,
});
```

## 修改语音消息的本地内容

`setMessageLocalContent()` 用于修改当前设备保存的完整消息内容，主要配合语音转文字能力，把识别结果写入语音消息的 `soundElem.text`。它不是本地插入方法，也不是服务端消息编辑接口。

完整参数、`soundElem.text` 更新示例和本地生效范围见[将音频转为文字](/sdk/wasm/message/composing-messages/transcribe-audio)。只修改 `localEx` 时使用 `setMessageLocalEx()`，不要为本地界面扩展重写完整 `MessageItem`。

## 使用边界

- 不要用本地插入实现正常聊天发送。
- 不要把本地插入当作服务端通知。
- 不要用 `setMessageLocalContent()` 伪造服务端消息修改结果。
- 如果消息需要多端同步、离线投递或服务端审核，应使用 `sendMessage()` 或服务端接口。

## 调用结果

两个本地插入方法的 Promise 成功表示当前客户端的本地数据库操作已经完成。它们不会触发 `OnRecvNewMessages`，也不会通知其他用户或其他设备。调用端可以在 Promise 成功后按 `clientMsgID` 更新当前消息列表，或重新读取对应会话的历史消息校准本地快照。

`setMessageLocalContent()` 同样只写入当前设备，但它的主要组合流程和结果说明归入[将音频转为文字](/sdk/wasm/message/composing-messages/transcribe-audio)，本页不重复参数和示例。

创建待插入消息的 `createTextMessage()` 同样只通过 Promise 返回 `MessageItem`，不会触发消息事件。只有之后明确调用 `sendMessage()`，消息才进入服务端投递和接收事件流程。
