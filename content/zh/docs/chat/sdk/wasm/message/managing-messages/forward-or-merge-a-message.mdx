转发消息时，先基于已有 `MessageItem` 调用 `createForwardMessage()` 创建新消息，再调用 `sendMessage()` 发送到目标单聊或群聊。转发后的消息会生成新的消息 ID 和发送状态，不会修改原消息。

## 转发单条消息

文本、引用、表情、图片、音视频和文件等消息使用相同的转发流程。传给 `createForwardMessage()` 的原消息必须已经发送成功，即 `status` 为 `MessageStatus.Succeed`；发送中或发送失败的消息不能转发。

下面把一条已有消息转发到单聊。`sourceMessage` 可以来自当前消息列表、历史消息查询结果或新消息事件。

```ts
const { data: forwardMessage } = await OpenIM.createForwardMessage(sourceMessage);

const { data: sentMessage } = await OpenIM.sendMessage(
  {
    recvID: targetUserID,
    groupID: '',
    message: forwardMessage,
  }
);

appendMessage(sentMessage);
```

转发到群聊时，将 `recvID` 设为空字符串并填写 `groupID`。无论原消息是什么类型，都应发送 `createForwardMessage()` 返回的新 `MessageItem`，不要复用原消息的 `clientMsgID`，也不要把原消息对象直接插入目标会话。重新上传本地文件属于创建并发送一条新的文件消息，不属于转发。

## 多条消息

多选转发或聊天记录分享可以使用 `createMergerMessage()`。`messageList` 是要合并的原始消息列表，`title` 是合并消息标题，`summaryList` 用于展示合并卡片摘要。

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `messageList` | `MessageItem[]` | 必填 | 要合并的已发送消息。 |
| `title` | `string` | 必填 | 合并消息卡片的标题。 |
| `summaryList` | `string[]` | 必填 | 合并卡片展示的摘要列表。 |

```ts
const summaryList = selectedMessages.map((message) =>
  buildMessageSummary(message)
);

const { data: mergerMessage } = await OpenIM.createMergerMessage({
  messageList: selectedMessages,
  title: '聊天记录',
  summaryList,
});

await OpenIM.sendMessage({
  recvID: '',
  groupID: targetGroupID,
  message: mergerMessage,
});
```

`summaryList` 是合并卡片的展示摘要，不应直接使用 `MessageItem.content`，因为该字段可能是序列化后的内部内容。应按消息类型提取可展示文本，并对文件、自定义消息等类型提供安全的降级文案。

## 调用结果与消息接收

`createForwardMessage()` 和 `createMergerMessage()` 的 Promise 只返回新的待发送 `MessageItem`，不会修改原消息，也不会触发接收事件。`sendMessage()` 的 Promise 成功后，发送端直接使用返回的 `MessageItem` 按新的 `clientMsgID` 合并目标会话列表。

其他客户端通过 `OnRecvNewMessages` 接收转发或合并消息，事件 `data` 是 `MessageItem[]`。应先根据每条消息的路由字段确定会话，再按 `clientMsgID` 去重；完整监听、清理和离线补偿见[接收消息](/sdk/wasm/message/receiving-messages/receive-messages)。Promise 成功和远端事件到达是两个独立阶段，重新登录后的消息变化由事件同步。

调用失败时，在 `catch` 中记录错误对象和目标用户或群组 ID，并确认当前用户具备目标群聊的发言权限。
