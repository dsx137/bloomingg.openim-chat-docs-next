清理消息历史时，先确定产品操作针对单个会话，还是当前账号的全部会话。单个会话清理适用于单聊会话和群聊会话；账号级清理会覆盖当前账号的所有会话，不应作为单个聊天窗口的“清空”按钮操作。

## 清理单个会话的消息

对目标 `conversationID` 调用 `clearConversationAndDeleteAllMsg()` 后，当前登录用户的该会话消息历史会被清空；它不会删除会话记录或解散群组，也不会清除单聊对方或其他群成员保存的消息。

无论会话类型如何，清理历史时都直接使用 `ConversationItem.conversationID`，不需要分别维护单聊和群聊的清理逻辑。

```ts
async function clearConversationHistory(conversationID: string) {
  await OpenIM.clearConversationAndDeleteAllMsg(conversationID);

  removeMessagesFromConversation(conversationID);
  resetConversationPreview(conversationID);
}
```

如果业务入口只有目标用户 ID 或群组 ID，应先按对应会话类型取得 `ConversationItem`，再把其中的 `conversationID` 传给上述方法。获取单聊和群聊会话的方式见[获取会话](/sdk/wasm/conversation/retrieving-conversations/retrieve-conversations)，本页不重复展开会话 ID 的获取流程。

### 调用后的状态变化

`clearConversationAndDeleteAllMsg()` 的 Promise 成功后，可以按 `conversationID` 立即清空当前窗口中的消息快照。会话摘要、未读数和最后一条消息随后可能通过 `OnConversationChanged` 更新；事件 `data` 是 `ConversationItem[]`，按 `conversationID` 合并。完整监听和清理代码见[获取会话列表](/sdk/wasm/conversation/retrieving-conversations/retrieve-conversation-list)。需要显示当前会话摘要时，再查询该会话获取快照；重新登录后的变化由会话事件同步。

该方法适合“保留会话入口，只清空当前聊天窗口历史”的用户动作。若产品动作是“删除此聊天”，使用 `deleteConversationAndDeleteAllMsg(conversationID)` 删除当前用户的会话记录和消息，具体流程见[删除或清空会话](/sdk/wasm/conversation/managing-conversations/delete-or-clear-conversation)。

清空历史调用成功后，确认当前聊天窗口不再展示旧消息，刷新页面后旧消息不会从本地缓存恢复。清空后收到的新消息仍应通过新消息事件进入对应会话，并作为新的历史起点展示。

## 清理当前账号的全部消息

`deleteAllMsgFromLocal()` 和 `deleteAllMsgFromLocalAndSvr()` 都不接收参数，并且都会清理当前账号所有会话中的本地消息。两者的关键区别是是否同时删除当前账号保存在服务端的消息。

### 只清理本地全部消息

只需要重置当前浏览器中的消息数据，同时希望服务端仍保留历史消息时，调用 `deleteAllMsgFromLocal()`：

```ts
await OpenIM.deleteAllMsgFromLocal();
resetAllMessageSnapshots();
```

Promise 成功表示当前账号的本地消息已经清理完成。此时应清空内存中的全部消息快照，不要只重置当前打开的会话。由于服务端消息仍然保留，之后清除站点数据、重新安装应用或重建本地数据库时，旧消息仍可能重新同步到本地。

### 同时清理本地和服务端全部消息

需要永久清理当前账号的全部历史消息时，调用 `deleteAllMsgFromLocalAndSvr()`：

```ts
await OpenIM.deleteAllMsgFromLocalAndSvr();
resetAllMessageSnapshots();
```

该方法先删除当前账号保存在服务端的全部消息，再清理本地消息。Promise 成功后，即使清除站点数据、重新安装应用或重建本地数据库，这些旧消息也不会再次同步回来。

这里的“服务端全部消息”只指当前账号保存的消息数据，不会删除单聊对方或其他群成员已经保存的消息。SDK 没有为这两个账号级清理操作提供独立的消息删除事件；其他已登录设备如何刷新本地副本，应以实际同步结果为准，不能把当前调用端的 Promise 成功当成其他设备已经完成清理。

账号级清理可能同时改变会话摘要和未读数。当前页面的消息快照在 Promise 成功后直接重置；后续会话变化继续通过 `OnConversationChanged` 按 `conversationID` 合并，完整监听代码见[获取会话列表](/sdk/wasm/conversation/retrieving-conversations/retrieve-conversation-list)。
