在 OpenIM 中，删除影响消息是否仍对当前账号可见，不会让其他会话成员看到“消息已撤回”。需要展示撤回态时，使用[撤回消息](/sdk/wasm/message/managing-messages/revoke-a-message)。修改已发送消息内容见[修改消息](/sdk/wasm/message/managing-messages/modify-a-message)。

`deleteMessageFromLocalStorage()` 只删除当前设备上的本地消息；`deleteMessages()` 可以删除当前账号在服务端保存的消息，并通过 `isSync` 决定是否请求把删除动作同步到其他客户端；`deleteUserAllMessagesInConv()` 按用户批量删除会话中的消息。

## 只删除当前设备上的本地消息

如果用户动作是“仅从这台设备删除”，使用 `deleteMessageFromLocalStorage()`。它只修改当前设备使用的本地消息库，不删除服务端消息，也不会向当前账号的其他设备或会话中的其他用户同步删除动作。

```ts
await OpenIM.deleteMessageFromLocalStorage({
  conversationID,
  clientMsgID,
});

removeMessageFromCurrentList(clientMsgID);
```

卸载应用、清除站点数据或更换浏览器后，本地消息库会重新建立。因为服务端仍保留这条消息，重新登录并同步历史消息时，它还会再次出现。发送失败或尚未取得服务端序号的消息只能使用本地删除。

删除的是当前会话最后一条消息时，通过 `CbEvents.OnConversationChanged` 更新会话的最后一条消息、未读数和排序。

## 删除当前账号保存的消息

单条和批量删除统一使用 `deleteMessages()`。删除单条消息时，`clientMsgIDs` 只传一个消息 ID：

### 参数说明

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `conversationID` | `string` | 必填 | 消息所属的会话 ID。 |
| `clientMsgIDs` | `string[]` | 必填 | 同一会话内要删除的消息 ID；删除单条消息时只传一个元素。 |
| `isSync` | `boolean` | 必填 | 是否请求把删除动作同步到其他客户端。 |

```ts
await OpenIM.deleteMessages({
  conversationID,
  clientMsgIDs: [clientMsgID],
  isSync: false,
});

removeMessageFromCurrentList(clientMsgID);
```

`isSync: false` 会删除当前设备的本地消息，并删除当前账号在服务端保存的对应消息。之后即使卸载重装或重新建立本地数据库，这条消息也不会再同步回来。

该调用不会删除当前账号其他设备上已经同步到本地的消息，也不会删除会话中其他用户已经收到的消息。它不是撤回，也不能用来让其他成员看到“消息已删除”。

## 同步删除到其他客户端

需要把删除动作同步到其他客户端时，将 `isSync` 设为 `true`：

```ts
await OpenIM.deleteMessages({
  conversationID,
  clientMsgIDs: selectedMessageIDs,
  isSync: true,
});
```

`isSync: true` 与普通的账号侧删除不同：SDK 会请求服务端把删除动作同步到其他客户端，其他客户端通过 `CbEvents.OnMsgDeleted` 删除已经同步到本地的消息。删除自己发送的消息或删除其他成员发送给自己的消息时，可同步的设备和用户范围以及操作权限由 OpenIM 服务端校验；调用失败时不要只更新本地列表。

## 删除指定用户在会话中的全部消息

`deleteUserAllMessagesInConv()` 是高影响范围的管控接口，按 `conversationID` 和 `userID` 删除该用户在会话中的全部消息：

```ts
await OpenIM.deleteUserAllMessagesInConv({
  conversationID,
  userID: targetUserID,
});
```

不要把它放在普通单条消息菜单中。调用权限和适用会话类型由服务端校验；执行前应二次确认目标会话和用户。其他客户端通过 `CbEvents.OnDeleteUserAllMsgsInConv` 获得 `{ conversationID, userID }`，再移除对应消息或重新读取历史记录。

## 监听删除相关事件

删除 API 的 Promise 成功表示服务端已完成当前请求。调用端可以先用返回值或请求参数更新当前列表，随后把事件作为幂等增量处理；事件也可能来自其他客户端或其他有权限的用户。

本页是 `OnMsgDeleted` 和 `OnDeleteUserAllMsgsInConv` 的完整监听示例归属页。撤回事件见[撤回消息](/sdk/wasm/message/managing-messages/revoke-a-message)。

```ts
import { CbEvents } from '@openim/wasm-client-sdk';

const handleMessageDeleted = ({ data }) => {
  removeMessage(resolveConversationID(data), data.clientMsgID);
};
const handleUserMessagesDeleted = ({ data }) => {
  removeUserMessages(data.conversationID, data.userID);
};

OpenIM.on(CbEvents.OnMsgDeleted, handleMessageDeleted);
OpenIM.on(CbEvents.OnDeleteUserAllMsgsInConv, handleUserMessagesDeleted);

function removeDeleteListeners() {
  OpenIM.off(CbEvents.OnMsgDeleted, handleMessageDeleted);
  OpenIM.off(CbEvents.OnDeleteUserAllMsgsInConv, handleUserMessagesDeleted);
}
```

`OnMsgDeleted` 的 `data` 是单个 `MessageItem`；`OnDeleteUserAllMsgsInConv` 的 `data` 是 `{ conversationID, userID }`。消息使用 `clientMsgID` 合并，并结合消息路由字段确定所属会话；按用户删除则直接使用事件中的 `conversationID:userID`。组件卸载、退出登录或切换账号时调用 `removeDeleteListeners()`；重新登录后的消息变化由消息事件同步。

## 删除结果

删除和本地删除调用失败时，Promise 会被拒绝。成功分支根据返回数据和后续事件更新界面；失败日志中记录错误对象、`conversationID` 和 `clientMsgID`。

| 用户动作                       | API                               | 结果                                                                    |
| ------------------------------ | --------------------------------- | ----------------------------------------------------------------------- |
| 仅删除当前设备上的本地消息     | `deleteMessageFromLocalStorage()` | 服务端仍保留消息，重新建立本地数据库后还会再次同步。                    |
| 删除当前账号保存的消息         | `deleteMessages({ isSync: false })` | 同时删除本地和当前账号的服务端消息，不影响其他客户端已经保存的副本。  |
| 请求把删除动作同步到其他客户端 | `deleteMessages({ isSync: true })`  | 通过服务端向其他客户端同步删除，实际范围和权限以服务端校验为准。      |
| 删除某用户的全部会话消息       | `deleteUserAllMessagesInConv()`   | 高影响管控操作，按会话和用户批量处理。                                  |
