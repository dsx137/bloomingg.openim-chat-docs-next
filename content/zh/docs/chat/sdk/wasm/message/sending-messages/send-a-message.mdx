OpenIMClientSDK 发送消息时先创建 `MessageItem`，再把它交给 `sendMessage()`。单聊消息填写 `recvID` 并把 `groupID` 设为空字符串；群聊消息填写 `groupID` 并把 `recvID` 设为空字符串。

调用发送 API 前，请确保浏览器端已经完成 SDK 初始化和 `login()`。调用失败时记录 `errCode`、`errMsg`、当前用户以及目标用户或群组 ID，便于排查发送链路。

## 消息类型

| 消息类型        | 创建方法                                          | 说明                                       |
| --------------- | ------------------------------------------------- | ------------------------------------------ |
| 文本消息        | `createTextMessage(text)`                         | 发送用户输入的纯文本内容。                 |
| @ 文本消息      | `createTextAtMessage(data)`                       | 群聊中提醒指定成员。                       |
| 图片消息        | `createImageMessageByFile(params)`                | 从浏览器 `File` 对象创建图片消息。         |
| 音频消息        | `createSoundMessageByFile(data)`                  | 发送语音或音频文件。                       |
| 视频消息        | `createVideoMessageByFile(data)`                  | 发送视频文件和快照图。                     |
| 文件消息        | `createFileMessageByFile(data)`                   | 发送普通附件。                             |
| 自定义消息      | `createCustomMessage(params)`                     | 承载卡片、邀请、订单等结构化业务数据。 |

## 参数说明

`sendMessage()` 和 `sendMessageNotOss()` 使用相同的发送对象；两者的区别是媒体消息是否进入 SDK 的 OSS 上传链路。

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `recvID` | `string` | 必填 | 单聊接收方用户 ID；发送群聊消息时传空字符串。 |
| `groupID` | `string` | 必填 | 群聊目标群组 ID；发送单聊消息时传空字符串。 |
| `message` | `MessageItem` | 必填 | 消息创建方法返回的待发送对象。 |
| `offlinePushInfo` | `OfflinePush` | 可选 | 当前消息的离线推送信息。 |
| `isOnlineOnly` | `boolean` | 可选 | 是否只向在线客户端投递。为 `true` 时，接收端通常通过 `OnRecvOnlineOnlyMessages` 收到；这类消息不写入 SDK 本地存储，也不能按历史回放。完整监听见[接收消息](/sdk/wasm/message/receiving-messages/receive-messages)。 |

创建方法返回的 `data` 就是发送时传入的 `message`。发送成功后，`sendMessage()` 返回服务端确认后的 `MessageItem`，应用按 `clientMsgID` 用返回对象替换本地待发送项，从而更新发送状态、服务端消息 ID 和时间等字段。

```ts
const { data: message } = await openimsdk.createTextMessage('你好，OpenIMSDK');

const { data: sentMessage } = await openimsdk.sendMessage({
  recvID: receiverUserID,
  groupID: '',
  message,
});

appendMessage(sentMessage);
```

发送群聊消息只需要切换目标字段。不要把 `conversationID` 当作发送目标；它主要用于历史消息、会话列表和未读状态。

```ts
const { data: message } = await openimsdk.createTextMessage('大家好');

await openimsdk.sendMessage({
  recvID: '',
  groupID,
  message,
});
```

## 发送定向群消息

`createTargetedGroupMessage()` 把已经创建的 `MessageItem` 转换为只面向指定群成员的定向群消息。它只负责创建消息对象，仍需调用 `sendMessage()` 发送到目标群组。

```ts
const { data: textMessage } = await openimsdk.createTextMessage('请两位负责人确认');

const { data: targetedMessage } = await openimsdk.createTargetedGroupMessage({
  message: textMessage,
  dstUserIDs: ['user_a', 'user_b'],
});

await openimsdk.sendMessage({
  recvID: '',
  groupID,
  message: targetedMessage,
});
```

`dstUserIDs` 应填写目标群内的用户 ID。定向群消息不是管理员消息，也不能绕过群成员关系、禁言或服务端权限；目标范围和离线投递结果以服务端校验为准。

## 发送文本消息

文本消息适合普通聊天内容。创建前可以在业务层完成长度、敏感词、空白字符和草稿状态处理；SDK 创建消息后，发送层只关心目标字段和 `message` 对象。

```ts
async function sendTextToUser(receiverUserID: string, text: string) {
  const normalizedText = text.trim();

  if (!normalizedText) {
    return;
  }

  const { data: message } = await openimsdk.createTextMessage(normalizedText);

  return openimsdk.sendMessage({
    recvID: receiverUserID,
    groupID: '',
    message,
  });
}
```

如果需要提醒群成员，使用[在消息中提及用户](/sdk/wasm/message/composing-messages/mention-users-in-a-message)中的 `createTextAtMessage()`；需要承载结构化业务数据时，使用自定义消息。

## 文件和自定义消息

文件、图片、音频和视频消息仍遵循“先创建、再发送”的模型；文件参数、上传关联和进度事件集中见[上传文件并跟踪进度](/sdk/wasm/message/sending-messages/upload-files-and-track-progress)。

自定义消息的业务数据设计、发送示例和接收端解析集中见[自定义消息与扩展数据](/sdk/wasm/message/composing-messages/custom-message-and-extra-data)，本页只说明发送目标和通用发送流程。

如果项目使用自定义文件上传链路，文件上传成功后应调用对应的 `create*MessageByURL()` 创建 `MessageItem`，再把同一个 `SendMsgParams` 结构传给 `sendMessageNotOss()`。它是发送入口，不是新的消息类型；文件、图片、音频和视频的完整组合示例见[创建媒体与富消息](/sdk/wasm/message/sending-messages/create-media-and-rich-messages#发送已经上传的媒体文件)。

## 调用结果与事件反馈

消息工厂和 `createTargetedGroupMessage()` 的 Promise 只返回待发送对象，不会触发新消息事件。`sendMessage()` 或 `sendMessageNotOss()` 的 Promise 成功表示服务端已确认本次发送；发送端直接使用返回的 `MessageItem`，不必等待接收事件更新自己的发送状态。

其他客户端通过 `OnRecvNewMessages` 接收普通消息；若发送时设置了 `isOnlineOnly: true`，接收端走 `OnRecvOnlineOnlyMessages`，且该消息不会进入本地消息存储。事件 `data` 都是 `MessageItem[]`。应根据消息路由字段确定会话，并按“目标会话 + `clientMsgID`”幂等合并；完整监听与清理代码见[接收消息](/sdk/wasm/message/receiving-messages/receive-messages)。文件上传进度见[上传文件并跟踪进度](/sdk/wasm/message/sending-messages/upload-files-and-track-progress)。Promise 成功与接收事件到达分别处理，重新登录后的消息变化由消息事件同步。
