OpenIM 发送消息分两步：先用对应的 `create*Message()` 创建 `MessageItem`，再调用发送方法。本页说明文件已通过业务上传服务上传时，如何组合使用 `create*MessageByURL()` 和 `sendMessageNotOss()`，以及如何创建名片、位置、引用和 Markdown 等富消息。

## 支持的方法

| 方法                           | 用途                                                |
| ------------------------------ | --------------------------------------------------- |
| `createSoundMessageByURL()`    | 创建已上传语音消息。                                |
| `createFileMessageByURL()`     | 创建已上传文件消息。                                |
| `createImageMessageByURL()`    | 使用原图、大图和缩略图信息创建图片消息。            |
| `createVideoMessageByURL()`    | 使用视频及快照地址创建视频消息。                    |
| `createFaceMessage()`          | 创建表情消息。                                      |
| `createLocationMessage()`      | 创建地理位置消息。                                  |
| `createCardMessage()`          | 创建名片消息。                                      |
| `createQuoteMessage()`         | 从序列化的原消息创建引用消息。                      |
| `createMarkdownMessage()`      | 创建 Markdown 消息。                                |

## 示例：发送名片消息

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `userID` | `string` | 必填 | 名片对应的用户 ID。 |
| `nickname` | `string` | 必填 | 名片展示名称。 |
| `faceURL` | `string` | 必填 | 名片头像地址。 |
| `ex` | `string` | 必填 | 名片扩展字段的完整值；没有扩展信息时传空字符串。 |

```ts
const { data: cardMessage } = await OpenIM.createCardMessage({
  userID: 'user_b',
  nickname: 'Alex',
  faceURL: 'https://example.com/avatar.png',
  ex: '',
});

await OpenIM.sendMessage({
  recvID: 'user_c',
  groupID: '',
  message: cardMessage,
});
```

## 示例：发送位置消息

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `description` | `string` | 必填 | 位置描述。 |
| `longitude` | `number` | 必填 | 经度。 |
| `latitude` | `number` | 必填 | 纬度。 |

```ts
const { data: locationMessage } = await OpenIM.createLocationMessage({
  description: '上海市浦东新区',
  longitude: 121.4737,
  latitude: 31.2304,
});

await OpenIM.sendMessage({
  recvID: '',
  groupID,
  message: locationMessage,
});
```

## URL 型媒体消息

URL 型方法适用于文件已经上传完成、客户端只需将资源地址及必要元数据写入消息的场景。发送前应保证 URL、文件大小、文件名、时长、尺寸和类型等元数据真实可用。

### 文件消息

#### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `filePath` | `string` | 必填 | 本地来源路径；只有远程地址时传空字符串。 |
| `fileName` | `string` | 必填 | 文件名。 |
| `uuid` | `string` | 必填 | 文件稳定标识。 |
| `sourceUrl` | `string` | 必填 | 已上传文件的可访问 URL。 |
| `fileSize` | `number` | 必填 | 文件大小，单位为字节。 |
| `fileType` | `string` | 可选 | 文件 MIME 类型。 |

```ts
const { data: fileMessage } = await OpenIM.createFileMessageByURL({
  filePath: '',
  fileName: 'report.pdf',
  uuid: crypto.randomUUID(),
  sourceUrl: 'https://example.com/report.pdf',
  fileSize: 1024 * 120,
  fileType: 'application/pdf',
});
```

### 图片消息

#### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `sourcePicture` | `PicBaseInfo` | 必填 | 原图信息。 |
| `bigPicture` | `PicBaseInfo` | 必填 | 大图信息。 |
| `snapshotPicture` | `PicBaseInfo` | 必填 | 缩略图信息。 |
| `sourcePath` | `string` | 必填 | 本地来源路径；只有远程地址时传空字符串。 |

`PicBaseInfo` 包含 `uuid`、`type`、`size`、`width`、`height` 和 `url`，所有值都应来自实际上传结果。

图片消息需要同时提供原图、大图和缩略图的 `PicBaseInfo`。即使三种图片规格暂时使用同一个地址，也要传入完整的宽、高、大小、类型和 UUID。

```ts
const picture = {
  uuid: crypto.randomUUID(),
  type: 'image/jpeg',
  size: 120_000,
  width: 1280,
  height: 720,
  url: 'https://example.com/photo.jpg',
};

const { data: imageMessage } = await OpenIM.createImageMessageByURL({
  sourcePicture: picture,
  bigPicture: picture,
  snapshotPicture: picture,
  sourcePath: '',
});
```

### 音频消息

#### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `uuid` | `string` | 必填 | 音频稳定标识。 |
| `soundPath` | `string` | 必填 | 本地来源路径；只有远程地址时传空字符串。 |
| `sourceUrl` | `string` | 必填 | 已上传音频的可访问 URL。 |
| `dataSize` | `number` | 必填 | 音频大小，单位为字节。 |
| `duration` | `number` | 必填 | 音频时长。 |
| `soundType` | `string` | 可选 | 音频 MIME 类型或业务约定的格式。 |

### 视频消息

#### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `videoPath` | `string` | 必填 | 本地视频来源路径；只有远程地址时传空字符串。 |
| `duration` | `number` | 必填 | 视频时长。 |
| `videoType` | `string` | 必填 | 视频类型。 |
| `videoUUID` | `string` | 必填 | 视频稳定标识。 |
| `videoUrl` | `string` | 必填 | 已上传视频的可访问 URL。 |
| `videoSize` | `number` | 必填 | 视频大小，单位为字节。 |
| `snapshotPath` | `string` | 必填 | 本地快照来源路径；只有远程地址时传空字符串。 |
| `snapshotUUID` | `string` | 必填 | 快照稳定标识。 |
| `snapshotSize` | `number` | 必填 | 快照大小，单位为字节。 |
| `snapshotUrl` | `string` | 必填 | 已上传快照的可访问 URL。 |
| `snapshotWidth` | `number` | 必填 | 快照宽度。 |
| `snapshotHeight` | `number` | 必填 | 快照高度。 |
| `snapShotType` | `string` | 可选 | 快照图片类型。 |

不要只传一个视频 URL；缺少视频元数据或快照尺寸、地址和 UUID，会让接收端无法稳定展示预览。

### 发送已经上传的媒体文件

文件、图片、音频或视频已经通过业务自己的上传服务取得可访问 URL 后，先调用对应的 `create*MessageByURL()` 创建完整的 `MessageItem`，再使用 `sendMessageNotOss()` 发送。这样可以直接使用消息中的远程地址，不再进入 SDK 的内置 OSS 上传链路。

```ts
const { data: fileMessage } = await OpenIM.createFileMessageByURL({
  filePath: '',
  fileName: 'report.pdf',
  uuid: crypto.randomUUID(),
  sourceUrl: uploadedFile.url,
  fileSize: uploadedFile.size,
  fileType: uploadedFile.contentType,
});

const { data: sentMessage } = await OpenIM.sendMessageNotOss({
  recvID: receiverUserID,
  groupID: '',
  message: fileMessage,
});

upsertMessage(sentMessage);
```

发送到群聊时，把 `recvID` 设为空字符串并传入 `groupID`。图片、音频和视频分别使用 `createImageMessageByURL()`、`createSoundMessageByURL()` 和 `createVideoMessageByURL()`，发送步骤仍使用同一个 `sendMessageNotOss()`。

URL 型消息的文件大小、类型、图片尺寸、音视频时长和快照信息应来自实际上传结果。不要为了通过参数校验填写虚假值，否则接收端可能无法正确展示、下载或播放媒体内容。

## 引用消息

`createQuoteMessage()` 的 `message` 参数是序列化后的原消息字符串。调用前先序列化要引用的原消息，不要手工拼接 `MessageItem` 字段。

```ts
const { data: quoteMessage } = await OpenIM.createQuoteMessage({
  text: '同意这个方案',
  message: JSON.stringify(originalMessage),
});

await OpenIM.sendMessage({
  recvID,
  groupID,
  message: quoteMessage,
});
```

## Markdown 消息

`createMarkdownMessage()` 接收 Markdown 源文本并返回 `MessageItem`。SDK 只负责承载内容；接收端仍需使用经过安全配置的 Markdown 渲染器，并过滤不可信链接、HTML 和脚本协议。

```ts
const { data: markdownMessage } = await OpenIM.createMarkdownMessage(
  '**发布完成**\n\n- Web\n- WASM'
);

await OpenIM.sendMessage({
  recvID: '',
  groupID,
  message: markdownMessage,
});
```

不支持 Markdown 展示的客户端应降级为纯文本。不要假定所有端都使用相同的 Markdown 方言。

## 调用结果与消息接收

本页所有 `create*Message()` 方法都只通过 Promise 返回待发送的 `MessageItem`。创建成功不代表消息已经发送，也不会触发新消息事件；调用端可以在发送前继续检查或补充发送参数。

`sendMessage()` 或 `sendMessageNotOss()` 的 Promise 成功后，发送端直接使用返回的 `MessageItem` 按 `clientMsgID` 合并到目标会话列表。其他客户端通过新消息事件 `OnRecvNewMessages` 接收消息，事件 `data` 是 `MessageItem[]`；完整监听、清理、会话路由和去重方式见[接收消息](/sdk/wasm/message/receiving-messages/receive-messages)。重新登录后的消息变化由消息事件同步。
