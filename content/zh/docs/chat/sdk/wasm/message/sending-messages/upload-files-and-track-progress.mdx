OpenIMClientSDK 的文件类消息在创建并发送时进入上传链路。应用应先注册 `CbEvents.OnProgress` 和 `CbEvents.UploadComplete`，再调用文件、图片、音频或视频消息创建方法，并通过 `sendMessage()` 发送生成的 `MessageItem`。

上传中的百分比、失败提示和重试按钮属于应用界面状态。SDK 事件负责提供进度和完成信号；应用需要把事件数据关联到本地上传项或消息气泡。

## 注册上传处理器

在开始发送文件类消息前注册事件处理器。处理器函数需要保持稳定引用，后续才能用同一个函数调用 `off()`。

```ts
import { CbEvents } from '@openim/wasm-client-sdk';

function handleUploadProgress({ data }) {
  updateMessageUploadProgress(data.clientMsgID, data.progress);
}

function handleUploadComplete({ data }) {
  markFileUploadComplete(data.uuid, {
    fileSize: data.fileSize,
    streamSize: data.streamSize,
  });
}

openimsdk.on(CbEvents.OnProgress, handleUploadProgress);
openimsdk.on(CbEvents.UploadComplete, handleUploadComplete);

function removeUploadListeners() {
  openimsdk.off(CbEvents.OnProgress, handleUploadProgress);
  openimsdk.off(CbEvents.UploadComplete, handleUploadComplete);
}
```

`OnProgress` 的 `data` 包含 `clientMsgID` 和 `progress`，按 `clientMsgID` 更新消息气泡。`UploadComplete` 的 `data` 包含 `uuid`、`fileSize` 和 `streamSize`。在 `@openim/wasm-client-sdk@3.8.5-hotfix.0` 中，`uploadFile()` 与 `create*MessageByFile()` 会在调用前把传入的 `uuid`（图片用 `sourcePicture.uuid`，视频用 `videoUUID` / `snapshotUUID`）改写为 `` `${uuid}/${file.name}` ``，再登记到 `fileMapSet`；因此完成事件里的 `uuid` 是改写后的值，应按创建后 `MessageItem` 内对应文件元素上的 UUID 关联，不要用创建前传入的原始 UUID。组件卸载、退出登录或切换账号时调用 `removeUploadListeners()`。

## 单独上传文件

只需要取得文件 URL、暂时不创建消息时，可以调用 `uploadFile()`。浏览器中传入 `File`、文件名、MIME 类型和本地生成的 `uuid`；包装层会改写为 `` `${uuid}/${file.name}` `` 后再上传。成功后 `data.url` 是上传后的资源地址。

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `name` | `string` | 必填 | 文件名。 |
| `contentType` | `string` | 必填 | 文件 MIME 类型。 |
| `uuid` | `string` | 必填 | 本次上传的本地标识；包装层会改写为 `` `${uuid}/${file.name}` ``，并用该值关联 `UploadComplete`。 |
| `file` | `File` | 浏览器上传时必填 | 浏览器文件对象。 |
| `filepath` | `string` | 按运行环境可选 | 非浏览器环境使用的文件路径；浏览器不要与 `file` 同时传入。 |
| `cause` | `string` | 可选 | 业务约定的上传原因。 |

```ts
const uuid = crypto.randomUUID();
const { data } = await openimsdk.uploadFile({
  name: file.name,
  contentType: file.type || 'application/octet-stream',
  uuid,
  file,
});

console.log(data.url);
```

`create*MessageByFile()` 与 `uploadFile()` 已在内部完成 `fileMapSet`。只有调用链明确要求预先登记文件时，才单独调用 `fileMapSet()`，并传入与后续上传一致的改写后 UUID（`` `${uuid}/${file.name}` ``）；不要先用原始 UUID 登记、再调用会改写 UUID 的创建或上传方法。

```ts
const mappedUUID = `${uuid}/${file.name}`;
await openimsdk.fileMapSet(mappedUUID, file);
```

## 创建并发送文件消息

文件消息仍然遵循 OpenIMSDK 的两步发送模型：先创建 `MessageItem`，再传给 `sendMessage()`。单聊填写 `recvID`，群聊填写 `groupID`。

```ts
type SendFileWithProgressParams = {
  file: File;
  recvID?: string;
  groupID?: string;
};

async function sendFileWithProgress({
  file,
  recvID = '',
  groupID = '',
}: SendFileWithProgressParams) {
  const uuid = crypto.randomUUID();

  const { data: message } = await openimsdk.createFileMessageByFile({
    file,
    filePath: file.name,
    fileName: file.name,
    uuid,
    sourceUrl: '',
    fileSize: file.size,
    fileType: file.type,
  });

  // UploadComplete 应按 message.fileElem.uuid（已是 uuid/file.name）关联
  const { data: sentMessage } = await openimsdk.sendMessage({
    recvID,
    groupID,
    message,
  });

  return sentMessage;
}
```

图片、音频和视频消息分别使用 `createImageMessageByFile()`、`createSoundMessageByFile()` 和 `createVideoMessageByFile()`。视频消息同时传入视频文件和快照文件，二者的 UUID 都会按各自文件名改写；创建方法返回 `MessageItem` 后，发送时仍使用同一个 `sendMessage()`。

## 关联进度到消息

如果界面允许同时上传多个文件，不要只按文件名匹配进度。同名文件、重试和多会话并发发送都可能造成状态串扰。创建上传项时保存本地上传队列 ID，创建消息成功后从返回的 `MessageItem` 读取改写后的文件 UUID 和 `clientMsgID`：完成事件用文件元素上的 UUID，进度事件和消息气泡用 `clientMsgID`。

```ts
type UploadState = {
  localUploadID: string;
  fileName: string;
  percent: number;
  clientMsgID?: string;
  status: 'pending' | 'uploading' | 'sent' | 'failed';
};

function attachSentMessage(localUploadID: string, sentMessage: MessageItem) {
  updateUploadState(localUploadID, {
    clientMsgID: sentMessage.clientMsgID,
    status: 'sent',
  });
}
```

发送失败时记录 `errCode`、`errMsg`、文件名和目标会话。如果业务支持重试，重试时重新创建消息或按你的上传策略复用已上传文件地址，避免把旧进度误写到新的消息气泡。

## 监听上传进度事件

本页是 `OnProgress` 和 `UploadComplete` 的完整监听示例归属页。上传事件在本页“注册上传处理器”一节的初始化阶段注册一次即可，不要在每次发送文件时再次注册同一组处理器。

`uploadFile()` 的 Promise 成功后直接使用 `data.url`；`fileMapSet()` 的 Promise 成功只表示本地文件映射已登记。消息工厂的 Promise 返回待发送 `MessageItem`，`sendMessage()` 的 Promise 返回服务端确认后的 `MessageItem`。这些调用结果与上传事件分别处理；需要显示当前消息列表时，再通过消息历史查询获取快照。

## 验证上传结果

- 确认 `login()` 已成功，并且发送前连接状态正常。
- 使用较大的图片或文件测试 `CbEvents.OnProgress`，确认界面状态会从等待、上传中变为完成。
- 确认 `sendMessage()` 返回的 `data` 已合并到当前会话消息列表。
- 发送多个同名文件，确认每个上传项的进度不会互相覆盖。
- 切换页面或账号后再次发送文件，确认事件处理器没有重复注册。
