在 OpenIMClientSDK 中，提及用户的消息是一种群聊 `MessageItem`。你需要先用 `createTextAtMessage()` 创建 @ 文本消息，再通过 `sendMessage()` 发送到目标群组。接收端可以从消息的 `atTextElem` 读取被提及的用户列表，并从会话的 `groupAtType` 判断当前用户是否有未处理的 @ 提醒。

提及逻辑应基于 OpenIMSDK 用户 ID，而不是昵称、头像或会话 ID。昵称只用于展示，真正触发提醒的是 `atUserIDList`。

## 按用户 ID 提及成员

`createTextAtMessage()` 接收要发送的文本、被提及用户 ID 列表，以及可选的群昵称信息。SDK 不保存 @ 文案在 `text` 中的位置信息；如果接收端需要把 @ 文案渲染为可点击的用户节点，建议在 `text` 中使用 `@userID` 作为稳定标记，再通过 `atUsersInfo` 映射展示名称。

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `text` | `string` | 必填 | 消息正文；需要渲染可点击成员时，使用 `@userID` 保存稳定标记。 |
| `atUserIDList` | `string[]` | 必填 | 需要触发提醒的用户 ID；单条消息不超过 10 个用户（SDK 底层限制）。 |
| `atUsersInfo` | `AtUsersInfoItem[]` | 可选 | 用户 ID 与群内展示名称的映射，应与 `atUserIDList` 使用同一份成员数据。 |
| `message` | `MessageItem` | 可选 | 需要同时引用的原消息。 |

```ts
const mentionedUsers = [
  { userID: 'alex-user-id', groupNickname: 'Alex' },
  { userID: 'lee-user-id', groupNickname: 'Lee' },
];

const { data: message } = await openimsdk.createTextAtMessage(
  {
    text: '@alex-user-id @lee-user-id 请看一下这条消息',
    atUserIDList: mentionedUsers.map((user) => user.userID),
    atUsersInfo: mentionedUsers.map((user) => ({
      atUserID: user.userID,
      groupNickname: user.groupNickname,
    })),
  }
);

const { data: sentMessage } = await openimsdk.sendMessage(
  {
    recvID: '',
    groupID,
    message,
  }
);

appendMessage(sentMessage);
```

发送前应校验正文不为空、提及用户仍在当前群内，并确保 `atUserIDList` 与 `atUsersInfo` 来自同一份成员数据。@ 消息面向群聊发送，因此 `sendMessage()` 中填写目标 `groupID`，并把 `recvID` 设为空字符串。

## 渲染可点击的 @ 成员

发送端的编辑器可以向用户显示 `@Alex`，但提交给 `createTextAtMessage()` 时应序列化为 `@alex-user-id`。接收端同时读取 `atTextElem.text` 和 `atTextElem.atUsersInfo`，只把能够与 `atUserID` 完整匹配的 `@userID` 标记转成 @ 节点：节点文案显示 `@groupNickname`，点击时使用对应的 `atUserID` 打开用户资料。

可以使用分词器或正则表达式完成标记切分，但候选 ID 必须来自 `atUsersInfo`，并在拼接正则前转义特殊字符、校验完整标记边界。不要直接把文本中所有 `@xxx` 都识别成用户，也不要用昵称反查用户 ID；群昵称可能重复、变更或包含空格和特殊字符。没有匹配到 `atUsersInfo` 的内容按普通文本显示。

## 提及全体成员

创建 @ 全体成员消息时，不要把群成员 ID 全部展开到 `atUserIDList`。应在业务层的 @ 候选元素中加入一个特殊成员，它的 `userID` 固定为 `AtAllTag`，再按普通提及对象的方式转换为 `createTextAtMessage()` 所需的参数。

```ts
const AT_ALL_TAG = 'AtAllTag';

const atAllElement = {
  userID: AT_ALL_TAG,
  groupNickname: '全体成员',
};

const { data: message } = await openimsdk.createTextAtMessage({
  text: '@AtAllTag 请查看这条消息',
  atUserIDList: [atAllElement.userID],
  atUsersInfo: [
    {
      atUserID: atAllElement.userID,
      groupNickname: atAllElement.groupNickname,
    },
  ],
});

await openimsdk.sendMessage({
  recvID: '',
  groupID,
  message,
});
```

`AtAllTag` 是 OpenIMSDK 识别 @ 全体成员的特殊用户 ID，不是真实群成员，也不是 `GroupAtType` 的枚举值。客户端应根据当前成员的角色和群权限决定是否展示“@ 全体成员”入口，服务端仍会对发送权限做最终校验。

## 发送带引用的 @ 消息

`createTextAtMessage()` 还可以接收可选的 `message` 字段，用来把一条已有消息作为引用消息带入 @ 文本消息中。

```ts
const { data: mentionReply } = await openimsdk.createTextAtMessage({
  text: '@alex-user-id 我回复的是这条消息',
  atUserIDList: ['alex-user-id'],
  atUsersInfo: [{ atUserID: 'alex-user-id', groupNickname: 'Alex' }],
  message: quotedMessage,
});

await openimsdk.sendMessage({
  recvID: '',
  groupID,
  message: mentionReply,
});
```

`quotedMessage` 应来自当前会话的消息列表、历史消息查询结果或新消息事件回调。不要手动拼接一个不完整的 `MessageItem`，否则接收端可能无法正确展示引用内容。

## 处理接收事件

`createTextAtMessage()` 的 Promise 只返回待发送的 `MessageItem`，不会触发新消息事件。`sendMessage()` 的 Promise 成功后，发送端直接使用返回的 `MessageItem` 按 `clientMsgID` 合并当前消息列表。

接收端通过 `OnRecvNewMessages` 收到 @ 消息后，从 `atTextElem` 的 `atUserList` 和 `atUsersInfo` 渲染内容。事件 `data` 是 `MessageItem[]`，通用事件注册、离线补偿和按 `conversationID + clientMsgID` 去重统一见[接收消息](/sdk/wasm/message/receiving-messages/receive-messages)，本页不重复实现消息列表处理器。重新登录后的消息变化由消息事件同步。

## 限制

### 单条消息的提及人数

单条 @ 消息中，`atUserIDList` 受 SDK 底层限制，最多 10 个用户。如果产品需要提醒更多成员，应改用经过权限控制的全体提醒策略或服务端通知能力。

### 群聊提醒状态

`GroupAtType` 只用于描述 `ConversationItem.groupAtType` 的状态，不参与 @ 消息的创建。`GroupAtType.AtMe`、`GroupAtType.AtAll` 和 `GroupAtType.AtAllAtMe` 分别表示当前用户在该群聊会话中的 @ 提醒；会话列表通过 `OnConversationChanged` 合并最新状态。完整监听和清理代码见[获取会话列表](/sdk/wasm/conversation/retrieving-conversations/retrieve-conversation-list)。

用户进入会话并处理完 @ 消息后，可以把该会话的 @ 状态重置为普通状态。

```ts
import { GroupAtType } from '@openim/wasm-client-sdk';

await openimsdk.setConversation({
  conversationID,
  groupAtType: GroupAtType.AtNormal,
});
```

Promise 成功表示服务端已完成本次会话设置请求。后续通过 `OnConversationChanged` 按 `conversationID` 合并该会话的最新状态。
