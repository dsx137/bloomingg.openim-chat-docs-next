OpenIMSDK 的消息扩展数据需要先确定作用范围：需要随消息同步给其他客户端的数据，可以在发送前写入 `MessageItem.ex`；只影响当前客户端展示状态的数据，写入 `localEx`。需要承载完整业务对象时，使用自定义消息通常更清晰。

OpenIMClientSDK 不提供一个发送后给任意消息追加共享键值对的客户端通用接口。需要跨用户、跨设备共享的业务数据，应作为自定义消息或业务服务状态的一部分设计。

## 选择数据落点

| 场景                                                 | 推荐方式                                             | 数据是否会同步给其他客户端 |
| ---------------------------------------------------- | ---------------------------------------------------- | -------------------------- |
| 订单卡片、任务卡片、邀请、投票等结构化业务消息       | `createCustomMessage()` 后调用 `sendMessage()`       | 是                         |
| 消息来源、展示版本等轻量消息扩展                     | 发送前写入 `MessageItem.ex`                          | 是                         |
| 当前客户端的选中、折叠、临时标记、来源追踪等界面状态 | `setMessageLocalEx()`                                | 否                         |

`CustomMsgParams` 包含 `data`、`extension` 和 `description`。建议把业务数据序列化到 `data`，用 `extension` 标识业务类型，并把 `description` 作为无法解析自定义数据时的降级展示文案。

## 发送带共享数据的消息

自定义消息适合承载可被接收端完整解析的业务对象。下面示例创建一条发票卡片消息，并发送到群聊。

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `data` | `string` | 必填 | 序列化后的业务数据。 |
| `extension` | `string` | 必填 | 业务消息类型标识，供接收端选择解析和渲染逻辑。 |
| `description` | `string` | 必填 | 无法解析业务数据时的降级展示文案。 |

```ts
const { data: message } = await openimsdk.createCustomMessage(
  {
    data: JSON.stringify({
      type: 'invoice-card',
      invoiceID,
      amount,
      currency: 'CNY',
    }),
    extension: 'invoice-card',
    description: '发票卡片',
  }
);

const { data: sentMessage } = await openimsdk.sendMessage(
  {
    recvID: '',
    groupID,
    message,
  }
);

mergeMessageIntoConversation(sentMessage);
```

接收端通过 `CbEvents.OnRecvNewMessages` 收到消息后，按 `extension` 或你项目中的消息类型判断是否进入自定义渲染流程，再解析 `data` 中的 JSON。事件注册、离线补偿和去重方式统一见[接收消息](/sdk/wasm/message/receiving-messages/receive-messages)。

不要把需要参与权限判断、支付状态或审批流的最终状态只放在客户端消息数据中。这类状态应由业务服务校验和持久化，消息只承载展示入口或快照。

## 写入共享扩展字段

`MessageItem.ex` 是随消息发送和存储的扩展字符串。接收端通过新消息事件获取消息，或之后查询历史消息时，都可以从同一条 `MessageItem` 读取 `ex`。适合保存消息来源、展示版本或其他轻量且不敏感的附加信息。

创建消息后、调用发送方法前，为 `ex` 写入完整字符串：

```ts
const { data: message } = await openimsdk.createTextMessage('请查看最新方案');

const messageWithEx = {
  ...message,
  ex: JSON.stringify({
    source: 'project-board',
    schemaVersion: 1,
  }),
};

const { data: sentMessage } = await openimsdk.sendMessage({
  recvID: receiverUserID,
  groupID: '',
  message: messageWithEx,
});
```

`ex` 没有自动合并对象的语义。多个业务模块共用该字段时，应约定字段命名和版本，并在写入前生成完整值。不要把令牌、密钥、隐私信息或需要服务端持续校验的权威状态放入 `ex`。

## 写入本地附加数据

`localEx` 是当前客户端的本地消息状态，不会随消息同步给其他用户或其他设备。它适合保存界面标记，例如当前用户是否选中过某条消息、是否折叠了某个卡片，或这条消息是否来自一次本地迁移流程。

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `conversationID` | `string` | 必填 | 目标消息所在的会话 ID。 |
| `clientMsgID` | `string` | 必填 | 要更新的消息 ID。 |
| `localEx` | `string` | 必填 | 本地扩展完整值；不会自动合并旧 JSON。 |

```ts
await openimsdk.setMessageLocalEx({
  conversationID,
  clientMsgID: sentMessage.clientMsgID,
  localEx: JSON.stringify({
    selected: true,
    source: 'invoice-panel',
  }),
});
```

调用 `setMessageLocalEx()` 前需要已经拿到目标消息的 `conversationID` 和 `clientMsgID`。如果这份状态要被多端、多用户共享，请不要使用 `localEx`，应改为发送一条新的业务消息，或把权威状态保存在你的业务服务中。

`setMessageLocalEx()` 写入的是完整的 `localEx` 字符串，不会自动合并旧 JSON。需要保留旧字段时，先读取并解析当前值，在业务层合并后再写回。

## 事件和校验

服务端向客户端发送的是自定义业务通知时，客户端可以监听 `CbEvents.OnRecvCustomBusinessMessage`。普通聊天自定义消息和文件消息仍应通过 `CbEvents.OnRecvNewMessages` 进入消息列表；本页不重复实现通用消息接收处理器。

| 事件                                   | 用途                                                   |
| -------------------------------------- | ------------------------------------------------------ |
| `CbEvents.OnRecvNewMessages`           | 接收普通消息和自定义消息，并更新消息列表             |
| `CbEvents.OnRecvCustomBusinessMessage` | 接收服务端下发的自定义业务通知                         |

`OnRecvCustomBusinessMessage` 的 `data` 是服务端下发的业务载荷，不是 `MessageItem`。应用应先校验并解析自有协议，再使用协议中的业务 ID 或幂等键更新状态；不要用事件到达顺序或数组位置去重。本页是该事件的完整监听示例归属页：

```ts
import { CbEvents } from '@openim/wasm-client-sdk';

const handleCustomBusinessMessage = ({ data }) => {
  const payload = parseAndValidateBusinessPayload(data);
  applyBusinessPayload(payload);
};

openimsdk.on(CbEvents.OnRecvCustomBusinessMessage, handleCustomBusinessMessage);

function removeCustomBusinessMessageListener() {
  openimsdk.off(CbEvents.OnRecvCustomBusinessMessage, handleCustomBusinessMessage);
}
```

组件卸载、退出登录或切换账号时调用 `removeCustomBusinessMessageListener()`。创建消息对象只会在 Promise 中返回本地待发送的 `MessageItem`，不会触发接收事件；`sendMessage()` 成功后直接使用返回的 `MessageItem` 合并到发送方的消息列表，其他客户端再通过接收事件获得消息。`setMessageLocalEx()` 只更新本地数据，不会产生共享消息事件。

验证附加数据时，重点确认这些结果：

- `createCustomMessage()`、`sendMessage()` 或 `setMessageLocalEx()` 成功时返回预期数据，失败时进入 `catch`。
- 发送前写入 `ex` 的消息，可以在其他客户端收到的 `MessageItem` 和之后查询到的历史消息中读取相同扩展值。
- 需要同步给其他成员的数据，可以在另一个已登录客户端通过 `CbEvents.OnRecvNewMessages` 收到。
- 只写入 `localEx` 的状态不会出现在其他用户或其他设备的共享消息内容中。
- 失败日志包含 `errCode`、`errMsg` 和必要的目标 ID，便于和服务端日志对齐。
