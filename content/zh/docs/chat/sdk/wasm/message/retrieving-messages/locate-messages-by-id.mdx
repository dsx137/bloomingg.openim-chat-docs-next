在消息列表、搜索结果、引用消息或通知跳转场景中，你可以使用 `conversationID` 和消息的 `clientMsgID` 定位一条 OpenIMSDK 消息。需要展示该消息前后的消息内容时，再以已获取的 `MessageItem` 作为起点读取周边消息。

## 按 clientMsgID 查询消息

`findMessageList()` 接收一组会话查询条件。每个条件使用 `conversationID` 指定会话，并通过 `clientMsgIDList` 传入一个或多个客户端消息 ID。

### 参数说明

`findMessageList()` 的参数是查询条件数组，每项包含以下字段：

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `conversationID` | `string` | 必填 | 目标消息所在的会话 ID。 |
| `clientMsgIDList` | `string[]` | 必填 | 要查询的客户端消息 ID；获取单条消息时只传一个 `clientMsgID`。 |

```ts
import { SessionType, ViewType } from '@openim/wasm-client-sdk';
import type { MessageItem } from '@openim/wasm-client-sdk';

const { data: result } = await openimsdk.findMessageList([
  {
    conversationID,
    clientMsgIDList: [clientMsgID],
  },
]);

const resultItems = result.findResultItems ?? result.searchResultItems ?? [];
const targetMessage = resultItems[0]?.messageList?.[0];
```

`clientMsgID` 是 OpenIMSDK 消息在客户端侧的稳定标识。建议在发送、接收、搜索和历史记录场景中把它随消息一起保存在应用状态中，后续即可用它重新定位消息。

### 返回结果

Promise 成功后，`data` 是 `SearchMessageResult`。匹配到的消息位于结果项的 `messageList` 中；没有匹配项时，应将其视为无结果处理，不要直接访问固定数组索引。

## 读取消息上下文

当用户从搜索结果、引用消息或外部通知跳转到某条消息时，通常需要同时加载该消息前后的若干条消息。先取得目标 `MessageItem`（搜索命中项可直接使用结果里的消息对象），再调用 `fetchSurroundingMessages()`。不要用 `findMessageList()` 代替周边上下文查询。

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `startMessage` | `MessageItem` | 必填 | 周边消息查询的起点，通常来自消息定位结果、历史消息列表或消息事件。 |
| `viewType` | `ViewType` | 必填 | 消息视图类型；读取普通历史上下文时使用 `ViewType.History`。 |
| `before` | `number` | 必填 | 起点消息之前最多读取多少条消息。 |
| `after` | `number` | 必填 | 起点消息之后最多读取多少条消息。 |

```ts
if (!targetMessage) {
  throw new Error('Message not found');
}

const { data: surrounding } = await openimsdk.fetchSurroundingMessages({
  startMessage: targetMessage,
  viewType: ViewType.History,
  before: 20,
  after: 20,
});

renderMessages(surrounding.messageList);
```

### 返回结果

Promise 成功后，`data.messageList` 是包含起点前后消息的 `MessageItem[]`。渲染时仍应按 `conversationID` 和 `clientMsgID` 合并，不要依赖返回数组位置标识消息。

`findMessageList()` 返回的匹配结果可能因本地缓存尚未同步、消息已删除或撤回而为空。消息页打开期间可以监听 `CbEvents.OnSyncServerFinish` 后重试定位，也可以监听 `CbEvents.OnNewRecvMessageRevoked` 更新撤回态。

同步生命周期事件的完整监听和清理代码见[事件概览](/sdk/wasm/events/overview-events)，撤回事件见[撤回消息](/sdk/wasm/message/managing-messages/revoke-a-message)。本页不重复注册这些事件。

## 获取群聊最新消息

如果只需要展示群聊最后一条消息，例如会话列表或群聊入口，可以读取群组对应的会话记录。`getOneConversation()` 返回的 `ConversationItem` 通常包含 `latestMsg`、未读数、名称和头像等会话展示信息。

```ts
const { data: conversation } = await openimsdk.getOneConversation({
  sourceID: groupID,
  sessionType: SessionType.WorkingGroup,
});

function parseLatestMessage(latestMsg: string): MessageItem | undefined {
  if (!latestMsg) return undefined;
  try {
    return JSON.parse(latestMsg) as MessageItem;
  } catch {
    return undefined;
  }
}

renderConversationPreview({
  conversationID: conversation.conversationID,
  latestMessage: parseLatestMessage(conversation.latestMsg),
  unreadCount: conversation.unreadCount,
});
```

`ConversationItem.latestMsg` 是序列化后的消息字符串，不是可直接访问字段的 `MessageItem`。解析失败时应降级为空预览，不要让会话列表渲染中断。需要打开这条消息所在上下文时：若已解析出 `MessageItem`，直接把它传给 `fetchSurroundingMessages()`；若只有 `clientMsgID`，先用 `findMessageList()` 取回消息对象，再读取周边上下文。

## 查询结果

`findMessageList()`、`fetchSurroundingMessages()` 和 `getOneConversation()` 的 Promise 成功后，`data` 分别是本次消息定位、周边消息和会话快照。三个查询都不会触发消息或会话事件，页面可以直接渲染返回值。

合并消息时使用目标 `conversationID` 与 `clientMsgID`，不要依赖结果数组位置。需要显示当前消息快照时重新执行原查询；重新查询与新消息、撤回等事件增量分别处理，重新登录后的变化由事件同步。
