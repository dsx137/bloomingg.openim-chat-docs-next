## 邀请群成员

`inviteUserToGroup()` 接收目标 `groupID`、邀请说明和用户 ID 列表：

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `groupID` | `string` | 必填 | 目标群组 ID。 |
| `reason` | `string` | 必填 | 邀请说明；没有说明时传空字符串。 |
| `userIDList` | `string[]` | 必填 | 要邀请的用户 ID 列表。 |

```ts
await openimsdk.inviteUserToGroup({
  groupID,
  reason: '邀请加入项目讨论组',
  userIDList: ['user-002', 'user-003'],
});
```

邀请结果取决于当前用户角色、群验证配置和 OpenIMServer 权限。Promise 成功表示 OpenIMServer 已接受本次邀请请求，不等于所有目标用户都已经出现在成员列表中。

## 移除群成员

有权限的群主或管理员调用 `kickGroupMember()`：

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `groupID` | `string` | 必填 | 目标群组 ID。 |
| `reason` | `string` | 必填 | 移除说明；没有说明时传空字符串。 |
| `userIDList` | `string[]` | 必填 | 要移除的成员用户 ID 列表。 |

```ts
await openimsdk.kickGroupMember({
  groupID,
  reason: '已离开项目',
  userIDList: [targetUserID],
});
```

不能用该方法移除群主。群主身份必须先通过 `transferGroupOwner()` 转让。OpenIMServer 会按当前操作者角色校验可移除的目标成员。

## 调用后的状态变化

用户直接加入群组后，客户端会通过 `OnGroupMemberAdded` 收到单个 `GroupMemberItem`；客户端应按 `groupID:userID` 合并成员列表。需要审核的邀请或申请流程可能先产生 `OnGroupApplicationAdded`，此时不要提前构造群成员记录。申请事件的完整监听方式见[处理入群申请](/sdk/wasm/group/managing-group-applications/manage-group-applications)。

`kickGroupMember()` 的 Promise 成功表示移除请求已经完成。成员列表随后可能收到 `OnGroupMemberDeleted`，其 `data` 是被删除的 `GroupMemberItem`，按 `groupID:userID` 移除即可。成员新增、删除事件的完整监听和清理代码统一放在[获取群成员](/sdk/wasm/group/retrieving-group-members/retrieve-group-members)。

事件也可能来自其他管理员或其他客户端的操作，不要把事件到达与当前调用的 Promise 完成视为同一步。首次进入成员列表或用户主动刷新时，再查询相应的成员列表以获取当前快照；重新登录后的变化由成员事件同步。调用失败时记录错误对象、目标群组和用户 ID。
