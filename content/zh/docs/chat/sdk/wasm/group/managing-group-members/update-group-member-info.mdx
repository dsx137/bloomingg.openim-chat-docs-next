使用 `setGroupMemberInfo()` 更新成员在指定群内的资料。它接收 `UpdateMemberInfoParams`，只需传入本次要修改的可选字段。

## 参数说明

| 字段        | 类型              | 是否必填 | 说明                                             |
| ----------- | ----------------- | -------- | ------------------------------------------------ |
| `groupID`   | `string`          | 是       | 目标群组 ID。                                    |
| `userID`    | `string`          | 是       | 目标成员用户 ID。                                |
| `nickname`  | `string`          | 否       | 群内昵称。                                       |
| `faceURL`   | `string`          | 否       | 群内头像。                                       |
| `roleLevel` | `GroupMemberRole` | 否       | 普通成员或管理员角色值；群主转让使用专门接口。   |
| `ex`        | `string`          | 否       | 群成员扩展字符串。                               |

```ts
import { GroupMemberRole } from '@openim/wasm-client-sdk';

await openimsdk.setGroupMemberInfo({
  groupID,
  userID: targetUserID,
  nickname: '项目负责人',
  roleLevel: GroupMemberRole.Admin,
});
```

群主负责设置或移除管理员。前端角色按钮只是交互控制，最终权限由 OpenIMServer 校验。不能通过把 `roleLevel` 改为 Owner 来替代 `transferGroupOwner()`。

## 更新昵称和角色

修改群内昵称时只传 `nickname`，修改成员角色时只传 `roleLevel`。两个字段需要同时变化时可以放在一次 `setGroupMemberInfo()` 调用中。角色变更必须遵守群主和管理员权限，转让群主仍应使用专门的群主转让流程。

`ex` 是完整的群成员扩展字符串，不会按 JSON 字段自动合并。多个业务模块共用时，应先读取目标成员资料，合并已有命名空间后再写回完整值。

## 调用后的状态变化

Promise 成功表示 OpenIMServer 已完成本次群成员资料更新。相关客户端随后可能收到 `OnGroupMemberInfoChanged`，事件 `data` 是更新后的单个 `GroupMemberItem`，应按 `groupID:userID` 覆盖本地成员记录。

该事件也可能来自其他管理员或其他客户端的修改，不能只把它当作当前调用的完成回调。完整监听和清理代码见[获取群成员](/sdk/wasm/group/retrieving-group-members/retrieve-group-members)；首次进入成员列表或用户主动刷新时，再查询目标成员获取当前资料，重新登录后的变化由成员事件同步。
