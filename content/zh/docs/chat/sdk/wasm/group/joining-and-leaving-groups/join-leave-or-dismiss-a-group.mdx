## 加入群组

拿到可加入的 `groupID` 后，调用 `joinGroup()`。`reqMsg` 是申请说明，`joinSource` 表示加入途径。

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `groupID` | `string` | 必填 | 要加入的群组 ID。 |
| `reqMsg` | `string` | 必填 | 申请说明；没有说明时传空字符串。 |
| `joinSource` | `GroupJoinSource` | 必填 | 加入途径，可选 `Invitation`、`Search` 或 `QrCode`。 |
| `ex` | `string` | 可选 | 申请加入时携带的业务扩展字符串。 |

```ts
import { GroupJoinSource } from '@openim/wasm-client-sdk';

await openimsdk.joinGroup(
  {
    groupID,
    reqMsg: '申请加入项目讨论组',
    joinSource: GroupJoinSource.Search,
  }
);
```

`joinSource` 只记录加入途径，不决定是否需要审核。如果群组允许直接加入，OpenIMClientSDK 会同步 `CbEvents.OnJoinedGroupAdded`；需要审核时，Promise 成功只表示申请已经提交，不能立即把当前用户写成群成员。

## 退出群组

普通成员和管理员调用 `quitGroup(groupID)` 退出：

```ts
await openimsdk.quitGroup(groupID);
```

退出后通常触发 `CbEvents.OnJoinedGroupDeleted`。历史会话可能继续保留，但不能再据此判断用户仍在群内。

群主不能直接退出，必须先调用 `transferGroupOwner()` 转让群主身份，再执行 `quitGroup()`。

## 解散群组

只有群主可以调用 `dismissGroup(groupID)`：

```ts
await openimsdk.dismissGroup(groupID);
```

解散会影响所有成员，并触发 `CbEvents.OnGroupDismissed`。它不同于删除或隐藏当前用户的会话；会话操作不会解散群组。

## 调用后的状态变化

| API | Promise 成功后 | 事件处理 |
| --- | --- | --- |
| `joinGroup()` | 直接加入时表示已经加入；需要审核时只表示申请已提交。 | 只有当前账号实际加入后，才通过 `OnJoinedGroupAdded` 写入已加入群组列表。 |
| `quitGroup()` | 当前账号已经退出目标群组。 | 通过 `OnJoinedGroupDeleted` 按 `groupID` 移除群组。 |
| `dismissGroup()` | 群主已经解散目标群组。 | 通过 `OnGroupDismissed` 将群组标记为已解散，并停止群成员管理入口。 |

不要在 `joinGroup()` 的 Promise 成功分支中无条件伪造 `GroupItem` 或 `GroupMemberItem`。这些事件的完整监听、清理和按 `groupID` 合并方式见[获取和搜索群组](/sdk/wasm/group/retrieving-groups/retrieve-and-search-groups)。历史会话是否保留属于会话数据，不应由群组事件处理器直接删除。
