## 创建群组

`createGroup()` 接收群资料以及初始普通成员、管理员和群主配置。

### 参数说明

| 字段            | 类型                 | 是否必填 | 说明                                                |
| --------------- | -------------------- | -------- | --------------------------------------------------- |
| `memberUserIDs` | `string[]`           | 是       | 创建后作为普通成员加入群组的用户 ID。               |
| `adminUserIDs`  | `string[]`           | 否       | 创建后作为管理员加入群组的用户 ID。                 |
| `ownerUserID`   | `string`             | 否       | 群主用户 ID；省略时使用当前登录用户。               |
| `groupInfo`     | `Partial<GroupItem>` | 是       | 群资料，`groupType` 使用 `GroupType.WorkingGroup`。 |

### 初始成员和管理员

`memberUserIDs` 和 `adminUserIDs` 分别表示普通成员与管理员，两组列表不能包含相同的用户 ID。构造参数时应先在客户端业务层完成去重和交集检查，再调用 `createGroup()`。群主通过 `ownerUserID` 单独指定，不要同时放入这两组列表。

```ts
import { AllowType, GroupType, GroupVerificationType } from '@openim/wasm-client-sdk';

const { data: group } = await OpenIM.createGroup(
  {
    memberUserIDs: ['user-003', 'user-004'],
    adminUserIDs: ['user-002'],
    ownerUserID: currentUserID,
    groupInfo: {
      groupName: '项目讨论组',
      groupType: GroupType.WorkingGroup,
      faceURL: 'https://example.com/group.png',
    },
  }
);
```

Promise 成功后，`data` 是新建群组的 `GroupItem`。

需要自定义 `groupID` 时可以在 `groupInfo` 中传入，但必须全局唯一；省略时由服务端生成。

## 更新群资料

`setGroupInfo()` 接收包含 `groupID` 的部分 `GroupItem` 字段。只传需要更新的字段，不要用旧快照覆盖刚同步到本地的新状态。

```ts
await OpenIM.setGroupInfo(
  {
    groupID,
    groupName: '新的群名称',
    notification: '本周五发布',
    introduction: '项目协作群',
    needVerification: GroupVerificationType.AllNeed,
    applyMemberFriend: AllowType.Allowed,
    lookMemberInfo: AllowType.Allowed,
    ex: JSON.stringify({ projectID: 'project-42' }),
  }
);
```

`ex` 是完整的字符串值，更新前必须保留已有业务字段。需要复杂查询、独立权限控制和审计的数据应由业务后端保存，不要依赖客户端遍历群组后解析 `ex`。

## 群权限和验证设置

`setGroupInfo()` 直接通过群资料字段管理以下能力：

| 字段                | 用途                                 |
| ------------------- | ------------------------------------ |
| `applyMemberFriend` | 控制群成员之间是否允许申请添加好友。 |
| `lookMemberInfo`    | 控制群成员资料的查看权限。           |
| `needVerification`  | 设置加入群组时的验证策略。           |

这些字段可以和群名称、公告等资料一起更新，也可以单独传入。不要把未修改字段的旧值一并提交，以免覆盖刚同步到本地的新设置。

SDK 只提供这些权限与验证字段的当前值，不会自动替客户端隐藏按钮、禁用入口或组织操作流程。

客户端应在业务层读取最新的 `GroupItem` 和当前用户的群角色，再根据 `applyMemberFriend`、`lookMemberInfo`、`needVerification` 等字段决定：是否展示相应功能、是否允许继续提交。

若允许提交，再选择直接执行还是进入申请审核流程。服务端仍负责最终权限校验，不能只依赖前端界面限制。

## 调用后的状态变化

- `createGroup()` 成功后，可以立即使用返回的 `GroupItem` 打开群资料或群聊页面。当前账号的已加入群组列表通过 `CbEvents.OnJoinedGroupAdded` 合并新群组。
- `setGroupInfo()` 的 Promise 成功表示服务端已完成本次群资料设置请求。SDK 通过 `CbEvents.OnGroupInfoChanged` 同步变化后的 `GroupItem`，应按 `groupID` 覆盖本地群资料。

两个事件的完整监听和清理方式见[获取和搜索群组](/sdk/wasm/group/retrieving-groups/retrieve-and-search-groups)。如果设置页面需要立即确认服务端最终值，可以在调用成功后使用 `getSpecifiedGroupsInfo([groupID])` 重新查询。重新查询失败不应被误报为群资料设置失败。
