## 获取指定群组

`getSpecifiedGroupsInfo()` 接收 `groupID[]`，返回匹配的 `GroupItem[]`：

```ts
const { data: groups } = await OpenIM.getSpecifiedGroupsInfo(
  groupIDs
);
```

读取单个群组时仍然传数组，并检查 `data[0]` 是否存在。群组没有 URL 或 slug 字段；业务路由应先解析为稳定的 `groupID`。

## 获取已加入群组

群组数量较少时调用无参数的 `getJoinedGroupList()`，Promise 成功后 `data` 是当前用户已加入的全部 `GroupItem[]`。

### 分页获取

群组数量较多时调用 `getJoinedGroupListPage()`。

#### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `offset` | `number` | 必填 | 起始偏移量，第一页传 `0`。 |
| `count` | `number` | 必填 | 本次读取数量。 |

```ts
const { data: firstPage } = await OpenIM.getJoinedGroupListPage({
  offset: 0,
  count: 50,
});
```

这两个方法只返回当前登录用户已加入的群组，不是全应用公开群目录。

## 检查成员状态

使用 `isJoinGroup(groupID)` 检查当前用户是否仍属于目标群组：

```ts
const { data: isJoined } = await OpenIM.isJoinGroup(groupID);
```

返回的 `data` 表示当前用户是否已经加入目标群组。需要校准成员关系时，再重新查询已加入群组列表。

## 搜索已加入群组

`searchGroups()` 只搜索当前用户已加入且已经同步到本地的群组。当前接口只使用 `keywordList` 中的第一个关键词，因此搜索框应传入一个去除首尾空格的非空关键词。

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `keywordList` | `string[]` | 必填 | 搜索关键词；当前只使用第一个元素。 |
| `isSearchGroupID` | `boolean` | 必填 | 是否匹配 `groupID`。 |
| `isSearchGroupName` | `boolean` | 必填 | 是否匹配群名称。 |

```ts
const { data: matchedGroups } = await OpenIM.searchGroups({
  keywordList: [keyword.trim()],
  isSearchGroupID: true,
  isSearchGroupName: true,
});
```

公开群发现、跨业务用户目录搜索、复杂分类和权限过滤必须由业务后端实现。`searchGroups()` 只搜索当前用户已加入并同步到本地的群组。

## 查询快照与群组事件

本页的查询方法只建立调用时的群组快照，不应描述为触发群组事件。群组列表首次进入时通过查询建立快照，之后监听当前用户加入或退出群组、群资料变化和群组解散事件。四个事件都应使用事件数据中的 `groupID` 定位群组：

```ts
import { CbEvents } from '@openim/wasm-client-sdk';

const handlers = {
  joinedAdded: ({ data }) => mergeJoinedGroup(data),
  joinedDeleted: ({ data }) => removeJoinedGroup(data.groupID),
  groupChanged: ({ data }) => mergeGroup(data),
  groupDismissed: ({ data }) => markGroupDismissed(data.groupID),
};

OpenIM.on(CbEvents.OnJoinedGroupAdded, handlers.joinedAdded);
OpenIM.on(CbEvents.OnJoinedGroupDeleted, handlers.joinedDeleted);
OpenIM.on(CbEvents.OnGroupInfoChanged, handlers.groupChanged);
OpenIM.on(CbEvents.OnGroupDismissed, handlers.groupDismissed);

function removeGroupListListeners() {
  OpenIM.off(CbEvents.OnJoinedGroupAdded, handlers.joinedAdded);
  OpenIM.off(CbEvents.OnJoinedGroupDeleted, handlers.joinedDeleted);
  OpenIM.off(CbEvents.OnGroupInfoChanged, handlers.groupChanged);
  OpenIM.off(CbEvents.OnGroupDismissed, handlers.groupDismissed);
}
```

事件可能因重连或多端操作重复到达，应按 `groupID` 幂等合并。如果已加入群组列表仍在继续分页，新增或删除事件会改变后续偏移位置，此时应重置分页并从第一页重新加载。用户主动刷新时，再重新查询当前可见的群组列表；重新登录后的群组变化由事件同步。
