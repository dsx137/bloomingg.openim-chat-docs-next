群组整体禁言和单个成员禁言是两个不同状态。群主和管理员可以按 OpenIMServer 权限执行这些操作。

## 操作权限

- 群主可以禁言管理员和普通成员，也可以开启或关闭群组整体禁言。
- 管理员只能禁言普通成员，不能禁言群主或其他管理员。
- 普通成员不能修改群组或其他成员的禁言状态。

前端可以根据 `GroupMemberRole` 隐藏无权限入口，但最终结果仍以 OpenIMServer 权限校验为准。

## 群组禁言

`changeGroupMute()` 的 `isMute` 为 `true` 时开启群禁言，为 `false` 时关闭：

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `groupID` | `string` | 必填 | 目标群组 ID。 |
| `isMute` | `boolean` | 必填 | `true` 开启群组禁言，`false` 关闭。 |
| `muteBypassUserIDs` | `string[]` | 可选 | 群组禁言期间仍允许发送消息的用户 ID。 |

```ts
await openimsdk.changeGroupMute({
  groupID,
  isMute: true,
  muteBypassUserIDs: [serviceAccountUserID],
});
```

`muteBypassUserIDs` 是可选字段。开启群组禁言时，可以用它列出仍允许发送消息的用户 ID；不需要例外用户时省略该字段。不要把当前群成员列表全部传入，否则会让群禁言失去作用。

开启后普通成员不能发送消息，群主和管理员仍可发送，因此这不表示群内所有用户都无法发送消息。

## 成员禁言

`changeGroupMemberMute()` 使用 `mutedSeconds` 指定时长，传 `0` 解除禁言：

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `groupID` | `string` | 必填 | 目标群组 ID。 |
| `userID` | `string` | 必填 | 要设置禁言状态的成员用户 ID。 |
| `mutedSeconds` | `number` | 必填 | 禁言时长，单位为秒；传 `0` 解除禁言。 |

```ts
await openimsdk.changeGroupMemberMute({
  groupID,
  userID: targetUserID,
  mutedSeconds: 3600,
});
```

成员资料中的 `muteEndTime` 是禁言截止时间。不同历史数据或 OpenIMServer 版本可能使用秒或毫秒，比较前先归一化为毫秒：

```ts
function normalizeMuteEndTime(value: number) {
  const timestamp = Number(value);
  if (!Number.isFinite(timestamp) || timestamp <= 0) return 0;
  return timestamp < 1_000_000_000_000 ? timestamp * 1000 : timestamp;
}

function isMemberMuted(member: GroupMemberItem, now = Date.now()) {
  return normalizeMuteEndTime(member.muteEndTime) > now;
}
```

## 调用后的状态变化

`changeGroupMute()` 的 Promise 成功表示 OpenIMServer 已完成群组禁言设置。相关客户端随后可能收到 `OnGroupInfoChanged`，事件 `data` 是更新后的 `GroupItem`，应按 `groupID` 合并群资料。完整监听和清理代码见[获取和搜索群组](/sdk/wasm/group/retrieving-groups/retrieve-and-search-groups)。

`changeGroupMemberMute()` 的 Promise 成功表示 OpenIMServer 已完成成员禁言设置。`OnGroupMemberInfoChanged` 的 `data` 是更新后的 `GroupMemberItem`，应按 `groupID:userID` 合并成员资料，再根据 OpenIMServer 返回的 `muteEndTime` 计算禁言状态，不要只按按钮提交时的 `mutedSeconds` 推算最终截止时间。完整监听和清理代码见[获取群成员](/sdk/wasm/group/retrieving-group-members/retrieve-group-members)。

两类事件都可能来自其他管理员或其他客户端的操作，不是当前 Promise 的专属完成回调。需要显示当前群资料或成员资料时，再分别调用对应查询方法获取快照；重新登录后的变化由群组事件同步。
