群成员资料由 `GroupMemberItem` 表示。调用者必须已经加入目标群组，才能读取群成员列表。

## 分页获取成员

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `groupID` | `string` | 必填 | 目标群组 ID。 |
| `filter` | `GroupMemberFilter` | 必填 | 成员角色筛选条件。 |
| `offset` | `number` | 必填 | 起始偏移量，第一页传 `0`。 |
| `count` | `number` | 必填 | 本次读取数量。 |

```ts
import { GroupMemberFilter } from '@openim/wasm-client-sdk';

const { data: members } = await openimsdk.getGroupMemberList({
  groupID,
  filter: GroupMemberFilter.All,
  offset: 0,
  count: 50,
});
```

`GroupMemberFilter` 可以筛选全部成员、群主、管理员或普通成员。继续加载时按已经请求的条目数量增加 `offset`，不要使用事件合并后的成员数组长度计算偏移。

## 查询被禁言成员

OpenIMClientSDK 没有“获取被禁言成员列表”的独立方法。需要这类管理视图时，分页读取全部群成员，再根据 `muteEndTime` 过滤。比较前兼容秒和毫秒时间戳：

```ts
function normalizeMuteEndTime(value: number) {
  const timestamp = Number(value);
  if (!Number.isFinite(timestamp) || timestamp <= 0) return 0;
  return timestamp < 1_000_000_000_000 ? timestamp * 1000 : timestamp;
}

function isMemberMuted(member: GroupMemberItem, now = Date.now()) {
  return normalizeMuteEndTime(member.muteEndTime) > now;
}

const mutedMembers = members.filter((member) => isMemberMuted(member));
```

群成员很多时应逐页加载，不要把第一页过滤结果误当作完整名单。需要展示禁言原因、操作人或审核记录时，从业务后端读取；这些字段不在 `GroupMemberItem` 中。

## 获取指定成员

`getSpecifiedGroupMembersInfo()` 返回指定用户的群内资料；只需要判断哪些用户仍在群内时，使用 `getUsersInGroup()` 返回命中的 `userID[]`。

该接口适合刷新消息发送者和群成员资料卡。同一个用户在不同群中的昵称、角色和禁言状态可能不同，本地缓存应使用 `groupID:userID` 作为键，不要用账号级公开资料覆盖 `GroupMemberItem`。

```ts
const { data: selectedMembers } =
  await openimsdk.getSpecifiedGroupMembersInfo({
    groupID,
    userIDList,
  });

for (const member of selectedMembers) {
  groupMembersByKey.set(`${member.groupID}:${member.userID}`, member);
}
```

## 搜索成员和获取 @ 候选人

`searchGroupMembers()` 只搜索指定群组的成员，适合在用户输入关键词时获取 @ 候选人。当前接口只使用 `keywordList` 中第一个关键词；搜索结果应按 `userID` 展示群内昵称，并排除当前产品不允许被提及的成员。

### 参数说明

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `groupID` | `string` | 必填 | 目标群组 ID。 |
| `keywordList` | `string[]` | 必填 | 搜索关键词；当前只使用第一个元素。 |
| `isSearchUserID` | `boolean` | 必填 | 是否匹配成员 `userID`。 |
| `isSearchMemberNickname` | `boolean` | 必填 | 是否匹配群内昵称。 |
| `offset` | `number` | 必填 | 起始偏移量，第一页传 `0`。 |
| `count` | `number` | 必填 | 本次读取数量。 |

```ts
const { data: matchedMembers } = await openimsdk.searchGroupMembers({
  groupID,
  keywordList: [keyword.trim()],
  isSearchUserID: true,
  isSearchMemberNickname: true,
  offset: 0,
  count: 50,
});
```

## 获取群主和管理员

只需要管理角色时，调用 `getGroupMemberOwnerAndAdmin(groupID)`，无需遍历全部成员：

```ts
const { data: ownerAndAdmins } =
  await openimsdk.getGroupMemberOwnerAndAdmin(groupID);
```

## 查询结果与成员变化

本页所有查询 API 的 Promise 成功后，`data` 就是本次查询得到的成员快照。读取列表、指定成员或管理角色不会触发成员事件，界面不需要等待事件后再展示查询结果。

成员加入、退出、被移除或群内资料发生变化时，OpenIMClientSDK 会通过成员事件发送增量。本页是以下事件的完整监听示例归属页：

- `OnGroupMemberAdded`：`data` 是新增的 `GroupMemberItem`，按 `groupID:userID` 写入列表。
- `OnGroupMemberDeleted`：`data` 是被删除的 `GroupMemberItem`，按 `groupID:userID` 从列表移除。
- `OnGroupMemberInfoChanged`：`data` 是变更后的 `GroupMemberItem`，按 `groupID:userID` 覆盖旧值。

```ts
import { CbEvents } from '@openim/wasm-client-sdk';

const handleGroupMemberAdded = ({ data }) => upsertGroupMember(data);
const handleGroupMemberDeleted = ({ data }) => {
  removeGroupMember(data.groupID, data.userID);
};
const handleGroupMemberInfoChanged = ({ data }) => upsertGroupMember(data);

openimsdk.on(CbEvents.OnGroupMemberAdded, handleGroupMemberAdded);
openimsdk.on(CbEvents.OnGroupMemberDeleted, handleGroupMemberDeleted);
openimsdk.on(CbEvents.OnGroupMemberInfoChanged, handleGroupMemberInfoChanged);

function removeGroupMemberListeners() {
  openimsdk.off(CbEvents.OnGroupMemberAdded, handleGroupMemberAdded);
  openimsdk.off(CbEvents.OnGroupMemberDeleted, handleGroupMemberDeleted);
  openimsdk.off(CbEvents.OnGroupMemberInfoChanged, handleGroupMemberInfoChanged);
}
```

组件卸载、退出登录或切换账号时调用 `removeGroupMemberListeners()`。事件到达不表示触发该变化的管理 API 刚刚完成，也可能来自其他管理员或其他客户端。如果成员列表仍在分页，新增或删除事件发生后应重置分页状态并从第一页重新加载；用户主动刷新时，再从第一页读取以建立当前快照，重新登录后的成员变化由事件同步。
