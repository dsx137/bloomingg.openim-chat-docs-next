## 概览

OpenIM 小程序 SDK 适用于小程序和轻量 Web 场景。它在用户、会话、消息、群组和事件模型上与其他 OpenIMClientSDK 保持一致，但不提供本地消息存储，适合客服入口、活动页、后台管理页、临时会话等不依赖本地历史缓存的轻量页面。

## 与 WASM 的关系

如果方法级 API 与 WASM SDK 保持一致，可以参考 WASM 文档理解用户、会话、消息和事件的调用模型。差异主要集中在初始化入口、网络适配、文件上传、平台生命周期，以及本地存储能力。

需要完整本地消息库、会话缓存和历史消息持久化的 Web 客户端，应优先使用 WASM SDK。小程序 SDK 更适合轻量页面，刷新、重新进入或进程被回收后，应通过服务端或 SDK 查询重新拉取必要数据。

## 运行环境注意事项

- 在测试登录前配置允许访问的 request 域名、WebSocket 域名和上传域名。
- 业务逻辑不要依赖 SDK 本地消息数据库或本地会话持久化；页面恢复后需要重新同步必要数据。
- 文件消息需要结合目标平台的文件选择、临时文件路径、上传 API 和文件大小限制测试。
- token 仍应由可信后端签发，客户端只接收短期凭证。
- 如果在 Web 中使用小程序 SDK，应限定在轻量场景；完整 Web IM 客户端仍建议使用 WASM SDK。
