`OnBatchMsgListener` 是 Android SDK 文档中展示 core binding 方法签名时使用的回调类型，完整类名为 `open_im_sdk_callback.OnBatchMsgListener`。core binding 批量消息监听器，用于接收多条新消息和离线消息。

## 回调方法

| 方法                     | 参数                          | 描述                                                                                                                                                                                                           |
| ------------------------ | ----------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| OnRecvNewMessages        | `messageList: string`（JSON） | SDK 在前台批量处理完新消息后触发。core 会过滤正在输入消息，并且只有待通知列表非空时才回调；`messageList` 是按发送顺序排列的 [`Message`](/docs/chat/sdk/v4/android/models/message) JSON 数组。                  |
| OnRecvOfflineNewMessages | `messageList: string`（JSON） | SDK 处于后台时批量同步到新消息，且全局与对应会话接收选项允许接收后触发；被屏蔽或设置为不接收的消息会被过滤。`messageList` 是需要通知业务层的 [`Message`](/docs/chat/sdk/v4/android/models/message) JSON 数组。 |

Go core 优先检查批量消息监听器：设置后，新消息只进入 `OnRecvNewMessages` 或 `OnRecvOfflineNewMessages`，不会再触发单条监听器的 `OnRecvNewMessage`、`OnRecvOfflineNewMessage` 和 `OnRecvOnlineOnlyMessage`。前台批量分支除正在输入消息外不会再区分仅在线消息，业务端不要期待两套新消息回调同时触发。
