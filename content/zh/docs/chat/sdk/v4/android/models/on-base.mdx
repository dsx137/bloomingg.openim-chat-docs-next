`OnBase` 是 Android SDK 的回调接口，完整类名为 `io.openim.android.sdk.listener.OnBase`。Android SDK 异步方法的通用成功和失败回调。

## 回调方法

| 方法        | 参数                       | 触发时机和字段含义                                                                                                                                                                                                                               |
| ----------- | -------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `onError`   | `int code`；`String error` | 异步方法在参数校验、本地数据库、网络请求或结果解析任一阶段失败时触发。`code` 是 SDK 错误码，`error` 是可用于日志和诊断的错误信息；Android 包装层反序列化返回数据失败时会使用 `code=-1`。收到该回调表示本次调用没有可用的成功结果。               |
| `onSuccess` | `T data`                   | 异步方法完成后触发。`T` 由具体方法声明决定，可以是模型对象、模型列表、字符串或无业务数据的占位结果；Android 包装层会先把 core 返回的 JSON 转换为该类型，再在主线程回调。应以具体方法“返回结果”章节为准，不要把不同方法的 `data` 当成同一种结构。 |

同一次异步调用只应进入 `onSuccess` 或 `onError` 之一。Android SDK 的 `OnBase<T>` 是 Java 层通用回调；文档中 [`Base`](/docs/chat/sdk/v4/android/models/base) 则表示 Go core binding 接收 JSON 字符串的底层回调，两者所处层级不同。
