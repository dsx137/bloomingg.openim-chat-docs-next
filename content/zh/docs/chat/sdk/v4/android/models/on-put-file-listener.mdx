`OnPutFileListener` 是 Android SDK 的回调接口，完整类名为 `io.openim.android.sdk.listener.OnPutFileListener`。调用 `uploadFile` 时，SDK 会通过它报告文件分片、哈希计算、分片上传和最终完成状态。

## 回调方法

| 方法                 | 参数                                                   | 触发时机和字段含义                                                                                                                                                                                                                |
| -------------------- | ------------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `open`               | `long size`                                            | SDK 打开待上传文件并取得文件大小后触发。`size` 是文件总字节数。                                                                                                                                                                   |
| `partSize`           | `long partSize`；`long num`                            | SDK 确定分片方案后触发。`partSize` 是每个标准分片的字节数，`num` 是分片总数；最后一个分片可能小于 `partSize`。Android 参数使用 `long`，底层 core 的 `num` 类型为 `int`。                                                          |
| `hashPartProgress`   | `long index`；`long size`；`String partHash`           | 一个分片完成哈希计算后触发。`index` 是从 `0` 开始的分片索引，`size` 是该分片的实际字节数，`partHash` 是该分片的 MD5 值。Android 的 `index` 使用 `long`，底层 core 类型为 `int`。                                                  |
| `hashPartComplete`   | `String partsHash`；`String fileHash`                  | 全部分片完成哈希计算后触发。`partsHash` 是按分片哈希合成的标识，用于查询或续传同一文件；`fileHash` 是完整文件的 MD5 值。                                                                                                          |
| `uploadID`           | `String uploadID`                                      | 服务端为一次分片上传创建标识后触发。`uploadID` 用于后续分片签名、续传和完成上传；若服务端发现相同文件并直接返回 URL，则可能不触发。                                                                                               |
| `uploadPartComplete` | `long index`；`long partSize`；`String partHash`       | 指定分片完成上传且哈希校验通过后触发。`index` 是从 `0` 开始的分片索引，`partSize` 是该分片的实际字节数，`partHash` 是分片 MD5 值。Android 的 `index` 使用 `long`，底层 core 类型为 `int`。                                        |
| `uploadComplete`     | `long fileSize`；`long streamSize`；`long storageSize` | 上传数据流有进展时触发。`fileSize` 是文件总字节数，`streamSize` 是本次任务已读取并上传的累计字节数，`storageSize` 是本次打开上传请求前服务端已保存的字节数，可用于识别续传起点。                                                  |
| `complete`           | `long size`；`String url`；`long type`                 | 文件可用后触发。`size` 是最终文件字节数，`url` 是服务端返回的文件 URL；当前 core 中 `type=0` 表示服务端已存在文件并直接复用，`type=1` 表示从零完成分片上传，`type=2` 表示在已有分片基础上续传完成。Android 使用 `long` 承载该值。 |

这些方法均为阶段通知，上传接口本身的最终成功或失败仍通过 [`OnBase<String>`](/docs/chat/sdk/v4/android/models/on-base) 返回。`OnBase.onSuccess` 收到 core 的 `{"url":"..."}` JSON 字符串，而本监听器的 `complete` 会直接给出 `url`。业务如果只需要最终地址，可以解析 `OnBase` 结果；需要展示进度或记录续传状态时再实现本监听器中的对应方法。
