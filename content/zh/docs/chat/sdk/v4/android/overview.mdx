本 Android 文档以 `open_im_sdk/*.go` 当前导出的 132 个包级方法为准。每个方法均提供独立页面，围绕当前 Android SDK 的调用方式、参数和返回结果说明。

这里记录的是 Android core binding 的真实能力边界。若项目使用上层 Android Manager 类，Manager 方法应映射到本页列出的 core 方法；当上层封装和 core 导出方法不一致时，以 core 导出方法为准。

每个导出方法都有独立页面说明方法签名、用途、关键参数、返回或事件和使用边界；能力概览页只保留目录和调用关系说明。

## 接入模型

| 模型      | 说明                                                                                                                                 |
| --------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| 初始化    | 调用 `InitSDK(listener, operationID, config)`，`config` 是 `sdk_struct.IMConfig` 的 JSON 字符串。                                    |
| 登录态    | 初始化后设置监听器，再调用 `Login(callback, operationID, userID, token)`。退出当前用户用 `Logout`，释放 SDK 全局资源用 `UnInitSDK`。 |
| 异步结果  | 大多数方法通过 [`Base`](/docs/chat/sdk/v4/android/models/base) 回调返回 `OnSuccess(data string)` 或 `OnError(errCode, errMsg)`。     |
| 同步创建  | `Create*Message`、`GetAtAllTag`、`GetConversationIDBySessionType` 直接返回字符串。                                                   |
| JSON 参数 | 列表、结构体和请求对象在 binding 层通常表现为 `string`，实际传入 JSON 字符串。                                                       |
| 示例代码  | 当前 Android API 页不放示例代码，只保留签名、参数语义、返回和注意事项。                                                              |

## 当前未暴露的能力

当前 core 未导出开放群组、投票、定时消息、消息线程、翻译、独立举报、群组元数据和计数器等独立 Android 方法。这些能力不会作为 Android 中文侧栏入口；如后续 SDK 增加导出方法，再按源码补充。

## 相关类型

- [底层通用回调](/docs/chat/sdk/v4/android/models/base)（`Base`）：core binding 异步结果回调，Android 调用层通常映射为 OnBase。
- [消息对象](/docs/chat/sdk/v4/android/models/message)（`Message`）：消息对象，包含消息 ID、发送者、会话、内容元素和状态等信息。
