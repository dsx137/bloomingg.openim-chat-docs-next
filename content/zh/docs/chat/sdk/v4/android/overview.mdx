本 Android 文档以 `open_im_sdk/*.go` 当前导出的 132 个包级方法为准。旧版文档只作为页面组织参考；旧文档里的业务示例代码已移除，避免把与当前 SDK 不一致的调用方式继续暴露给开发者。

这里记录的是 Android core binding 的真实能力边界。若项目使用上层 Android wrapper 或 Manager 类，wrapper 方法应映射到本页列出的 core 方法；当 wrapper 文档和 core 导出方法不一致时，以 core 导出方法为准。

每个导出方法都有独立页面说明方法签名、用途、关键参数、返回或事件和使用边界；能力概览页只保留目录和调用关系说明。

## 接入模型

| 模型      | 说明                                                                                                                                 |
| --------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| 初始化    | 调用 `InitSDK(listener, operationID, config)`，`config` 是 `sdk_struct.IMConfig` 的 JSON 字符串。                                    |
| 登录态    | 初始化后设置监听器，再调用 `Login(callback, operationID, userID, token)`。退出当前用户用 `Logout`，释放 SDK 全局资源用 `UnInitSDK`。 |
| 异步结果  | 大多数方法通过 `Base` 回调返回 `OnSuccess(data string)` 或 `OnError(errCode, errMsg)`。                                              |
| 同步创建  | `Create*Message`、`GetAtAllTag`、`GetConversationIDBySessionType` 直接返回字符串。                                                   |
| JSON 参数 | 列表、结构体和请求对象在 binding 层通常表现为 `string`，实际传入 JSON 字符串。                                                       |
| 示例代码  | 当前 Android API 页不放示例代码，只保留签名、参数语义、返回和注意事项。                                                              |

## 不再沿用的生成内容

当前 core 未导出 open channel、poll、scheduled message、message thread、translation、report、channel metadata、metacounter 等独立 Android 方法。这些旧生成页不会作为 Android 中文侧栏入口；如后续 SDK 增加导出方法，再按源码补充。
