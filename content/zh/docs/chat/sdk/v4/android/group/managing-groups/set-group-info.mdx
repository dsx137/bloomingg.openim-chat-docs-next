更新群资料，例如群名称、头像、公告或扩展字段。

群组方法通常要求当前用户已登录，且群主、管理员、成员权限由服务端校验；群组变化通过 `SetGroupListener` 接收。

## Core Binding 方法

| 项目     | 说明                                                                |
| -------- | ------------------------------------------------------------------- |
| 方法签名 | `SetGroupInfo(callback Base, operationID string, groupInfo string)` |
| 所属能力 | 群组                                                                |
| 源码位置 | `open_im_sdk/group.go`                                              |

## Android 调用参考

Android SDK v3.8.3 的 Java 方法原型如下，示例代码也基于该调用方式。Core binding 方法签名用于说明底层 `open_im_sdk` 能力和参数映射。

```java showLineNumbers
public void setGroupInfo(GroupInfo groupInfo, OnBase<String> callBack)
```

## 输入参数

| 参数名称  | 参数类型                                                   | 是否必填 | 描述     |
| --------- | ---------------------------------------------------------- | -------- | -------- |
| callBack  | [`OnBase`](/docs/chat/sdk/v4/android/models/on-base)       | 是       | 回调接口 |
| groupInfo | [`GroupInfo`](/docs/chat/sdk/v4/android/models/group-info) | 是       | 群 信息  |

## 返回结果

`onSuccess` 表示请求中的群资料字段已通过服务端校验并完成本地群同步，回调字符串不含业务数据；无修改权限、群不存在或字段无效时调用 `onError`。

## 代码示例

以下示例展示 Android SDK v3.8.3 的调用方式。

```java showLineNumbers
OpenIMClient.getInstance().groupManager.setGroupInfo(groupInfo, new OnBase<String>() {
    @Override
    public void onError(int code, String error) {
        // TODO: 处理错误
    }

    @Override
    public void onSuccess(String data) {
        // TODO: 处理成功结果
    }
});
```

## 相关类型

- [底层通用回调](/docs/chat/sdk/v4/android/models/base)（`Base`）：core binding 异步结果回调，Android 调用层通常映射为 OnBase。
- [通用回调](/docs/chat/sdk/v4/android/models/on-base)（`OnBase`）：Android SDK 异步方法的通用成功和失败回调。
- [群组信息](/docs/chat/sdk/v4/android/models/group-info)（`GroupInfo`）：群组资料对象，包含群名、公告、群主、成员数和入群设置。
