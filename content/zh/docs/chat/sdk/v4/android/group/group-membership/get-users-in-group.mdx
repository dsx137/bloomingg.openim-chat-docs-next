查询一批用户是否在指定群组内，适合邀请前过滤已入群用户。

群组方法通常要求当前用户已登录，且群主、管理员、成员权限由服务端校验；群组变化通过 `SetGroupListener` 接收。

## Core Binding 方法

| 项目     | 说明                                                                             |
| -------- | -------------------------------------------------------------------------------- |
| 方法签名 | `GetUsersInGroup(callback Base, operationID string, groupID, userIDList string)` |
| 所属能力 | 群组                                                                             |
| 源码位置 | `open_im_sdk/group.go`                                                           |

## Android 调用参考

Android SDK v3.8.3 的 Java 方法原型如下，示例代码也基于该调用方式。Core binding 方法签名用于说明底层 `open_im_sdk` 能力和参数映射。

```java showLineNumbers
public void getUsersInGroup(OnBase<List<String>> base, String groupID, List<String> userIdList)
```

## 输入参数

| 参数名称   | 参数类型                                                             | 是否必填 | 描述                   |
| ---------- | -------------------------------------------------------------------- | -------- | ---------------------- |
| base       | [`OnBase`](/docs/chat/sdk/v4/android/models/on-base)`<List<String>>` | 是       | 查询结果回调。         |
| groupID    | String                                                               | 是       | 群 ID。                |
| userIdList | `List<String>`                                                       | 是       | 待检查的用户 ID 列表。 |

## 返回结果

`onSuccess` 返回输入 `userIdList` 中实际存在于目标群的用户 ID 列表，并保持输入遍历顺序；不在群内的用户不会返回，当前用户本身不在该群时底层可能返回 `null`。查询失败时调用 `onError`。

## 代码示例

以下示例展示 Android SDK v3.8.3 的调用方式。

```java showLineNumbers
OpenIMClient.getInstance().groupManager.getUsersInGroup(
    new OnBase<List<String>>() {
    @Override
    public void onError(int code, String error) {
        // TODO: 处理错误
    }

    @Override
    public void onSuccess(List<String> data) {
        // TODO: 处理在群内的用户 ID 列表
    }
},
    groupID,
    userIdList
);
```
