获取当前用户作为群管理者收到的入群申请列表。

群组方法通常要求当前用户已登录，且群主、管理员、成员权限由服务端校验；群组变化通过 `SetGroupListener` 接收。

## Core Binding 方法

| 项目     | 说明                                                                         |
| -------- | ---------------------------------------------------------------------------- |
| 方法签名 | `GetGroupApplicationListAsRecipient(callback Base, operationID, req string)` |
| 所属能力 | 群组                                                                         |
| 源码位置 | `open_im_sdk/group.go`                                                       |

## Android 调用参考

Android SDK v3.8.3 的 Java 方法原型如下，示例代码也基于该调用方式。Core binding 方法签名用于说明底层 `open_im_sdk` 能力和参数映射。

```java showLineNumbers
public void getRecvGroupApplicationList(OnBase<List<GroupApplicationInfo>> base, GetGroupApplicationListAsRecipientReq recipientReq)
```

## 输入参数

| 参数名称     | 参数类型                                                                                                                                          | 是否必填 | 描述                 |
| ------------ | ------------------------------------------------------------------------------------------------------------------------------------------------- | -------- | -------------------- |
| base         | [`OnBase`](/docs/chat/sdk/v4/android/models/on-base)`<List<`[`GroupApplicationInfo`](/docs/chat/sdk/v4/android/models/group-application-info)`>>` | 是       | 查询结果回调。       |
| recipientReq | [`GetGroupApplicationListAsRecipientReq`](/docs/chat/sdk/v4/android/models/get-group-application-list-as-recipient-req)                           | 是       | 分页和筛选请求参数。 |

## 返回结果

`onSuccess` 返回 `List<`[`GroupApplicationInfo`](/docs/chat/sdk/v4/android/models/group-application-info)`>`，只包含当前页需要当前用户查看或处理的收到申请；查询或反序列化失败时调用 `onError`。

## 代码示例

以下示例展示 Android SDK v3.8.3 的调用方式。

```java showLineNumbers
OpenIMClient.getInstance().groupManager.getRecvGroupApplicationList(
    new OnBase<List<GroupApplicationInfo>>() {
    @Override
    public void onError(int code, String error) {
        // TODO: 处理错误
    }

    @Override
    public void onSuccess(List<GroupApplicationInfo> data) {
        // TODO: 处理收到的入群申请列表
    }
},
    recipientReq
);
```

## 相关类型

- [底层通用回调](/docs/chat/sdk/v4/android/models/base)（`Base`）：core binding 异步结果回调，Android 调用层通常映射为 OnBase。
- [通用回调](/docs/chat/sdk/v4/android/models/on-base)（`OnBase`）：Android SDK 异步方法的通用成功和失败回调。
- [群申请信息](/docs/chat/sdk/v4/android/models/group-application-info)（`GroupApplicationInfo`）：入群申请记录，包含群资料、申请人、处理状态和来源。
- [收到的群申请查询条件](/docs/chat/sdk/v4/android/models/get-group-application-list-as-recipient-req)（`GetGroupApplicationListAsRecipientReq`）：查询当前用户收到的入群申请时使用的分页和过滤条件。
