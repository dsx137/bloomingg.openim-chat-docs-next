## 功能介绍

通知 SDK 应用进入前台或后台，便于 SDK 调整连接、同步和推送相关行为。

初始化类方法应在应用进程内按需调用；用户态方法通常要求 `InitSDK` 已成功执行，并通过 `operationID` 串联日志。

## Core Binding 方法

| 项目     | 说明                                                                           |
| -------- | ------------------------------------------------------------------------------ |
| 方法签名 | `SetAppBackgroundStatus(callback Base, operationID string, isBackground bool)` |
| 所属能力 | 初始化与登录                                                                   |
| 源码位置 | `open_im_sdk/init_login.go`                                                    |

## Android 调用参考

当前 Android SDK v3.8.3.5 未提供该 core 方法对应的公开 Java 调用方法。Core binding 方法签名仅用于说明底层 `open_im_sdk` 能力。

## 输入参数

| 参数名称       | 类型     | 描述                       |
| -------------- | -------- | -------------------------- |
| `callback`     | `Base`   | 异步结果回调。             |
| `operationID`  | `string` | 本次操作的追踪 ID。        |
| `isBackground` | `bool`   | 按 core binding 签名传入。 |

## 返回结果

通过回调返回结果：成功走 `OnSuccess`，失败走 `OnError`；发送或上传类方法还可能通过进度回调返回进度。
