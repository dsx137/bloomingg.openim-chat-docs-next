通知 SDK 网络状态已变化，当前实现会触发长连接关闭并进入后续重连处理。

初始化类方法应在应用进程内按需调用；用户态方法通常要求 `InitSDK` 已成功执行，并通过 `operationID` 串联日志。

## Core Binding 方法

| 项目     | 说明                                                      |
| -------- | --------------------------------------------------------- |
| 方法签名 | `NetworkStatusChanged(callback Base, operationID string)` |
| 所属能力 | SDK 生命周期                                              |
| 源码位置 | `open_im_sdk/init_login.go`                               |

## Android 调用参考

Android SDK v3.8.3 的 Java 方法原型如下，示例代码也基于该调用方式。Core binding 方法签名用于说明底层 `open_im_sdk` 能力和参数映射。

```java showLineNumbers
public void networkChanged()
```

## 输入参数

无。

## 返回结果

无返回值。SDK 在内部把网络变化通知给 core，连接恢复结果通过连接监听器返回。

## 代码示例

以下示例展示 Android SDK v3.8.3 的调用方式。

```java showLineNumbers
OpenIMClient.getInstance().networkChanged();
```
