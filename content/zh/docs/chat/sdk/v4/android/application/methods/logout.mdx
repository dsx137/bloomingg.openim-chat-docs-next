退出当前登录用户，断开当前用户态会话并清理登录相关状态。它不等同于销毁 SDK 全局实例。

初始化类方法应在应用进程内按需调用；用户态方法通常要求 `InitSDK` 已成功执行，并通过 `operationID` 串联日志。

## Core Binding 方法

| 项目     | 说明                                        |
| -------- | ------------------------------------------- |
| 方法签名 | `Logout(callback Base, operationID string)` |
| 所属能力 | SDK 生命周期                                |
| 源码位置 | `open_im_sdk/init_login.go`                 |

## Android 调用参考

Android SDK v3.8.3 的 Java 方法原型如下，示例代码也基于该调用方式。Core binding 方法签名用于说明底层 `open_im_sdk` 能力和参数映射。

```java showLineNumbers
public void logout(OnBase<String> base)
```

## 输入参数

| 参数名称 | Java 类型                                                      | 是否必填 | 描述               |
| -------- | -------------------------------------------------------------- | -------- | ------------------ |
| base     | [`OnBase`](/docs/chat/sdk/v4/android/models/on-base)`<String>` | 是       | 退出登录结果回调。 |

## 返回结果

`onSuccess` 表示当前账号已退出并结束用户态连接，回调字符串不含业务数据；退出失败时，`onError` 返回错误码和描述。退出成功后如需销毁全局 SDK，还应单独调用 `unInit`。

## 代码示例

以下示例展示 Android SDK v3.8.3 的调用方式。

```java showLineNumbers
OpenIMClient.getInstance().logout(new OnBase<String>() {
    @Override
    public void onError(int code, String error) {
        // TODO: 处理错误
    }

    @Override
    public void onSuccess(String data) {
        // TODO: 登出成功
    }
});
```

## 相关类型

- [通用回调](/docs/chat/sdk/v4/android/models/on-base)（`OnBase`）：Android SDK 异步方法的通用成功和失败回调。
