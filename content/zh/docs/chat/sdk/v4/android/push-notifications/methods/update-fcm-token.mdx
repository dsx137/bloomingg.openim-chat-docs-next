更新当前设备的 FCM token，用于服务端向 Android 设备发送离线推送。

推送通知能力至少要求 SDK 已初始化。

## Core Binding 方法

| 项目     | 说明                                                                            |
| -------- | ------------------------------------------------------------------------------- |
| 方法签名 | `UpdateFcmToken(callback Base, operationID, fcmToken string, expireTime int64)` |
| 所属能力 | 推送通知                                                                        |
| 源码位置 | `open_im_sdk/third.go`                                                          |

## Android 调用参考

Android SDK v3.8.3 的 Java 方法原型如下，示例代码也基于该调用方式。Core binding 方法签名用于说明底层 `open_im_sdk` 能力和参数映射。

```java showLineNumbers
public void updateFcmToken(OnBase<String> base, String fcmToken, long expireTime)
```

## 输入参数

| 参数名称   | Java 类型                                                      | 是否必填 | 描述                             |
| ---------- | -------------------------------------------------------------- | -------- | -------------------------------- |
| base       | [`OnBase`](/docs/chat/sdk/v4/android/models/on-base)`<String>` | 是       | FCM token 更新结果回调。         |
| fcmToken   | `String`                                                       | 是       | Firebase Cloud Messaging token。 |
| expireTime | `long`                                                         | 是       | 该 token 的过期时间戳。          |

## 返回结果

`onSuccess` 表示 FCM token、当前用户、平台和过期时间已提交给服务端，回调字符串不含业务数据；token 为空、未登录或网络请求失败时调用 `onError`。该方法不负责向 Firebase 申请 token。

## 代码示例

以下示例展示 Android SDK v3.8.3 的调用方式。

```java showLineNumbers
OpenIMClient.getInstance().updateFcmToken(new OnBase<String>() {
    @Override
    public void onError(int code, String error) {
        // TODO: 处理错误
    }

    @Override
    public void onSuccess(String data) {
        // TODO: 处理成功结果
    }
}, fcmToken, expireTime);
```

## 相关类型

- [底层通用回调](/docs/chat/sdk/v4/android/models/base)（`Base`）：core binding 异步结果回调，Android 调用层通常映射为 OnBase。
- [通用回调](/docs/chat/sdk/v4/android/models/on-base)（`OnBase`）：Android SDK 异步方法的通用成功和失败回调。
