向 OpenIM 服务端同步当前用户的应用未读数，供后续推送角标策略使用；该方法不直接修改 Android launcher 图标角标。

推送通知能力至少要求 SDK 已初始化。

## Core Binding 方法

| 项目     | 说明                                                                   |
| -------- | ---------------------------------------------------------------------- |
| 方法签名 | `SetAppBadge(callback Base, operationID string, appUnreadCount int32)` |
| 所属能力 | 推送通知                                                               |
| 源码位置 | `open_im_sdk/third.go`                                                 |

## Android 调用参考

当前 Android SDK v3.8.3 未公开该 core 方法对应的 Java 调用入口。底层请求会自动带上当前登录用户 ID，并把 `appUnreadCount` 提交到 `/third/set_app_badge`；它只更新服务端状态，不调用 Android 厂商桌面角标 API。

## 输入参数

| 参数名称         | 类型                                            | 描述                                                           |
| ---------------- | ----------------------------------------------- | -------------------------------------------------------------- |
| `callback`       | [`Base`](/docs/chat/sdk/v4/android/models/base) | 异步结果回调。                                                 |
| `operationID`    | `string`                                        | 本次操作的追踪 ID。                                            |
| `appUnreadCount` | `int32`                                         | 应用当前希望同步到服务端的总未读数。应传非负值，清零时传 `0`。 |

## 返回结果

服务端接受未读数后调用底层 `OnSuccess`，请求失败时调用 `OnError`，没有业务返回数据。Android v3.8.3 没有公开 Java 入口，因此本页不提供无法运行的调用示例；launcher 角标仍需由应用使用目标厂商或系统支持的 Android API 单独处理。
