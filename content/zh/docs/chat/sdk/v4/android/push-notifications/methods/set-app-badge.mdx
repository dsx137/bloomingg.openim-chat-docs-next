设置应用角标未读数，适合未读数变化后同步系统角标。

## 功能介绍

设置应用角标未读数，适合未读数变化后同步系统角标。

第三方能力至少要求 SDK 已初始化；日志和文件上传的进度通过对应 progress callback 返回。

## Core Binding 方法

| 项目     | 说明                                                                   |
| -------- | ---------------------------------------------------------------------- |
| 方法签名 | `SetAppBadge(callback Base, operationID string, appUnreadCount int32)` |
| 所属能力 | 推送、日志与文件                                                       |
| 源码位置 | `open_im_sdk/third.go`                                                 |

## Android 调用参考

当前 Android SDK v3.8.3.5 未提供该 core 方法对应的公开 Java 调用方法。Core binding 方法签名仅用于说明底层 `open_im_sdk` 能力。

## 输入参数

| 参数名称         | 类型                                            | 描述                       |
| ---------------- | ----------------------------------------------- | -------------------------- |
| `callback`       | [`Base`](/docs/chat/sdk/v4/android/models/base) | 异步结果回调。             |
| `operationID`    | `string`                                        | 本次操作的追踪 ID。        |
| `appUnreadCount` | `int32`                                         | 按 core binding 签名传入。 |

## 返回结果

通过回调返回结果：成功走 `OnSuccess`，失败走 `OnError`；发送或上传类方法还可能通过进度回调返回进度。
