获取未处理好友申请数量，适合好友入口角标和后台同步。

好友关系变化通过 `SetFriendListener` 设置的 `OnFriendshipListener` 接收；涉及列表和请求对象的字符串参数应传入对应结构的 JSON。

## Core Binding 方法

| 项目     | 说明                                                                         |
| -------- | ---------------------------------------------------------------------------- |
| 方法签名 | `GetFriendApplicationUnhandledCount(callback Base, operationID, req string)` |
| 所属能力 | 好友                                                                         |
| 源码位置 | `open_im_sdk/relation.go`                                                    |

## Android 调用参考

当前 Android SDK v3.8.3 未公开该 core 方法对应的 Java 调用入口。下列参数描述的是底层 JSON 契约，用于核对版本能力，不能作为该版本可直接调用的 Java 方法。

## 输入参数

| 参数名称      | 类型                                            | 描述                                                                                                             |
| ------------- | ----------------------------------------------- | ---------------------------------------------------------------------------------------------------------------- |
| `callback`    | [`Base`](/docs/chat/sdk/v4/android/models/base) | 异步结果回调。                                                                                                   |
| `operationID` | `string`                                        | 本次操作的追踪 ID。                                                                                              |
| `req`         | `string`                                        | JSON 字符串，结构为 `{"time": long}`；`time` 是统计未处理好友申请的起算时间，core 会将该整数值原样提交给服务端。 |

## 返回结果

成功时底层 `OnSuccess` 返回 `int32` 对应的 JSON 整数，即从 `time` 起仍未处理的好友申请数量；失败时调用 `OnError`。Android v3.8.3 没有公开 Java 入口，因此本页不提供无法运行的调用示例。

- [好友关系监听器](/docs/chat/sdk/v4/android/models/on-friendship-listener)（`OnFriendshipListener`）：好友、好友申请和黑名单变化监听器。
