搜索好友，适合按用户 ID、昵称、备注等当前搜索结构支持的字段查找好友。

## 功能介绍

搜索好友，适合按用户 ID、昵称、备注等当前搜索结构支持的字段查找好友。

好友关系变化通过 `SetFriendListener` 设置的 `OnFriendshipListener` 接收；涉及列表和请求对象的字符串参数应传入对应结构的 JSON。

## Core Binding 方法

| 项目     | 说明                                                                   |
| -------- | ---------------------------------------------------------------------- |
| 方法签名 | `SearchFriends(callback Base, operationID string, searchParam string)` |
| 所属能力 | 好友                                                                   |
| 源码位置 | `open_im_sdk/relation.go`                                              |

## Android 调用参考

Android SDK v3.8.3.5 的 Java 方法原型如下，示例代码也基于该调用方式。Core binding 方法签名用于说明底层 `open_im_sdk` 能力和参数映射。

```java showLineNumbers
public void searchFriends(OnBase<List<FriendInfo>> base, List<String> keywordList, boolean isSearchUserID, boolean isSearchNickname, boolean isSearchRemark)
```

## 输入参数

| 参数名称         | 参数类型                 | 是否必填 | 描述                                           |
| ---------------- | ------------------------ | -------- | ---------------------------------------------- |
| callBack         | OnBase<List<FriendInfo>> | 是       | 回调接口                                       |
| keywordList      | List<String>             | 是       | 搜索关键词，目前仅支持一个关键词搜索，不能为空 |
| isSearchUserID   | Boolean                  | 是       | 是否以关键词搜索 UserID                        |
| isSearchNickname | Boolean                  | 是       | 是否以关键词搜索昵称，默认 false               |
| isSearchRemark   | Boolean                  | 是       | 是否以关键词搜索备注，默认 false               |

## 返回结果

通过回调返回结果：成功走 `OnSuccess`，失败走 `OnError`；发送或上传类方法还可能通过进度回调返回进度。

## 代码示例

以下示例展示 Android SDK v3.8.3.5 的调用方式。

```java showLineNumbers
OpenIMClient.getInstance().friendshipManager.searchFriends(
    new OnBase<List<FriendInfo>>() {
    @Override
    public void onError(int code, String error) {
        // TODO: 处理错误
    }

    @Override
    public void onSuccess(List<FriendInfo> data) {
        // TODO: 处理好友搜索结果
    }
},
    keywordList,
    isSearchUserID,
    isSearchNickname,
    isSearchRemark
);
```
