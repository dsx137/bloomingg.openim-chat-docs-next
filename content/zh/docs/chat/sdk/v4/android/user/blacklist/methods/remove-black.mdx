将指定用户移出黑名单，恢复与该用户的普通关系能力。

黑名单新增和移除通过 `SetFriendListener` 设置的 `OnFriendshipListener` 接收；屏蔽后的业务效果以当前服务端策略为准。

## Core Binding 方法

| 项目     | 说明                                                                  |
| -------- | --------------------------------------------------------------------- |
| 方法签名 | `RemoveBlack(callback Base, operationID string, removeUserID string)` |
| 所属能力 | 黑名单                                                                |
| 源码位置 | `open_im_sdk/relation.go`                                             |

## Android 调用参考

Android SDK v3.8.3 的 Java 方法原型如下，示例代码也基于该调用方式。Core binding 方法签名用于说明底层 `open_im_sdk` 能力和参数映射。

```java showLineNumbers
public void removeBlacklist(OnBase<String> base, String uid)
```

## 输入参数

| 参数名称 | 参数类型                                             | 是否必填 | 描述     |
| -------- | ---------------------------------------------------- | -------- | -------- |
| callBack | [`OnBase`](/docs/chat/sdk/v4/android/models/on-base) | 是       | 回调接口 |
| uid      | String                                               | 是       | 用户 ID  |

## 返回结果

`onSuccess` 表示目标用户已移出当前用户黑名单并完成本地关系同步，回调字符串不含业务数据；目标不存在或服务端请求失败时调用 `onError`。关系变化也会通过好友监听器通知。

## 代码示例

以下示例展示 Android SDK v3.8.3 的调用方式。

```java showLineNumbers
OpenIMClient.getInstance().friendshipManager.removeBlacklist(new OnBase<String>() {
    @Override
    public void onError(int code, String error) {
        // TODO: 处理错误
    }

    @Override
    public void onSuccess(String data) {
        // TODO: 处理成功结果
    }
}, uid);
```
