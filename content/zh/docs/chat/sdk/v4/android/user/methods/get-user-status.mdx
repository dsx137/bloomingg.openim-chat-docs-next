获取指定用户在线状态。当前源码实现复用订阅状态接口，因此调用后也可能产生订阅效果。

用户资料和在线状态方法通常要求当前用户已登录；在线状态变化通过 `SetUserListener` 设置的监听器接收。

## Core Binding 方法

| 项目     | 说明                                                               |
| -------- | ------------------------------------------------------------------ |
| 方法签名 | `GetUserStatus(callback Base, operationID string, userIDs string)` |
| 所属能力 | 用户                                                               |
| 源码位置 | `open_im_sdk/online.go`                                            |

## Android 调用参考

Android SDK v3.8.3 的 Java 方法原型如下，示例代码也基于该调用方式。Core binding 方法签名用于说明底层 `open_im_sdk` 能力和参数映射。

```java showLineNumbers
public void getUserStatus(OnBase<List<UsersOnlineStatus>> callback, List<String> ids)
```

## 输入参数

| 参数名称 | Java 类型                                                                                                                                   | 是否必填 | 描述                       |
| -------- | ------------------------------------------------------------------------------------------------------------------------------------------- | -------- | -------------------------- |
| callback | [`OnBase`](/docs/chat/sdk/v4/android/models/on-base)`<List<`[`UsersOnlineStatus`](/docs/chat/sdk/v4/android/models/users-online-status)`>>` | 是       | 用户在线状态查询结果回调。 |
| ids      | `List<String>`                                                                                                                              | 是       | 要查询的用户 ID 列表。     |

## 返回结果

`onSuccess` 返回 `List<`[`UsersOnlineStatus`](/docs/chat/sdk/v4/android/models/users-online-status)`>`，每项给出用户当前在线平台；当前 core 实现会同时订阅这些用户的状态，查询或订阅失败时调用 `onError`。

## 代码示例

以下示例展示 Android SDK v3.8.3 的调用方式。

```java showLineNumbers
OpenIMClient.getInstance().userInfoManager.getUserStatus(new OnBase<List<UsersOnlineStatus>>() {
    @Override
    public void onError(int code, String error) {
        // TODO: 处理错误
    }

    @Override
    public void onSuccess(List<UsersOnlineStatus> data) {
        // TODO: 处理用户在线状态
    }
}, ids);
```
