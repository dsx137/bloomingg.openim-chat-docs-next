更新当前登录用户资料，例如昵称、头像或扩展字段。只应修改当前账号允许自行修改的字段。

用户资料和在线状态方法通常要求当前用户已登录；在线状态变化通过 `SetUserListener` 设置的监听器接收。

## Core Binding 方法

| 项目     | 说明                                                              |
| -------- | ----------------------------------------------------------------- |
| 方法签名 | `SetSelfInfo(callback Base, operationID string, userInfo string)` |
| 所属能力 | 用户                                                              |
| 源码位置 | `open_im_sdk/user.go`                                             |

## Android 调用参考

Android SDK v3.8.3 的 Java 方法原型如下，示例代码也基于该调用方式。Core binding 方法签名用于说明底层 `open_im_sdk` 能力和参数映射。

```java showLineNumbers
public void setSelfInfo(OnBase<String> base, UserInfoReq userInfoReq)
public void setSelfInfo(OnBase<String> base, String nickname, String faceURL, int gender,
                        int appMangerLevel, String phoneNumber, long birth,
                        String email, String ex)
```

## 输入参数

推荐使用接收 `UserInfoReq` 的重载；Android SDK 只会序列化其中非空字段。

| 参数名称    | Java 类型                                                       | 是否必填 | 描述                     |
| ----------- | --------------------------------------------------------------- | -------- | ------------------------ |
| base        | [`OnBase`](/docs/chat/sdk/v4/android/models/on-base)`<String>`  | 是       | 用户资料更新结果回调。   |
| userInfoReq | [`UserInfoReq`](/docs/chat/sdk/v4/android/models/user-info-req) | 是       | 本次需要更新的资料字段。 |

另一个重载会序列化 `nickname`、`faceURL`、`gender`、`appMangerLevel`、`phoneNumber`、`birth`、`email` 和 `ex`。但当前 core 的 `UserInfoWithEx` 只接收 `nickname`、`faceURL`、`ex` 和 `globalRecvMsgOpt`，其中 `gender`、`appMangerLevel`、`phoneNumber`、`birth`、`email` 会作为未知 JSON 字段被忽略；新代码应使用 `UserInfoReq` 重载，并只依赖当前 core 明确支持的字段。

## 返回结果

`onSuccess` 表示当前 core 支持的用户资料字段已提交并完成本地自身资料同步，回调字符串不含业务数据；字段无效或服务端请求失败时调用 `onError`。资料变化会通过用户监听器继续通知业务层。

## 代码示例

以下示例展示 Android SDK v3.8.3 的调用方式。

```java showLineNumbers
OpenIMClient.getInstance().userInfoManager.setSelfInfo(new OnBase<String>() {
    @Override
    public void onError(int code, String error) {
        // TODO: 处理错误
    }

    @Override
    public void onSuccess(String data) {
        // TODO: 处理成功结果
    }
}, userInfoReq);
```
