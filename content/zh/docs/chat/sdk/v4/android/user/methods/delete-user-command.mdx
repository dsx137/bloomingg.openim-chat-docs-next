删除指定类型和 UUID 的用户命令数据，用于撤销收藏、移除用户侧命令或清理本地同步数据。

用户资料和在线状态方法通常要求当前用户已登录；在线状态变化通过 `SetUserListener` 设置的监听器接收。

## Core Binding 方法

| 项目     | 说明                                                                            |
| -------- | ------------------------------------------------------------------------------- |
| 方法签名 | `DeleteUserCommand(callback Base, operationID string, Type int32, uuid string)` |
| 所属能力 | 用户                                                                            |
| 源码位置 | `open_im_sdk/user.go`                                                           |

## Android 调用参考

当前 Android SDK v3.8.3 未公开该方法的 Java 包装。提供的 Go core 源码还存在调用契约不一致：导出函数把 `Type`、`uuid` 作为两个独立参数传给内部 `ProcessUserCommandDelete`，而内部函数实际只接收一个 `ProcessUserCommandDeleteReq` 对象。按当前源码调用会在反射参数校验阶段失败，不能据此编写可运行的 Android 示例。

## 输入参数

| 参数名称      | 类型                                            | 描述                                                                                     |
| ------------- | ----------------------------------------------- | ---------------------------------------------------------------------------------------- |
| `callback`    | [`Base`](/docs/chat/sdk/v4/android/models/base) | 异步结果回调。                                                                           |
| `operationID` | `string`                                        | 本次操作的追踪 ID。                                                                      |
| `Type`        | `int32`                                         | 要删除记录的业务分类值，必须与新增记录时使用的 `Type` 一致；当前 core 没有公开固定枚举。 |
| `uuid`        | `string`                                        | 要删除记录的唯一标识；core 使用 `Type` 与 `uuid` 共同定位命令记录。                      |

## 返回结果

按内部函数的预期契约，服务端删除记录并完成本地命令同步后调用 `OnSuccess`，失败时调用 `OnError`，没有业务返回数据。但当前导出签名无法正确构造内部请求，且 Android v3.8.3 没有公开 Java 包装；应使用已修复该契约的 SDK 版本后再接入。
