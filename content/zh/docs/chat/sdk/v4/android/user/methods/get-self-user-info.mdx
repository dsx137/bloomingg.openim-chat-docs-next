获取当前登录用户自己的资料，适合进入个人资料页、启动后刷新当前用户信息或账号切换后校验本地缓存。

## 功能介绍

获取当前登录用户自己的资料，适合进入个人资料页、启动后刷新当前用户信息或账号切换后校验本地缓存。

用户资料和在线状态方法通常要求当前用户已登录；在线状态变化通过 `SetUserListener` 设置的监听器接收。

## Core Binding 方法

| 项目     | 说明                                                 |
| -------- | ---------------------------------------------------- |
| 方法签名 | `GetSelfUserInfo(callback Base, operationID string)` |
| 所属能力 | 用户                                                 |
| 源码位置 | `open_im_sdk/user.go`                                |

## Android 调用参考

Android SDK v3.8.3.5 的 Java 方法原型如下，示例代码也基于该调用方式。Core binding 方法签名用于说明底层 `open_im_sdk` 能力和参数映射。

```java showLineNumbers
public void getSelfUserInfo(OnBase<UserInfo> base)
```

## 输入参数

| 参数名称 | 参数类型         | 是否必填 | 描述     |
| -------- | ---------------- | -------- | -------- |
| callBack | OnBase<UserInfo> | 是       | 回调接口 |

## 返回结果

通过回调返回结果：成功走 `OnSuccess`，失败走 `OnError`；发送或上传类方法还可能通过进度回调返回进度。

## 代码示例

以下示例展示 Android SDK v3.8.3.5 的调用方式。

```java showLineNumbers
OpenIMClient.getInstance().userInfoManager.getSelfUserInfo(new OnBase<UserInfo>() {
    @Override
    public void onError(int code, String error) {
        // TODO: 处理错误
    }

    @Override
    public void onSuccess(UserInfo data) {
        // TODO: 处理当前用户资料
    }
});
```
