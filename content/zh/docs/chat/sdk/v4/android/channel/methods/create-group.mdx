创建一个 OpenIM 群组。适合发起群聊、创建项目群或从业务组织关系生成群。

## 功能介绍

创建一个 OpenIM 群组。适合发起群聊、创建项目群或从业务组织关系生成群。

群组方法通常要求当前用户已登录，且群主、管理员、成员权限由服务端校验；群组变化通过 `SetGroupListener` 接收。

## Core Binding 方法

| 项目     | 说明                                                                  |
| -------- | --------------------------------------------------------------------- |
| 方法签名 | `CreateGroup(callback Base, operationID string, groupReqInfo string)` |
| 所属能力 | 群组                                                                  |
| 源码位置 | `open_im_sdk/group.go`                                                |

## Android 调用参考

Android SDK v3.8.3.5 的 Java 方法原型如下，示例代码也基于该调用方式。Core binding 方法签名用于说明底层 `open_im_sdk` 能力和参数映射。

```java showLineNumbers
public void createGroup(List<String> memberUserIDs, List<String> adminUserIDs,
                        GroupInfo groupInfo, String ownerUserID, OnBase<GroupInfo> callBack)
```

## 输入参数

| 参数名称      | 参数类型          | 是否必填 | 描述                           |
| ------------- | ----------------- | -------- | ------------------------------ |
| memberUserIDs | List<String>      | 是       | 初始化群成员 ID 列表。         |
| adminUserIDs  | List<String>      | 否       | 初始化管理员 ID 列表。         |
| groupInfo     | GroupInfo         | 是       | 群组基础资料。                 |
| ownerUserID   | String            | 否       | 群主用户 ID。                  |
| callBack      | OnBase<GroupInfo> | 是       | 创建结果回调，成功返回群资料。 |

## 返回结果

通过回调返回结果：成功走 `OnSuccess`，失败走 `OnError`；发送或上传类方法还可能通过进度回调返回进度。

## 代码示例

以下示例展示 Android SDK v3.8.3.5 的调用方式。

```java showLineNumbers
OpenIMClient.getInstance().groupManager.createGroup(
    memberUserIDs,
    adminUserIDs,
    groupInfo,
    ownerUserID,
    new OnBase<GroupInfo>() {
    @Override
    public void onError(int code, String error) {
        // TODO: 处理错误
    }

    @Override
    public void onSuccess(GroupInfo data) {
        // TODO: 创建成功，处理群资料
    }
}
);
```
