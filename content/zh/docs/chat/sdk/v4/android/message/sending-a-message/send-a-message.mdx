发送消息由两个阶段组成：先用 `Create*Message` 同步创建消息 JSON，再把该消息 JSON 传给 `SendMessage` 或 `SendMessageNotOss`。本页只说明关系和入口，具体方法说明见独立页面。

## 相关方法

| 方法                                                                                                              | 说明                                                                                                           |
| ----------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------- |
| [`GetAtAllTag`](/docs/chat/sdk/v4/android/message/methods/get-at-all-tag)                                         | 获取群聊 @ 所有人使用的特殊标记。发送 @ 所有人消息时，应使用该标记而不是业务侧随意拼接字符串。                 |
| [`CreateTextMessage`](/docs/chat/sdk/v4/android/message/methods/create-text-message)                              | 创建普通文本消息。适合纯文本聊天内容，发送前可由业务层先完成长度、空白和敏感词处理。                           |
| [`CreateAdvancedTextMessage`](/docs/chat/sdk/v4/android/message/methods/create-advanced-text-message)             | 创建带实体信息的文本消息。适合文本中需要携带链接、富文本实体或业务范围标记的场景。                             |
| [`CreateTextAtMessage`](/docs/chat/sdk/v4/android/message/methods/create-text-at-message)                         | 创建群聊 @ 文本消息。可提醒指定成员，也可以配合引用消息展示上下文。                                            |
| [`CreateLocationMessage`](/docs/chat/sdk/v4/android/message/methods/create-location-message)                      | 创建位置消息。适合发送地图位置、门店位置或业务地点。                                                           |
| [`CreateCustomMessage`](/docs/chat/sdk/v4/android/message/methods/create-custom-message)                          | 创建自定义消息。适合订单卡片、邀请卡片、审批状态等 SDK 没有内置内容类型的业务消息。                            |
| [`CreateQuoteMessage`](/docs/chat/sdk/v4/android/message/methods/create-quote-message)                            | 创建引用消息。用于回复某条历史消息并携带新的文本内容。                                                         |
| [`CreateAdvancedQuoteMessage`](/docs/chat/sdk/v4/android/message/methods/create-advanced-quote-message)           | 创建带实体信息的引用消息。适合引用回复中仍需要链接、富文本实体或业务标记。                                     |
| [`CreateCardMessage`](/docs/chat/sdk/v4/android/message/methods/create-card-message)                              | 创建名片消息。通常用于分享用户资料卡片或联系人卡片。                                                           |
| [`CreateFaceMessage`](/docs/chat/sdk/v4/android/message/methods/create-face-message)                              | 创建表情消息。适合业务自定义表情包或贴纸消息。                                                                 |
| [`CreateForwardMessage`](/docs/chat/sdk/v4/android/message/methods/create-forward-message)                        | 创建单条转发消息。用于把已有消息转发给另一个用户或群。                                                         |
| [`CreateMergerMessage`](/docs/chat/sdk/v4/android/message/methods/create-merger-message)                          | 创建合并转发消息。适合转发一组聊天记录，并展示标题和摘要。                                                     |
| [`CreateImageMessage`](/docs/chat/sdk/v4/android/message/methods/create-image-message)                            | 通过图片路径创建图片消息。SDK 会根据本地图片路径生成消息体。                                                   |
| [`CreateImageMessageFromFullPath`](/docs/chat/sdk/v4/android/message/methods/create-image-message-from-full-path) | 通过完整图片路径创建图片消息。适合调用方持有绝对路径的场景。                                                   |
| [`CreateImageMessageByURL`](/docs/chat/sdk/v4/android/message/methods/create-image-message-by-url)                | 通过已上传图片 URL 创建图片消息。适合资源已由业务侧上传完成的场景。                                            |
| [`CreateSoundMessage`](/docs/chat/sdk/v4/android/message/methods/create-sound-message)                            | 通过音频路径创建语音消息。                                                                                     |
| [`CreateSoundMessageFromFullPath`](/docs/chat/sdk/v4/android/message/methods/create-sound-message-from-full-path) | 通过完整音频路径创建语音消息。适合调用方已持有完整本地路径。                                                   |
| [`CreateSoundMessageByURL`](/docs/chat/sdk/v4/android/message/methods/create-sound-message-by-url)                | 通过已上传音频 URL 信息创建语音消息。                                                                          |
| [`CreateVideoMessage`](/docs/chat/sdk/v4/android/message/methods/create-video-message)                            | 通过视频路径和快照路径创建视频消息。                                                                           |
| [`CreateVideoMessageFromFullPath`](/docs/chat/sdk/v4/android/message/methods/create-video-message-from-full-path) | 通过完整视频路径和完整快照路径创建视频消息。                                                                   |
| [`CreateVideoMessageByURL`](/docs/chat/sdk/v4/android/message/methods/create-video-message-by-url)                | 通过已上传视频 URL 信息创建视频消息。                                                                          |
| [`CreateFileMessage`](/docs/chat/sdk/v4/android/message/methods/create-file-message)                              | 通过文件路径创建普通文件消息。                                                                                 |
| [`CreateFileMessageFromFullPath`](/docs/chat/sdk/v4/android/message/methods/create-file-message-from-full-path)   | 通过完整文件路径创建普通文件消息。                                                                             |
| [`CreateFileMessageByURL`](/docs/chat/sdk/v4/android/message/methods/create-file-message-by-url)                  | 通过已上传文件 URL 信息创建文件消息。                                                                          |
| [`SendMessage`](/docs/chat/sdk/v4/android/message/methods/send-message)                                           | 发送由 Create 方法创建的消息，并使用 SDK 内置上传和发送链路。单聊与群聊只能选择一个目标字段。                  |
| [`SendMessageNotOss`](/docs/chat/sdk/v4/android/message/methods/send-message-not-oss)                             | 发送不走 SDK OSS 上传链路的消息。适合业务自定义上传、资源 URL 已经准备好或使用 `Create*ByURL` 方法创建的消息。 |

## 参数选择

| 目标 | recvID      | groupID  | 说明                         |
| ---- | ----------- | -------- | ---------------------------- |
| 单聊 | 接收用户 ID | 空字符串 | 向一个用户发送消息。         |
| 群聊 | 空字符串    | 群 ID    | 向一个 OpenIM 群组发送消息。 |

## 结果事件

| 事件     | 来源                                                                               | 说明                                     |
| -------- | ---------------------------------------------------------------------------------- | ---------------------------------------- |
| 发送成功 | `SendMsgCallBack.OnSuccess`                                                        | 返回服务端确认后的消息 JSON。            |
| 发送进度 | `SendMsgCallBack.OnProgress`                                                       | 文件、图片、视频等可能产生进度。         |
| 新消息   | `OnAdvancedMsgListener.OnRecvNewMessage` 或 `OnBatchMsgListener.OnRecvNewMessages` | 接收端通过监听器收到消息。               |
| 会话变化 | `OnConversationListener.OnNewConversation`、`OnConversationChanged`                | 发送或接收消息可能改变会话列表和未读数。 |

## 注意事项

- 单聊和群聊只选择一个目标字段：单聊传 `recvID`，群聊传 `groupID`。
- `Create*MessageByURL` 类方法适合资源已上传或由业务侧提供 URL 的场景。
- 发送前必须已登录，并已设置消息和会话监听器。
- 不要把旧文档里的跨平台示例直接复制到 Android core binding；应按独立方法页签名映射。
