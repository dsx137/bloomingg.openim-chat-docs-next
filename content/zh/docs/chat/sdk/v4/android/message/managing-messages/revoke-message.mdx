撤回指定消息，通常受时间窗口和权限限制。

创建消息类方法同步返回消息 JSON；发送、查询、删除、已读和本地写入方法通过回调返回。消息与会话事件分别通过消息监听器和会话监听器接收。

## Core Binding 方法

| 项目     | 说明                                                                                   |
| -------- | -------------------------------------------------------------------------------------- |
| 方法签名 | `RevokeMessage(callback Base, operationID string, conversationID, clientMsgID string)` |
| 所属能力 | 会话与消息                                                                             |
| 源码位置 | `open_im_sdk/conversation_msg.go`                                                      |

## Android 调用参考

Android SDK v3.8.3 的 Java 方法原型如下，示例代码也基于该调用方式。Core binding 方法签名用于说明底层 `open_im_sdk` 能力和参数映射。

```java showLineNumbers
public void revokeMessageV2(OnBase<String> callBack, String conversationID, String clientMsgID)
```

## 输入参数

| 参数名称       | 参数类型                                                       | 是否必填 | 描述            |
| -------------- | -------------------------------------------------------------- | -------- | --------------- |
| callBack       | [`OnBase`](/docs/chat/sdk/v4/android/models/on-base)`<String>` | 是       | 撤回结果回调。  |
| conversationID | String                                                         | 是       | 会话 ID。       |
| clientMsgID    | String                                                         | 是       | 客户端消息 ID。 |

## 返回结果

`onSuccess` 表示服务端已接受撤回并更新本地原消息；其他端会通过撤回事件同步。消息不存在、超过撤回限制或当前用户无权限时调用 `onError`。

## 代码示例

以下示例展示 Android SDK v3.8.3 的调用方式。

```java showLineNumbers
OpenIMClient.getInstance().messageManager.revokeMessageV2(new OnBase<String>() {
    @Override
    public void onError(int code, String error) {
        // TODO: 处理错误
    }

    @Override
    public void onSuccess(String data) {
        // TODO: 处理成功结果
    }
}, conversationID, clientMsgID);
```
