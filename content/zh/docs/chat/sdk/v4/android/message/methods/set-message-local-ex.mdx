设置消息本地扩展字段，只影响本地展示或业务状态，不应当作服务端消息内容。

## 功能介绍

设置消息本地扩展字段，只影响本地展示或业务状态，不应当作服务端消息内容。

创建消息类方法同步返回消息 JSON；发送、查询、删除、已读和本地写入方法通过回调返回。消息与会话事件分别通过消息监听器和会话监听器接收。

## Core Binding 方法

| 项目     | 说明                                                                                                |
| -------- | --------------------------------------------------------------------------------------------------- |
| 方法签名 | `SetMessageLocalEx(callback Base, operationID string, conversationID, clientMsgID, localEx string)` |
| 所属能力 | 会话与消息                                                                                          |
| 源码位置 | `open_im_sdk/conversation_msg.go`                                                                   |

## Android 调用参考

Android SDK v3.8.3.5 的 Java 方法原型如下，示例代码也基于该调用方式。Core binding 方法签名用于说明底层 `open_im_sdk` 能力和参数映射。

```java showLineNumbers
public void setMessageLocalEx(OnBase<String> callBack, String conversationID, String clientMsgID, String localEx)
```

## 输入参数

| 参数名称       | 参数类型 | 是否必填 | 描述             |
| -------------- | -------- | -------- | ---------------- |
| OnBase         | OnBase   | 是       | 会话 ID          |
| conversationID | String   | 是       | 会话 ID          |
| clientMsgID    | String   | 是       | 消息 ID          |
| localEx        | String   | 是       | 要设置的 ex 信息 |

## 返回结果

通过回调返回结果：成功走 `OnSuccess`，失败走 `OnError`；发送或上传类方法还可能通过进度回调返回进度。

## 代码示例

以下示例展示 Android SDK v3.8.3.5 的调用方式。

```java showLineNumbers
OpenIMClient.getInstance().messageManager.setMessageLocalEx(new OnBase<String>() {
    @Override
    public void onError(int code, String error) {
        // TODO: 处理错误
    }

    @Override
    public void onSuccess(String data) {
        // TODO: 处理成功结果
    }
}, conversationID, clientMsgID, localEx);
```
