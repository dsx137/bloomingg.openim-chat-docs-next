根据目标 ID 和会话类型生成 OpenIM 会话 ID，适合从用户或群跳转到会话前构造会话标识。

创建消息类方法同步返回消息 JSON；发送、查询、删除、已读和本地写入方法通过回调返回。消息与会话事件分别通过消息监听器和会话监听器接收。

## Core Binding 方法

| 项目     | 说明                                                                                          |
| -------- | --------------------------------------------------------------------------------------------- |
| 方法签名 | `GetConversationIDBySessionType(operationID string, sourceID string, sessionType int) string` |
| 所属能力 | 会话与消息                                                                                    |
| 源码位置 | `open_im_sdk/conversation_msg.go`                                                             |

## Android 调用参考

Android SDK v3.8.3 的 Java 方法原型如下，示例代码也基于该调用方式。Core binding 方法签名用于说明底层 `open_im_sdk` 能力和参数映射。

```java showLineNumbers
public String getConversationIDBySessionType(String sourceId, int sessionType)
```

## 输入参数

| 参数名称    | 参数类型 | 是否必填 | 描述                                   |
| ----------- | -------- | -------- | -------------------------------------- |
| sourceId    | String   | 是       | 单聊为用户 ID，群聊为群 ID。           |
| sessionType | int      | 是       | 会话类型，单聊通常为 1，群聊通常为 2。 |

## 返回结果

| 名称           | 类型   | 描述     |
| -------------- | ------ | -------- |
| conversationID | String | 成功返回 |

## 代码示例

以下示例展示 Android SDK v3.8.3 的调用方式。

```java showLineNumbers
String conversationID = OpenIMClient.getInstance().conversationManager.getConversationIDBySessionType(sourceId, sessionType);
```
