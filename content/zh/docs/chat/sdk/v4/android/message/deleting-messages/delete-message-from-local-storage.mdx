仅从本地存储删除指定消息，不表示服务端消息被删除。

## 功能介绍

仅从本地存储删除指定消息，不表示服务端消息被删除。

创建消息类方法同步返回消息 JSON；发送、查询、删除、已读和本地写入方法通过回调返回。消息与会话事件分别通过消息监听器和会话监听器接收。

## Core Binding 方法

| 项目     | 说明                                                                                                          |
| -------- | ------------------------------------------------------------------------------------------------------------- |
| 方法签名 | `DeleteMessageFromLocalStorage(callback Base, operationID string, conversationID string, clientMsgID string)` |
| 所属能力 | 会话与消息                                                                                                    |
| 源码位置 | `open_im_sdk/conversation_msg.go`                                                                             |

## Android 调用参考

Android SDK v3.8.3.5 的 Java 方法原型如下，示例代码也基于该调用方式。Core binding 方法签名用于说明底层 `open_im_sdk` 能力和参数映射。

```java showLineNumbers
public void deleteMessageFromLocalStorage(String conversationID, String clientMsgID, OnBase<String> callBack)
```

## 输入参数

| 参数名称       | 参数类型                                                           | 是否必填 | 描述            |
| -------------- | ------------------------------------------------------------------ | -------- | --------------- |
| conversationID | String                                                             | 是       | 会话 ID。       |
| clientMsgID    | String                                                             | 是       | 客户端消息 ID。 |
| callBack       | [`OnBase`](/docs/chat/sdk/v4/android/models/on-base)&lt;String&gt; | 是       | 删除结果回调。  |

## 返回结果

通过回调返回结果：成功走 `OnSuccess`，失败走 `OnError`；发送或上传类方法还可能通过进度回调返回进度。

## 代码示例

以下示例展示 Android SDK v3.8.3.5 的调用方式。

```java showLineNumbers
OpenIMClient.getInstance().messageManager.deleteMessageFromLocalStorage(conversationID, clientMsgID, new OnBase<String>() {
    @Override
    public void onError(int code, String error) {
        // TODO: 处理错误
    }

    @Override
    public void onSuccess(String data) {
        // TODO: 处理成功结果
    }
```
