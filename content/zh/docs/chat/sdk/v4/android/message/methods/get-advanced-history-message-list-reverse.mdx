按反向方向获取历史消息，适合围绕游标向另一方向加载。

## 功能介绍

按反向方向获取历史消息，适合围绕游标向另一方向加载。

创建消息类方法同步返回消息 JSON；发送、查询、删除、已读和本地写入方法通过回调返回。消息与会话事件分别通过消息监听器和会话监听器接收。

## Core Binding 方法

| 项目     | 说明                                                                                                |
| -------- | --------------------------------------------------------------------------------------------------- |
| 方法签名 | `GetAdvancedHistoryMessageListReverse(callback Base, operationID string, getMessageOptions string)` |
| 所属能力 | 会话与消息                                                                                          |
| 源码位置 | `open_im_sdk/conversation_msg.go`                                                                   |

## Android 调用参考

Android SDK v3.8.3.5 的 Java 方法原型如下，示例代码也基于该调用方式。Core binding 方法签名用于说明底层 `open_im_sdk` 能力和参数映射。

```java showLineNumbers
public void getAdvancedHistoryMessageListReverse(OnBase<AdvancedMessage> callBack, String conversationID, Message startMsg, int count, ViewType viewType)
```

## 输入参数

| 参数名称            | 类型                                            | 描述                                                        |
| ------------------- | ----------------------------------------------- | ----------------------------------------------------------- |
| `callback`          | [`Base`](/docs/chat/sdk/v4/android/models/base) | 异步结果回调。                                              |
| `operationID`       | `string`                                        | 本次操作的追踪 ID。                                         |
| `getMessageOptions` | `string`                                        | 通常为 JSON 字符串或上层封装对象，具体结构以 SDK 定义为准。 |

## 返回结果

通过回调返回结果：成功走 `OnSuccess`，失败走 `OnError`；发送或上传类方法还可能通过进度回调返回进度。

## 代码示例

以下示例展示 Android SDK v3.8.3.5 的调用方式。

```java showLineNumbers
OpenIMClient.getInstance().messageManager.getAdvancedHistoryMessageListReverse(
    new OnBase<AdvancedMessage>() {
    @Override
    public void onError(int code, String error) {
        // TODO: 处理错误
    }

    @Override
    public void onSuccess(AdvancedMessage data) {
        // TODO: 处理反向拉取的历史消息
    }
},
    conversationID,
    startMsg,
    count,
    ViewType.History
);
```
