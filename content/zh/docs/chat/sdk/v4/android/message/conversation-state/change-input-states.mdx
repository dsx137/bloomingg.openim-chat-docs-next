更新当前用户在指定会话中的输入状态。

创建消息类方法同步返回消息 JSON；发送、查询、删除、已读和本地写入方法通过回调返回。消息与会话事件分别通过消息监听器和会话监听器接收。

## Core Binding 方法

| 项目     | 说明                                                                                      |
| -------- | ----------------------------------------------------------------------------------------- |
| 方法签名 | `ChangeInputStates(callback Base, operationID string, conversationID string, focus bool)` |
| 所属能力 | 会话与消息                                                                                |
| 源码位置 | `open_im_sdk/conversation_msg.go`                                                         |

## Android 调用参考

Android SDK v3.8.3 的 Java 方法原型如下，示例代码也基于该调用方式。Core binding 方法签名用于说明底层 `open_im_sdk` 能力和参数映射。

```java showLineNumbers
public void changeInputStates(OnBase<String> base, String conversationId, boolean hasFocus)
```

## 输入参数

| 参数名称       | Java 类型                                                      | 是否必填 | 描述                                        |
| -------------- | -------------------------------------------------------------- | -------- | ------------------------------------------- |
| base           | [`OnBase`](/docs/chat/sdk/v4/android/models/on-base)`<String>` | 是       | 输入状态更新结果回调。                      |
| conversationId | `String`                                                       | 是       | 输入状态所属的会话 ID。                     |
| hasFocus       | `boolean`                                                      | 是       | `true` 表示正在输入，`false` 表示停止输入。 |

## 返回结果

`onSuccess` 表示本次开始或停止输入状态已发送，回调字符串不含业务数据；会话不存在、会话 ID 为空或发送超时时调用 `onError`。对同一状态的重复调用可能直接成功而不重复发送。

## 代码示例

以下示例展示 Android SDK v3.8.3 的调用方式。

```java showLineNumbers
OpenIMClient.getInstance().conversationManager.changeInputStates(new OnBase<String>() {
    @Override
    public void onError(int code, String error) {
        // TODO: 处理错误
    }

    @Override
    public void onSuccess(String data) {
        // TODO: 处理成功结果
    }
}, conversationId, hasFocus);
```
