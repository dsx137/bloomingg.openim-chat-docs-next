通过视频路径和快照路径创建视频消息。

## 功能介绍

通过视频路径和快照路径创建视频消息。

创建消息类方法同步返回消息 JSON；发送、查询、删除、已读和本地写入方法通过回调返回。消息与会话事件分别通过消息监听器和会话监听器接收。

## Core Binding 方法

| 项目     | 说明                                                                                                                     |
| -------- | ------------------------------------------------------------------------------------------------------------------------ |
| 方法签名 | `CreateVideoMessage(operationID string, videoPath string, videoType string, duration int64, snapshotPath string) string` |
| 所属能力 | 会话与消息                                                                                                               |
| 源码位置 | `open_im_sdk/conversation_msg.go`                                                                                        |

## Android 调用参考

Android SDK v3.8.3.5 的 Java 方法原型如下，示例代码也基于该调用方式。Core binding 方法签名用于说明底层 `open_im_sdk` 能力和参数映射。

```java showLineNumbers
public Message createVideoMessage(String videoPath, String videoType, long duration, String snapshotPath)
```

## 输入参数

| 参数名称       | 类型     | 描述                       |
| -------------- | -------- | -------------------------- |
| `operationID`  | `string` | 本次操作的追踪 ID。        |
| `videoPath`    | `string` | 按 core binding 签名传入。 |
| `videoType`    | `string` | 按 core binding 签名传入。 |
| `duration`     | `int64`  | 按 core binding 签名传入。 |
| `snapshotPath` | `string` | 按 core binding 签名传入。 |

## 返回结果

同步返回 `string`。

## 代码示例

以下示例展示 Android SDK v3.8.3.5 的调用方式。

```java showLineNumbers
Message videoMessage = OpenIMClient.getInstance().messageManager.createVideoMessage(videoPath, videoType, duration, snapshotPath);
```
