按消息 ID 精确标记已读，适合只确认一部分消息已读的场景。

## 功能介绍

按消息 ID 精确标记已读，适合只确认一部分消息已读的场景。

创建消息类方法同步返回消息 JSON；发送、查询、删除、已读和本地写入方法通过回调返回。消息与会话事件分别通过消息监听器和会话监听器接收。

## Core Binding 方法

| 项目     | 说明                                                                                                       |
| -------- | ---------------------------------------------------------------------------------------------------------- |
| 方法签名 | `MarkMessagesAsReadByMsgID(callback Base, operationID string, conversationID string, clientMsgIDs string)` |
| 所属能力 | 会话与消息                                                                                                 |
| 源码位置 | `open_im_sdk/conversation_msg.go`                                                                          |

## Android 调用参考

Android SDK v3.8.3.5 的 Java 方法原型如下，示例代码也基于该调用方式。Core binding 方法签名用于说明底层 `open_im_sdk` 能力和参数映射。

```java showLineNumbers
public void markMessagesAsReadByMsgID(String conversationID, List<String> clientMsgIDs, OnBase<String> callBack)
```

该方法在 Android SDK 中已标记为 `@Deprecated`，优先使用 `markConversationMessageAsRead` 按会话标记已读。

## 输入参数

| 参数名称         | 类型                                            | 描述                |
| ---------------- | ----------------------------------------------- | ------------------- |
| `callback`       | [`Base`](/docs/chat/sdk/v4/android/models/base) | 异步结果回调。      |
| `operationID`    | `string`                                        | 本次操作的追踪 ID。 |
| `conversationID` | `string`                                        | 业务标识参数。      |
| `clientMsgIDs`   | `string`                                        | 业务标识参数。      |

## 返回结果

通过回调返回结果：成功走 `OnSuccess`，失败走 `OnError`；发送或上传类方法还可能通过进度回调返回进度。

## 代码示例

以下示例展示 Android SDK v3.8.3.5 的调用方式。

```java showLineNumbers

        // TODO: 处理错误
    }

    @Override

        // TODO: 处理成功结果
```
