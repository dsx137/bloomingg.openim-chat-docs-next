接收好友和黑名单相关事件，用于更新通讯录、好友申请入口、好友资料和屏蔽关系状态。

## 功能介绍

好友事件适合驱动通讯录和关系状态更新。好友申请、好友新增或删除、好友资料变化、黑名单新增或移除，都应通过该监听器同步到业务层。

该监听器是单实例替换式注册。建议由关系模块统一维护好友和黑名单缓存，再通知通讯录、资料页和申请列表刷新。

## Core Binding 方法

| 项目     | 说明                                               |
| -------- | -------------------------------------------------- |
| 方法签名 | `SetFriendListener(listener OnFriendshipListener)` |
| 所属能力 | 监听器                                             |
| 源码位置 | `open_im_sdk/listener.go`                          |

## Android 调用参考

Android SDK v3.8.3 的 Java 方法原型如下，示例代码也基于该调用方式。Core binding 方法签名用于说明底层 `open_im_sdk` 能力和参数映射。

```java showLineNumbers
public void setOnFriendshipListener(OnFriendshipListener listener)
```

## 输入参数

| 参数名称 | Java 类型                                                                         | 是否必填 | 描述                           |
| -------- | --------------------------------------------------------------------------------- | -------- | ------------------------------ |
| listener | [`OnFriendshipListener`](/docs/chat/sdk/v4/android/models/on-friendship-listener) | 是       | 好友、好友申请和黑名单监听器。 |

## 返回结果

无

## 代码示例

以下示例展示 Android SDK v3.8.3 的调用方式。

```java showLineNumbers
OpenIMClient.getInstance().friendshipManager.setOnFriendshipListener(new OnFriendshipListener() {
    @Override
    public void onFriendAdded(FriendInfo info) {
        // TODO: 处理新增好友
    }

    @Override
    public void onFriendDeleted(FriendInfo info) {
        // TODO: 处理好友删除
    }

    @Override
    public void onBlacklistAdded(BlacklistInfo info) {
        // TODO: 处理加入黑名单
    }
});
```

## 相关类型

- [好友关系监听器](/docs/chat/sdk/v4/android/models/on-friendship-listener)（`OnFriendshipListener`）：好友、好友申请和黑名单变化监听器。
- [好友信息](/docs/chat/sdk/v4/android/models/friend-info)（`FriendInfo`）：好友资料，包含昵称、头像、备注和添加来源等信息。
- [黑名单用户信息](/docs/chat/sdk/v4/android/models/blacklist-info)（`BlacklistInfo`）：黑名单用户资料，用于黑名单列表和黑名单事件回调。
