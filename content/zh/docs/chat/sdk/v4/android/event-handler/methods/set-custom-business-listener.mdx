设置自定义业务消息监听器，用于接收业务自定义通道传入的消息。

## 功能介绍

设置自定义业务消息监听器，用于接收业务自定义通道传入的消息。

`Set*Listener` 会替换当前 SDK 保存的同类监听器；如果多个模块都要消费事件，应在业务层封装统一分发。

## Core Binding 方法

| 项目     | 说明                                                           |
| -------- | -------------------------------------------------------------- |
| 方法签名 | `SetCustomBusinessListener(listener OnCustomBusinessListener)` |
| 所属能力 | 监听器                                                         |
| 源码位置 | `open_im_sdk/listener.go`                                      |

## Android 调用参考

Android SDK v3.8.3.5 的 Java 方法原型如下，示例代码也基于该调用方式。Core binding 方法签名用于说明底层 `open_im_sdk` 能力和参数映射。

```java showLineNumbers
public void setCustomBusinessListener(OnCustomBusinessListener customBusinessListener)
```

## 输入参数

| 参数名称   | 类型                                                                                       | 描述             |
| ---------- | ------------------------------------------------------------------------------------------ | ---------------- |
| `listener` | [`OnCustomBusinessListener`](/docs/chat/sdk/v4/android/models/on-custom-business-listener) | 事件监听器实例。 |

## 返回结果

无直接返回值。

## 代码示例

以下示例展示 Android SDK v3.8.3.5 的调用方式。

```java showLineNumbers
OpenIMClient.getInstance().setCustomBusinessListener(new OnCustomBusinessListener() {
    @Override
    public void onRecvCustomBusinessMessage(String message) {
        // TODO: 处理自定义业务消息
    }
});
```
