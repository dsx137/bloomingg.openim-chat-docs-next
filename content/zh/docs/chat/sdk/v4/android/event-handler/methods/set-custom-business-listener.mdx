接收自定义业务消息，用于处理业务侧通过 OpenIM 自定义通道下发的扩展事件。

## 功能介绍

自定义业务事件不属于标准消息、会话、群组或好友事件，适合承载业务扩展通知，例如审核结果、运营指令或应用自定义同步信号。

该监听器是单实例替换式注册。收到事件后应先做业务类型解析，再交给对应模块处理，避免所有页面直接解析同一段自定义 payload。

## Core Binding 方法

| 项目     | 说明                                                           |
| -------- | -------------------------------------------------------------- |
| 方法签名 | `SetCustomBusinessListener(listener OnCustomBusinessListener)` |
| 所属能力 | 监听器                                                         |
| 源码位置 | `open_im_sdk/listener.go`                                      |

## Android 调用参考

Android SDK v3.8.3 的 Java 方法原型如下，示例代码也基于该调用方式。Core binding 方法签名用于说明底层 `open_im_sdk` 能力和参数映射。

```java showLineNumbers
public void setCustomBusinessListener(OnCustomBusinessListener customBusinessListener)
```

## 输入参数

| 参数名称               | Java 类型                                                                                  | 是否必填 | 描述                       |
| ---------------------- | ------------------------------------------------------------------------------------------ | -------- | -------------------------- |
| customBusinessListener | [`OnCustomBusinessListener`](/docs/chat/sdk/v4/android/models/on-custom-business-listener) | 是       | 自定义业务消息监听器实例。 |

## 返回结果

无直接返回值。

## 代码示例

以下示例展示 Android SDK v3.8.3 的调用方式。

```java showLineNumbers
OpenIMClient.getInstance().setCustomBusinessListener(new OnCustomBusinessListener() {
    @Override
    public void onRecvCustomBusinessMessage(String message) {
        // TODO: 处理自定义业务消息
    }
});
```
