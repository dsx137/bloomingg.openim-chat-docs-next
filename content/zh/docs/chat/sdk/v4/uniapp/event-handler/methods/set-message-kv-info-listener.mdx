设置消息 KV 信息变化监听器，用于接收消息扩展键值信息变化。

## 功能介绍

设置消息 KV 信息变化监听器，用于接收消息扩展键值信息变化。

`Set*Listener` 会替换当前 SDK 保存的同类监听器；如果多个模块都要消费事件，应在业务层封装统一分发。

## Core Binding 方法

| 项目     | 说明                                                         |
| -------- | ------------------------------------------------------------ |
| 方法签名 | `SetMessageKvInfoListener(listener OnMessageKvInfoListener)` |
| 所属能力 | 监听器                                                       |
| 源码位置 | `open_im_sdk/listener.go`                                    |

## uni-app 调用参考

uni-app SDK v3.8.3 的 JavaScript 方法原型如下。Core binding 方法签名用于说明底层能力和参数映射。

```js showLineNumbers
openIM.setMessageKvInfoListener(operationID, /* ... */, callback);
```

## 输入参数

| 参数名称 | 参数类型   | 是否必填   | 描述                 |
| -------- | ---------- | ---------- | -------------------- |
| 参数     | 见方法原型 | 按方法原型 | 按上方调用原型传入。 |

## 返回结果

同步创建类方法直接返回值；异步方法通过 callback 返回成功或失败。建议为每次调用传入可追踪的 operationID。

## 代码示例

以下示例展示 uni-app SDK v3.8.3 的调用骨架，请按业务上下文补充真实参数、错误处理和 UI 状态更新。

```js showLineNumbers
const openIM = uni.requireNativePlugin('OpenIMUniPlugin');

openIM.setMessageKvInfoListener('operationID', callback);
```
