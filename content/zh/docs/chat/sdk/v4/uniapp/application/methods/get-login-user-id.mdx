获取当前 SDK 实例记录的登录用户 ID，常用于日志、会话归属和账号切换判断。

## 功能介绍

获取当前 SDK 实例记录的登录用户 ID，常用于日志、会话归属和账号切换判断。

初始化类方法应在应用进程内按需调用；用户态方法通常要求 `InitSDK` 已成功执行，并通过 `operationID` 串联日志。

## Core Binding 方法

| 项目     | 说明                        |
| -------- | --------------------------- |
| 方法签名 | `GetLoginUserID() string`   |
| 所属能力 | 初始化与登录                |
| 源码位置 | `open_im_sdk/init_login.go` |

## uni-app 调用参考

uni-app SDK v3.8.3 的 JavaScript 方法原型如下。Core binding 方法签名用于说明底层能力和参数映射。

源码位置：`ios/OpenIMUniPlugin/OpenIMUniPlugin/Classes/OpenIMModule.m`。

```js showLineNumbers
openIM.getLoginUserID(operationID);
```

## 输入参数

| 参数名称 | 参数类型   | 是否必填   | 描述                 |
| -------- | ---------- | ---------- | -------------------- |
| 参数     | 见方法原型 | 按方法原型 | 按上方调用原型传入。 |

## 返回结果

同步创建类方法直接返回值；异步方法通过 callback 返回成功或失败。建议为每次调用传入可追踪的 operationID。

## 代码示例

以下示例展示 uni-app SDK v3.8.3 的调用方式。

```java showLineNumbers
String loginUserID = OpenIMClient.getInstance().getLoginUserID();
```
