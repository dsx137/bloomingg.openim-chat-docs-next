OpenIM WASM SDK 使用 `groupID` 标识群组，使用 `conversationID` 标识聊天列表中的本地会话记录。OpenIM 群组没有内置的频道 URL 字段；如果应用路由使用 slug 或 URL，先由业务侧把它解析为 `groupID`，再调用 SDK 读取频道数据。

如果你的入口只保存了 `groupID`，先读取群资料，再读取对应会话。群组资料用于标题、头像、成员入口和群设置；会话记录用于未读数、最后一条消息、置顶和草稿等聊天列表状态。

## 群组频道

`getSpecifiedGroupsInfo()` 接收一个 `groupID` 数组，返回匹配到的 `GroupItem[]`。获取单个群组时传入一个元素即可。

```ts
const groupResult = await OpenIM.getSpecifiedGroupsInfo([groupID]);
const group = groupResult.data?.[0];

if (!group) {
  console.error('getSpecifiedGroupsInfo failed', {
    errCode: groupResult.    errMsg: groupResult.    operationID: groupResult.operationID,
    groupID,
  });
  throw new Error(groupResult.'Group not found');
}

renderGroupHeader({
  groupID: group.groupID,
  title: group.groupName,
  faceURL: group.faceURL,
});
```

当前用户需要拥有读取该群组的权限。群组不存在、用户不在群内或本地同步尚未完成时，返回结果可能为空；失败时记录 `operationID`，便于和服务端日志对应。

## 加载频道会话

群组会话使用 `sourceID: groupID` 和 `sessionType: 3` 读取。`getOneConversation()` 返回 `ConversationItem`，适合打开聊天页时补齐会话状态。

```ts
const { data: conversation, operationID } = await OpenIM.getOneConversation({
  sourceID: group.groupID,
  sessionType: 3,
});

if (!conversation) {
  console.error('getOneConversation failed', {    operationID,
    groupID: group.groupID,
  });
  throw new Error('Conversation not found');
}

openChannel({
  group,
  conversationID: conversation.conversationID,
  unreadCount: conversation.unreadCount,
  latestMessage: conversation.latestMsg,
});
```

如果只需要稳定保存会话 ID，可以用同一组参数调用 `getConversationIDBySessionType()`。

```ts
const { data: conversationID } = await OpenIM.getConversationIDBySessionType({
  sourceID: groupID,
  sessionType: 3,
});
```

## 使用 conversationID 恢复频道

当路由或本地状态已经保存了 `conversationID`，可以直接用 `getMultipleConversation()` 读取会话记录。这个方法接收会话 ID 数组，即使只读取一个会话也需要传数组。

```ts
const conversationResult = await OpenIM.getMultipleConversation([conversationID]);
const conversation = conversationResult.data?.[0];

if (!conversation) {
  console.error('getMultipleConversation failed', {
    errCode: conversationResult.    errMsg: conversationResult.    operationID: conversationResult.operationID,
    conversationID,
  });
  throw new Error(conversationResult.'Conversation not found');
}

setActiveConversation(conversation);
```

如果该会话是群聊，并且界面还需要群头像、群公告或成员入口，请从会话中保存的 `groupID` 或业务状态中的 `groupID` 再调用 `getSpecifiedGroupsInfo()`。

## 处理更新事件

读取结果只代表当前本地状态。频道页打开后，继续监听会话和群组事件，把后续变化合并进应用状态。

```ts
import { CbEvents } from '@openim/wasm-client-sdk';

const handleConversationChanged = ({ data }) => {
  mergeConversations(data);
};

const handleGroupInfoChanged = ({ data }) => {
  mergeGroup(data);
};

OpenIM.on(CbEvents.OnConversationChanged, handleConversationChanged);
OpenIM.on(CbEvents.OnGroupInfoChanged, handleGroupInfoChanged);
```

## 标识说明

| 参数             | 说明                                                           |
| ---------------- | -------------------------------------------------------------- |
| `groupID`        | 群组 ID，用于读取群资料、群成员和发送群消息。                  |
| `conversationID` | 本地会话记录 ID，用于恢复聊天列表状态和打开消息页。            |
| `sourceID`       | `getOneConversation()` 的目标 ID；群聊传 `groupID`。           |
| `sessionType`    | 会话类型；群聊传 `3`，单聊传 `1`。                             |
| `operationID`    | 可选的请求追踪 ID；排查失败时和 `errCode`、`errMsg` 一起记录。 |

## 校验读取结果

- 确认调用读取方法前 `login()` 已成功，并且当前浏览器会话已收到 `CbEvents.OnConnectSuccess`。
- 确认 `getSpecifiedGroupsInfo()` 返回的 `data[0].groupID` 与请求的 `groupID` 一致。
- 确认 `getOneConversation()` 或 `getMultipleConversation()` 返回可用的 `conversationID`，并且聊天页能用它继续加载消息。
- 修改群名称、头像或会话状态后，确认当前页面能通过 `CbEvents.OnGroupInfoChanged` 或 `CbEvents.OnConversationChanged` 更新展示。
