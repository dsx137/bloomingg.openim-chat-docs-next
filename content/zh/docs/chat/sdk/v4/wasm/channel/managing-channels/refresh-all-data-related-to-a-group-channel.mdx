OpenIM WASM SDK 会在在线状态下通过事件持续同步会话、群资料和成员变化。应用仍然需要在重连、切回前台、用户重新进入群聊详情页，或本地状态可能已经过期时主动刷新当前频道相关数据。

OpenIM 没有 `channel.refresh()` 这种单个频道对象方法。群聊频道通常由三类数据共同组成：`conversationID` 对应聊天列表和未读数，`groupID` 对应群资料，成员列表由 `getGroupMemberList()` 分页读取。刷新时只重新加载当前界面需要展示的数据，不需要重新创建 SDK 实例或重新登录。

## 刷新频道快照

如果你已经保存了 `groupID`，可以用 `getOneConversation()` 读取群聊会话，用 `getSpecifiedGroupsInfo()` 读取群资料，再按页面需要读取成员列表。群聊会话的 `sessionType` 使用 `3`。

```ts
type RefreshGroupChannelParams = {
  groupID: string;
  memberOffset?: number;
  memberCount?: number;
};

const refreshGroupChannel = async ({
  groupID,
  memberOffset = 0,
  memberCount = 50,
}: RefreshGroupChannelParams) => {
  const operationID = crypto.randomUUID();

  const [conversationResult, groupResult, membersResult] = await Promise.all([
    OpenIM.getOneConversation(
      {
        sourceID: groupID,
        sessionType: 3,
      },
      operationID,
    ),
    OpenIM.getSpecifiedGroupsInfo([groupID], operationID),
    OpenIM.getGroupMemberList(
      {
        groupID,
        filter: 0,
        offset: memberOffset,
        count: memberCount,
      },
      operationID,
    ),
  ]);

  for (const result of [conversationResult, groupResult, membersResult]) {
    if (result.errCode !== 0) {
      console.error('refresh group channel failed', {
        errCode: result.errCode,
        errMsg: result.errMsg,
        operationID: result.operationID,
        groupID,
      });
      throw new Error(result.errMsg || 'Failed to refresh group channel');
    }
  }

  replaceConversation(conversationResult.data);
  replaceGroup(groupResult.data[0]);
  replaceVisibleMembers(groupID, membersResult.data);

  return {
    conversation: conversationResult.data,
    group: groupResult.data[0],
    members: membersResult.data,
  };
};
```

如果你的界面已经持有 `conversationID`，也可以调用 `getMultipleConversation([conversationID])` 只刷新对应会话行。只持有 `groupID` 时，`getOneConversation({ sourceID: groupID, sessionType: 3 })` 更直接。

## 重连后刷新频道

网络恢复后，SDK 会继续触发同步和会话事件。为了让当前打开的群聊页尽快展示最新状态，可以在连接成功或服务端同步完成后刷新当前频道。处理器里应读取当前可见的 `groupID`，避免刷新已经离开的页面。

```ts
import { CbEvents } from '@openim/wasm-client-sdk';

const refreshVisibleGroupChannel = () => {
  const groupID = getVisibleGroupID();

  if (!groupID) return;

  void refreshGroupChannel({ groupID }).catch((error) => {
    console.error('refresh visible group channel failed', { groupID, error });
  });
};

OpenIM.on(CbEvents.OnConnectSuccess, refreshVisibleGroupChannel);
OpenIM.on(CbEvents.OnSyncServerFinish, refreshVisibleGroupChannel);
```

页面卸载、退出登录或切换账号时，用同一个函数引用移除处理器：

```ts
OpenIM.off(CbEvents.OnConnectSuccess, refreshVisibleGroupChannel);
OpenIM.off(CbEvents.OnSyncServerFinish, refreshVisibleGroupChannel);
```

如果你的应用在 `login()` 前已经注册了连接事件，重连后的刷新逻辑可以复用同一套连接状态管理。不要在重连事件里再次调用 `login()`；只刷新当前页面需要的数据。

## 合并实时变化

主动刷新负责拿到一次完整快照，实时事件负责把后续变化合并进本地状态。群聊频道页面通常至少监听会话和群资料事件；如果页面展示成员列表，再监听成员事件。

```ts
OpenIM.on(CbEvents.OnConversationChanged, ({ data }) => {
  mergeConversations(data);
});

OpenIM.on(CbEvents.OnGroupInfoChanged, ({ data }) => {
  mergeGroups(data);
});

OpenIM.on(CbEvents.OnGroupMemberAdded, ({ data }) => {
  mergeGroupMembers(data);
});

OpenIM.on(CbEvents.OnGroupMemberDeleted, ({ data }) => {
  removeGroupMembers(data);
});

OpenIM.on(CbEvents.OnGroupMemberInfoChanged, ({ data }) => {
  mergeGroupMembers(data);
});
```

当事件只携带局部数据，或你需要以服务端记录为准时，可以在事件处理器中记录受影响的 `groupID`，再对 `refreshGroupChannel()` 做防抖或节流。这样既能保持 UI 实时更新，也能避免短时间内重复请求同一个群聊频道。

## 刷新结果的使用方式

`getOneConversation()` 返回的 `ConversationItem` 适合更新聊天列表、当前会话标题、未读数和本地展示状态。`getSpecifiedGroupsInfo()` 返回的 `GroupItem[]` 通常只取当前 `groupID` 对应的第一项，用于更新群名称、头像、公告、群主、扩展字段和禁言状态。`getGroupMemberList()` 返回的成员数据是分页结果，应只替换当前可见页或按你的状态模型合并。

失败时记录 `errCode`、`errMsg` 和 `operationID`。同一次刷新可以共用一个 `operationID`，便于把三个请求和浏览器日志、服务端日志对应起来。

## 校验刷新流程

- 确认 `login()` 已成功，并且浏览器收到 `CbEvents.OnConnectSuccess` 后再调用刷新函数。
- 修改群名称、头像、公告或成员信息后，确认 `CbEvents.OnGroupInfoChanged`、`CbEvents.OnGroupMemberInfoChanged` 或成员增删事件能更新页面。
- 断网后恢复连接，确认 `CbEvents.OnConnectSuccess` 或 `CbEvents.OnSyncServerFinish` 会触发当前群聊频道刷新。
- 确认聊天列表中的会话状态能通过 `getOneConversation()`、`getMultipleConversation()` 或 `CbEvents.OnConversationChanged` 更新。
- 出现错误时，确认日志中包含 `groupID`、`errCode`、`errMsg` 和 `operationID`。
