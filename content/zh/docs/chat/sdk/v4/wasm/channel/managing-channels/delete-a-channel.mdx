OpenIM WASM SDK 中，频道式群聊由群组记录和会话记录共同组成。删除频道前先确认产品动作的影响范围：`dismissGroup()` 会解散群组，所有成员都不应再继续使用这个群组；会话删除和隐藏类 API 只影响当前用户的聊天列表或消息记录，不会删除群组本身。

OpenIM 没有开放频道类，因此本页只覆盖群组频道和当前用户会话记录两类删除语义。如果你的界面提供“删除频道”按钮，通常只应让群主或具备相应权限的管理员调用 `dismissGroup()`。如果用户只是想整理聊天列表，使用会话级 API 更合适。

## 群组频道

调用 `dismissGroup(groupID)` 会解散指定群组。执行前客户端应已经完成登录并收到 `CbEvents.OnConnectSuccess`，同时当前用户需要具备服务端允许的群组操作权限。

```ts
import { CbEvents } from '@openim/wasm-client-sdk';

const handleGroupDismissed = ({ data, operationID }) => {
  markGroupAsDismissed(data.groupID);
  removeConversationByGroupID(data.groupID);
  console.debug('group dismissed', { groupID: data.groupID, operationID });
};

const handleJoinedGroupDeleted = ({ data }) => {
  removeJoinedGroup(data.groupID);
};

const handleConversationChanged = ({ data }) => {
  mergeConversations(data);
};

OpenIM.on(CbEvents.OnGroupDismissed, handleGroupDismissed);
OpenIM.on(CbEvents.OnJoinedGroupDeleted, handleJoinedGroupDeleted);
OpenIM.on(CbEvents.OnConversationChanged, handleConversationChanged);

const operationID = crypto.randomUUID();

const { errCode, errMsg } = await OpenIM.dismissGroup(groupID, operationID);

if (errCode !== 0) {
  console.error('dismissGroup failed', { errCode, errMsg, groupID, operationID });
  throw new Error(errMsg || 'Failed to dismiss group');
}

closeChannelView(groupID);
```

`dismissGroup()` 的目标是 `groupID`，不是 `conversationID`。调用成功后，当前客户端可以先关闭正在展示的频道详情页，并等待群组和会话事件把列表状态同步到最终结果。

## 当前用户的会话记录

如果用户只想从自己的聊天列表中移除频道，不要调用 `dismissGroup()`。这类操作不会影响其他群成员，也不会删除群组资料。

```ts
const operationID = crypto.randomUUID();

const { errCode, errMsg } = await OpenIM.deleteConversationAndDeleteAllMsg(
  conversationID,
  operationID,
);

if (errCode !== 0) {
  console.error('deleteConversationAndDeleteAllMsg failed', {
    errCode,
    errMsg,
    conversationID,
    operationID,
  });
  throw new Error(errMsg || 'Failed to delete conversation');
}

removeConversation(conversationID);
```

如果产品只是需要把会话从列表中临时隐藏，可以使用 `hideConversation(conversationID)`。如果需要保留会话行但清空消息历史，可以使用 `clearConversationAndDeleteAllMsg(conversationID)`。这些 API 的影响范围都不同于解散群组。

| 用户动作                         | 建议 API                                            | 影响范围                                       |
| -------------------------------- | --------------------------------------------------- | ---------------------------------------------- |
| 删除整个群组频道                 | `dismissGroup(groupID)`                             | 群组被解散，所有成员都应看到群组不可继续使用。 |
| 从当前账号聊天列表删除会话和消息 | `deleteConversationAndDeleteAllMsg(conversationID)` | 仅影响当前用户的会话记录和对应消息展示。       |
| 从当前账号聊天列表隐藏会话       | `hideConversation(conversationID)`                  | 仅隐藏当前用户的会话入口。                     |
| 清空当前会话消息历史             | `clearConversationAndDeleteAllMsg(conversationID)`  | 清理当前用户在该会话下的消息历史。             |

## 删除后的频道状态

删除或解散频道后，前端不要只依赖按钮点击结果修改所有状态。Promise 响应用于判断本次调用是否提交成功；最终的聊天列表、群资料和成员状态仍应以 SDK 事件或重新查询结果为准。

重点监听这些事件：

- `CbEvents.OnGroupDismissed`：群组被解散时更新群资料页、频道入口和权限状态。
- `CbEvents.OnJoinedGroupDeleted`：当前用户的已加入群组列表发生删除时移除对应记录。
- `CbEvents.OnConversationChanged`：会话列表、未读数、最后一条消息或展示状态变化时合并会话数据。

事件载荷可能是单个对象，也可能是列表，取决于具体事件和 SDK 版本。合并状态时应按业务模型做归一化处理，并在错误路径记录 `errCode`、`errMsg` 和 `operationID`。

```ts
const toArray = <T>(value: T | T[]) => (Array.isArray(value) ? value : [value]);

const handleConversationChanged = ({ data, operationID }) => {
  for (const conversation of toArray(data)) {
    mergeConversation(conversation);
  }

  console.debug('conversation changed after delete action', { operationID });
};
```

## 校验删除结果

- 确认调用删除动作前 `login()` 已成功，并且浏览器收到 `CbEvents.OnConnectSuccess`。
- 调用 `dismissGroup()` 时确认当前用户有解散群组的权限，并且传入的是正确的 `groupID`。
- 确认 SDK 响应满足 `errCode === 0`；失败时记录 `errCode`、`errMsg`、`operationID` 和目标 ID。
- 在另一个已登录成员会话中确认解散群组能触发群组删除相关事件，并且旧 `groupID` 不再作为可用聊天入口展示。
- 对会话级删除，确认其他成员仍然可以访问群组，只有当前账号的聊天列表或消息历史发生变化。
