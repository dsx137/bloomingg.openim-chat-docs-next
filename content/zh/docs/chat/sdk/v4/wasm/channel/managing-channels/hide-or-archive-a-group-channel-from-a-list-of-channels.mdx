OpenIM WASM SDK 没有独立的群组频道对象。频道列表通常展示 `ConversationItem`，群组资料和成员关系则由 `groupID` 对应的群组记录管理。因此，隐藏或归档一个群组频道时，实际操作对象是当前用户本地的群会话 `conversationID`，不是删除群组本身。

如果只是让当前用户的聊天列表不再显示该群组，调用 `hideConversation()`。OpenIM 不提供单独的隐藏参数对象或内置归档状态；如果产品需要“已归档”列表或恢复入口，可以先把归档状态写入 `ConversationItem.ex`，再按业务策略隐藏会话或清空本地消息历史。

## 隐藏或归档状态

| 目标                                       | 推荐做法                                                                                              |
| ------------------------------------------ | ----------------------------------------------------------------------------------------------------- |
| 只从当前用户默认列表移除                   | 调用 `hideConversation(conversationID)`。                                                             |
| 从默认列表移除，并保留可恢复的归档状态     | 先用 `setConversation()` 写入 `ex.archivedAt`，再调用 `hideConversation()` 或由应用列表筛选掉该会话。 |
| 从默认列表移除，并清空当前用户本地历史消息 | 调用 `clearConversationAndDeleteAllMsg(conversationID)`。                                             |
| 解散群组，让所有成员都无法继续使用         | 使用群组管理 API，例如 `dismissGroup(groupID)`；这不是归档操作。                                      |

这些操作不会把当前用户踢出群组。只要群组仍存在，后续新消息、重新同步或你的应用恢复逻辑都可能让会话再次进入可见列表。生产代码应把最终展示规则放在自己的列表过滤逻辑中，而不是只依赖按钮点击后的临时 UI 状态。

## 隐藏当前用户的群会话

对群组会话调用 `getConversationIDBySessionType()` 时，`sourceID` 传 `groupID`，`sessionType` 传 `3`。拿到 `conversationID` 后，再调用 `hideConversation()`。

```ts
import { CbEvents } from '@openim/wasm-client-sdk';

OpenIM.on(CbEvents.OnConversationChanged, ({ data }) => {
  mergeConversations(data);
});

async function hideGroupChannel(groupID: string) {
  const operationID = crypto.randomUUID();

  const conversationIDResponse = await OpenIM.getConversationIDBySessionType(
    {
      sourceID: groupID,
      sessionType: 3,
    },
    operationID,
  );

  const conversationID = conversationIDResponse.data;
  const hideResponse = await OpenIM.hideConversation(conversationID, operationID);

  removeConversationFromDefaultList(conversationID);
}
```

`hideConversation()` 影响当前登录用户的会话列表，不会删除群组、不会修改群成员，也不会替其他成员隐藏这个群组。需要通知其他页面刷新时，监听 `CbEvents.OnConversationChanged`，并在响应成功后主动更新当前页面状态。

## 标记为已归档

OpenIM 没有内置的 `isArchived` 字段。需要归档列表时，把紧凑的业务状态写入 `ConversationItem.ex`，例如 `archivedAt`。这个字段适合保存客户端展示状态；如果归档规则需要跨设备强一致或后台审计，应由你的业务后端保存权威状态。

```ts
function readConversationEx(ex?: string): Record<string, unknown> {
  if (!ex) return {};

  try {
    return JSON.parse(ex) as Record<string, unknown>;
  } catch {
    return {};
  }
}

async function archiveGroupChannel(groupID: string) {
  const operationID = crypto.randomUUID();

  const { data: conversation } = await OpenIM.getOneConversation(
    {
      sourceID: groupID,
      sessionType: 3,
    },
    operationID,
  );

  const ex = {
    ...readConversationEx(conversation.ex),
    archivedAt: new Date().toISOString(),
  };

  const archiveResponse = await OpenIM.setConversation(
    {
      conversationID: conversation.conversationID,
      ex: JSON.stringify(ex),
    },
    operationID,
  );

  const hideResponse = await OpenIM.hideConversation(conversation.conversationID, operationID);

  moveConversationToArchiveList(conversation.conversationID, ex);
}
```

如果归档动作还要求清空当前用户的本地历史消息，把最后一步替换为 `clearConversationAndDeleteAllMsg(conversation.conversationID, operationID)`。不要用 `dismissGroup()` 实现归档；它表达的是解散群组，会影响所有成员。

## 筛选默认列表和归档列表

加载聊天列表时，默认列表应排除已归档会话；归档页则只展示带有 `archivedAt` 的会话。隐藏后的会话可能不会继续出现在普通列表查询结果中，因此很多产品会把归档索引同时保存在业务后端或当前用户资料中，用于跨设备恢复归档页。

```ts
function isArchivedConversation(conversation: { ex?: string }) {
  return Boolean(readConversationEx(conversation.ex).archivedAt);
}

const { data: conversations } = await OpenIM.getConversationListSplit({
  offset: 0,
  count: 50,
});

const defaultConversations = conversations.filter((conversation) => {
  return !isArchivedConversation(conversation);
});

const archivedConversations = conversations.filter(isArchivedConversation);
```

要恢复归档状态，读取原会话后用 `setConversation()` 移除 `archivedAt`，再让列表按最新 `ConversationItem.ex` 重新渲染。归档只应改变当前用户的列表展示，不应改写群名称、群成员或群消息发送权限。

## 校验隐藏或归档结果

- 确认 SDK 已登录成功，并且在调用会话 API 前收到 `CbEvents.OnConnectSuccess`。
- 确认 `groupID` 是当前用户已加入的群组，`getConversationIDBySessionType()` 或 `getOneConversation()` 能返回群会话。
- 确认调用成功 resolve，并且返回数据或后续事件符合预期。
- 确认默认频道列表不再展示目标 `conversationID`，归档列表按你的 `ex.archivedAt` 或后端归档记录展示。
- 确认另一个成员的客户端不受影响，除非你调用的是解散群组等全局群组管理操作。
