OpenIM 的频道列表通常由两类数据组成：`ConversationItem` 用于聊天列表、未读数、置顶、草稿和最后一条消息；`GroupItem` 用于群名称、头像、公告、群主和成员状态。单聊只需要会话记录即可展示，群聊则通常把 `conversationID` 对应的会话和 `groupID` 对应的群资料合并后展示。

读取列表前，浏览器端应已经完成 SDK 初始化和 `login()`，并且收到 `CbEvents.OnConnectSuccess`。如果页面只需要当前用户的聊天列表，优先读取会话列表；如果页面还要展示群资料、群设置入口或已加入群组管理，再读取已加入群组列表。

## 会话列表

`getAllConversationList()` 返回当前登录用户本地可见的全部会话。它适合会话数量不大、或应用启动后需要一次性恢复聊天列表状态的场景。

```ts
const { data: conversations, errCode, errMsg, operationID } = await OpenIM.getAllConversationList();

if (errCode !== 0) {
  console.error('getAllConversationList failed', { errCode, errMsg, operationID });
  throw new Error(errMsg || 'Failed to load conversations');
}

replaceConversationList(conversations);
```

会话记录中的 `conversationID` 是后续打开聊天、读取历史消息、标记已读和更新未读数时最常用的标识。单聊会话还会关联对方用户，群聊会话会关联 `groupID`；业务状态中建议同时保存这些标识，避免从 UI 标题或路由字符串反推目标。

## 分页加载会话

列表较长时使用 `getConversationListSplit()` 分页读取。`offset` 表示起始位置，`count` 表示本页数量。滚动加载下一页时，把 `offset` 增加已加载数量即可。

```ts
const pageSize = 30;

const loadConversationPage = async (pageIndex: number) => {
  const { data, errCode, errMsg, operationID } = await OpenIM.getConversationListSplit({
    offset: pageIndex * pageSize,
    count: pageSize,
  });

  if (errCode !== 0) {
    console.error('getConversationListSplit failed', { errCode, errMsg, operationID });
    return [];
  }

  return data;
};

const firstPage = await loadConversationPage(0);
appendConversations(firstPage);
```

分页返回的是当前 SDK 已同步到本地的会话列表切片。列表页仍应监听会话事件，因为新消息、置顶、草稿、未读数和会话删除等变化可能在分页请求之后到达。

## 群组频道列表

`getJoinedGroupList()` 返回当前用户已加入的全部群组。它适合群组数量不大、或需要在设置页、转发页、群组选择器中一次性展示当前用户可访问群组的场景。

```ts
const { data: groups, errCode, errMsg, operationID } = await OpenIM.getJoinedGroupList();

if (errCode !== 0) {
  console.error('getJoinedGroupList failed', { errCode, errMsg, operationID });
  throw new Error(errMsg || 'Failed to load groups');
}

replaceJoinedGroups(groups);
```

群组记录的核心标识是 `groupID`。发送群消息、读取群成员、修改群资料和打开群会话时都使用同一个 `groupID`。如果你已经从会话列表拿到了群聊会话，可以用会话上的群标识和 `GroupItem.groupID` 合并展示名称、头像和群状态。

## 分页加载群组频道

群组列表较长时使用 `getJoinedGroupListPage()`。它和会话分页一样接收 `offset` 和 `count`。

```ts
const groupPageSize = 50;

const {
  data: groupPage,
  errCode,
  errMsg,
  operationID,
} = await OpenIM.getJoinedGroupListPage({
  offset: 0,
  count: groupPageSize,
});

if (errCode !== 0) {
  console.error('getJoinedGroupListPage failed', { errCode, errMsg, operationID });
} else {
  appendJoinedGroups(groupPage);
}
```

如果产品需要按群名称搜索、按成员筛选或展示全应用频道目录，不要只依赖浏览器端的已加入群组列表。客户端列表表示当前用户已加入的群组；跨用户目录、复杂搜索和权限过滤通常应由业务后端按你的产品规则返回结果，再让浏览器使用 `groupID` 打开或加入目标群组。

## 合并群组和会话数据

频道列表可以先以会话为主，再用 `groupID` 补齐群资料。这样未读数、最后一条消息和置顶状态始终来自 `ConversationItem`，群名称、头像和群设置状态来自 `GroupItem`。

```ts
const [{ data: conversations }, { data: groups }] = await Promise.all([
  OpenIM.getConversationListSplit({ offset: 0, count: 50 }),
  OpenIM.getJoinedGroupListPage({ offset: 0, count: 50 }),
]);

const groupsByID = new Map(groups.map((group) => [group.groupID, group]));

const channelItems = conversations.map((conversation) => {
  const group = conversation.groupID ? groupsByID.get(conversation.groupID) : undefined;

  return {
    conversationID: conversation.conversationID,
    groupID: conversation.groupID,
    title: group?.groupName || conversation.showName,
    faceURL: group?.faceURL || conversation.faceURL,
    unreadCount: conversation.unreadCount,
    latestMsg: conversation.latestMsg,
    isPinned: conversation.isPinned,
  };
});

renderChannelList(channelItems);
```

不要把群资料当作聊天列表的唯一来源。用户加入了群组但本地尚未产生会话时，群组列表中可能有 `GroupItem`，而聊天列表中暂时没有对应 `ConversationItem`。这种情况下可以在群组选择器中展示该群组；真正进入聊天页时，再用 `getOneConversation()` 或后续会话事件补齐会话状态。

## 筛选群组频道

OpenIM 没有 `GroupChannelFilter` 对象。会话分页和已加入群组分页负责返回当前用户可见的数据；名称、未读、置顶、归档、自定义标签等筛选通常在应用层处理。需要跨全量群组、权限控制或多字段排序时，应由业务后端返回候选 `groupID`，浏览器再用 SDK 补齐资料。

OpenIM 的会话分页负责读取当前用户的会话切片；常见的展示筛选可以在应用层完成。例如只展示群聊、只展示未读会话、或按置顶和最后消息时间排序。

```ts
const visibleGroupChannels = channelItems
  .filter((item) => Boolean(item.groupID))
  .filter((item) => item.title.toLowerCase().includes(keyword.toLowerCase()))
  .sort((a, b) => {
    if (a.isPinned !== b.isPinned) return a.isPinned ? -1 : 1;
    return Number(b.latestMsg?.sendTime ?? 0) - Number(a.latestMsg?.sendTime ?? 0);
  });
```

对于大规模列表，筛选条件应尽量下沉到后端或业务索引服务。浏览器端只适合处理已经加载到当前用户本地状态中的小范围筛选。

## 监听列表变化

频道列表打开期间，注册会话和群组事件，把增量变化合并到当前列表。组件卸载、退出登录或切换账号时，使用同一个函数引用调用 `off()` 清理处理器。

```ts
import { CbEvents } from '@openim/wasm-client-sdk';

const handleNewConversation = ({ data }) => {
  mergeConversations(data);
};

const handleConversationChanged = ({ data }) => {
  mergeConversations(data);
};

const handleJoinedGroupAdded = ({ data }) => {
  mergeGroups(data);
};

const handleJoinedGroupDeleted = ({ data }) => {
  removeJoinedGroups(data);
};

const handleGroupInfoChanged = ({ data }) => {
  mergeGroups(data);
};

const handleTotalUnreadChanged = ({ data }) => {
  updateTotalUnreadCount(data);
};

const registerChannelListEvents = () => {
  OpenIM.on(CbEvents.OnNewConversation, handleNewConversation);
  OpenIM.on(CbEvents.OnConversationChanged, handleConversationChanged);
  OpenIM.on(CbEvents.OnJoinedGroupAdded, handleJoinedGroupAdded);
  OpenIM.on(CbEvents.OnJoinedGroupDeleted, handleJoinedGroupDeleted);
  OpenIM.on(CbEvents.OnGroupInfoChanged, handleGroupInfoChanged);
  OpenIM.on(CbEvents.OnTotalUnreadMessageCountChanged, handleTotalUnreadChanged);

  return () => {
    OpenIM.off(CbEvents.OnNewConversation, handleNewConversation);
    OpenIM.off(CbEvents.OnConversationChanged, handleConversationChanged);
    OpenIM.off(CbEvents.OnJoinedGroupAdded, handleJoinedGroupAdded);
    OpenIM.off(CbEvents.OnJoinedGroupDeleted, handleJoinedGroupDeleted);
    OpenIM.off(CbEvents.OnGroupInfoChanged, handleGroupInfoChanged);
    OpenIM.off(CbEvents.OnTotalUnreadMessageCountChanged, handleTotalUnreadChanged);
  };
};

const cleanupChannelListEvents = registerChannelListEvents();
```

`OnNewConversation` 通常用于插入新会话，`OnConversationChanged` 用于更新已有会话的未读数、最后消息、置顶或草稿状态。`OnJoinedGroupAdded`、`OnJoinedGroupDeleted` 和 `OnGroupInfoChanged` 用于维护群资料缓存。群组被解散时还可以监听 `CbEvents.OnGroupDismissed`，把相关入口置为不可用或从列表中移除。

## 校验列表结果

- 确认 `login()` 已成功，并且在读取列表前收到 `CbEvents.OnConnectSuccess`。
- 确认列表请求响应满足 `errCode === 0`，失败时记录 `errCode`、`errMsg` 和 `operationID`。
- 确认会话列表能拿到预期的 `conversationID`，群聊会话能关联到正确的 `groupID`。
- 确认分页加载不会重复插入同一个 `conversationID` 或 `groupID`。
- 在另一个已登录浏览器会话中发送消息、创建群组或修改群资料，确认当前页面能通过事件更新列表。
- 刷新页面或断线重连后，确认列表能通过重新查询和事件同步恢复到正确状态。
