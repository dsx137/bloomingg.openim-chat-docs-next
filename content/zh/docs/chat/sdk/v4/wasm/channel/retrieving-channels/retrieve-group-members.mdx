`getGroupMemberList()` 更适合放在群组/频道文档中，而不是 User 文档中。原因是这个接口的主语是群组：请求必须携带 `groupID`，返回值是群范围内的 `GroupMemberItem`，字段包括群内昵称、群角色、禁言状态和入群信息。

用户资料页可以按 `userID` 使用 `getUsersInfo()` 或 `getSpecifiedFriendsInfo()`；群成员列表、@ 候选人、群资料页和成员管理面板应按 `groupID` 使用本页接口。

## 分页读取群成员

通过 SDK 实例调用 `getGroupMemberList()`，可以分页读取指定群的成员。第一页把 `offset` 设为 `0`，`count` 设为页面大小；继续加载时把 `offset` 增加到已经合并的成员数量。返回数量小于 `count` 时，当前筛选条件下通常没有更多成员。

```ts
const pageSize = 50;

async function loadGroupMembers(groupID: string, offset = 0) {
  const operationID = crypto.randomUUID();
  const { data } = await OpenIM.getGroupMemberList(
    {
      groupID,
      filter: 0,
      offset,
      count: pageSize,
    },
    operationID,
  );

  mergeGroupMembers(groupID, data);
  return data;
}
```

`filter` 的枚举值由 SDK 定义。普通成员列表通常使用 `0` 读取全部成员；如果产品需要只展示某类角色，可以按当前 SDK 枚举值或返回后的 `roleLevel` 做筛选。

## 获取指定群成员

如果界面已经持有一组 `userID`，例如打开成员资料卡或收到成员事件后刷新单个成员，可以调用 `getSpecifiedGroupMembersInfo()` 获取这些用户在当前群里的成员资料。

```ts
const { data: selectedMembers } = await OpenIM.getSpecifiedGroupMembersInfo({
    groupID,
    userIDList: selectedUserIDs,
  });

mergeGroupMembers(groupID, selectedMembers);
```

只需要判断一批用户是否仍在群内时，使用 `getUsersInGroup()` 返回命中的 `userID[]`。如果要展示昵称、头像、角色或禁言状态，仍应读取 `GroupMemberItem`。

```ts
const { data: existingUserIDs } = await OpenIM.getUsersInGroup({
  groupID,
  userIDList: candidateUserIDs,
});
```

## 搜索群成员

成员搜索使用 `searchGroupMembers()`。搜索范围是指定 `groupID` 下的群成员，不是应用级用户搜索。

```ts
const { data: matchedMembers } = await OpenIM.searchGroupMembers({
  groupID,
  keywordList: [keyword.trim()],
  isSearchUserID: true,
  isSearchMemberNickname: true,
  offset: 0,
  count: 50,
});
```

## 成员属性

`getGroupMemberList()`、`getSpecifiedGroupMembersInfo()` 和 `searchGroupMembers()` 返回的成员都是 `GroupMemberItem`。这些字段属于群范围：同一个 `userID` 在不同群里可以有不同昵称、头像、角色和禁言状态。

| 字段 | 说明 |
| --- | --- |
| `groupID` | 成员所属群组 ID。 |
| `userID` | 成员用户 ID，适合作为资料卡、事件合并和权限判断的键。 |
| `nickname` | 群内展示昵称。 |
| `faceURL` | 群内展示头像地址。 |
| `roleLevel` | 群内角色，例如普通成员、管理员和群主。 |
| `joinTime` | 成员入群时间；渲染前确认当前 SDK 返回的时间单位。 |
| `muteEndTime` | 成员禁言截止时间，大于当前时间时可视为仍在禁言期。 |
| `inviterUserID` | 邀请该成员入群的用户 ID。 |
| `operatorUserID` | 最近一次成员变更操作的操作者用户 ID。 |
| `ex` | 群成员扩展字段，用于存放业务约定的数据。 |

渲染群成员时优先使用 `GroupMemberItem` 的群内字段。`PublicUserItem` 适合补充账号级公开信息，但不应覆盖群内昵称、群角色或禁言状态。

```ts
function toGroupMemberRow(member: GroupMemberItem) {
  return {
    id: member.userID,
    name: member.nickname || member.userID,
    avatar: member.faceURL,
    roleLevel: member.roleLevel,
    muted: Number(member.muteEndTime) > Date.now(),
  };
}
```

## 同步成员变化

成员列表应结合群成员事件保持同步。首次进入页面、刷新或重新登录后先调用 `getGroupMemberList()` 建立快照，之后再通过事件合并新增、删除和资料变化。

```ts
OpenIM.on(CbEvents.OnGroupMemberAdded, ({ data }) => {
  mergeGroupMembers(data.groupID, [data]);
});

OpenIM.on(CbEvents.OnGroupMemberDeleted, ({ data }) => {
  removeGroupMember(`${data.groupID}:${data.userID}`);
});

OpenIM.on(CbEvents.OnGroupMemberInfoChanged, ({ data }) => {
  mergeGroupMembers(data.groupID, [data]);
});
```
