OpenIM 的群管理员由群成员角色表示。常见角色值为：群主 `100`、管理员 `60`、普通成员 `20`。注册管理员就是把目标成员在指定 `groupID` 下的 `roleLevel` 更新为管理员；移除管理员就是把该成员恢复为普通成员角色。

执行角色变更前，浏览器客户端应已经完成 SDK 初始化和登录，并且当前登录用户具备调整群成员角色的权限。生产环境建议为每次角色变更传入并记录 `operationID`，便于和 OpenIM 服务端日志对应。

## 注册管理员

调用 `setGroupMemberInfo()`，传入目标群组、目标用户和管理员角色值 `60`。下面的示例会检查 SDK 响应中的 `errCode`，并在失败时带上本次调用的 `operationID`。

```ts
const GROUP_ROLE_LEVEL = {
  NORMAL: 20,
  ADMIN: 60,
} as const;

async function addGroupAdmin(groupID: string, userID: string) {
  const operationID = crypto.randomUUID();
  const { data } = await OpenIM.setGroupMemberInfo(
    {
      groupID,
      userID,
      roleLevel: GROUP_ROLE_LEVEL.ADMIN,
    },
    operationID,
  );

  return data;
}
```

## 移除管理员

移除管理员时仍然调用 `setGroupMemberInfo()`，但把目标成员的 `roleLevel` 改回普通成员值 `20`。不要把群主当作普通管理员处理；群主角色是 `100`，应按你的产品权限流程单独控制。

```ts
async function removeGroupAdmin(groupID: string, userID: string) {
  const operationID = crypto.randomUUID();
  const { data } = await OpenIM.setGroupMemberInfo(
    {
      groupID,
      userID,
      roleLevel: GROUP_ROLE_LEVEL.NORMAL,
    },
    operationID,
  );

  return data;
}
```

## 展示管理员列表

如果页面需要展示管理员列表，先使用 [获取群成员列表](/docs/chat/sdk/v4/wasm/channel/retrieving-channels/retrieve-group-members) 读取成员，再按 `roleLevel` 过滤。这样可以和普通成员列表共用同一份 `GroupMemberItem` 缓存。

```ts
const { data: members } = await OpenIM.getGroupMemberList({
  groupID,
  filter: 0,
  offset: 0,
  count: 100,
});

const admins = members.filter((member) => member.roleLevel === GROUP_ROLE_LEVEL.ADMIN);
```

如果群成员数量很大，不要为了展示管理员而一次性拉取所有成员。可以由业务后端维护管理员索引，或按产品需要分页加载后在当前页内展示角色标识。

## 同步成员角色变化

角色变更会影响成员列表、管理员入口和权限按钮状态。除了处理 `setGroupMemberInfo()` 的返回值，还应监听 `CbEvents.OnGroupMemberInfoChanged`，让当前页面和其他已登录会话都能刷新目标成员的角色。

```ts
import { CbEvents } from '@openim/wasm-client-sdk';

function handleGroupMemberInfoChanged({ data }) {
  updateGroupMember(data);
}

OpenIM.on(CbEvents.OnGroupMemberInfoChanged, handleGroupMemberInfoChanged);
```

## 校验结果

- 确认 `login()` 已成功，并且在调用角色变更前收到 `CbEvents.OnConnectSuccess`。
- 注册管理员后，确认目标成员的 `roleLevel` 为 `60`。
- 移除管理员后，确认目标成员的 `roleLevel` 已恢复为 `20`。
- 使用另一个已登录客户端验证 `CbEvents.OnGroupMemberInfoChanged` 会触发，并且相关权限按钮、成员列表和管理员列表同步刷新。
