迁移到 OpenIM WASM SDK 时，优先迁移业务模型，而不是逐个查找旧方法名的替代品。OpenIM 的浏览器流程是：发布 WASM 资源，创建 SDK 实例，使用可信后端返回的 `userID` 和 token 登录，围绕会话、群组和消息更新 UI，并通过 `CbEvents` 接收实时变化。

## 迁移目标

把旧聊天代码迁移到 OpenIM 时，建议先完成以下目标：

- 用 `userID` 表示当前用户、消息接收方和用户资料对象。
- 用 `conversationID` 表示聊天列表和历史消息分页使用的本地会话记录。
- 用 `recvID` 发送单聊消息；单聊不需要提前创建频道。
- 用 `groupID` 表示多人聊天、群成员、群资料和群消息发送目标。
- 用 `create*Message()` 创建 `MessageItem`，再用 `sendMessage()` 发送。
- 用 `OpenIM.on(CbEvents.*)` 同步消息、会话、未读数、用户和连接状态。

OpenIM WASM SDK 没有开放频道、投票、定时消息、消息 reaction、消息线程或集合对象的完全同构客户端对象。迁移这些功能时，应使用 OpenIM 群组、会话、消息、自定义消息和可信后端组合实现，并把权威状态放在后端。

## 概念映射

| 旧聊天代码中的概念     | OpenIM 中的实现方式                                                                                    |
| ---------------------- | ------------------------------------------------------------------------------------------------------ |
| 应用级客户端实例       | 通过 `getSDK()` 创建并在当前浏览器会话中复用的 SDK 实例                                                |
| 当前用户               | `login()` 使用的 `userID` 和 token                                                                     |
| 单聊频道               | 不需要创建频道；发送消息时设置 `recvID`，`groupID` 为空                                                |
| 群组频道或多人会话     | `createGroup()` 创建的群组，核心标识是 `groupID`                                                       |
| 聊天列表条目           | `ConversationItem`，核心标识是 `conversationID`                                                        |
| 文本、文件、自定义消息 | `createTextMessage()`、`createFileMessageByFile()`、`createCustomMessage()` 等方法返回的 `MessageItem` |
| 新消息和会话更新监听   | `CbEvents.OnRecvNewMessages`、`CbEvents.OnNewConversation`、`CbEvents.OnConversationChanged`           |
| 未读数更新             | `CbEvents.OnTotalUnreadMessageCountChanged` 和会话记录中的未读状态                                     |

## 迁移应用和认证

浏览器端需要安装 `@openim/wasm-client-sdk`，并把 `openIM.wasm`、`sql-wasm.wasm` 和 `wasm_exec.js` 放到前端可访问的静态资源目录。创建 SDK 实例时指定 WASM 路径，然后使用当前用户的 `userID`、token、`apiAddr` 和 `wsAddr` 登录。浏览器端登录使用 `platformID: 5`。

```ts
import { CbEvents, getSDK } from '@openim/wasm-client-sdk';

const OpenIM = getSDK({
  coreWasmPath: '/openIM.wasm',
  sqlWasmPath: '/sql-wasm.wasm',
});

OpenIM.on(CbEvents.OnConnectSuccess, () => {
  setConnectionState('connected');
});

OpenIM.on(CbEvents.OnConnectFailed, ({ operationID }) => {
  reportConnectionError({ operationID });
});

await OpenIM.login({
  userID,
  token,
  platformID: 5,
  apiAddr,
  wsAddr,
});
```

生产环境中，token 应由可信后端签发。不要在浏览器端保存管理员凭据，也不要把用户创建、权限授予或敏感配置逻辑放到前端。

## 迁移会话和群组

OpenIM 的消息目标和会话记录是两个概念。单聊发送使用目标用户的 `recvID`；聊天列表、历史消息和未读数通常围绕 `conversationID` 更新。群聊发送使用 `groupID`，群会话可以通过会话 API 读取。

```ts
const { data: conversations } = await OpenIM.getAllConversationList();

const { data: directConversationID } = await OpenIM.getConversationIDBySessionType({
  sourceID: receiverUserID,
  sessionType: 1,
});

const { data: groupConversationID } = await OpenIM.getConversationIDBySessionType({
  sourceID: groupID,
  sessionType: 3,
});
```

如果旧代码在创建多人聊天时创建频道，迁移到 OpenIM 后应创建群组。`createGroup()` 返回的 `GroupItem.groupID` 是后续发送群消息、读取群资料和管理群成员的核心标识。

```ts
const operationID = crypto.randomUUID();

const response = await OpenIM.createGroup(
  {
    memberUserIDs,
    ownerUserID: currentUserID,
    adminUserIDs: [],
    groupInfo: {
      groupName: 'Project chat',
      groupType: 2,
    },
  },
  operationID,
);

const groupID = response.data.groupID;
```

单聊无需对应的创建步骤。用户点击“开始聊天”时，可以先打开临时会话界面；第一条消息发送成功后，再用返回的消息和会话事件补全本地状态。

## 迁移消息收发

OpenIM 发送消息分两步：先创建一个 `MessageItem`，再把它传给 `sendMessage()`。单聊消息设置 `recvID`，群聊消息设置 `groupID`，另一个目标字段保持空字符串。

```ts
const { data: message } = await OpenIM.createTextMessage('你好，OpenIM');

const { data: sentMessage } = await OpenIM.sendMessage({
  recvID: receiverUserID,
  groupID: '',
  message,
});

appendMessage(sentMessage);
```

群聊发送使用同一个 `sendMessage()`，只切换目标字段。

```ts
const { data: message } = await OpenIM.createTextMessage('大家好');

await OpenIM.sendMessage({
  recvID: '',
  groupID,
  message,
});
```

文件、图片、卡片和业务通知也遵循同一模式。先用对应的 `create*Message()` 方法生成消息对象；旧代码中的自定义 payload 可以迁移到 `createCustomMessage()`，把业务数据放在 `data`、`extension` 和 `description` 中。

```ts
const { data: customMessage } = await OpenIM.createCustomMessage({
  data: JSON.stringify({
    type: 'task-card',
    taskID,
    title: '任务已分配',
  }),
  extension: 'task',
  description: '任务卡片',
});

await OpenIM.sendMessage({
  recvID: '',
  groupID,
  message: customMessage,
});
```

## 迁移实时事件和本地状态

如果旧代码依赖频道集合、消息集合或本地缓存集合，迁移时应改为显式读取 OpenIM 会话和消息，并用事件把增量数据合并到应用状态。

历史消息使用 `getAdvancedHistoryMessageList()` 分页读取。第一页可以把 `startClientMsgID` 设为空字符串，下一页使用当前列表最后一条消息的 `clientMsgID` 作为游标。

```ts
const { data: firstPage } = await OpenIM.getAdvancedHistoryMessageList({
  conversationID,
  startClientMsgID: '',
  count: 30,
  viewType: 0,
});

const nextCursor = firstPage.messageList.at(-1)?.clientMsgID ?? '';

const { data: nextPage } = await OpenIM.getAdvancedHistoryMessageList({
  conversationID,
  startClientMsgID: nextCursor,
  count: 30,
  viewType: 0,
});
```

实时更新通过事件处理器进入应用状态。连接事件建议在 `login()` 前注册；消息和会话事件可以在聊天主界面加载时注册，并集中合并到应用状态。

```ts
function handleNewMessages({ data }) {
  mergeMessages(data);
}

function handleConversationChanged({ data }) {
  mergeConversations(data);
}

OpenIM.on(CbEvents.OnRecvNewMessages, handleNewMessages);
OpenIM.on(CbEvents.OnNewConversation, handleConversationChanged);
OpenIM.on(CbEvents.OnConversationChanged, handleConversationChanged);
OpenIM.on(CbEvents.OnTotalUnreadMessageCountChanged, ({ data }) => {
  updateTotalUnreadCount(data);
});

```

## 功能差异

| 旧功能形态           | OpenIM 迁移建议                                                            |
| -------------------- | -------------------------------------------------------------------------- |
| 开放房间或公开频道   | 使用 OpenIM 群组承载聊天成员和消息；公开发现、加入规则和审核策略放在后端。 |
| 投票、表单或消息反馈 | 使用自定义消息和后端业务表存储结果，前端通过消息事件刷新展示。             |
| 定时消息             | 由后端任务调度在指定时间调用 OpenIM 服务端能力或业务发送流程。             |
| 消息 reaction 或线程 | 用自定义消息、扩展字段或后端关系表实现，并明确多端同步策略。               |
| 集合对象自动分页     | 改为显式调用会话、历史消息和本地搜索 API，再通过事件合并增量。             |

## 上线检查

- `openIM.wasm`、`sql-wasm.wasm` 和 `wasm_exec.js` 在生产静态资源路径下可访问。
- `login()` 成功返回，并且连接事件触发 `CbEvents.OnConnectSuccess`。
- 单聊发送时 `recvID` 有值、`groupID` 为空；群聊发送时 `groupID` 有值、`recvID` 为空。
- 写操作响应满足你部署环境的成功约定，并在失败时进入 `catch`。
- 另一个已登录浏览器会话能通过 `CbEvents.OnRecvNewMessages` 收到消息。
- 会话列表能通过 `getAllConversationList()`、`getOneConversation()` 或会话事件拿到最新 `ConversationItem`。
- 失败时同时记录 `errCode`、`errMsg` 和 `operationID`，再结合 OpenIM 服务端日志定位。
