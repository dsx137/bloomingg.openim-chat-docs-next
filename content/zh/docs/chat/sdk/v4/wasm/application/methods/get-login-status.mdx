查询 SDK 当前登录状态，用于业务层判断是否可以继续调用用户态 API。

## 功能介绍

查询 SDK 当前登录状态，用于业务层判断是否可以继续调用用户态 API。

初始化类方法应在应用进程内按需调用；用户态方法通常要求 `InitSDK` 已成功执行，并通过 `operationID` 串联日志。

## 底层能力

| 项目     | 说明                                     |
| -------- | ---------------------------------------- |
| 方法签名 | `GetLoginStatus(operationID string) int` |
| 所属能力 | SDK 生命周期                             |
| 源码位置 | `open_im_sdk/init_login.go`              |

## WASM 调用参考

WASM SDK v3.8.3 暴露以下平台方法原型。底层能力表用于说明 OpenIM core 能力和参数映射。

源码位置：`src/sdk/index.ts`。

```ts showLineNumbers
getLoginStatus((operationID = uuidv4()));
```

## 输入参数

| 参数名称    | 参数类型   | 是否必填 | 描述                               |
| ----------- | ---------- | -------- | ---------------------------------- |
| operationID | As defined | 否       | 链路追踪 ID；不传时 SDK 自动生成。 |

## 返回结果

返回 `Promise` 或注册回调结果。调用失败时 Promise reject；建议在业务层使用 `try/catch` 或 `.catch()` 处理错误。

## 代码示例

以下示例展示 WASM SDK v3.8.3 的调用骨架，请按业务上下文补充真实参数、错误处理和 UI 状态更新。

```ts showLineNumbers
import { getSDK } from '@openim/wasm-client-sdk';

const openIM = getSDK({
  coreWasmPath: '/openIM.wasm',
  sqlWasmPath: '/sql-wasm.wasm',
});

const result = await openIM.getLoginStatus();
```
