应用级页面说明浏览器客户端如何进入 OpenIM Chat：初始化 SDK、登录用户、维护连接状态、处理 token 生命周期，并在调用消息、会话、群组和用户 API 前确认当前客户端已经处于可用状态。

OpenIM WASM SDK 的浏览器包由 `@openim/wasm-client-sdk` 提供。应用需要先发布 `openIM.wasm`、`sql-wasm.wasm` 和 `wasm_exec.js`，再通过 `getSDK()` 创建 SDK 实例，并使用可信后端签发的 `userID` 和 token 调用 `login()`。

## 功能按主题

#### 用户认证

[用户认证](/docs/chat/sdk/v4/wasm/application/authenticating-a-user/authentication) 介绍从 SDK 初始化到登录、连接事件、token 过期、token 无效、被踢下线、退出登录和登录状态查询的完整流程。

应用通常会在 `login()` 前注册以下事件，并把它们映射到全局连接状态、登录态和问题诊断日志：

```ts
CbEvents.OnConnecting;
CbEvents.OnConnectSuccess;
CbEvents.OnConnectFailed;
CbEvents.OnUserTokenExpired;
CbEvents.OnUserTokenInvalid;
CbEvents.OnKickedOffline;
```

登录成功后，可以使用 `getLoginStatus()` 和 `getLoginUserID()` 校验当前 SDK 状态，避免在切换账号、刷新页面或恢复连接时把旧用户状态写入新会话。

#### 理解调用限制

调用限制说明应用应如何看待前端节流、服务端拒绝和 SDK 错误响应。OpenIM WASM SDK 不把限流做成单独的浏览器接口；前端应把非零 `errCode`、连接事件和同步失败事件交给统一错误处理层。

浏览器端可以限制按钮连点、合并重复请求和延迟重试，但最终的产品级限流、权限校验和风控策略应由可信后端、网关或 OpenIM 服务端策略承担。当前端收到非零 `errCode`、连接失败或同步失败事件时，应记录 `errMsg` 和 `operationID`，再结合服务端日志判断是限流、权限、token、网络还是参数问题。
