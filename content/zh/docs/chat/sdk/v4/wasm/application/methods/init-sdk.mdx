初始化 SDK 全局实例、日志、长连接管理、本地数据目录和基础资源。应用进程内通常只调用一次。

## 功能介绍

初始化 SDK 全局实例、日志、长连接管理、本地数据目录和基础资源。应用进程内通常只调用一次。

初始化类方法应在应用进程内按需调用；用户态方法通常要求 `InitSDK` 已成功执行，并通过 `operationID` 串联日志。

## 底层能力

| 项目     | 说明                                                                       |
| -------- | -------------------------------------------------------------------------- |
| 方法签名 | `InitSDK(listener OnConnListener, operationID string, config string) bool` |
| 所属能力 | SDK 生命周期                                                               |
| 源码位置 | `open_im_sdk/init_login.go`                                                |

## WASM 调用参考

WASM SDK v3.8.3 通过 `getSDK(config)` 初始化浏览器 SDK 实例。该方法会创建或返回当前浏览器上下文中的单例 SDK，并配置运行时需要加载的 WASM 资源路径。

源码位置：`src/sdk/index.ts`。

```ts showLineNumbers
getSDK(config?: WasmPathConfig)
```

## 输入参数

| 参数名称 | 参数类型       | 是否必填 | 描述                                                                               |
| -------- | -------------- | -------- | ---------------------------------------------------------------------------------- |
| config   | WasmPathConfig | 否       | WASM 资源路径和调试配置。按运行环境传入 `coreWasmPath`、`sqlWasmPath` 和 `debug`。 |

## 返回结果

返回 SDK 实例。当前浏览器上下文中重复调用会返回同一个实例。

## 代码示例

以下示例展示 WASM SDK v3.8.3 的初始化方式。调用 `getSDK()` 前，请先把 WASM 资源发布到 Web 应用可访问的静态路径。

```ts showLineNumbers
import { getSDK } from '@openim/wasm-client-sdk';

const openIM = getSDK({
  coreWasmPath: '/openIM.wasm',
  sqlWasmPath: '/sql-wasm.wasm',
  debug: true,
});
```
