OpenIM WASM SDK 的错误信息主要来自两类数据：SDK 方法返回的 `WsResponse<T>`，以及运行时事件回调收到的事件载荷。业务代码应把 `errCode` 作为可判断的错误码，把 `errMsg` 作为诊断文本，并在失败日志中保留 `operationID`，用于把浏览器行为与 OpenIM 服务端日志对应起来。

多数业务方法调用失败时会 reject；在 `catch` 中记录错误对象、`operationID` 和相关业务 ID，成功分支只处理返回的 `data` 与后续事件。

```ts
type WsResponse<T> = {
  event: string;
  errCode: number;
  errMsg: string;
  data: T;
  operationID: string;
};

type WSEvent<T = unknown> = {
  event: CbEvents;
  data: T;
  errCode: number;
  errMsg: string;
  operationID: string;
};
```

## 客户端错误码

客户端侧问题通常发生在 SDK 请求发出前、WASM 资源加载阶段、浏览器运行时或连接生命周期中。它们可能表现为 Promise 异常、连接失败事件、同步失败事件，或包含非零 `errCode` 的 SDK 响应。

| 来源                     | 典型表现                                                           | 处理方式                                                             |
| ------------------------ | ------------------------------------------------------------------ | -------------------------------------------------------------------- |
| WASM 资源缺失            | `openIM.wasm`、`sql-wasm.wasm` 或 `wasm_exec.js` 请求 404          | 修正 public 目录和 `coreWasmPath`、`sqlWasmPath`。                   |
| SDK 在非浏览器环境初始化 | SSR、Node.js 或 Electron 主进程中访问浏览器能力失败                | 只在客户端组件、浏览器生命周期或 Electron 渲染进程中创建 SDK。       |
| 参数缺失或状态不一致     | `recvID`、`groupID`、`conversationID` 或消息对象不符合当前调用要求 | 在调用前校验目标 ID、登录状态和消息对象来源。                        |
| 连接失败                 | `CbEvents.OnConnectFailed` 触发                                    | 检查 `apiAddr`、`wsAddr`、token、`platformID: 5` 和网络连通性。      |
| 同步失败                 | `CbEvents.OnSyncServerFailed` 触发                                 | 保留 `operationID`，避免立即覆盖本地列表，必要时重新拉取会话或消息。 |

为关键写操作生成 `operationID`，并把它传给 SDK 方法。Promise 返回后先判断 `errCode`，再使用 `data` 更新本地状态。

```ts
function logOpenIMError(
  action: string,
  error: Pick<WsResponse<unknown>, 'errCode' | 'errMsg' | 'operationID'>,
) {
  console.error(`OpenIM ${action} failed`, {
    errCode: error.    errMsg: error.    operationID: error.operationID,
  });
}

const operationID = crypto.randomUUID();
const response = await OpenIM.sendMessage(
  {
    recvID,
    groupID: '',
    message,
  },
  operationID,
);

appendMessage(response.data);
```

对读取类 API 也使用同一套检查方式，避免把失败响应误当成空结果。

```ts
const history = await OpenIM.getAdvancedHistoryMessageList(
  {
    conversationID,
    startClientMsgID: '',
    count: 20,
  },
  operationID,
);

renderMessages(history.data.messageList);
```

## 服务端错误码

服务端错误码来自 OpenIM 服务端或你的业务后端策略。前端不应只根据数字猜测业务含义，而应同时记录 `errCode`、`errMsg`、`operationID`、当前用户、目标 ID 和正在执行的动作。

| 来源           | 典型表现                                        | 处理方式                                                        |
| -------------- | ----------------------------------------------- | --------------------------------------------------------------- |
| token 过期     | `CbEvents.OnUserTokenExpired` 触发              | 停止依赖当前连接状态，从可信后端刷新 token 或引导用户重新登录。 |
| token 无效     | `CbEvents.OnUserTokenInvalid` 触发              | 清理当前用户状态，重新走认证流程。                              |
| 权限不足       | 群组、好友、黑名单或消息 API 返回非零 `errCode` | 检查当前用户角色、群成员状态、好友关系和服务端策略。            |
| 目标不存在     | 用户、群组、会话或消息 ID 无法匹配服务端状态    | 重新读取目标信息，避免继续使用过期缓存。                        |
| 频率或策略限制 | 写操作被服务端拒绝，`errMsg` 指向限流或策略原因 | 前端节流按钮和重试频率，最终策略以服务端为准。                  |

连接、token 和同步类失败会通过事件通知业务层。连接事件建议在 `login()` 之前注册，这样登录阶段的失败也能被记录。

```ts
import { CbEvents } from '@openim/wasm-client-sdk';

OpenIM.on(CbEvents.OnConnectFailed, ({ errCode, errMsg, operationID }) => {
  logOpenIMError('connect', { errCode, errMsg, operationID });
  setConnectionState('failed');
});

OpenIM.on(CbEvents.OnUserTokenExpired, ({ errCode, errMsg, operationID }) => {
  logOpenIMError('token expired', { errCode, errMsg, operationID });
  redirectToSignIn();
});

OpenIM.on(CbEvents.OnUserTokenInvalid, ({ errCode, errMsg, operationID }) => {
  logOpenIMError('token invalid', { errCode, errMsg, operationID });
  redirectToSignIn();
});

OpenIM.on(CbEvents.OnSyncServerFailed, ({ errCode, errMsg, operationID }) => {
  logOpenIMError('sync server', { errCode, errMsg, operationID });
});
```

`login()` 本身的公开签名返回 `Promise<any>`，连接结果应结合连接事件判断。

```ts
await OpenIM.login(
  {
    userID,
    token,
    platformID: 5,
    apiAddr,
    wsAddr,
  },
  operationID,
);
```

## 日志字段

| 字段          | 用途                                                           |
| ------------- | -------------------------------------------------------------- |
| `errCode`     | SDK 或服务端返回的错误码。业务逻辑应优先判断这个字段。         |
| `errMsg`      | 错误说明，适合写入诊断日志；面向用户的提示建议转换成业务文案。 |
| `operationID` | 请求追踪 ID。关键调用建议主动传入，失败时必须随日志记录。      |
| `data`        | 成功时的业务数据；失败时不要依赖其中内容更新 UI 状态。         |
| `event`       | 响应或回调对应的事件名称，可用于区分连接、同步和业务操作。     |
