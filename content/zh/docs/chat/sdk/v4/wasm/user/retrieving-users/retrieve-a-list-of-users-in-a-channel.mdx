在 OpenIM WASM SDK 中，群聊频道中的用户以 `GroupMemberItem` 表示。成员列表、群资料页、@ 候选人和成员管理面板通常都应按 `groupID` 调用 `getGroupMemberList()` 分页读取，而不是把账号级公开资料当作群成员资料使用。

## 获取成员列表

调用 `getGroupMemberList()` 读取指定群的成员。第一页把 `offset` 设为 `0`，`count` 设为页面大小；继续加载时把 `offset` 增加到已经合并的成员数量。返回数量小于 `count` 时，当前筛选条件下通常没有更多成员。

```ts
const pageSize = 50;

async function loadMembers(groupID: string, offset = 0) {
  const operationID = crypto.randomUUID();
  const { data, errCode, errMsg } = await OpenIM.getGroupMemberList(
    {
      groupID,
      filter: 0,
      offset,
      count: pageSize,
    },
    operationID,
  );

  if (errCode !== 0) {
    console.error('getGroupMemberList failed', {
      errCode,
      errMsg,
      operationID,
      groupID,
      offset,
      count: pageSize,
    });
    throw new Error(errMsg || 'Failed to load group members');
  }

  mergeMembers(groupID, data);
  return data;
}

const firstPage = await loadMembers(groupID);
```

`filter` 的枚举含义由当前 SDK 版本定义。普通成员列表通常使用 `0`；如果只需要群主和管理员，请使用 [获取运营者列表](/docs/chat/sdk/v4/wasm/user/retrieving-users/retrieve-a-list-of-operators) 中的 `getGroupMemberOwnerAndAdmin(groupID)`。

如果界面已经持有一组 `userID`，例如打开成员资料卡或收到成员事件后刷新单个成员，可以调用 `getSpecifiedGroupMembersInfo()` 获取这些用户在当前群里的成员资料：

```ts
const { data: selectedMembers } = await OpenIM.getSpecifiedGroupMembersInfo({
  groupID,
  userIDList: selectedUserIDs,
});

mergeMembers(groupID, selectedMembers);
```

只需要判断一批用户是否仍在群内时，使用 `getUsersInGroup()` 返回命中的 `userID[]`。如果要展示昵称、头像、角色或禁言状态，仍应读取 `GroupMemberItem`。

```ts
const { data: existingUserIDs } = await OpenIM.getUsersInGroup({
  groupID,
  userIDList: candidateUserIDs,
});
```

成员列表应结合群成员事件保持同步。首次进入页面、刷新或重新登录后先调用 `getGroupMemberList()` 建立快照，之后再通过事件合并新增、删除和资料变化。

```ts
OpenIM.on(CbEvents.OnGroupMemberAdded, ({ data }) => {
  mergeMembers(data.groupID, [data]);
});

OpenIM.on(CbEvents.OnGroupMemberDeleted, ({ data }) => {
  removeMember(`${data.groupID}:${data.userID}`);
});

OpenIM.on(CbEvents.OnGroupMemberInfoChanged, ({ data }) => {
  mergeMembers(data.groupID, [data]);
});
```

## 成员属性

`getGroupMemberList()` 和 `getSpecifiedGroupMembersInfo()` 返回的成员都是 `GroupMemberItem`。这些字段属于群范围：同一个 `userID` 在不同群里可以有不同昵称、头像、角色和禁言状态。

| 字段             | 说明                                                  |
| ---------------- | ----------------------------------------------------- |
| `groupID`        | 成员所属群组 ID。                                     |
| `userID`         | 成员用户 ID，适合作为资料卡、事件合并和权限判断的键。 |
| `nickname`       | 群内展示昵称。                                        |
| `faceURL`        | 群内展示头像地址。                                    |
| `roleLevel`      | 群内角色，常见值包括普通成员、管理员和群主。          |
| `joinTime`       | 成员入群时间；渲染前确认当前 SDK 返回的时间单位。     |
| `muteEndTime`    | 成员禁言截止时间，大于当前时间时可视为仍在禁言期。    |
| `inviterUserID`  | 邀请该成员入群的用户 ID。                             |
| `operatorUserID` | 最近一次成员变更操作的操作者用户 ID。                 |
| `ex`             | 群成员扩展字段，用于存放业务约定的数据。              |

渲染群成员时优先使用 `GroupMemberItem` 的群内字段。`PublicUserItem` 适合补充账号级公开信息，但不应覆盖群内昵称、群角色或禁言状态。

```ts
function toMemberRow(member: GroupMemberItem) {
  return {
    id: member.userID,
    name: member.nickname || member.userID,
    avatar: member.faceURL,
    roleLevel: member.roleLevel,
    muted: Number(member.muteEndTime) > Date.now(),
  };
}
```
