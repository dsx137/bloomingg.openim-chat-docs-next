OpenIM 的成员禁言是群组范围内的管理操作。对同一个 `userID`，禁言只影响指定 `groupID` 下的发言能力，不会改变该用户在其他群组或单聊中的状态。

## 禁言用户

调用 `changeGroupMemberMute()`，传入目标 `groupID`、目标成员 `userID` 和禁言时长 `mutedSeconds`。禁言时长以秒为单位，服务端会据此更新该成员资料中的 `muteEndTime`。

```ts
async function muteGroupMember(groupID: string, userID: string, mutedSeconds: number) {
  const operationID = crypto.randomUUID();
  const { errCode, errMsg } = await OpenIM.changeGroupMemberMute(
    {
      groupID,
      userID,
      mutedSeconds,
    },
    operationID,
  );

  if (errCode !== 0) {
    console.error('changeGroupMemberMute failed', {
      errCode,
      errMsg,
      operationID,
      groupID,
      userID,
      mutedSeconds,
    });
    throw new Error(errMsg || 'Failed to mute group member');
  }
}

await muteGroupMember(groupID, targetUserID, 30 * 60);
```

不要只根据按钮点击立即把成员显示为已禁言。写操作返回成功后，可以等待成员资料事件，或主动读取目标成员资料来确认最终的 `muteEndTime`。

## 解除禁言用户

解除禁言仍然调用 `changeGroupMemberMute()`，并把 `mutedSeconds` 设为 `0`。

```ts
async function unmuteGroupMember(groupID: string, userID: string) {
  const operationID = crypto.randomUUID();
  const { errCode, errMsg } = await OpenIM.changeGroupMemberMute(
    {
      groupID,
      userID,
      mutedSeconds: 0,
    },
    operationID,
  );

  if (errCode !== 0) {
    console.error('changeGroupMemberMute failed', {
      errCode,
      errMsg,
      operationID,
      groupID,
      userID,
    });
    throw new Error(errMsg || 'Failed to unmute group member');
  }
}

await unmuteGroupMember(groupID, targetUserID);
```

如果产品需要展示禁言原因、操作者或管理备注，应把这些业务字段保存到自己的后端。OpenIM SDK 的成员禁言接口负责变更群成员禁言状态，不负责保存产品侧审计信息。

## 读取禁言状态

成员禁言状态保存在群成员资料中。`GroupMemberItem.muteEndTime` 大于当前时间时，表示该成员仍处于禁言期；为 `0` 或早于当前时间时，可视为未禁言。

```ts
function isMutedMember(member: GroupMemberItem) {
  return Number(member.muteEndTime) > Date.now();
}

const { data: members } = await OpenIM.getSpecifiedGroupMembersInfo({
  groupID,
  userIDList: [targetUserID],
});

const [member] = members;

if (member) {
  setMemberMuted(groupID, member.userID, isMutedMember(member));
}
```

成员列表页可以使用 `getGroupMemberList()` 分页读取群成员，并按 `muteEndTime` 过滤当前仍被禁言的成员。完整列表读取方式见 [获取被禁言用户列表](/docs/chat/sdk/v4/wasm/user/moderating-a-user/retrieve-a-list-of-muted-users)。

## 同步禁言变化

成员禁言和解除禁言会触发群成员资料变化。管理页、成员列表和聊天输入区可以监听 `CbEvents.OnGroupMemberInfoChanged`，把事件中的成员记录合并进本地状态。

```ts
function handleGroupMemberInfoChanged({ data }) {
  if (data.groupID !== groupID || data.userID !== targetUserID) {
    return;
  }

  mergeGroupMember(data);
  setMemberMuted(data.groupID, data.userID, isMutedMember(data));
}

OpenIM.on(CbEvents.OnGroupMemberInfoChanged, handleGroupMemberInfoChanged);

// 页面卸载、退出登录或切换账号时移除。
OpenIM.off(CbEvents.OnGroupMemberInfoChanged, handleGroupMemberInfoChanged);
```

禁言状态属于群成员状态，不是当前用户黑名单。用户侧拉黑和解除拉黑见 [拉黑和取消拉黑其他成员](/docs/chat/sdk/v4/wasm/user/moderating-a-user/block-and-unblock-other-members)。
