好友列表用于构建当前登录用户的通讯录、联系人选择器和好友关系缓存。页面需要展示好友备注、置顶状态或好友关系扩展字段时，应优先使用 `getFriendListPage()` 分页读取 `FriendUserItem`。

## 分页读取好友列表

通过 SDK 实例调用 `getFriendListPage()`，按 `offset` 和 `count` 读取当前用户的好友关系。即使当前好友数量不大，也建议统一使用分页接口，后续扩展滚动加载、刷新和增量更新时更稳定。

#### 请求参数

| 参数名 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `offset` | `number` | 必填 | 起始偏移量。第一页传 `0`。 |
| `count` | `number` | 必填 | 本次读取数量。 |
| `filterBlack` | `boolean` | 可选 | 是否过滤当前用户黑名单中的用户。普通通讯录通常传 `true`。 |
| `operationID` | `string` | 可选 | 链路追踪 ID，便于对应客户端和服务端日志。 |

#### 返回值

返回 `WsResponse<FriendUserItem[]>`。`data` 是当前用户的一页好友关系。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `userID` | `string` | 好友用户 ID。 |
| `nickname` | `string` | 好友账号级公开昵称。 |
| `faceURL` | `string` | 好友账号级公开头像。 |
| `remark` | `string` | 当前用户为该好友设置的备注。 |
| `isPinned` | `boolean` | 当前用户是否置顶该好友。 |
| `attachedInfo` | `string` | SDK 返回的附加信息。 |
| `ex` | `string` | 好友关系扩展字段。 |

```ts
const pageSize = 50;
const operationID = crypto.randomUUID();

const { data: friends } = await OpenIM.getFriendListPage(
  {
    offset: 0,
    count: pageSize,
    filterBlack: true,
  },
  operationID,
);

const rows = friends.map((friend) => ({
  userID: friend.userID,
  displayName: friend.remark || friend.nickname || friend.userID,
  avatar: friend.faceURL,
  pinned: friend.isPinned,
}));
```

## 分页状态

继续加载下一页时，把 `offset` 增加到已经合并的好友数量。若本次返回数量小于 `count`，通常表示已经没有更多好友。

```ts
function getNextFriendOffset(currentFriends: FriendUserItem[]) {
  return currentFriends.length;
}

function hasMoreFriends(lastPage: FriendUserItem[], pageSize: number) {
  return lastPage.length === pageSize;
}
```

添加好友、删除好友或修改好友资料后，优先通过好友事件中的 `data` 增量合并本地列表。页面首次进入、手动刷新或重连恢复后，再重置分页状态并重新读取第一页来校准完整列表。

## 搜索好友

在通讯录、转发选择器或发起单聊前选择联系人时，可以调用 `searchFriends()` 搜索当前用户的好友关系。它返回的仍然是好友数据，展示名、头像和关系字段的处理方式应与好友列表保持一致。

#### 请求参数

| 参数名 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `keywordList` | `string[]` | 必填 | 搜索关键字列表。调用前应去掉空字符串，并根据产品需要限制长度。 |
| `isSearchUserID` | `boolean` | 可选 | 是否匹配好友的 `userID`。 |
| `isSearchNickname` | `boolean` | 可选 | 是否匹配好友账号级昵称。 |
| `isSearchRemark` | `boolean` | 可选 | 是否匹配当前用户设置的好友备注。 |
| `operationID` | `string` | 可选 | 链路追踪 ID，便于对应客户端和服务端日志。 |

#### 返回值

返回匹配到的好友关系数据。渲染时优先使用 `remark` 作为联系人显示名，再回退到 `nickname` 或 `userID`。

```ts
async function searchFriends(keyword: string) {
  const normalizedKeyword = keyword.trim();

  if (!normalizedKeyword) {
    return [];
  }

  const { data: friends } = await OpenIM.searchFriends({
    keywordList: [normalizedKeyword],
    isSearchUserID: true,
    isSearchNickname: true,
    isSearchRemark: true,
  });

  return friends.map((friend) => ({
    userID: friend.userID,
    displayName: friend.remark || friend.nickname || friend.userID,
    avatar: friend.faceURL,
  }));
}
```

如果产品流程是“搜索并添加新好友”，通常先通过业务后端或精确 `userID` 找到候选用户，再使用 [查询应用用户](/docs/chat/sdk/v4/wasm/user/retrieving-users/retrieve-a-list-of-users-in-an-application) 展示公开资料，随后进入好友申请流程。

## 同步列表变化

好友列表同步关注好友关系和好友资料事件。黑名单有独立页面和独立状态，黑名单事件应在黑名单相关状态中处理。

```ts
function handleFriendAdded({ data }) {
  upsertFriend(data);
}

function handleFriendDeleted({ data }) {
  removeFriend(data.userID);
}

function handleFriendInfoChanged({ data }) {
  upsertFriend(data);
}

OpenIM.on(CbEvents.OnFriendAdded, handleFriendAdded);
OpenIM.on(CbEvents.OnFriendDeleted, handleFriendDeleted);
OpenIM.on(CbEvents.OnFriendInfoChanged, handleFriendInfoChanged);
```
