账号级封禁列表属于应用管理数据，不是 OpenIM WASM SDK 的浏览器端列表能力。WASM SDK 可以读取当前用户黑名单、群成员列表和群成员禁言状态；平台级“被封禁用户列表”应从可信后端或管理后台接口读取。

如果页面是面向管理员的封禁列表，前端应调用自己的后端接口。后端负责校验当前管理员权限、分页查询封禁记录、脱敏展示字段，并记录审计日志。

## 与相近列表的区别

| 列表 | 数据范围 | 读取方式 |
| --- | --- | --- |
| 被封禁用户列表 | 应用或租户范围，通常由管理员维护 | 可信后端或管理后台接口。 |
| 黑名单用户列表 | 当前登录用户拉黑的人 | `getBlackList()`，见 [获取黑名单用户列表](/docs/chat/sdk/v4/wasm/user/moderating-a-user/retrieve-a-list-of-blocked-users)。 |
| 被禁言成员列表 | 某个群内当前仍处于禁言期的成员 | `getGroupMemberList()` 后按 `muteEndTime` 过滤，见 [获取被禁言成员列表](/docs/chat/sdk/v4/wasm/user/moderating-a-user/retrieve-a-list-of-muted-users)。 |
| 群成员移除记录 | 某个群的成员变化 | 由群管理操作和后端审计记录维护。 |

不要把当前用户黑名单当作平台封禁列表。黑名单只影响当前用户视角，不能代表账号被管理员封禁。

## 读取后端封禁列表

后端封禁列表通常需要分页、搜索和状态筛选。前端只传递筛选条件，最终权限判断应在后端完成。

```ts
type BannedUserRecord = {
  userID: string;
  reason: string;
  operatorID: string;
  bannedAt: number;
  expiresAt?: number;
};

async function loadBannedUsers(cursor?: string) {
  const operationID = crypto.randomUUID();

  try {
    const page = await backend.listBannedUsers({
      cursor,
      limit: 50,
      operationID,
    });

    appendBannedUsers(page.items);
    return page;
  } catch (error) {
    console.error('listBannedUsers failed', { error, operationID, cursor });
    throw error;
  }
}
```

展示时优先使用后端封禁记录里的 `userID`、封禁原因、操作者和时间字段。昵称、头像等公开资料可以按需用 `getUsersInfo()` 读取，但它们不是封禁状态的权威来源。

```ts
async function attachPublicProfiles(records: BannedUserRecord[]) {
  const operationID = crypto.randomUUID();
  const userIDs = records.map((record) => record.userID);

  try {
    const response = await OpenIM.getUsersInfo(userIDs, operationID);
    const users = response.data ?? [];
    const profileMap = new Map(users.map((user) => [user.userID, user]));

    return records.map((record) => ({
      ...record,
      profile: profileMap.get(record.userID),
    }));
  } catch (error) {
    console.error('getUsersInfo failed', { error, operationID, userIDs });
    throw error;
  }
}
```

如果封禁列表里包含敏感原因、内部备注或风控标签，应由后端按管理员角色返回不同字段，前端不要在本地做权限裁剪。

## 同步封禁变化

账号级封禁变化没有对应的 OpenIM WASM SDK 客户端事件。管理页可以使用后端推送、轮询或操作成功后的局部更新来同步列表。

```ts
function handleBackendBanAdded(record: BannedUserRecord) {
  upsertBannedUser(record);
}

function handleBackendBanRemoved(userID: string) {
  removeBannedUser(userID);
}

backendEvents.on('ban-added', handleBackendBanAdded);
backendEvents.on('ban-removed', handleBackendBanRemoved);
```

如果当前页面同时支持群内踢人，可以监听 `CbEvents.OnGroupMemberDeleted` 来更新群成员列表；但这个事件只说明成员从某个群移除，不代表账号进入平台封禁列表。
