OpenIM WASM SDK 将当前登录用户资料和其他用户公开资料分开处理。当前用户资料通过 `getSelfUserInfo()` 读取，并可通过 `setSelfInfo()` 更新昵称、头像和扩展字段等由用户自己维护的信息。其他用户资料通过 `getUsersInfo()` 按 `userID` 批量读取，浏览器端不应把它当作修改其他账号资料的入口。

如果业务需要管理员代改资料、维护组织通讯录或同步应用侧用户目录，应通过可信后端完成权限校验后再调用服务端能力。

## 读取当前用户资料

完成 SDK 初始化和 `login()` 后，调用 `getSelfUserInfo()` 获取当前账号资料。它适合初始化个人资料页、设置页顶部头像、消息发送者本地展示名，以及账号切换后的本地状态恢复。

```ts
async function loadCurrentUser() {
  const operationID = crypto.randomUUID();
  const { data, errCode, errMsg } = await OpenIM.getSelfUserInfo(operationID);

  if (errCode !== 0) {
    console.error('getSelfUserInfo failed', {
      errCode,
      errMsg,
      operationID,
    });
    throw new Error(errMsg || 'Failed to load current user');
  }

  setCurrentUser(data);
  return data;
}

const me = await loadCurrentUser();
```

## 更新当前用户资料

`setSelfInfo()` 只应用于当前登录用户。常见可由用户自己更新的字段包括 `nickname`、`faceURL` 和 `ex`。更新时只提交发生变化的字段；头像地址应使用可信上传结果，`ex` 建议保存 JSON 字符串。

```ts
async function updateMyProfile(input: {
  nickname?: string;
  faceURL?: string;
  extra?: Record<string, unknown>;
}) {
  const payload: PartialUserItem = {};

  if (input.nickname !== undefined) {
    payload.nickname = input.nickname.trim();
  }

  if (input.faceURL !== undefined) {
    payload.faceURL = input.faceURL.trim();
  }

  if (input.extra !== undefined) {
    payload.ex = JSON.stringify(input.extra);
  }

  if (Object.keys(payload).length === 0) {
    return;
  }

  const operationID = crypto.randomUUID();
  const { errCode, errMsg } = await OpenIM.setSelfInfo(payload, operationID);

  if (errCode !== 0) {
    console.error('setSelfInfo failed', {
      errCode,
      errMsg,
      operationID,
      fields: Object.keys(payload),
    });
    throw new Error(errMsg || 'Failed to update current user');
  }

  return loadCurrentUser();
}
```

不要在浏览器端构造“目标用户 ID + 新资料”的管理式编辑流程。`setSelfInfo()` 没有用于修改任意用户资料的目标 `userID` 参数，业务 UI 也不应暗示当前用户可以直接修改其他账号资料。

## 同步资料变化

当前账号资料变化后，SDK 会通过 `CbEvents.OnSelfInfoUpdated` 通知客户端。资料设置页、应用顶部用户菜单和全局用户状态层都可以监听这个事件，并把新的 `SelfUserInfo` 合并到本地状态。

```ts
function handleSelfInfoUpdated({ data }) {
  setCurrentUser(data);
}

OpenIM.on(CbEvents.OnSelfInfoUpdated, handleSelfInfoUpdated);

// 退出登录、切换账号或组件卸载时移除。
OpenIM.off(CbEvents.OnSelfInfoUpdated, handleSelfInfoUpdated);
```

如果资料设置页希望提交后立即展示服务端最终结果，可以在 `setSelfInfo()` 成功后重新调用 `getSelfUserInfo()`；如果页面已经统一处理 `OnSelfInfoUpdated`，也可以由事件驱动刷新。

## 读取其他用户公开资料

需要展示其他用户时，使用 `getUsersInfo()` 按已知 `userID` 批量读取公开资料。这个接口适合资料卡、会话标题、@ 候选人或联系人补全；公开资料通常只用于展示，不应覆盖群成员昵称、好友备注或应用后端维护的组织信息。

```ts
function uniqueUserIDs(userIDs: string[]) {
  return [...new Set(userIDs.filter(Boolean))];
}

async function loadPublicUsers(userIDs: string[]) {
  const targetUserIDs = uniqueUserIDs(userIDs);

  if (targetUserIDs.length === 0) {
    return [];
  }

  const { data } = await OpenIM.getUsersInfo(targetUserIDs);
  mergePublicUsers(data);
  return data;
}
```

更多用户对象的区别见 [用户概览](/docs/chat/sdk/v4/wasm/user/overview-user)，在线状态读取见 [获取用户在线状态](/docs/chat/sdk/v4/wasm/user/retrieving-and-updating-user-information/retrieve-the-online-status-of-a-user)。
