账号级封禁是应用管理能力，不应由浏览器端 WASM SDK 直接决定。OpenIM WASM SDK 可以读取当前用户关系、执行群组内踢人或禁言等群范围操作，但“封禁某个账号登录、发消息或进入应用”需要由可信后端完成鉴权、审计和策略落库。

如果你的产品界面里有“封禁用户”按钮，浏览器端应只把操作者、目标用户和原因提交给后端。后端确认当前操作者有权限后，再调用服务端能力或更新业务侧封禁状态。

## 能力边界

| 需求 | 推荐实现 |
| --- | --- |
| 平台级封禁或解封账号 | 可信后端或管理后台处理，浏览器端只提交请求。 |
| 从某个群移除成员 | 使用 `kickGroupMember()`，只影响指定 `groupID`。 |
| 禁止某个成员在群里发言 | 使用 `changeGroupMemberMute()`，只影响指定群成员。 |
| 当前用户屏蔽另一个用户 | 使用 `addBlack()` 和 `removeBlack()`，这是当前用户维度的黑名单。 |

封禁状态通常还需要记录原因、操作者、封禁时间、到期时间和审计日志。这些字段应保存在业务后端；不要把它们只放在浏览器本地状态或 SDK 事件里。

## 提交封禁请求

浏览器端可以把操作提交给自己的后端。`operationID` 建议随请求一起传递，便于把前端日志、后端日志和管理后台审计记录串起来。

```ts
async function banUser(targetUserID: string, reason: string) {
  const operationID = crypto.randomUUID();

  try {
    await backend.banUser({
      targetUserID,
      reason,
      operationID,
    });

    markUserBanned(targetUserID);
  } catch (error) {
    console.error('banUser failed', { error, operationID, targetUserID });
    throw error;
  }
}
```

解封同样应走后端权限校验。前端收到成功结果后再更新当前管理页状态。

```ts
async function unbanUser(targetUserID: string) {
  const operationID = crypto.randomUUID();

  try {
    await backend.unbanUser({
      targetUserID,
      operationID,
    });

    markUserUnbanned(targetUserID);
  } catch (error) {
    console.error('unbanUser failed', { error, operationID, targetUserID });
    throw error;
  }
}
```

## 群内替代操作

如果产品需求只是把用户移出某个群，不需要平台级封禁，可以调用 `kickGroupMember()`。这个操作只改变群成员关系，不代表账号被封禁。

```ts
async function removeMemberFromGroup(groupID: string, targetUserID: string) {
  const operationID = crypto.randomUUID();

  try {
    await OpenIM.kickGroupMember(
      {
        groupID,
        reason: 'Removed by moderator',
        userIDList: [targetUserID],
      },
      operationID,
    );
  } catch (error) {
    console.error('kickGroupMember failed', { error, operationID, groupID, targetUserID });
    throw error;
  }
}
```

如果只是临时限制发言，使用 [禁言和解除禁言用户](/docs/chat/sdk/v4/wasm/user/moderating-a-user/mute-and-unmute-a-user)。如果只是当前用户不想再接收某人的消息，使用 [拉黑和解除拉黑其他用户](/docs/chat/sdk/v4/wasm/user/moderating-a-user/block-and-unblock-other-members)。

## 同步界面状态

账号级封禁和解封由后端维护，管理页应通过后端返回结果、轮询、WebSocket 或服务端推送更新状态。OpenIM WASM SDK 没有“账号被平台封禁列表变化”的客户端事件。

群成员被移出时，可以监听群成员删除事件，并只处理事件里携带的增量数据。

```ts
function handleGroupMemberDeleted({ data }) {
  if (data.groupID !== groupID) {
    return;
  }

  removeGroupMember(data.userID);
}

OpenIM.on(CbEvents.OnGroupMemberDeleted, handleGroupMemberDeleted);
```

需要展示所有被封禁账号时，读取你的后端封禁列表。完整列表页见 [获取被封禁用户列表](/docs/chat/sdk/v4/wasm/user/moderating-a-user/retrieve-a-list-of-banned-users)。
