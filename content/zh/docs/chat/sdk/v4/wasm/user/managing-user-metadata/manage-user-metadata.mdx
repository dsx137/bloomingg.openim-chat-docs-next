OpenIM WASM SDK 没有独立的用户元数据对象。用户相关的扩展信息需要按数据归属拆开处理：当前用户资料可以写入 `SelfUserInfo.ex`，好友关系信息可以写入 `FriendUserItem.ex`，应用侧的组织、标签、风控或运营字段应保存在可信后端。

不要把所有业务字段都塞进 SDK 的 `ex`。`ex` 更适合保存轻量、低敏感度、和聊天展示强相关的 JSON 字符串；权限、审计、搜索索引和后台管理字段应由业务后端维护。

## 数据归属

| 数据 | 推荐位置 | 说明 |
| --- | --- | --- |
| 当前用户可自维护的昵称、头像、少量展示扩展 | `setSelfInfo()` / `getSelfUserInfo()` | 只修改当前登录用户允许自己维护的字段。 |
| 好友备注、置顶、好友关系扩展字段 | `updateFriends()` / `getSpecifiedFriendsInfo()` | 属于当前用户视角下的好友关系。 |
| 其他用户公开资料 | `getUsersInfo()` | 按已知 `userID` 读取公开资料，不能用于浏览器端修改其他账号。 |
| 组织、角色、封禁、内部标签、审计信息 | 可信后端 | 后端负责鉴权、脱敏、审计和搜索。 |

如果一个字段会影响权限、计费、风控或运营后台，不应由浏览器端直接写入。

## 更新当前用户扩展字段

`setSelfInfo()` 只应用于当前登录用户。写入 `ex` 前建议先把业务对象整理成稳定结构，再序列化为 JSON 字符串。

```ts
type ProfileExtra = {
  title?: string;
  department?: string;
};

async function updateMyProfileExtra(extra: ProfileExtra) {
  const operationID = crypto.randomUUID();

  try {
    await OpenIM.setSelfInfo(
      {
        ex: JSON.stringify(extra),
      },
      operationID,
    );

    const { data } = await OpenIM.getSelfUserInfo(operationID);
    setCurrentUser(data);
    return data;
  } catch (error) {
    console.error('updateMyProfileExtra failed', { error, operationID });
    throw error;
  }
}
```

读取 `ex` 时要做好空值和 JSON 解析失败的兜底，避免旧数据或非 JSON 字符串影响页面渲染。

```ts
function parseProfileExtra(ex?: string) {
  if (!ex) {
    return {};
  }

  try {
    return JSON.parse(ex) as ProfileExtra;
  } catch {
    return {};
  }
}
```

## 更新好友关系扩展字段

好友备注、置顶状态和好友关系扩展字段属于当前用户自己的联系人关系。需要给好友写备注或关系侧扩展信息时，使用 `updateFriends()`。

```ts
async function updateFriendExtra(friendUserID: string, extra: Record<string, unknown>) {
  const operationID = crypto.randomUUID();

  try {
    await OpenIM.updateFriends(
      {
        friendUserIDs: [friendUserID],
        ex: JSON.stringify(extra),
      },
      operationID,
    );
  } catch (error) {
    console.error('updateFriendExtra failed', { error, operationID, friendUserID });
    throw error;
  }
}
```

好友资料页展示时，优先使用好友关系里的 `remark`、`isPinned` 和 `ex`；公开资料里的昵称、头像只作为展示补充。

## 读取公开资料扩展字段

需要展示其他用户公开资料时，按已知 `userID` 调用 `getUsersInfo()`。如果你的产品需要按手机号、组织、邮箱或标签搜索用户，应先通过后端搜索出允许当前用户查看的 `userID`，再把这些 ID 传给 SDK 读取公开资料。

```ts
async function loadPublicProfileExtras(userIDs: string[]) {
  if (userIDs.length === 0) {
    return [];
  }

  const operationID = crypto.randomUUID();

  try {
    const response = await OpenIM.getUsersInfo(userIDs, operationID);
    const users = response.data ?? [];

    return users.map((user) => ({
      userID: user.userID,
      extra: parseProfileExtra(user.ex),
    }));
  } catch (error) {
    console.error('getUsersInfo failed', { error, operationID, userIDs });
    throw error;
  }
}
```

会话列表和聊天页标题仍应优先使用 `ConversationItem.showName`；不要为了补标题而额外批量调用 `getUsersInfo()`。

## 同步元数据变化

当前用户资料变化后，监听 `CbEvents.OnSelfInfoUpdated`。好友关系扩展字段变化后，监听 `CbEvents.OnFriendInfoChanged`，并把事件里的增量数据合并到本地缓存。

```ts
function handleSelfInfoUpdated({ data }) {
  setCurrentUser(data);
}

function handleFriendInfoChanged({ data }) {
  mergeFriend(data);
}

OpenIM.on(CbEvents.OnSelfInfoUpdated, handleSelfInfoUpdated);
OpenIM.on(CbEvents.OnFriendInfoChanged, handleFriendInfoChanged);
```

如果元数据由后端维护，前端应以后端响应或后端推送为准。更多当前用户资料更新方式见 [更新用户资料](/docs/chat/sdk/v4/wasm/user/retrieving-and-updating-user-information/update-user-profile)，好友关系字段见 [更新或删除好友](/docs/chat/sdk/v4/wasm/user/managing-friends/update-or-delete-friends)。
