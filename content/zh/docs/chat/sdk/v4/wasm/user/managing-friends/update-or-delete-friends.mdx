好友备注、置顶状态和好友关系扩展字段都属于当前用户的关系链数据。使用 `updateFriends()` 更新当前用户视角下的好友资料，使用 `deleteFriend()` 删除当前用户与目标用户之间的好友关系。

## 更新好友关系资料

通过 SDK 实例调用 `updateFriends()`，可以更新好友备注、置顶状态和好友关系扩展字段。一次调用会把提交的字段应用到 `friendUserIDs` 指定的目标好友；如果不同好友需要写入不同备注，建议按好友分别调用。

#### 请求参数

| 参数名 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `friendUserIDs` | `string[]` | 必填 | 要更新的好友用户 ID 列表。 |
| `remark` | `string` | 可选 | 好友备注。 |
| `isPinned` | `boolean` | 可选 | 是否置顶该好友。 |
| `ex` | `string` | 可选 | 好友关系扩展字段。 |
| `operationID` | `string` | 可选 | 链路追踪 ID，便于对应客户端和服务端日志。 |

#### 返回值

返回 `WsResponse<unknown>`。调用成功后，好友列表和联系人详情页可以等待 `CbEvents.OnFriendInfoChanged`，或重新读取相关好友资料确认最终状态。

```ts
async function updateFriendRemark(friendUserID: string, remark: string) {
  const operationID = crypto.randomUUID();

  try {
    await OpenIM.updateFriends(
      {
        friendUserIDs: [friendUserID],
        remark: remark.trim(),
        ex: JSON.stringify({ updatedFrom: 'contact-detail' }),
      },
      operationID,
    );
  } catch (error) {
    console.error('updateFriends failed', { error, operationID, friendUserID });
    throw error;
  }
}
```

## 删除好友关系

通过 SDK 实例调用 `deleteFriend()`，可以删除当前用户与目标用户的好友关系。删除后应更新好友列表、好友资料卡和相关入口状态。

```ts
async function deleteFriend(friendUserID: string) {
  const operationID = crypto.randomUUID();

  try {
    await OpenIM.deleteFriend(friendUserID, operationID);
  } catch (error) {
    console.error('deleteFriend failed', { error, operationID, friendUserID });
    throw error;
  }
}
```

删除成功后，本地列表可以先把目标 `userID` 从好友集合中移除，再通过事件或重新读取第一页校准状态。资料卡上的“发消息”“修改备注”等好友关系入口也应一起刷新。

## 事件同步

好友关系资料变化后，SDK 会通过事件通知当前账号的已登录端：

| 事件 | 适用场景 |
| --- | --- |
| `CbEvents.OnFriendInfoChanged` | 好友备注、置顶状态或扩展字段变化。 |
| `CbEvents.OnFriendDeleted` | 好友关系被删除。 |

```ts
function handleFriendInfoChanged({ data }) {
  mergeFriend(data);
}

function handleFriendDeleted({ data }) {
  removeFriend(data.userID);
  resetContactDetailIfNeeded(data.userID);
}

OpenIM.on(CbEvents.OnFriendInfoChanged, handleFriendInfoChanged);
OpenIM.on(CbEvents.OnFriendDeleted, handleFriendDeleted);
```

如果更新或删除后没有收到预期事件，可以调用 `getSpecifiedFriendsInfo()` 或 `checkFriend()` 对目标用户做一次确认。
