OpenIM WASM SDK 使用 `conversationID` 和 `clientMsgID` 游标读取历史消息。进入会话时先请求第一页；用户上拉或滚动到边界时，再把上一页边界消息的 `clientMsgID` 作为下一次请求的 `startClientMsgID`。

OpenIM 的历史消息读取以会话为中心。单聊和群聊使用同一套分页模型；群聊只是 `sessionType` 为 `3` 的会话，不需要为“开放频道”和“群频道”维护两套消息列表接口。

## 按会话读取消息

常规聊天页优先使用 `getAdvancedHistoryMessageList()`。第一页没有游标时，把 `startClientMsgID` 设为空字符串；继续加载时，使用当前历史列表边界消息的 `clientMsgID`。

```ts
import { ViewType } from '@openim/wasm-client-sdk';
import type { MessageItem } from '@openim/wasm-client-sdk';

type HistoryPage = {
  messages: MessageItem[];
  nextCursor: string;
  isEnd: boolean;
};

const loadHistoryPage = async (
  conversationID: string,
  startClientMsgID = '',
): Promise<HistoryPage> => {
  const operationID = crypto.randomUUID();

  const { data, errCode, errMsg } = await OpenIM.getAdvancedHistoryMessageList(
    {
      conversationID,
      startClientMsgID,
      count: 30,
      viewType: ViewType.History,
    },
    operationID,
  );

  if (errCode !== 0 || data.errCode !== 0) {
    console.error('getAdvancedHistoryMessageList failed', {
      errCode,
      errMsg,
      dataErrCode: data.errCode,
      dataErrMsg: data.errMsg,
      operationID,
      conversationID,
      startClientMsgID,
    });
    throw new Error(data.errMsg || errMsg || 'Failed to load messages');
  }

  return {
    messages: data.messageList,
    nextCursor: data.messageList.at(-1)?.clientMsgID ?? '',
    isEnd: data.isEnd,
  };
};
```

调用方可以把第一页用于初始化消息列表，把后续页追加到列表的历史方向。合并消息时建议以 `clientMsgID` 去重，避免登录同步、新消息事件和历史分页返回同一条消息。

```ts
let historyCursor = '';
let reachedHistoryEnd = false;

async function loadOlderMessages(conversationID: string) {
  if (reachedHistoryEnd) {
    return;
  }

  const page = await loadHistoryPage(conversationID, historyCursor);

  prependMessages(page.messages);
  historyCursor = page.nextCursor;
  reachedHistoryEnd = page.isEnd || !page.nextCursor;
}
```

#### 历史消息参数

| 参数               | 说明                                                                                                              |
| ------------------ | ----------------------------------------------------------------------------------------------------------------- |
| `conversationID`   | 要读取历史消息的会话 ID。它来自会话列表、会话事件，或通过 `getConversationIDBySessionType()` 根据用户或群组换取。 |
| `startClientMsgID` | 分页游标。第一页传空字符串；后续请求传入上一页边界消息的 `clientMsgID`。                                          |
| `count`            | 每页消息数量。聊天页常用 20 到 50 条，避免一次性渲染过多历史消息。                                                |
| `viewType`         | 读取视图类型。历史列表使用 `ViewType.History`，对应数值 `0`。                                                     |
| `operationID`      | 可选的请求追踪 ID；排查失败时和 `errCode`、`errMsg` 一起记录。                                                    |

返回值中的 `data.messageList` 是本页消息，`data.isEnd` 表示是否已经到达历史边界。

```ts
type AdvancedGetMessageResult = {
  isEnd: boolean;
  errCode: number;
  errMsg: string;
  messageList: MessageItem[];
};
```

## 群聊消息列表

如果业务入口只有 `groupID`，先取得群组对应的 `conversationID`，再使用同一个历史消息接口读取列表。

```ts
const { data: conversationID } = await OpenIM.getConversationIDBySessionType({
  sourceID: groupID,
  sessionType: 3,
});

const { data: firstPage } = await OpenIM.getAdvancedHistoryMessageList({
  conversationID,
  startClientMsgID: '',
  count: 30,
  viewType: ViewType.History,
});
```

也可以通过 `getOneConversation()` 读取群会话状态，再使用返回的 `conversationID` 打开聊天页。

```ts
const { data: conversation } = await OpenIM.getOneConversation({
  sourceID: groupID,
  sessionType: 3,
});

openGroupMessageList(conversation.conversationID);
```

群聊页面打开期间，新消息不需要等下一次历史分页，可以直接通过事件合并。

```ts
function handleNewMessages({ data }: { data: MessageItem[] }) {
  mergeMessagesByClientMsgID(data.filter((message) => message.conversationID === conversationID));
}

function refreshAfterSync() {
  reloadVisibleConversationIfNeeded();
}

OpenIM.on(CbEvents.OnRecvNewMessages, handleNewMessages);
OpenIM.on(CbEvents.OnSyncServerFinish, refreshAfterSync);

OpenIM.off(CbEvents.OnRecvNewMessages, handleNewMessages);
OpenIM.off(CbEvents.OnSyncServerFinish, refreshAfterSync);
```

## 选择读取方向

`getAdvancedHistoryMessageList()` 和 `getAdvancedHistoryMessageListReverse()` 使用同一组参数，适合已经持有 `conversationID` 的聊天页。两者返回的都是 `AdvancedGetMessageResult`，包含 `messageList` 和 `isEnd`。

```ts
const { data: result } = await OpenIM.getAdvancedHistoryMessageListReverse({
  conversationID,
  startClientMsgID: cursorClientMsgID,
  count: 30,
  viewType: ViewType.History,
});
```

如果旧代码只有单聊用户 ID 或群组 ID，也可以使用 `getHistoryMessageListReverse()`。该方法仍建议传入 `conversationID`；单聊时填写 `userID`，群聊时填写 `groupID`，另一侧目标字段保持为空字符串。

```ts
const { data: groupHistory } = await OpenIM.getHistoryMessageListReverse({
  userID: '',
  groupID,
  conversationID,
  startClientMsgID: cursorClientMsgID,
  count: 30,
});
```

新页面通常不需要在高级历史接口和旧接口之间混用。统一一种读取方向和一种游标规则，可以减少重复消息、顺序反转和滚动位置跳动。

## 游标和空列表

当 `messageList` 为空且 `isEnd` 为 `true` 时，可以停止继续请求历史页。当 `messageList` 为空但界面仍需要等待同步完成时，先保留当前游标，并在收到 `CbEvents.OnSyncServerFinish` 后重新尝试当前会话。

不要用数组下标作为分页游标。消息撤回、删除、本地同步和多端登录都可能改变当前列表长度；`clientMsgID` 才是历史读取和去重应使用的稳定标识。
