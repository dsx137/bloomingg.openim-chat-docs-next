在群聊中，消息已读状态用于判断某条消息已经被哪些成员阅读。OpenIM WASM SDK 不返回整个群频道的用户已读时间字典；它以单条群消息为维度，通过 `getGroupMessageReaderList()` 查询已读或未读成员列表。

查询前需要准备消息所在的 `conversationID` 和目标消息的 `clientMsgID`。`filter: 0` 查询已读成员，`filter: 1` 查询未读成员。

```ts
const { data: readMembers } = await OpenIM.getGroupMessageReaderList({
  conversationID,
  clientMsgID,
  filter: 0,
  offset: 0,
  count: 50,
});

renderReadMembers(readMembers);
```

## 查询未读成员

同一个 API 可以查询仍未读该消息的群成员。未读列表适合消息详情页、发送后的触达状态，或管理员查看群内消息阅读情况。

```ts
const { data: unreadMembers } = await OpenIM.getGroupMessageReaderList({
  conversationID,
  clientMsgID,
  filter: 1,
  offset: 0,
  count: 50,
});

renderUnreadMembers(unreadMembers);
```

返回值是当前页的 `GroupMemberItem[]`。如果需要全量精确数量，继续分页直到返回数量小于 `count`；如果界面只需要当前页列表或入口数字，可以按页面大小增量加载。

## 同步已读变化

成员读过群消息后，需要在阅读端调用 `sendGroupMessageReadReceipt()` 上报回执。发送端或其他在线客户端可以监听 `CbEvents.OnRecvGroupReadReceipt`，把事件中的已读人数、未读人数和已读成员合并到本地消息状态。

```ts
import { CbEvents } from '@openim/wasm-client-sdk';

function handleGroupReadReceipt({ data }) {
  for (const item of data.groupMessageReadInfo) {
    updateMessageReadState(item.clientMsgID, {
      readCount: item.hasReadCount,
      unreadCount: item.unreadCount,
      readMembers: item.readMembers,
    });
  }
}

OpenIM.on(CbEvents.OnRecvGroupReadReceipt, handleGroupReadReceipt);
```

群消息读状态和会话未读数不是同一个概念。`markConversationMessageAsRead(conversationID)` 用于清理当前用户在该会话中的未读数；`sendGroupMessageReadReceipt()` 和 `getGroupMessageReaderList()` 才用于某条群消息的成员级已读状态。

## 参数列表

| 参数             | 说明                                 |
| ---------------- | ------------------------------------ |
| `conversationID` | 群消息所在的会话 ID。                |
| `clientMsgID`    | 要查询读状态的消息 ID。              |
| `filter`         | `0` 查询已读成员，`1` 查询未读成员。 |
| `offset`         | 分页起始位置，第一页传 `0`。         |
| `count`          | 本次查询数量。                       |
