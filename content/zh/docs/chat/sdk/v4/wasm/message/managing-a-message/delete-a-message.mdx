在 OpenIM 中，删除消息前需要先确认产品动作的影响范围。`deleteMessage()` 按 OpenIM 服务端规则删除指定消息；`deleteMessageFromLocalStorage()` 只移除当前客户端的本地缓存；`revokeMessage()` 会让会话成员通过撤回事件看到消息已撤回。

删除指定消息时，传入消息所在的 `conversationID` 和消息的 `clientMsgID`。如果当前用户无权删除该消息，SDK 会通过响应中的 `errCode` 和 `errMsg` 返回失败原因。

```ts
const operationID = crypto.randomUUID();

await OpenIM.deleteMessage(
  {
    conversationID,
    clientMsgID,
  },
  operationID,
);

markMessageDeleted(clientMsgID);
```

## 本地删除

如果用户动作是“只从我这里删除”，使用 `deleteMessageFromLocalStorage()`。这个操作只影响当前客户端缓存，不应让其他成员或其他设备据此更新消息状态。

```ts
await OpenIM.deleteMessageFromLocalStorage({
  conversationID,
  clientMsgID,
});

removeMessageFromCurrentList(clientMsgID);
```

删除的是当前会话最后一条消息时，聊天列表的最后消息、未读数和排序应以 `CbEvents.OnConversationChanged` 或重新查询到的会话数据为准。

## 撤回消息

如果界面需要让其他会话成员看到“消息已撤回”，不要用普通删除替代撤回。调用 `revokeMessage()` 后，在线客户端会收到 `CbEvents.OnNewRecvMessageRevoked`，接收端应根据事件里的 `clientMsgID` 把对应消息更新为撤回态。

```ts
import { CbEvents } from '@openim/wasm-client-sdk';

function handleMessageRevoked({ data }) {
  markMessageRevoked(data.clientMsgID);
}

OpenIM.on(CbEvents.OnNewRecvMessageRevoked, handleMessageRevoked);

const { data } = await OpenIM.revokeMessage({
  conversationID,
  clientMsgID,
});

markMessageRevoked(data?.clientMsgID ?? clientMsgID);
```

## 删除结果

删除、撤回和本地删除调用失败时会 reject。成功分支根据返回数据和后续事件更新 UI；失败日志中记录错误对象、`operationID`、`conversationID` 和 `clientMsgID`。

| 用户动作                   | API                               | 结果                                                        |
| -------------------------- | --------------------------------- | ----------------------------------------------------------- |
| 删除指定消息               | `deleteMessage()`                 | 按服务端和当前用户权限处理，适合产品定义为“删除”的入口。    |
| 仅从当前设备的消息列表移除 | `deleteMessageFromLocalStorage()` | 只影响当前客户端本地缓存。                                  |
| 让会话成员看到撤回态       | `revokeMessage()`                 | 触发 `CbEvents.OnNewRecvMessageRevoked`，用于同步撤回展示。 |
