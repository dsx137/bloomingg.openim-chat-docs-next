OpenIM WASM SDK 发送消息时先创建 `MessageItem`，再把它交给 `sendMessage()`。单聊消息填写 `recvID` 并把 `groupID` 设为空字符串；群聊消息填写 `groupID` 并把 `recvID` 设为空字符串。

调用发送 API 前，请确保浏览器端已经完成 SDK 初始化和 `login()`。如果需要排查链路问题，可以为创建和发送调用传入同一个 `operationID`，并在失败日志中记录 `errCode`、`errMsg` 和 `operationID`。

#### 消息类型列表

| 消息类型          | 创建方法                                          | 说明                                           |
| ----------------- | ------------------------------------------------- | ---------------------------------------------- |
| 文本消息          | `createTextMessage(text, operationID?)`           | 发送用户输入的纯文本内容。                     |
| 富文本/实体文本   | `createAdvancedTextMessage(params, operationID?)` | 文本中需要携带链接、实体或业务范围标记。       |
| @ 文本消息        | `createTextAtMessage(data, operationID?)`         | 群聊中提醒指定成员。                           |
| 图片消息          | `createImageMessageByFile(params, operationID?)`  | 从浏览器 `File` 对象创建图片消息。             |
| 音频消息          | `createSoundMessageByFile(data, operationID?)`    | 发送语音或音频文件。                           |
| 视频消息          | `createVideoMessageByFile(data, operationID?)`    | 发送视频文件和快照图。                         |
| 文件消息          | `createFileMessageByFile(data, operationID?)`     | 发送普通附件。                                 |
| 自定义消息        | `createCustomMessage(params, operationID?)`       | 承载卡片、邀请、订单等结构化业务 payload。     |
| 不走 OSS 发送链路 | `sendMessageNotOss(params, operationID?)`         | 复用 `SendMsgParams`，用于项目自定义上传链路。 |

创建方法返回的 `data` 就是发送时传入的 `message`。发送成功后，`sendMessage()` 返回服务端确认后的 `MessageItem`，应用可以用它更新本地列表中的发送状态、`clientMsgID` 和会话信息。

```ts
const operationID = crypto.randomUUID();

const { data: message } = await OpenIM.createTextMessage('你好，OpenIM', operationID);

const { data: sentMessage } = await OpenIM.sendMessage(
  {
    recvID: receiverUserID,
    groupID: '',
    message,
  },
  operationID,
);

appendMessage(sentMessage);
```

发送群聊消息只需要切换目标字段。不要把 `conversationID` 当作发送目标；它主要用于历史消息、会话列表和未读状态。

```ts
const { data: message } = await OpenIM.createTextMessage('大家好');

await OpenIM.sendMessage({
  recvID: '',
  groupID,
  message,
});
```

## 发送文本消息

文本消息适合普通聊天内容。创建前可以在业务层完成长度、敏感词、空白字符和草稿状态处理；SDK 创建消息后，发送层只关心目标字段和 `message` 对象。

```ts
async function sendTextToUser(receiverUserID: string, text: string) {
  const operationID = crypto.randomUUID();
  const normalizedText = text.trim();

  if (!normalizedText) {
    return;
  }

  const { data: message } = await OpenIM.createTextMessage(normalizedText, operationID);

  return OpenIM.sendMessage(
    {
      recvID: receiverUserID,
      groupID: '',
      message,
    },
    operationID,
  );
}
```

如果文本中需要携带可点击实体、业务范围或结构化标记，使用 `createAdvancedTextMessage()`。如果需要提醒群成员，使用[在消息中提及用户](/docs/chat/sdk/v4/wasm/message/mentioning-other-users-in-a-message/mention-other-users-in-a-message)中的 `createTextAtMessage()`。

## 发送文件消息

文件、图片、音频和视频消息同样遵循创建再发送的模型。普通文件消息需要浏览器 `File` 对象和文件元数据。

```ts
const { data: fileMessage } = await OpenIM.createFileMessageByFile({
  file,
  filePath: file.name,
  fileName: file.name,
  uuid: crypto.randomUUID(),
  sourceUrl: '',
  fileSize: file.size,
  fileType: file.type,
});

await OpenIM.sendMessage({
  recvID: '',
  groupID,
  message: fileMessage,
});
```

如果界面需要展示上传进度，先注册上传事件，再创建并发送文件类消息。

```ts
OpenIM.on(CbEvents.OnProgress, ({ data }) => {
  updateUploadProgress(data);
});

OpenIM.on(CbEvents.UploadComplete, ({ data }) => {
  markUploadComplete(data);
});
```

完整进度处理见[使用事件处理器跟踪文件上传进度](/docs/chat/sdk/v4/wasm/message/sending-a-message/track-file-upload-progress-using-a-handler)。

## 发送自定义消息

自定义消息适合承载需要被接收端完整解析的业务卡片。`data` 建议使用 JSON 字符串，`extension` 用于标识业务类型，`description` 可作为无法解析 payload 时的降级展示文案。

```ts
const { data: customMessage } = await OpenIM.createCustomMessage({
  data: JSON.stringify({
    type: 'order-card',
    orderID,
    title: '订单已创建',
  }),
  extension: 'order',
  description: '订单卡片',
});

await OpenIM.sendMessage({
  recvID: '',
  groupID,
  message: customMessage,
});
```

接收端仍通过 `CbEvents.OnRecvNewMessages` 收到这条消息，再按 `extension` 或业务 payload 渲染卡片。

## 处理发送结果

`WsResponse<T>` 通常包含 `errCode`、`errMsg`、`data` 和 `operationID`。Promise 返回后建议检查业务成功状态，并把响应中的 `data` 合并到当前会话消息列表。

```ts
type WsResponse<T> = {
  event: string;
  errCode: number;
  errMsg: string;
  data: T;
  operationID: string;
};
```

发送链路中常用事件如下：

| 事件                         | 用途                                       |
| ---------------------------- | ------------------------------------------ |
| `CbEvents.OnRecvNewMessages` | 当前用户收到新消息时触发，用于更新消息列表 |
| `CbEvents.OnProgress`        | 文件类消息上传过程中触发，用于更新进度条   |
| `CbEvents.UploadComplete`    | 文件上传完成时触发，用于收尾上传状态       |

组件卸载、退出登录或切换账号前，使用同一个处理函数引用调用 `off()`，避免重复消费消息。
