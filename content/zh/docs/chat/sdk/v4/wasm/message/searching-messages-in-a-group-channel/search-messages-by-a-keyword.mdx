OpenIM WASM SDK 通过 `searchLocalMessages()` 搜索当前用户本地可见的消息。群组消息搜索的目标参数是群聊对应的 `conversationID`，不是发送消息时使用的 `groupID`。如果界面只保存了 `groupID`，先用 `getConversationIDBySessionType()` 取得群会话 ID。

搜索范围来自 SDK 已同步到浏览器本地缓存的消息。需要跨用户审计、全量服务端检索、复杂权限过滤或全局排序时，应由你的后端搜索服务承担，再把命中的 `conversationID` 和 `clientMsgID` 返回给客户端定位。

## 创建搜索查询

群聊会话的 `sessionType` 为 `3`。拿到 `conversationID` 后，把它和 `groupID` 一起保存在聊天页状态中，后续读取历史、搜索消息和标记已读都可以复用。

```ts
const getGroupConversationID = async (groupID: string) => {
  const operationID = crypto.randomUUID();

  const { data, errCode, errMsg } = await OpenIM.getConversationIDBySessionType(
    {
      sourceID: groupID,
      sessionType: 3,
    },
    operationID,
  );

  if (errCode !== 0) {
    console.error('getConversationIDBySessionType failed', {
      errCode,
      errMsg,
      operationID,
      groupID,
    });
    throw new Error(errMsg || 'Failed to get group conversation');
  }

  return data;
};
```

`keywordList` 接收一个或多个关键词。搜索框通常只代表一次用户输入，建议先去掉首尾空格并过滤空值，避免把空关键词传给 SDK。

```ts
import { MessageType } from '@openim/wasm-client-sdk';

type SearchGroupMessagesOptions = {
  conversationID: string;
  keyword: string;
  pageIndex?: number;
  count?: number;
  senderUserIDList?: string[];
};

const searchGroupMessagesByKeyword = async ({
  conversationID,
  keyword,
  pageIndex = 1,
  count = 20,
  senderUserIDList,
}: SearchGroupMessagesOptions) => {
  const normalizedKeyword = keyword.trim();

  if (!normalizedKeyword) {
    return {
      totalCount: 0,
      messages: [],
    };
  }

  const operationID = crypto.randomUUID();

  const { data, errCode, errMsg } = await OpenIM.searchLocalMessages(
    {
      conversationID,
      keywordList: [normalizedKeyword],
      senderUserIDList,
      messageTypeList: [MessageType.TextMessage, MessageType.AtTextMessage],
      pageIndex,
      count,
    },
    operationID,
  );

  if (errCode !== 0) {
    console.error('searchLocalMessages failed', {
      errCode,
      errMsg,
      operationID,
      conversationID,
      keyword: normalizedKeyword,
    });
    throw new Error(errMsg || 'Failed to search group messages');
  }

  return toSearchRows(data);
};
```

## 高级搜索

`searchLocalMessages()` 支持在关键词之外限制发送者、消息类型和时间窗口。你可以用这些条件实现群成员筛选、只搜文本消息、按时间段缩小范围等高级搜索能力。

```ts
const { data } = await OpenIM.searchLocalMessages({
  conversationID,
  keywordList: ['release'],
  senderUserIDList: [senderUserID],
  messageTypeList: [MessageType.TextMessage],
  searchTimePosition,
  searchTimePeriod,
  pageIndex: 1,
  count: 20,
});
```

如果你的搜索入口允许搜索图片、文件或自定义消息，把对应的 `MessageType` 加入 `messageTypeList`。如果不需要限制消息类型，可以省略该字段，让 SDK 在本地缓存中按默认范围搜索。

#### 搜索参数

| 参数                   | 说明                                                            |
| ---------------------- | --------------------------------------------------------------- |
| `conversationID`       | 要搜索的群聊会话 ID。                                           |
| `keywordList`          | 关键词列表；单个搜索框通常传入一个清理后的关键词。              |
| `keywordListMatchType` | 多关键词匹配方式，使用 SDK 的数字约定；不需要特殊匹配时可省略。 |
| `senderUserIDList`     | 只搜索这些成员发送的消息。                                      |
| `messageTypeList`      | 只搜索指定类型的消息，例如文本消息和 @ 文本消息。               |
| `searchTimePosition`   | 搜索时间窗口的起点时间戳。                                      |
| `searchTimePeriod`     | 从 `searchTimePosition` 开始的时间跨度。                        |
| `pageIndex`            | 搜索结果页码。                                                  |
| `count`                | 每页返回数量。                                                  |
| `operationID`          | 可选的请求追踪 ID；排查失败时建议和 `errCode`、`errMsg` 记录。  |

## 处理分页结果

返回的 `SearchMessageResult` 包含 `totalCount` 和 `searchResultItems`。每个结果项对应一个会话，并在 `messageList` 中携带匹配到的 `MessageItem[]`。

```ts
import type { SearchMessageResult } from '@openim/wasm-client-sdk';

const toSearchRows = (result: SearchMessageResult) => {
  return {
    totalCount: result.totalCount,
    messages: (result.searchResultItems ?? []).flatMap((item) => {
      return item.messageList.map((message) => ({
        conversationID: item.conversationID,
        clientMsgID: message.clientMsgID,
        senderUserID: message.sendID,
        sentAt: message.sendTime,
        message,
      }));
    }),
  };
};
```

分页时继续传入相同的 `conversationID`、`keywordList` 和筛选条件，只递增 `pageIndex`。如果用户修改关键词、成员筛选或时间范围，应把 `pageIndex` 重置为 `1`，并清空旧结果，避免不同条件的结果混在一起。

## 保持结果可用

搜索命中的消息可能在页面打开后被撤回、删除或由新同步数据补充。聊天页应监听消息事件，把结果列表和当前会话消息列表一起更新。

```ts
import { CbEvents } from '@openim/wasm-client-sdk';

const handleNewMessages = ({ data }: { data: MessageItem[] }) => {
  mergeMessagesIntoVisibleConversation(data);
  refreshSearchIfNeeded(data);
};

const handleMessageRevoked = ({ data }: { data: MessageItem }) => {
  removeRevokedMessageFromSearch(data.clientMsgID);
};

OpenIM.on(CbEvents.OnRecvNewMessages, handleNewMessages);
OpenIM.on(CbEvents.OnNewRecvMessageRevoked, handleMessageRevoked);
OpenIM.on(CbEvents.OnSyncServerFinish, rerunCurrentSearch);

OpenIM.off(CbEvents.OnRecvNewMessages, handleNewMessages);
OpenIM.off(CbEvents.OnNewRecvMessageRevoked, handleMessageRevoked);
OpenIM.off(CbEvents.OnSyncServerFinish, rerunCurrentSearch);
```

跳转到某条搜索结果时，使用 `clientMsgID` 和 `conversationID` 重新定位消息；需要展示上下文时，再调用 `findMessageList()` 或 `fetchSurroundingMessages()` 读取目标消息附近的聊天记录。
