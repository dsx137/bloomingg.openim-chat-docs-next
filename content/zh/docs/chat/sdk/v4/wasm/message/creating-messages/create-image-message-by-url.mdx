## 功能介绍

通过已上传图片 URL 创建图片消息。适合资源已由业务侧上传完成的场景。

创建消息类方法同步返回消息 JSON；发送、查询、删除、已读和本地写入方法通过回调返回。消息与会话事件分别通过消息监听器和会话监听器接收。

## 底层能力

| 项目     | 说明                                                                                                                       |
| -------- | -------------------------------------------------------------------------------------------------------------------------- |
| 方法签名 | `CreateImageMessageByURL(operationID string, sourcePath string, sourcePicture, bigPicture, snapshotPicture string) string` |
| 所属能力 | 会话与消息                                                                                                                 |
| 源码位置 | `open_im_sdk/conversation_msg.go`                                                                                          |

## WASM 调用参考

WASM SDK v3.8.3 暴露以下平台方法原型。底层能力表用于说明 OpenIM core 能力和参数映射。

源码位置：`src/sdk/index.ts`。

```ts showLineNumbers
createImageMessageByURL(params: ImageMsgParamsByURL, operationID = uuidv4())
```

## 输入参数

| 参数名称    | 参数类型            | 是否必填 | 描述                               |
| ----------- | ------------------- | -------- | ---------------------------------- |
| params      | ImageMsgParamsByURL | 是       | 按平台类型定义传入。               |
| operationID | As defined          | 否       | 链路追踪 ID；不传时 SDK 自动生成。 |

## 返回结果

返回 `Promise` 或注册回调结果。调用失败时 Promise reject；建议在业务层使用 `try/catch` 或 `.catch()` 处理错误。
