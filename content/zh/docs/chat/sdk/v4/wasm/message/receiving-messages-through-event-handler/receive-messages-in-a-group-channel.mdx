群聊消息页通常同时处理实时推送的新消息、登录或重连后补到本地的离线消息，以及首次进入会话时主动读取的历史消息。实时和离线数据通过 `CbEvents` 事件进入应用状态；历史列表通过 `getAdvancedHistoryMessageList()` 按 `conversationID` 分页读取。

在同一个 SDK 实例上注册事件处理器，并在组件卸载、退出登录或切换账号前用同一个函数引用调用 `off()`，避免同一批消息被重复合并。

## 消息类型

`CbEvents.OnRecvNewMessages` 返回的是 `MessageItem[]`。每条消息可以按内容元素或消息类型进入不同渲染分支，例如文本、自定义消息、@ 文本、图片、音频、视频和文件。

```ts
function renderGroupMessage(message: MessageItem) {
  if (message.textElem) {
    return renderTextMessage(message);
  }

  if (message.atTextElem) {
    return renderMentionMessage(message);
  }

  if (message.customElem) {
    return renderCustomMessage(message);
  }

  if (message.pictureElem || message.soundElem || message.videoElem || message.fileElem) {
    return renderFileLikeMessage(message);
  }

  return renderUnsupportedMessage(message);
}
```

群聊事件可能包含当前用户不在查看的会话消息。回调中应先按 `conversationID` 过滤，再按 `clientMsgID` 去重。

```ts
function messagesForCurrentConversation(messages: MessageItem[]) {
  return messages.filter((message) => message.conversationID === conversationID);
}
```

### 文件消息和文件类消息

OpenIM WASM SDK 用不同元素表示图片、音频、视频和普通文件。接收端不需要重新上传文件，只需要读取消息中已有的文件地址、大小、名称、时长或快照图等字段，并按你的 UI 规则展示。

如果你的产品需要一次发送多个文件，常见实现是连续发送多条文件消息，或发送一条自定义消息承载文件组 payload。接收端仍应以每条 `MessageItem` 的 `clientMsgID` 作为渲染和状态更新的稳定标识。

## 事件处理器

`CbEvents.OnRecvNewMessages` 是群聊消息页最常用的新消息事件。处理器中把当前群聊的消息合并到列表，并根据用户滚动位置决定是否自动滚到底部。

```ts
function handleNewMessages({ data }: { data: MessageItem[] }) {
  const messages = messagesForCurrentConversation(data);

  if (messages.length === 0) {
    return;
  }

  mergeMessagesByClientMsgID(messages);
  scrollToLatestMessageIfNeeded();
}

OpenIM.on(CbEvents.OnRecvNewMessages, handleNewMessages);
```

离线期间收到的消息、弱网重连后的补偿消息，可以通过 `CbEvents.OnRecvOfflineNewMessages` 进入客户端。处理方式和实时新消息相同：筛选当前会话、按 `clientMsgID` 去重，并保持时间顺序。

```ts
function handleOfflineMessages({ data }: { data: MessageItem[] }) {
  mergeMessagesByClientMsgID(messagesForCurrentConversation(data));
}

OpenIM.on(CbEvents.OnRecvOfflineNewMessages, handleOfflineMessages);
```

只在线分发的消息可监听 `CbEvents.OnRecvOnlineOnlyMessages`。这类消息通常用于临时提示或业务通知，是否进入消息流应以你的产品约定为准。

```ts
function handleOnlineOnlyMessages({ data }: { data: MessageItem[] }) {
  handleTransientGroupEvents(messagesForCurrentConversation(data));
}

OpenIM.on(CbEvents.OnRecvOnlineOnlyMessages, handleOnlineOnlyMessages);
```

当群成员撤回消息时，接收端通过 `CbEvents.OnNewRecvMessageRevoked` 得到更新。建议把对应气泡更新为撤回态，而不是直接从列表中删除。

```ts
function handleMessageRevoked({ data }: { data: MessageItem }) {
  if (data.conversationID !== conversationID) {
    return;
  }

  markMessageRevoked(data.clientMsgID);
}

OpenIM.on(CbEvents.OnNewRecvMessageRevoked, handleMessageRevoked);
```

## 首次进入会话时读取历史

事件只负责推送新到达的消息。首次进入群聊、向上翻页或需要补齐断线期间的列表时，使用 `getAdvancedHistoryMessageList()` 读取当前用户可见并已同步的历史消息。

```ts
const { data: firstPage } = await OpenIM.getAdvancedHistoryMessageList({
  conversationID,
  startClientMsgID: '',
  count: 30,
  viewType: 0,
});

replaceMessageList(firstPage.messageList);
```

继续加载更早消息时，使用当前列表边界上的 `clientMsgID` 作为游标。历史分页和事件推送可能覆盖同一条消息，渲染层应继续按 `clientMsgID` 去重。

```ts
const nextCursor = messageList.at(-1)?.clientMsgID ?? '';

const { data: nextPage } = await OpenIM.getAdvancedHistoryMessageList({
  conversationID,
  startClientMsgID: nextCursor,
  count: 30,
  viewType: 0,
});

appendOlderMessages(nextPage.messageList);
```

## 标记群聊为已读

用户进入群聊并看到最新消息后，可以调用 `markConversationMessageAsRead(conversationID)` 清理该会话的未读数。这个调用处理的是会话未读数，不等同于群消息成员级已读回执。

```ts
await OpenIM.markConversationMessageAsRead(conversationID);
```

会话列表和总未读角标仍建议通过事件做最终同步。

```ts
OpenIM.on(CbEvents.OnConversationChanged, ({ data }) => {
  mergeConversations(data);
});

OpenIM.on(CbEvents.OnTotalUnreadMessageCountChanged, ({ data }) => {
  setTotalUnreadCount(data);
});
```

## 移除事件处理器

`off()` 必须接收注册时同一个函数引用。组件卸载、退出登录或切换账号前移除监听器，可以避免重复渲染、重复清理未读数和内存泄漏。

```ts
OpenIM.off(CbEvents.OnRecvNewMessages, handleNewMessages);
OpenIM.off(CbEvents.OnRecvOfflineNewMessages, handleOfflineMessages);
OpenIM.off(CbEvents.OnRecvOnlineOnlyMessages, handleOnlineOnlyMessages);
OpenIM.off(CbEvents.OnNewRecvMessageRevoked, handleMessageRevoked);
```

如果事件注册在全局消息状态层，只在 SDK 销毁或账号切换时清理；不要在每次进入同一个群聊页面时重复注册全局处理器。

## 验证接收流程

- 在另一个已登录账号中向目标 `groupID` 发送消息，确认当前浏览器通过 `CbEvents.OnRecvNewMessages` 收到消息。
- 确认回调使用当前群聊的 `conversationID` 过滤消息，并且列表按 `clientMsgID` 去重后只渲染一次。
- 断开当前账号或刷新页面后重新登录，确认离线期间的消息可以通过离线事件或历史分页补齐。
- 撤回一条群消息，确认接收端通过 `CbEvents.OnNewRecvMessageRevoked` 把对应 `clientMsgID` 更新为撤回态。
- 进入群聊后调用 `markConversationMessageAsRead(conversationID)`，确认会话未读数和总未读角标随事件更新。
