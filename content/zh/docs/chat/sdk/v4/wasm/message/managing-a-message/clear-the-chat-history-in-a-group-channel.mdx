在 OpenIM 中，清空群组频道聊天历史是当前用户维度的会话操作。对目标群聊会话调用 `clearConversationAndDeleteAllMsg(conversationID)` 后，当前登录用户的该会话消息历史会被清空；群组不会被解散，其他成员的聊天视图也不会因此删除历史消息。

如果业务入口只有 `groupID`，先用群聊 `sessionType: 3` 换取会话 ID，再清空对应会话。调用成功后，可以立即清空当前消息列表，并等待 `CbEvents.OnConversationChanged` 或重新查询会话来校正会话摘要、未读数和最后一条消息。

```ts
import { CbEvents } from '@openim/wasm-client-sdk';

function handleConversationChanged({ data }) {
  const conversations = Array.isArray(data) ? data : [data];
  conversations.forEach((conversation) => {
    mergeConversation(conversation);
  });
}

OpenIM.on(CbEvents.OnConversationChanged, handleConversationChanged);

async function clearGroupChannelHistory(groupID: string) {
  const operationID = crypto.randomUUID();

  const { data: conversationID } = await OpenIM.getConversationIDBySessionType(
    {
      sourceID: groupID,
      sessionType: 3,
    },
    operationID,
  );

  const clearResult = await OpenIM.clearConversationAndDeleteAllMsg(conversationID, operationID);

  removeMessagesFromConversation(conversationID);
  resetConversationPreview(conversationID);
}
```

如果页面已经从会话列表拿到了 `ConversationItem`，可以直接使用其中的 `conversationID`，不需要再次调用 `getConversationIDBySessionType()`。

`clearConversationAndDeleteAllMsg()` 适合“保留群聊入口，只清空当前聊天窗口历史”的用户动作。若产品动作是“删除此聊天”，使用 `deleteConversationAndDeleteAllMsg(conversationID)` 删除当前用户的会话记录和消息；若是账号级重置，才考虑 `deleteAllMsgFromLocal()` 或 `deleteAllMsgFromLocalAndSvr()`。这些账号级 API 不接收 `groupID` 或 `conversationID`，不应挂在单个群组频道的清空按钮上。

清空历史调用成功 resolve 后，确认当前聊天窗口不再展示旧消息，刷新页面后旧消息不会从本地缓存恢复。清空后收到的新群消息仍应通过新消息事件进入该会话，并作为新的历史起点展示。
