在 OpenIM 中，某个频道的未读消息数保存在对应的 `ConversationItem.unreadCount` 上。群聊通常用 `groupID` 标识群组，用 `conversationID` 标识当前用户聊天列表中的会话记录；读取未读数时优先使用 `conversationID`。

```ts
const {
  data: conversations,
  errCode,
  errMsg,
} = await OpenIM.getMultipleConversation([conversationID]);

const conversation = conversations?.[0];

if (errCode !== 0 || !conversation) {
  throw new Error(errMsg || 'Conversation not found');
}

renderChannelUnreadCount(conversation.conversationID, conversation.unreadCount);
```

## 只有 groupID 时

如果业务入口只有 `groupID`，调用 `getOneConversation()` 读取该群组对应的会话。群聊的 `sessionType` 传 `3`，`sourceID` 传目标 `groupID`。

```ts
const { data: conversation } = await OpenIM.getOneConversation({
  sourceID: groupID,
  sessionType: 3,
});

setActiveChannelState({
  groupID,
  conversationID: conversation.conversationID,
  unreadCount: conversation.unreadCount,
});
```

新加入的群组或本地缓存尚未同步完成时，目标会话可能暂时不存在。此时保留 `groupID`，等待 `CbEvents.OnNewConversation` 或 `CbEvents.OnConversationChanged` 后再重新读取。

## 监听变化

收到新消息、标记已读、同步离线消息或会话状态变化时，`unreadCount` 都可能更新。频道页和聊天列表应监听 `CbEvents.OnConversationChanged`，把事件中的会话记录合并到本地状态。

```ts
import { CbEvents } from '@openim/wasm-client-sdk';

function handleConversationChanged({ data }) {
  const conversations = Array.isArray(data) ? data : [data];
  const updated = conversations.find((item) => item.conversationID === conversationID);

  if (updated) {
    renderChannelUnreadCount(updated.conversationID, updated.unreadCount);
  }
}

OpenIM.on(CbEvents.OnConversationChanged, handleConversationChanged);
```

`CbEvents.OnTotalUnreadMessageCountChanged` 返回当前用户所有会话的总未读数，适合应用角标；不要把它当作某个频道的未读数。

## 清理当前频道未读数

用户进入频道并看到消息后，调用 `markConversationMessageAsRead(conversationID)` 清理当前会话的未读数。调用成功后可以先把当前频道角标置为 `0`，最终状态仍以会话事件同步为准。

```ts
await OpenIM.markConversationMessageAsRead(conversationID);

renderChannelUnreadCount(conversationID, 0);
```

这个 API 只处理会话未读数。如果产品还需要展示某条群消息的已读或未读成员列表，使用 `sendGroupMessageReadReceipt()` 和 `getGroupMessageReaderList()`，不要用频道未读数替代消息回执状态。
