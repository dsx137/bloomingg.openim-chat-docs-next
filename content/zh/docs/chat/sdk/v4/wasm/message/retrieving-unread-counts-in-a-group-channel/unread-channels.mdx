OpenIM WASM SDK 不提供单独的“有未读消息的群频道数量”接口。未读状态保存在每个 `ConversationItem.unreadCount` 上，因此需要读取会话列表，再统计 `unreadCount > 0` 的会话数量。

如果产品只需要群聊数量，同时过滤 `groupID`；如果需要所有未读会话数量，则直接统计全部会话。

```ts
const { data: conversations } = await OpenIM.getAllConversationList();

const unreadConversationCount = conversations.filter(
  (conversation) => conversation.unreadCount > 0,
).length;

const unreadGroupConversationCount = conversations.filter(
  (conversation) => conversation.groupID && conversation.unreadCount > 0,
).length;

renderUnreadConversationCount(unreadConversationCount);
renderUnreadGroupConversationCount(unreadGroupConversationCount);
```

## 分页统计

会话数量较多时，用 `getConversationListSplit()` 分页读取，再累加每一页中的未读会话数量。

```ts
let offset = 0;
const count = 100;
let unreadGroupConversationCount = 0;

while (true) {
  const { data: conversations } = await OpenIM.getConversationListSplit({
    offset,
    count,
  });

  unreadGroupConversationCount += conversations.filter(
    (conversation) => conversation.groupID && conversation.unreadCount > 0,
  ).length;

  if (conversations.length < count) {
    break;
  }

  offset += count;
}

renderUnreadGroupConversationCount(unreadGroupConversationCount);
```

## 监听变化

新消息、标记已读或会话同步都会改变 `unreadCount`。监听 `CbEvents.OnConversationChanged` 后，把事件中的会话合并到本地列表，再重新统计有未读消息的会话数量。

```ts
import { CbEvents } from '@openim/wasm-client-sdk';

const conversationMap = new Map();

function countUnreadGroups() {
  return Array.from(conversationMap.values()).filter(
    (conversation) => conversation.groupID && conversation.unreadCount > 0,
  ).length;
}

function handleConversationChanged({ data }) {
  const conversations = Array.isArray(data) ? data : [data];
  conversations.forEach((conversation) => {
    conversationMap.set(conversation.conversationID, conversation);
  });

  renderUnreadGroupConversationCount(countUnreadGroups());
}

OpenIM.on(CbEvents.OnConversationChanged, handleConversationChanged);
```

`CbEvents.OnTotalUnreadMessageCountChanged` 返回的是所有会话未读消息条数总和，不是有未读消息的会话数量。不要把这两个数字混用。
