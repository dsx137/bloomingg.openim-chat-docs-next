OpenIM WASM SDK 使用会话输入状态实现“正在输入”提示。当前用户开始输入时调用 `changeInputStates({ conversationID, focus: true })`，发送消息、失焦、切换会话或停止输入时再调用 `focus: false`。输入状态绑定到 `conversationID`，不会写入消息列表，也不用于保存草稿文本。

```ts
import { CbEvents } from '@openim/wasm-client-sdk';

async function startTyping(conversationID: string) {
  await OpenIM.changeInputStates({
    conversationID,
    focus: true,
  });
}

async function stopTyping(conversationID: string) {
  await OpenIM.changeInputStates({
    conversationID,
    focus: false,
  });
}
```

实际界面中建议对 `focus: true` 做节流或状态去重，例如只在输入状态从“未输入”变为“输入中”时调用一次。`focus: false` 可以由发送消息、输入框失焦、切换会话、页面卸载或一段时间没有键盘输入共同触发。

## 接收输入状态

其他成员输入状态变化时，SDK 会触发 `CbEvents.OnConversationUserInputStatusChanged`。事件中的 `data.conversationID` 表示发生变化的会话，`data.userID` 表示输入用户，`data.platformIDs` 非空时表示该用户仍在输入。

```ts
function handleInputStatusChanged({ data }) {
  if (data.conversationID !== activeConversationID) {
    return;
  }

  if (data.platformIDs.length > 0) {
    showTypingIndicator(data.userID);
    return;
  }

  hideTypingIndicator(data.userID);
}

OpenIM.on(CbEvents.OnConversationUserInputStatusChanged, handleInputStatusChanged);
```

## 查询当前状态

进入会话页时，可以用 `getInputstates()` 查询某个成员在当前会话中仍处于输入状态的平台列表。返回数组非空时展示该成员正在输入。

```ts
const { data: platformIDs } = await OpenIM.getInputstates({
  conversationID,
  userID: memberUserID,
});

if (platformIDs.length > 0) {
  showTypingIndicator(memberUserID);
}
```

群聊中如果需要展示多名成员正在输入，优先通过 `CbEvents.OnConversationUserInputStatusChanged` 维护当前会话的输入用户集合；只在进入页面或需要补偿状态时，对重点成员调用 `getInputstates()`。

`typingStatusUpdate()` 仍可能出现在类型定义中，但已标记为废弃。新代码应使用 `changeInputStates()` 和 `getInputstates()`。
