接收新消息、消息撤回、已读回执变化和消息扩展更新。

## 功能介绍

单条消息事件应更新当前聊天页、会话预览、未读数和本地消息缓存。它适合处理实时新消息以及影响单条消息状态的变化。

## 底层能力

| 项目     | 说明                                                     |
| -------- | -------------------------------------------------------- |
| 方法签名 | `SetAdvancedMsgListener(listener OnAdvancedMsgListener)` |
| 所属能力 | 监听器                                                   |
| 源码位置 | `open_im_sdk/listener.go`                                |

## WASM 调用参考

WASM SDK v3.8.3 暴露以下平台方法原型。底层能力表用于说明 OpenIM core 能力和参数映射。

源码位置：`src/sdk/index.ts`。

```ts showLineNumbers
on(event: CbEvents, listener: (event) => void)
```

## 输入参数

| 参数名称 | 参数类型 | 是否必填 | 描述                   |
| -------- | -------- | -------- | ---------------------- |
| event    | CbEvents | 是       | 需要订阅的事件名称。   |
| listener | Function | 是       | 事件触发时执行的回调。 |

## 返回结果

返回 `Promise` 或注册回调结果。调用失败时 Promise reject；建议在业务层使用 `try/catch` 或 `.catch()` 处理错误。

## 代码示例

WASM 使用事件订阅方式接收回调。按需要替换事件名和处理函数。

```ts showLineNumbers
import { CbEvents, getSDK } from '@openim/wasm-client-sdk';

const openIM = getSDK({
  coreWasmPath: '/openIM.wasm',
  sqlWasmPath: '/sql-wasm.wasm',
});

openIM.on(CbEvents.OnRecvNewMessages, (event) => {
  console.log(event);
});
```
