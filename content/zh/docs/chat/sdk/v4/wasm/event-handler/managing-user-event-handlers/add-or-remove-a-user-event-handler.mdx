本指南介绍如何在浏览器应用中为用户相关事件添加和移除处理器。用户事件通常会影响当前用户资料、好友列表、黑名单列表和在线状态，因此应在应用状态层统一注册，再把变化同步到具体页面。

事件处理器需要使用同一个函数引用完成注册和移除。不要在需要清理的页面里直接传入匿名函数，否则调用 `off()` 时无法准确解除对应监听。

## 用户事件类型

OpenIM WASM SDK 通过 `CbEvents` 枚举派发用户、好友、黑名单和在线状态事件。下面是用户界面中最常用的一组事件：

#### 用户事件列表

| 事件                                        | 触发场景                         | 常见处理方式                                           |
| ------------------------------------------- | -------------------------------- | ------------------------------------------------------ |
| `CbEvents.OnSelfInfoUpdated`                | 当前登录用户资料变化。           | 刷新头像、昵称和扩展资料。                             |
| `CbEvents.OnUserStatusChanged`              | 已订阅用户的在线状态变化。       | 按 `userID` 更新会话列表、成员列表或资料卡中的状态点。 |
| `CbEvents.OnFriendAdded`                    | 好友关系新增。                   | 把用户加入好友列表或刷新好友计数。                     |
| `CbEvents.OnFriendDeleted`                  | 好友关系删除。                   | 从好友列表移除用户，并更新相关入口状态。               |
| `CbEvents.OnFriendInfoChanged`              | 好友资料变化。                   | 合并好友昵称、头像或备注等展示字段。                   |
| `CbEvents.OnBlackAdded`                     | 用户被加入黑名单。               | 更新黑名单列表，并在聊天入口中隐藏或禁用相关操作。     |
| `CbEvents.OnBlackDeleted`                   | 用户从黑名单移除。               | 从黑名单列表删除用户，并恢复允许的交互。               |
| `CbEvents.OnTotalUnreadMessageCountChanged` | 当前用户所有会话的总未读数变化。 | 更新应用级角标、导航栏提示或浏览器标题。               |

在线状态事件只会对已订阅的用户有意义。只订阅当前页面实际展示的用户 ID，例如会话列表、成员列表或资料卡中可见的用户。

```ts
const { data: me } = await OpenIM.getSelfUserInfo();
const { data: friends } = await OpenIM.getFriendList();
const { data: blockedUsers } = await OpenIM.getBlackList();

setCurrentUser(me);
replaceFriends(friends);
replaceBlockedUsers(blockedUsers);

await OpenIM.subscribeUsersStatus(visibleUserIDs);
```

`CbEvents.OnUserStatusChanged` 的 `data` 是单个 `UserOnlineState`。`subscribeUsersStatus()` 和 `getSubscribeUsersStatus()` 的响应 `data` 才是状态数组，合并本地状态时不要混用这两种形状。

## 添加用户事件处理器

先定义稳定的处理函数，再通过 `OpenIM.on()` 注册。这样同一组函数既可以复用，也可以在页面卸载或用户退出登录时准确移除。

```ts
import { CbEvents, getSDK } from '@openim/wasm-client-sdk';

const OpenIM = getSDK({
  coreWasmPath: '/openIM.wasm',
  sqlWasmPath: '/sql-wasm.wasm',
});

const handleSelfInfoUpdated = ({ data }) => {
  setCurrentUser(data);
};

const handleUserStatusChanged = ({ data }) => {
  mergePresence(data);
};

const handleFriendAdded = ({ data }) => {
  upsertFriend(data);
};

const handleFriendDeleted = ({ data }) => {
  removeFriend(data);
};

const handleFriendInfoChanged = ({ data }) => {
  upsertFriend(data);
};

const handleBlackAdded = ({ data }) => {
  upsertBlockedUser(data);
};

const handleBlackDeleted = ({ data }) => {
  removeBlockedUser(data);
};

const handleTotalUnreadChanged = ({ data }) => {
  setTotalUnreadCount(data);
};

OpenIM.on(CbEvents.OnSelfInfoUpdated, handleSelfInfoUpdated);
OpenIM.on(CbEvents.OnUserStatusChanged, handleUserStatusChanged);
OpenIM.on(CbEvents.OnFriendAdded, handleFriendAdded);
OpenIM.on(CbEvents.OnFriendDeleted, handleFriendDeleted);
OpenIM.on(CbEvents.OnFriendInfoChanged, handleFriendInfoChanged);
OpenIM.on(CbEvents.OnBlackAdded, handleBlackAdded);
OpenIM.on(CbEvents.OnBlackDeleted, handleBlackDeleted);
OpenIM.on(CbEvents.OnTotalUnreadMessageCountChanged, handleTotalUnreadChanged);

await OpenIM.subscribeUsersStatus(visibleUserIDs);
```

将用户状态按 `userID` 归一化保存，可以让资料页、通讯录、黑名单页、会话列表和成员列表共用同一份状态。事件处理器里只做轻量合并；如果页面需要服务端权威数据，可以在事件到达后节流调用 `getSelfUserInfo()`、`getFriendList()` 或 `getBlackList()` 刷新。

## 移除用户事件处理器

调用 `OpenIM.off()` 时传入和注册时完全相同的事件名与函数引用。适合在页面销毁、组件卸载、切换工作区或退出登录前执行。

```ts
OpenIM.off(CbEvents.OnSelfInfoUpdated, handleSelfInfoUpdated);
OpenIM.off(CbEvents.OnUserStatusChanged, handleUserStatusChanged);
OpenIM.off(CbEvents.OnFriendAdded, handleFriendAdded);
OpenIM.off(CbEvents.OnFriendDeleted, handleFriendDeleted);
OpenIM.off(CbEvents.OnFriendInfoChanged, handleFriendInfoChanged);
OpenIM.off(CbEvents.OnBlackAdded, handleBlackAdded);
OpenIM.off(CbEvents.OnBlackDeleted, handleBlackDeleted);
OpenIM.off(CbEvents.OnTotalUnreadMessageCountChanged, handleTotalUnreadChanged);

await OpenIM.unsubscribeUsersStatus(visibleUserIDs);
```

如果你的应用使用 React，可以把注册和清理封装在同一个 effect 中，避免路由切换后重复注册：

```tsx
useEffect(() => {
  OpenIM.on(CbEvents.OnSelfInfoUpdated, handleSelfInfoUpdated);
  OpenIM.on(CbEvents.OnUserStatusChanged, handleUserStatusChanged);
  OpenIM.on(CbEvents.OnFriendAdded, handleFriendAdded);
  OpenIM.on(CbEvents.OnFriendDeleted, handleFriendDeleted);
  OpenIM.on(CbEvents.OnFriendInfoChanged, handleFriendInfoChanged);
  OpenIM.on(CbEvents.OnBlackAdded, handleBlackAdded);
  OpenIM.on(CbEvents.OnBlackDeleted, handleBlackDeleted);
  OpenIM.on(CbEvents.OnTotalUnreadMessageCountChanged, handleTotalUnreadChanged);

  void OpenIM.subscribeUsersStatus(visibleUserIDs);

  return () => {
    OpenIM.off(CbEvents.OnSelfInfoUpdated, handleSelfInfoUpdated);
    OpenIM.off(CbEvents.OnUserStatusChanged, handleUserStatusChanged);
    OpenIM.off(CbEvents.OnFriendAdded, handleFriendAdded);
    OpenIM.off(CbEvents.OnFriendDeleted, handleFriendDeleted);
    OpenIM.off(CbEvents.OnFriendInfoChanged, handleFriendInfoChanged);
    OpenIM.off(CbEvents.OnBlackAdded, handleBlackAdded);
    OpenIM.off(CbEvents.OnBlackDeleted, handleBlackDeleted);
    OpenIM.off(CbEvents.OnTotalUnreadMessageCountChanged, handleTotalUnreadChanged);

    void OpenIM.unsubscribeUsersStatus(visibleUserIDs);
  };
}, [visibleUserIDs]);
```

页面重复进入、可见用户集合变化、退出登录或切换账号时，都应先移除旧处理器，并取消旧的在线状态订阅，再为新的页面范围或账号注册。这样可以避免旧页面继续写入状态，也能减少不必要的在线状态事件。
