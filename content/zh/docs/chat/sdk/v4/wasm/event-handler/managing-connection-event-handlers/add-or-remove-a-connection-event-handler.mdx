连接事件用于把 OpenIM WASM SDK 的登录和 WebSocket 状态同步到应用界面。你可以用它们显示连接中、已连接、连接失败、token 失效或账号在其他端登录等状态。

在浏览器应用中，建议创建一个 SDK 实例并在当前会话中复用。连接事件应在调用 `login()` 前注册，这样应用不会错过登录过程中触发的连接中、连接成功或连接失败事件。

## 连接事件类型

OpenIM WASM SDK 通过 `OpenIM.on()` 注册事件处理器，通过 `OpenIM.off()` 移除事件处理器。连接相关事件包括：

| 事件                          | 触发时机                | 常见用途                                                     |
| ----------------------------- | ----------------------- | ------------------------------------------------------------ |
| `CbEvents.OnConnecting`       | SDK 正在建立连接。      | 显示连接中状态，禁用依赖在线状态的操作。                     |
| `CbEvents.OnConnectSuccess`   | SDK 已成功连接。        | 标记客户端可用，继续加载会话或发送消息。                     |
| `CbEvents.OnConnectFailed`    | SDK 连接失败。          | 展示重试入口，记录 `errCode`、`errMsg` 和 `operationID`。    |
| `CbEvents.OnKickedOffline`    | 当前账号被踢下线。      | 停止当前会话，提示用户账号已在其他端登录或会话被服务端终止。 |
| `CbEvents.OnUserTokenExpired` | 当前用户 token 已过期。 | 向业务后端请求新 token，并按你的登录流程恢复会话。           |
| `CbEvents.OnUserTokenInvalid` | 当前用户 token 无效。   | 清理登录态，引导用户重新认证。                               |

事件回调通常接收 `WSEvent` 形状的数据。处理连接失败、token 异常或被踢下线时，至少记录 `errCode`、`errMsg` 和 `operationID`，便于和 OpenIM 服务端日志对应。

```ts
type WSEvent<T = unknown> = {
  event: CbEvents;
  data: T;
  errCode: number;
  errMsg: string;
  operationID: string;
};
```

## 添加连接事件处理器

使用 `OpenIM.on(event, handler)` 添加连接事件处理器。先创建 SDK 实例，再注册连接事件，最后调用 `login()`。浏览器端登录通常使用 `platformID: 5`，并传入从可信后端获取的 `userID` 和 token。

```ts
import { CbEvents, getSDK } from '@openim/wasm-client-sdk';

const OpenIM = getSDK({
  coreWasmPath: '/openIM.wasm',
  sqlWasmPath: '/sql-wasm.wasm',
});

const handleConnecting = () => {
  setConnectionState('connecting');
};

const handleConnectSuccess = () => {
  setConnectionState('connected');
  loadInitialConversationState();
};

const handleConnectFailed = ({ operationID }) => {
  setConnectionState('failed');
  reportConnectionError({ operationID });
};

const handleTokenExpired = ({ operationID }) => {
  console.warn('OpenIM token expired', { operationID });
  requestFreshToken();
};

const handleTokenInvalid = ({ operationID }) => {
  console.error('OpenIM token invalid', { operationID });
  clearLocalSession();
  redirectToSignIn();
};

const handleKickedOffline = ({ operationID }) => {
  console.warn('OpenIM kicked offline', { operationID });
  showSignedInElsewhereDialog();
};

OpenIM.on(CbEvents.OnConnecting, handleConnecting);
OpenIM.on(CbEvents.OnConnectSuccess, handleConnectSuccess);
OpenIM.on(CbEvents.OnConnectFailed, handleConnectFailed);
OpenIM.on(CbEvents.OnUserTokenExpired, handleTokenExpired);
OpenIM.on(CbEvents.OnUserTokenInvalid, handleTokenInvalid);
OpenIM.on(CbEvents.OnKickedOffline, handleKickedOffline);

await OpenIM.login({
  userID,
  token,
  platformID: 5,
  apiAddr,
  wsAddr,
});
```

`login()` 返回成功不等于所有业务数据都已经加载完成。通常做法是用 `CbEvents.OnConnectSuccess` 切换在线状态，再继续读取会话列表、同步未读数或恢复发送队列。

如果你的应用使用 React、Vue 或其他组件框架，事件处理函数应保持稳定引用。这样组件卸载、切换账号或销毁 SDK 会话时，才能用同一个函数引用移除监听。

## 移除连接事件处理器

调用 `OpenIM.off()` 时，需要传入注册时使用的同一个事件名和同一个函数引用。不要在 `off()` 中重新创建匿名函数，否则 SDK 无法匹配到已注册的处理器。

```ts
// 组件卸载、页面离开或切换账号前移除处理器。
OpenIM.off(CbEvents.OnConnecting, handleConnecting);
OpenIM.off(CbEvents.OnConnectSuccess, handleConnectSuccess);
OpenIM.off(CbEvents.OnConnectFailed, handleConnectFailed);
OpenIM.off(CbEvents.OnUserTokenExpired, handleTokenExpired);
OpenIM.off(CbEvents.OnUserTokenInvalid, handleTokenInvalid);
OpenIM.off(CbEvents.OnKickedOffline, handleKickedOffline);
```

如果同一个浏览器会话会多次进入聊天模块，重复注册同一类处理器前应先确认旧处理器已经移除，避免同一事件触发多次 UI 更新或重复上报日志。

切换账号前先移除当前账号绑定的处理器，并调用 `logout()` 结束当前会话。新账号登录时再注册新的处理器，避免旧账号的 UI 状态或 token 刷新逻辑影响新账号。
