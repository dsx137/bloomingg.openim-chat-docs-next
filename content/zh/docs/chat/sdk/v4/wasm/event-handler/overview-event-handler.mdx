OpenIM WASM SDK 通过事件处理器把连接状态、消息、会话、用户资料、好友关系、群组和信令变化推送给浏览器应用。业务界面不需要轮询这些状态，而是在 SDK 实例上订阅对应的 `CbEvents`，再把事件 `data` 合并到本地状态或缓存中。

`getSDK()` 返回的 SDK 实例提供统一的 `OpenIM.on()` 和 `OpenIM.off()`。OpenIM 不需要为不同模块创建独立的 handler 对象；你只需要保存处理函数引用，按页面或账号生命周期注册、处理回调，并在不再需要时移除。

> OpenIM WASM SDK 同时适用于 TypeScript 和 JavaScript 项目。示例使用 TypeScript 语法展示事件枚举和回调形状。

## 事件处理器功能

#### 管理频道事件处理器

频道相关界面通常由会话、消息和群组状态共同驱动。注册这些事件后，聊天列表、群聊详情、成员列表和未读数可以随着服务端变化保持同步。

| 功能                                                                                                                                   | 说明                                                                                                                         | 单聊会话 | 群聊会话 |
| -------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------- | -------- | -------- |
| [添加或移除频道事件处理器](/docs/chat/sdk/v4/wasm/event-handler/managing-channel-event-handlers/add-or-remove-a-channel-event-handler) | 使用 `CbEvents.OnNewConversation`、`CbEvents.OnConversationChanged`、`CbEvents.OnRecvNewMessages` 以及群组事件更新频道界面。 | 适用     | 适用     |

频道事件处理器通常在进入聊天列表、群聊页或成员管理页时添加，在页面卸载、切换会话、退出登录或切换账号前移除。

#### 管理用户事件处理器

用户事件用于同步当前用户资料、好友关系、黑名单和已订阅用户的在线状态。只订阅当前页面实际展示的用户，可以减少不必要的状态更新。

| 功能                                                                                                                             | 说明                                                                                                      |
| -------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------- |
| [添加或移除用户事件处理器](/docs/chat/sdk/v4/wasm/event-handler/managing-user-event-handlers/add-or-remove-a-user-event-handler) | 使用 `CbEvents.OnSelfInfoUpdated`、`CbEvents.OnUserStatusChanged`、好友事件和黑名单事件更新用户相关界面。 |

用户事件处理器适合在应用状态层或联系人页面注册。离开联系人、成员列表、资料页或退出登录时，应使用同一个函数引用调用 `OpenIM.off()` 清理。

#### 管理连接事件处理器

连接事件用于同步登录、WebSocket 连接、token 失效和被踢下线等应用级状态。它们通常比页面级事件更早注册。

| 功能                                                                                                                                         | 说明                                                                                                                                         |
| -------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| [添加或移除连接事件处理器](/docs/chat/sdk/v4/wasm/event-handler/managing-connection-event-handlers/add-or-remove-a-connection-event-handler) | 在 `login()` 前注册 `CbEvents.OnConnecting`、`CbEvents.OnConnectSuccess`、`CbEvents.OnConnectFailed`、token 和踢下线事件，驱动全局连接状态。 |

连接事件处理器建议由应用根组件或会话管理模块维护。切换账号前先移除旧账号绑定的处理器，再登录新账号，避免旧会话继续写入当前界面。

## 基本生命周期

所有 OpenIM 事件处理器都遵循同一套生命周期：定义稳定的处理函数，调用 `OpenIM.on()` 添加处理器，在回调中处理 `data`，最后用同一个函数引用调用 `OpenIM.off()` 移除处理器。

```ts
import { CbEvents, getSDK } from '@openim/wasm-client-sdk';

const OpenIM = getSDK({
  coreWasmPath: '/openIM.wasm',
  sqlWasmPath: '/sql-wasm.wasm',
});

const handleMessages = ({ data, operationID }) => {
  mergeMessages(data);
  console.debug('received messages', { operationID });
};

OpenIM.on(CbEvents.OnRecvNewMessages, handleMessages);

// 页面卸载、退出登录或切换账号前清理。
OpenIM.off(CbEvents.OnRecvNewMessages, handleMessages);
```

事件回调通常包含 `data`、`errCode`、`errMsg` 和 `operationID`。业务代码应把 `data` 合并到本地状态，并在异常路径中记录 `operationID`，方便把浏览器日志和服务端日志关联起来。

```ts
type WSEvent<T = unknown> = {
  event: CbEvents;
  data: T;
  errCode: number;
  errMsg: string;
  operationID: string;
};
```
