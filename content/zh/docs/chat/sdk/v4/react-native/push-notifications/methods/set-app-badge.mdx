设置应用角标未读数，适合未读数变化后同步系统角标。

## 功能介绍

设置应用角标未读数，适合未读数变化后同步系统角标。

第三方能力至少要求 SDK 已初始化；日志和文件上传的进度通过对应 progress callback 返回。

## Core Binding 方法

| 项目     | 说明                                                                   |
| -------- | ---------------------------------------------------------------------- |
| 方法签名 | `SetAppBadge(callback Base, operationID string, appUnreadCount int32)` |
| 所属能力 | 推送、日志与文件                                                       |
| 源码位置 | `open_im_sdk/third.go`                                                 |

## React Native 调用参考

React Native SDK v3.8.3-patch.12.4 的 TypeScript 方法原型如下。Core binding 方法签名用于说明底层能力和参数映射。

源码位置：`src/sdk.ts`。

```ts showLineNumbers
setAppBadge(appUnreadCount: number, operationID: string = id())
```

## 输入参数

| 参数名称       | 参数类型 | 是否必填 | 描述       |
| -------------- | -------- | -------- | ---------- |
| appUnreadCount | number   | 是       | 分页参数。 |

## 返回结果

返回 `Promise`。调用失败时 Promise reject，错误会被封装为 OpenIMApiError。

## 代码示例

以下示例展示 React Native SDK v3.8.3-patch.12.4 的调用骨架，请按业务上下文补充真实参数、错误处理和 UI 状态更新。

```ts showLineNumbers
import OpenIMSDK from '@openim/rn-client-sdk';

const result = await OpenIMSDK.setAppBadge(0);
```
