上传 SDK 日志，适合用户反馈、故障排查或线上问题定位。

## 功能介绍

上传 SDK 日志，适合用户反馈、故障排查或线上问题定位。

第三方能力至少要求 SDK 已初始化；日志和文件上传的进度通过对应 progress callback 返回。

## 底层能力

| 项目     | 说明                                                                                             |
| -------- | ------------------------------------------------------------------------------------------------ |
| 方法签名 | `UploadLogs(callback Base, operationID string, line int, ex string, progress UploadLogProgress)` |
| 所属能力 | 日志 |
| 源码位置 | `open_im_sdk/third.go`                                                                           |

## Electron 调用参考

Electron SDK v3.8.3 暴露以下平台方法原型。底层能力表用于说明 OpenIM core 能力和参数映射。

源码位置：`src/core/index.ts`。

```ts showLineNumbers
uploadLogs(params: UploadLogsParams, opid = uuidV4())
```

## 输入参数

| 参数名称 | 参数类型         | 是否必填 | 描述                               |
| -------- | ---------------- | -------- | ---------------------------------- |
| params   | UploadLogsParams | 是       | 按平台类型定义传入。               |
| opid     | As defined       | 否       | 链路追踪 ID；不传时 SDK 自动生成。 |

## 返回结果

返回 `Promise` 或注册回调结果。调用失败时 Promise reject；建议在业务层使用 `try/catch` 或 `.catch()` 处理错误。

## 代码示例

以下示例展示 Electron SDK v3.8.3 的调用骨架，请按业务上下文补充真实参数、错误处理和 UI 状态更新。

```ts showLineNumbers
import { getWithRenderProcess } from '@openim/electron-client-sdk/lib/render';

const { instance: openIM } = getWithRenderProcess();

const result = await openIM.uploadLogs({});
```
