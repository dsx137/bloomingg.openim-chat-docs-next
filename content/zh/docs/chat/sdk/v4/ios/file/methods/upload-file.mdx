## 功能介绍

上传文件到 SDK 配置的对象存储链路，适合发送文件、图片、音视频前先完成资源上传。

第三方能力至少要求 SDK 已初始化；日志和文件上传的进度通过对应 progress callback 返回。

## Core Binding 方法

| 项目     | 说明                                                                                     |
| -------- | ---------------------------------------------------------------------------------------- |
| 方法签名 | `UploadFile(callback Base, operationID string, req string, progress UploadFileCallback)` |
| 所属能力 | 文件上传 |
| 源码位置 | `open_im_sdk/third.go`                                                                   |

## iOS 调用参考

iOS Core SDK v3.8.3 的 Objective-C 方法原型如下。Core binding 方法签名用于说明底层能力和参数映射。

源码位置：`Framework/OpenIMCore.xcframework/.../Open_im_sdk.objc.h`。

```objective-c showLineNumbers
FOUNDATION_EXPORT void Open_im_sdkUploadFile(id<Open_im_sdk_callbackBase> _Nullable callback, NSString* _Nullable operationID, NSString* _Nullable req, id<Open_im_sdk_callbackUploadFileCallback> _Nullable progress);
```

## 输入参数

| 参数名称 | 参数类型                                   | 是否必填   | 描述                                   |
| -------- | ------------------------------------------ | ---------- | -------------------------------------- |
| callback | id<Open_im_sdk_callbackBase>               | 是         | 回调或监听器。                         |
| req      | NSString\*                                 | 按方法原型 | 请求对象，字段以对应平台类型定义为准。 |
| progress | id<Open_im_sdk_callbackUploadFileCallback> | 按方法原型 | 按方法原型传入。                       |

## 返回结果

同步创建类方法直接返回值；异步方法通过 callback 返回成功或失败。建议为每次调用传入可追踪的 operationID。

## 代码示例

以下示例展示 iOS Core SDK v3.8.3 的调用骨架，请按业务上下文补充真实参数、错误处理和 UI 状态更新。

```objective-c showLineNumbers
Open_im_sdkUploadFile(callback, @"operationID", @"req", progress);
```
