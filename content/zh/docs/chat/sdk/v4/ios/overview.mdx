本 iOS 文档按照当前 OpenIM SDK 能力结构组织，并以 iOS Core SDK 3.8.3-hotfix.14-dynamic.1 的公开导出方法为准。每个方法均提供独立页面，围绕当前 iOS 调用方式、参数、返回结果和使用边界说明。

这里记录的是 iOS 调用层与 OpenIM core 能力的对应关系。不同平台的方法命名和参数封装可能不同；当平台调用层与 core 导出方法不一致时，以本页标注的 iOS Core SDK 3.8.3-hotfix.14-dynamic.1 方法原型为准。

## 接入模型

| 模型        | 说明                                                                             |
| ----------- | -------------------------------------------------------------------------------- |
| 初始化      | 配置 API 地址、WebSocket 地址、日志和本地数据目录，并注册连接监听器。            |
| 登录态      | 登录前建议先注册消息、会话、好友、群组和用户监听器，避免漏掉同步事件。           |
| 异步结果    | 异步方法通过 callback 返回结果。                                                 |
| 消息创建    | `Create*Message` / `create*Message` 类方法创建本地消息对象，再交给发送方法发送。 |
| operationID | 建议每次调用生成唯一 operationID，便于客户端和服务端日志串联。                   |

## 当前未暴露的能力

当前 iOS Core SDK 3.8.3-hotfix.14-dynamic.1 未按独立方法暴露开放群组、投票、定时消息、消息线程、翻译、独立举报、群组元数据和计数器等能力。后续 SDK 增加导出方法后，再按源码补充。
