## 功能介绍

主动写入一条客户端日志，适合业务层把关键 IM 操作写入 SDK 日志链路。

第三方能力至少要求 SDK 已初始化；日志和文件上传的进度通过对应 progress callback 返回。

## Core Binding 方法

| 项目     | 说明                                                                                                                        |
| -------- | --------------------------------------------------------------------------------------------------------------------------- |
| 方法签名 | `Logs(callback Base, operationID string, logLevel int, file string, line int, msgs string, err string, keyAndValue string)` |
| 所属能力 | 日志 |
| 源码位置 | `open_im_sdk/third.go`                                                                                                      |

## iOS 调用参考

iOS Core SDK v3.8.3 的 Objective-C 方法原型如下。Core binding 方法签名用于说明底层能力和参数映射。

源码位置：`Framework/OpenIMCore.xcframework/.../Open_im_sdk.objc.h`。

```objective-c showLineNumbers
FOUNDATION_EXPORT void Open_im_sdkLogs(id<Open_im_sdk_callbackBase> _Nullable callback, NSString* _Nullable operationID, long logLevel, NSString* _Nullable file, long line, NSString* _Nullable msgs, NSString* _Nullable err, NSString* _Nullable keyAndValue);
```

## 输入参数

| 参数名称    | 参数类型                     | 是否必填   | 描述                          |
| ----------- | ---------------------------- | ---------- | ----------------------------- |
| callback    | id<Open_im_sdk_callbackBase> | 是         | 回调或监听器。                |
| logLevel    | long                         | 按方法原型 | 按方法原型传入。              |
| file        | NSString\*                   | 按方法原型 | 按方法原型传入。              |
| line        | long                         | 按方法原型 | 按方法原型传入。              |
| msgs        | NSString\*                   | 按方法原型 | 消息对象、消息内容或消息 ID。 |
| err         | NSString\*                   | 按方法原型 | 按方法原型传入。              |
| keyAndValue | NSString\*                   | 按方法原型 | 按方法原型传入。              |

## 返回结果

同步创建类方法直接返回值；异步方法通过 callback 返回成功或失败。建议为每次调用传入可追踪的 operationID。

## 代码示例

以下示例展示 iOS Core SDK v3.8.3 的调用骨架，请按业务上下文补充真实参数、错误处理和 UI 状态更新。

```objective-c showLineNumbers
Open_im_sdkLogs(callback, @"operationID", 0, @"file", 0, @"msgs", @"err", @"keyAndValue");
```
