初始化 SDK 全局实例、日志、长连接管理、本地数据目录和基础资源。应用进程内通常只调用一次。

## 功能介绍

初始化 SDK 全局实例、日志、长连接管理、本地数据目录和基础资源。应用进程内通常只调用一次。

初始化类方法应在应用进程内按需调用；用户态方法通常要求 `InitSDK` 已成功执行，并通过 `operationID` 串联日志。

## Core Binding 方法

| 项目     | 说明                                                                       |
| -------- | -------------------------------------------------------------------------- |
| 方法签名 | `InitSDK(listener OnConnListener, operationID string, config string) bool` |
| 所属能力 | 初始化与登录                                                               |
| 源码位置 | `open_im_sdk/init_login.go`                                                |

## iOS 调用参考

iOS Core SDK 3.8.3-hotfix.14-dynamic.1 的 Objective-C 方法原型如下。Core binding 方法签名用于说明底层能力和参数映射。

源码位置：`Framework/OpenIMCore.xcframework/.../Open_im_sdk.objc.h`。

```objective-c showLineNumbers
FOUNDATION_EXPORT BOOL Open_im_sdkInitSDK(id<Open_im_sdk_callbackOnConnListener> _Nullable listener, NSString* _Nullable operationID, NSString* _Nullable config);
```

## 输入参数

| 参数名称 | 参数类型                               | 是否必填   | 描述                                   |
| -------- | -------------------------------------- | ---------- | -------------------------------------- |
| listener | id<Open_im_sdk_callbackOnConnListener> | 按方法原型 | 回调或监听器。                         |
| config   | NSString\*                             | 按方法原型 | 请求对象，字段以对应平台类型定义为准。 |

## 返回结果

同步创建类方法直接返回值；异步方法通过 callback 返回成功或失败。建议为每次调用传入可追踪的 operationID。

以下示例展示 iOS Core SDK 3.8.3-hotfix.14-dynamic.1 的调用骨架，请按业务上下文补充真实参数、错误处理和 UI 状态更新。
