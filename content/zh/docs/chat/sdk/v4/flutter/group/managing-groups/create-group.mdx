## 功能介绍

创建一个 OpenIM 群组。适合发起群聊、创建项目群或从业务组织关系生成群。

群组方法通常要求当前用户已登录，且群主、管理员、成员权限由服务端校验；群组变化通过 `SetGroupListener` 接收。

## Core Binding 方法

| 项目     | 说明                                                                  |
| -------- | --------------------------------------------------------------------- |
| 方法签名 | `CreateGroup(callback Base, operationID string, groupReqInfo string)` |
| 所属能力 | 群组                                                                  |
| 源码位置 | `open_im_sdk/group.go`                                                |

## Flutter 调用参考

Flutter SDK v3.8.3 的 Dart 方法原型如下。Core binding 方法签名用于说明底层能力和参数映射。

源码位置：`lib/src/manager/im_group_manager.dart`。

```dart showLineNumbers
Future<GroupInfo> createGroup({ required GroupInfo groupInfo, List<String> memberUserIDs = const [], List<String> adminUserIDs = const [], String? ownerUserID, String? operationID, })
```

## 输入参数

| 参数名称      | 参数类型                                                 | 是否必填 | 描述                                   |
| ------------- | -------------------------------------------------------- | -------- | -------------------------------------- |
| groupInfo     | [GroupInfo](/docs/chat/sdk/v4/flutter/models/group-info) | 是       | 请求对象，字段以对应平台类型定义为准。 |
| memberUserIDs | List<String>                                             | 否       | 用户 ID 或用户 ID 列表。               |
| adminUserIDs  | List<String>                                             | 否       | 用户 ID 或用户 ID 列表。               |
| ownerUserID   | String?                                                  | 否       | 用户 ID 或用户 ID 列表。               |

## 返回结果

返回 `Future<GroupInfo>`。调用失败时 Future 抛出异常，建议在业务层捕获并记录 operationID。

## 代码示例

以下示例展示 Flutter SDK v3.8.3 的调用骨架，请按业务上下文补充真实参数、错误处理和 UI 状态更新。

```dart showLineNumbers
final result = await OpenIM.iMManager.groupManager.createGroup(
  groupInfo: {},
  memberUserIDs: [],
  adminUserIDs: [],
  ownerUserID: 'ownerUserID',

```
