分页获取当前用户已加入群组，适合群数量较多的账号。

## 功能介绍

分页获取当前用户已加入群组，适合群数量较多的账号。

群组方法通常要求当前用户已登录，且群主、管理员、成员权限由服务端校验；群组变化通过 `SetGroupListener` 接收。

## Core Binding 方法

| 项目     | 说明                                                                             |
| -------- | -------------------------------------------------------------------------------- |
| 方法签名 | `GetJoinedGroupListPage(callback Base, operationID string, offset, count int32)` |
| 所属能力 | 群组                                                                             |
| 源码位置 | `open_im_sdk/group.go`                                                           |

## Flutter 调用参考

Flutter SDK v3.8.3 的 Dart 方法原型如下。Core binding 方法签名用于说明底层能力和参数映射。

源码位置：`lib/src/manager/im_group_manager.dart`。

```dart showLineNumbers
Future<List<GroupInfo>> getJoinedGroupListPage({String? operationID, int offset = 0, int count = 40})
```

## 输入参数

| 参数名称 | 参数类型 | 是否必填 | 描述       |
| -------- | -------- | -------- | ---------- |
| offset   | int      | 否       | 分页参数。 |
| count    | int      | 否       | 分页参数。 |

## 返回结果

返回 `Future<List<GroupInfo>>`。调用失败时 Future 抛出异常，建议在业务层捕获并记录 operationID。

## 代码示例

以下示例展示 Flutter SDK v3.8.3 的调用骨架，请按业务上下文补充真实参数、错误处理和 UI 状态更新。

final result = await OpenIM.iMManager.groupManager.getJoinedGroupListPage(
  offset: 0,
