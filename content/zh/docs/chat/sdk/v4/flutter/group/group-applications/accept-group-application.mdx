同意指定用户的入群申请。

## 功能介绍

同意指定用户的入群申请。

群组方法通常要求当前用户已登录，且群主、管理员、成员权限由服务端校验；群组变化通过 `SetGroupListener` 接收。

## Core Binding 方法

| 项目     | 说明                                                                                                             |
| -------- | ---------------------------------------------------------------------------------------------------------------- |
| 方法签名 | `AcceptGroupApplication(callback Base, operationID string, groupID string, fromUserID string, handleMsg string)` |
| 所属能力 | 群组                                                                                                             |
| 源码位置 | `open_im_sdk/group.go`                                                                                           |

## Flutter 调用参考

Flutter SDK v3.8.3 的 Dart 方法原型如下。Core binding 方法签名用于说明底层能力和参数映射。

源码位置：`lib/src/manager/im_group_manager.dart`。

```dart showLineNumbers
Future<dynamic> acceptGroupApplication({ required String groupID, required String userID, String? handleMsg, String? operationID, })
```

## 输入参数

| 参数名称  | 参数类型 | 是否必填 | 描述                          |
| --------- | -------- | -------- | ----------------------------- |
| groupID   | String   | 是       | 群组 ID 或群组 ID 列表。      |
| userID    | String   | 是       | 用户 ID 或用户 ID 列表。      |
| handleMsg | String?  | 否       | 消息对象、消息内容或消息 ID。 |

## 返回结果

返回 `Future<dynamic>`。调用失败时 Future 抛出异常，建议在业务层捕获并记录 operationID。

## 代码示例

以下示例展示 Flutter SDK v3.8.3 的调用骨架，请按业务上下文补充真实参数、错误处理和 UI 状态更新。

```dart showLineNumbers
final result = await OpenIM.iMManager.groupManager.acceptGroupApplication(
  groupID: 'groupID',
  userID: 'userID',
  handleMsg: 'handleMsg',
);
```
