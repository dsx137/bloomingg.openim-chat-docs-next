## 功能介绍

将一批成员移出群组，通常要求操作者具备管理权限。

群组方法通常要求当前用户已登录，且群主、管理员、成员权限由服务端校验；群组变化通过 `SetGroupListener` 接收。

## Core Binding 方法

| 项目     | 说明                                                                                                   |
| -------- | ------------------------------------------------------------------------------------------------------ |
| 方法签名 | `KickGroupMember(callback Base, operationID string, groupID string, reason string, userIDList string)` |
| 所属能力 | 群组                                                                                                   |
| 源码位置 | `open_im_sdk/group.go`                                                                                 |

## Flutter 调用参考

Flutter SDK v3.8.3 的 Dart 方法原型如下。Core binding 方法签名用于说明底层能力和参数映射。

源码位置：`lib/src/manager/im_group_manager.dart`。

```dart showLineNumbers
Future kickGroupMember({ required String groupID, required List<String> userIDList, String? reason, String? operationID, })
```

## 输入参数

| 参数名称   | 参数类型     | 是否必填 | 描述                     |
| ---------- | ------------ | -------- | ------------------------ |
| groupID    | String       | 是       | 群组 ID 或群组 ID 列表。 |
| userIDList | List<String> | 是       | 用户 ID 或用户 ID 列表。 |
| reason     | String?      | 否       | 按方法原型传入。         |

## 返回结果

返回 `Future`。调用失败时 Future 抛出异常，建议在业务层捕获并记录 operationID。

## 代码示例

以下示例展示 Flutter SDK v3.8.3 的调用骨架，请按业务上下文补充真实参数、错误处理和 UI 状态更新。

```dart showLineNumbers
final result = await OpenIM.iMManager.groupManager.kickGroupMember(
  groupID: 'groupID',
  userIDList: [],
  reason: 'reason',
);
```
