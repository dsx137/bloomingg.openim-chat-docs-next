订阅一组用户的在线状态，适合好友列表、会话页和在线状态角标需要持续更新的场景。

## 功能介绍

订阅一组用户的在线状态，适合好友列表、会话页和在线状态角标需要持续更新的场景。

用户资料和在线状态方法通常要求当前用户已登录；在线状态变化通过 `SetUserListener` 设置的监听器接收。

## Core Binding 方法

| 项目     | 说明                                                                      |
| -------- | ------------------------------------------------------------------------- |
| 方法签名 | `SubscribeUsersStatus(callback Base, operationID string, userIDs string)` |
| 所属能力 | 用户                                                                      |
| 源码位置 | `open_im_sdk/online.go`                                                   |

## Flutter 调用参考

Flutter SDK 3.8.3+hotfix.12 的 Dart 方法原型如下。Core binding 方法签名用于说明底层能力和参数映射。

源码位置：`lib/src/manager/im_user_manager.dart`。

```dart showLineNumbers
Future<List<UserStatusInfo>> subscribeUsersStatus(List<String> userIDs, { String? operationID, })
```

## 输入参数

| 参数名称 | 参数类型     | 是否必填   | 描述                     |
| -------- | ------------ | ---------- | ------------------------ |
| userIDs  | List<String> | 按方法原型 | 用户 ID 或用户 ID 列表。 |

## 返回结果

返回 `Future<List<UserStatusInfo>>`。调用失败时 Future 抛出异常，建议在业务层捕获并记录 operationID。

## 代码示例

以下示例展示 Flutter SDK 3.8.3+hotfix.12 的调用骨架，请按业务上下文补充真实参数、错误处理和 UI 状态更新。

final result = await OpenIM.iMManager.userManager.subscribeUsersStatus(
  userIDs: [],
