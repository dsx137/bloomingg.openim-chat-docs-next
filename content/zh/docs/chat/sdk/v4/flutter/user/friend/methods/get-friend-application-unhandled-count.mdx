## 功能介绍

获取未处理好友申请数量，适合好友入口角标和后台同步。

好友关系变化通过 `SetFriendListener` 设置的 `OnFriendshipListener` 接收；涉及列表和请求对象的字符串参数应传入对应结构的 JSON。

## Core Binding 方法

| 项目     | 说明                                                                         |
| -------- | ---------------------------------------------------------------------------- |
| 方法签名 | `GetFriendApplicationUnhandledCount(callback Base, operationID, req string)` |
| 所属能力 | 好友                                                                         |
| 源码位置 | `open_im_sdk/relation.go`                                                    |

## Flutter 调用参考

Flutter SDK v3.8.3 的 Dart 方法原型如下。Core binding 方法签名用于说明底层能力和参数映射。

源码位置：`lib/src/manager/im_friendship_manager.dart`。

```dart showLineNumbers
Future<int> getFriendApplicationUnhandledCount(GetFriendApplicationUnhandledCountReq req, {String? operationID})
```

## 输入参数

| 参数名称 | 参数类型                              | 是否必填   | 描述                                   |
| -------- | ------------------------------------- | ---------- | -------------------------------------- |
| req      | GetFriendApplicationUnhandledCountReq | 按方法原型 | 请求对象，字段以对应平台类型定义为准。 |

## 返回结果

返回 `Future<int>`。调用失败时 Future 抛出异常，建议在业务层捕获并记录 operationID。
