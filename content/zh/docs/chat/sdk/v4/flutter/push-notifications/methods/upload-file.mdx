## 功能介绍

上传文件到 SDK 配置的对象存储链路，适合发送文件、图片、音视频前先完成资源上传。

第三方能力至少要求 SDK 已初始化；日志和文件上传的进度通过对应 progress callback 返回。

## Core Binding 方法

| 项目     | 说明                                                                                     |
| -------- | ---------------------------------------------------------------------------------------- |
| 方法签名 | `UploadFile(callback Base, operationID string, req string, progress UploadFileCallback)` |
| 所属能力 | 推送、日志与文件                                                                         |
| 源码位置 | `open_im_sdk/third.go`                                                                   |

## Flutter 调用参考

Flutter SDK 3.8.3+hotfix.12 的 Dart 方法原型如下。Core binding 方法签名用于说明底层能力和参数映射。

源码位置：`lib/src/manager/im_manager.dart`。

```dart showLineNumbers
Future uploadFile({ required String id, required String filePath, required String fileName, String? contentType, String? cause, String? operationID, })
```

## 输入参数

| 参数名称    | 参数类型 | 是否必填 | 描述             |
| ----------- | -------- | -------- | ---------------- |
| id          | String   | 是       | 按方法原型传入。 |
| filePath    | String   | 是       | 按方法原型传入。 |
| fileName    | String   | 是       | 按方法原型传入。 |
| contentType | String?  | 否       | 按方法原型传入。 |
| cause       | String?  | 否       | 按方法原型传入。 |

## 返回结果

返回 `Future`。调用失败时 Future 抛出异常，建议在业务层捕获并记录 operationID。

## 代码示例

以下示例展示 Flutter SDK 3.8.3+hotfix.12 的调用骨架，请按业务上下文补充真实参数、错误处理和 UI 状态更新。

```dart showLineNumbers
final result = await OpenIM.iMManager.uploadFile(
  id: 'id',
  filePath: 'filePath',
  fileName: 'fileName',
  contentType: 'contentType',
  cause: 'cause',

```
