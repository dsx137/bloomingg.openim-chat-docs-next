查询 SDK 当前登录状态，用于业务层判断是否可以继续调用用户态 API。

## 功能介绍

查询 SDK 当前登录状态，用于业务层判断是否可以继续调用用户态 API。

初始化类方法应在应用进程内按需调用；用户态方法通常要求 `InitSDK` 已成功执行，并通过 `operationID` 串联日志。

## Core Binding 方法

| 项目     | 说明                                     |
| -------- | ---------------------------------------- |
| 方法签名 | `GetLoginStatus(operationID string) int` |
| 所属能力 | SDK 生命周期                             |
| 源码位置 | `open_im_sdk/init_login.go`              |

## Flutter 调用参考

Flutter SDK v3.8.3 的 Dart 方法原型如下。Core binding 方法签名用于说明底层能力和参数映射。

源码位置：`lib/src/manager/im_manager.dart`。

```dart showLineNumbers
Future<int?> getLoginStatus({ String? operationID, })
```

## 输入参数

| 参数名称 | 参数类型   | 是否必填   | 描述                 |
| -------- | ---------- | ---------- | -------------------- |
| 参数     | 见方法原型 | 按方法原型 | 按上方调用原型传入。 |

## 返回结果

返回 `Future<int?>`。调用失败时 Future 抛出异常，建议在业务层捕获并记录 operationID。

## 代码示例

以下示例展示 Flutter SDK v3.8.3 的调用骨架，请按业务上下文补充真实参数、错误处理和 UI 状态更新。
