使用业务服务端签发的 token 登录指定用户。登录成功后才能可靠调用会话、消息、群组、好友等用户态接口。

## 功能介绍

使用业务服务端签发的 token 登录指定用户。登录成功后才能可靠调用会话、消息、群组、好友等用户态接口。

初始化类方法应在应用进程内按需调用；用户态方法通常要求 `InitSDK` 已成功执行，并通过 `operationID` 串联日志。

## Core Binding 方法

| 项目     | 说明                                                             |
| -------- | ---------------------------------------------------------------- |
| 方法签名 | `Login(callback Base, operationID string, userID, token string)` |
| 所属能力 | 初始化与登录                                                     |
| 源码位置 | `open_im_sdk/init_login.go`                                      |

## Flutter 调用参考

Flutter SDK 3.8.3+hotfix.12 的 Dart 方法原型如下。Core binding 方法签名用于说明底层能力和参数映射。

源码位置：`lib/src/manager/im_manager.dart`。

```dart showLineNumbers
Future<UserInfo> login({ required String userID, required String token, String? operationID, Future<UserInfo> Function()? defaultValue, bool checkLoginStatus = true, })
```

## 输入参数

| 参数名称         | 参数类型                                                                   | 是否必填 | 描述                     |
| ---------------- | -------------------------------------------------------------------------- | -------- | ------------------------ |
| userID           | String                                                                     | 是       | 用户 ID 或用户 ID 列表。 |
| token            | String                                                                     | 是       | 按方法原型传入。         |
| defaultValue     | Future<[UserInfo](/docs/chat/sdk/v4/flutter/models/user-info)> Function()? | 否       | 按方法原型传入。         |
| checkLoginStatus | bool                                                                       | 否       | 按方法原型传入。         |

## 返回结果

返回 `Future<UserInfo>`。调用失败时 Future 抛出异常，建议在业务层捕获并记录 operationID。

## 代码示例

以下示例展示 Flutter SDK 3.8.3+hotfix.12 的调用骨架，请按业务上下文补充真实参数、错误处理和 UI 状态更新。

```dart showLineNumbers
final result = await OpenIM.iMManager.login(
  userID: 'userID',
  token: 'token',
  defaultValue: 'defaultValue',
  checkLoginStatus: false,

```
