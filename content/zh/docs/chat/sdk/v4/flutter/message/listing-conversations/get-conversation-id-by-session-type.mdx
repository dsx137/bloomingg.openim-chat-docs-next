根据目标 ID 和会话类型生成 OpenIM 会话 ID，适合从用户或群跳转到会话前构造会话标识。

## 功能介绍

根据目标 ID 和会话类型生成 OpenIM 会话 ID，适合从用户或群跳转到会话前构造会话标识。

创建消息类方法同步返回消息 JSON；发送、查询、删除、已读和本地写入方法通过回调返回。消息与会话事件分别通过消息监听器和会话监听器接收。

## Core Binding 方法

| 项目     | 说明                                                                                          |
| -------- | --------------------------------------------------------------------------------------------- |
| 方法签名 | `GetConversationIDBySessionType(operationID string, sourceID string, sessionType int) string` |
| 所属能力 | 会话与消息                                                                                    |
| 源码位置 | `open_im_sdk/conversation_msg.go`                                                             |

## Flutter 调用参考

Flutter SDK v3.8.3 的 Dart 方法原型如下。Core binding 方法签名用于说明底层能力和参数映射。

源码位置：`lib/src/manager/im_conversation_manager.dart`。

```dart showLineNumbers
Future<dynamic> getConversationIDBySessionType({ required String sourceID, required int sessionType, String? operationID, })
```

## 输入参数

| 参数名称    | 参数类型 | 是否必填 | 描述             |
| ----------- | -------- | -------- | ---------------- |
| sourceID    | String   | 是       | 按方法原型传入。 |
| sessionType | int      | 是       | 按方法原型传入。 |

## 返回结果

返回 `Future<dynamic>`。调用失败时 Future 抛出异常，建议在业务层捕获并记录 operationID。

## 代码示例

以下示例展示 Flutter SDK v3.8.3 的调用方式。

```java showLineNumbers
String conversationID = OpenIMClient.getInstance().conversationManager.getConversationIDBySessionType(sourceId, sessionType);
```
