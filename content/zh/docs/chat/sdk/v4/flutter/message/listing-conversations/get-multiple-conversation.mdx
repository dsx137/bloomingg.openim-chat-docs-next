按会话 ID 批量获取多个会话，适合局部刷新或从推送跳转后补齐会话。

## 功能介绍

按会话 ID 批量获取多个会话，适合局部刷新或从推送跳转后补齐会话。

创建消息类方法同步返回消息 JSON；发送、查询、删除、已读和本地写入方法通过回调返回。消息与会话事件分别通过消息监听器和会话监听器接收。

## Core Binding 方法

| 项目     | 说明                                                                                    |
| -------- | --------------------------------------------------------------------------------------- |
| 方法签名 | `GetMultipleConversation(callback Base, operationID string, conversationIDList string)` |
| 所属能力 | 会话与消息                                                                              |
| 源码位置 | `open_im_sdk/conversation_msg.go`                                                       |

## Flutter 调用参考

Flutter SDK v3.8.3 的 Dart 方法原型如下。Core binding 方法签名用于说明底层能力和参数映射。

源码位置：`lib/src/manager/im_conversation_manager.dart`。

```dart showLineNumbers
Future<List<ConversationInfo>> getMultipleConversation({ required List<String> conversationIDList, String? operationID, })
```

## 输入参数

| 参数名称           | 参数类型     | 是否必填 | 描述                     |
| ------------------ | ------------ | -------- | ------------------------ |
| conversationIDList | List<String> | 是       | 会话 ID 或会话 ID 列表。 |

## 返回结果

返回 `Future<List<ConversationInfo>>`。调用失败时 Future 抛出异常，建议在业务层捕获并记录 operationID。

final result = await OpenIM.iMManager.conversationManager.getMultipleConversation(
