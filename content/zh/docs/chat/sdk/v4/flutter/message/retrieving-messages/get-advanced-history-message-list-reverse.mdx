按反向方向获取历史消息，适合围绕游标向另一方向加载。

## 功能介绍

按反向方向获取历史消息，适合围绕游标向另一方向加载。

创建消息类方法同步返回消息 JSON；发送、查询、删除、已读和本地写入方法通过回调返回。消息与会话事件分别通过消息监听器和会话监听器接收。

## Core Binding 方法

| 项目     | 说明                                                                                                |
| -------- | --------------------------------------------------------------------------------------------------- |
| 方法签名 | `GetAdvancedHistoryMessageListReverse(callback Base, operationID string, getMessageOptions string)` |
| 所属能力 | 会话与消息                                                                                          |
| 源码位置 | `open_im_sdk/conversation_msg.go`                                                                   |

## Flutter 调用参考

Flutter SDK v3.8.3 的 Dart 方法原型如下。Core binding 方法签名用于说明底层能力和参数映射。

源码位置：`lib/src/manager/im_message_manager.dart`。

```dart showLineNumbers
Future<AdvancedMessage> getAdvancedHistoryMessageListReverse({ String? conversationID, Message? startMsg, GetHistoryViewType viewType = GetHistoryViewType.history, int? count, String? operationID, })
```

## 输入参数

| 参数名称       | 参数类型                                             | 是否必填 | 描述                          |
| -------------- | ---------------------------------------------------- | -------- | ----------------------------- |
| conversationID | String?                                              | 否       | 会话 ID 或会话 ID 列表。      |
| startMsg       | [Message](/docs/chat/sdk/v4/flutter/models/message)? | 否       | 消息对象、消息内容或消息 ID。 |
| viewType       | GetHistoryViewType                                   | 否       | 按方法原型传入。              |
| count          | int?                                                 | 否       | 分页参数。                    |

## 返回结果

返回 `Future<AdvancedMessage>`。调用失败时 Future 抛出异常，建议在业务层捕获并记录 operationID。

## 代码示例

以下示例展示 Flutter SDK v3.8.3 的调用骨架，请按业务上下文补充真实参数、错误处理和 UI 状态更新。

```dart showLineNumbers
final result = await OpenIM.iMManager.messageManager.getAdvancedHistoryMessageListReverse(
  conversationID: 'conversationID',
  startMsg: 'startMsg',
  viewType: 'viewType',

```
