创建带实体信息的引用消息。适合引用回复中仍需要链接、富文本实体或业务标记。

## 功能介绍

创建带实体信息的引用消息。适合引用回复中仍需要链接、富文本实体或业务标记。

创建消息类方法同步返回消息 JSON；发送、查询、删除、已读和本地写入方法通过回调返回。消息与会话事件分别通过消息监听器和会话监听器接收。

## Core Binding 方法

| 项目     | 说明                                                                                                    |
| -------- | ------------------------------------------------------------------------------------------------------- |
| 方法签名 | `CreateAdvancedQuoteMessage(operationID string, text string, message, messageEntityList string) string` |
| 所属能力 | 会话与消息                                                                                              |
| 源码位置 | `open_im_sdk/conversation_msg.go`                                                                       |

## Flutter 调用参考

Flutter SDK v3.8.3 的 Dart 方法原型如下。Core binding 方法签名用于说明底层能力和参数映射。

源码位置：`lib/src/manager/im_message_manager.dart`。

```dart showLineNumbers
Future<Message> createAdvancedQuoteMessage({ required String text, required Message quoteMsg, List<RichMessageInfo> list = const [], String? operationID, })
```

## 输入参数

| 参数名称 | 参数类型                                            | 是否必填 | 描述                          |
| -------- | --------------------------------------------------- | -------- | ----------------------------- |
| text     | String                                              | 是       | 按方法原型传入。              |
| quoteMsg | [Message](/docs/chat/sdk/v4/flutter/models/message) | 是       | 消息对象、消息内容或消息 ID。 |
| list     | List<RichMessageInfo>                               | 否       | 按方法原型传入。              |

## 返回结果

返回 `Future<Message>`。调用失败时 Future 抛出异常，建议在业务层捕获并记录 operationID。

## 代码示例

以下示例展示 Flutter SDK v3.8.3 的调用骨架，请按业务上下文补充真实参数、错误处理和 UI 状态更新。

```dart showLineNumbers
final result = await OpenIM.iMManager.messageManager.createAdvancedQuoteMessage(
  text: 'text',
  quoteMsg: 'quoteMsg',
  list: [],
);
```
