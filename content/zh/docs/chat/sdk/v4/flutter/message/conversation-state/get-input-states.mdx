获取指定用户在会话中的输入状态，适合恢复页面时查询当前状态。

## 功能介绍

获取指定用户在会话中的输入状态，适合恢复页面时查询当前状态。

创建消息类方法同步返回消息 JSON；发送、查询、删除、已读和本地写入方法通过回调返回。消息与会话事件分别通过消息监听器和会话监听器接收。

## Core Binding 方法

| 项目     | 说明                                                                                      |
| -------- | ----------------------------------------------------------------------------------------- |
| 方法签名 | `GetInputStates(callback Base, operationID string, conversationID string, userID string)` |
| 所属能力 | 会话与消息                                                                                |
| 源码位置 | `open_im_sdk/conversation_msg.go`                                                         |

## Flutter 调用参考

Flutter SDK v3.8.3 的 Dart 方法原型如下。Core binding 方法签名用于说明底层能力和参数映射。

源码位置：`lib/src/manager/im_conversation_manager.dart`。

```dart showLineNumbers
Future<List<int>?> getInputStates(String conversationID, String userID, { String? operationID, })
```

## 输入参数

| 参数名称       | 参数类型 | 是否必填   | 描述                     |
| -------------- | -------- | ---------- | ------------------------ |
| conversationID | String   | 按方法原型 | 会话 ID 或会话 ID 列表。 |
| userID         | String   | 按方法原型 | 用户 ID 或用户 ID 列表。 |

## 返回结果

返回 `Future<List<int>?>`。调用失败时 Future 抛出异常，建议在业务层捕获并记录 operationID。

## 代码示例

以下示例展示 Flutter SDK v3.8.3 的调用骨架，请按业务上下文补充真实参数、错误处理和 UI 状态更新。

```dart showLineNumbers
final result = await OpenIM.iMManager.conversationManager.getInputStates(
  conversationID: 'conversationID',
  userID: 'userID',
);
```
