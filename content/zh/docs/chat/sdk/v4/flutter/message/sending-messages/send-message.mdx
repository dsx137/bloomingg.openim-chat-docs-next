发送由 Create 方法创建的消息，并使用 SDK 内置上传和发送链路。单聊与群聊只能选择一个目标字段。

## 功能介绍

发送由 Create 方法创建的消息，并使用 SDK 内置上传和发送链路。单聊与群聊只能选择一个目标字段。

创建消息类方法同步返回消息 JSON；发送、查询、删除、已读和本地写入方法通过回调返回。消息与会话事件分别通过消息监听器和会话监听器接收。

## Core Binding 方法

| 项目     | 说明                                                                                                                      |
| -------- | ------------------------------------------------------------------------------------------------------------------------- |
| 方法签名 | `SendMessage(callback SendMsgCallBack, operationID, message, recvID, groupID, offlinePushInfo string, isOnlineOnly bool)` |
| 所属能力 | 会话与消息                                                                                                                |
| 源码位置 | `open_im_sdk/conversation_msg.go`                                                                                         |

## Flutter 调用参考

Flutter SDK 3.8.3+hotfix.12 的 Dart 方法原型如下。Core binding 方法签名用于说明底层能力和参数映射。

源码位置：`lib/src/manager/im_message_manager.dart`。

```dart showLineNumbers
Future<Message> sendMessage({ required Message message, required OfflinePushInfo offlinePushInfo, String? userID, String? groupID, bool isOnlineOnly = false, String? operationID, })
```

## 输入参数

| 参数名称        | 参数类型                                                              | 是否必填 | 描述                                   |
| --------------- | --------------------------------------------------------------------- | -------- | -------------------------------------- |
| message         | [Message](/docs/chat/sdk/v4/flutter/models/message)                   | 是       | 消息对象、消息内容或消息 ID。          |
| offlinePushInfo | [OfflinePushInfo](/docs/chat/sdk/v4/flutter/models/offline-push-info) | 是       | 请求对象，字段以对应平台类型定义为准。 |
| userID          | String?                                                               | 否       | 用户 ID 或用户 ID 列表。               |
| groupID         | String?                                                               | 否       | 群组 ID 或群组 ID 列表。               |
| isOnlineOnly    | bool                                                                  | 否       | 按方法原型传入。                       |

## 返回结果

返回 `Future<Message>`。调用失败时 Future 抛出异常，建议在业务层捕获并记录 operationID。

以下示例展示 Flutter SDK 3.8.3+hotfix.12 的调用骨架，请按业务上下文补充真实参数、错误处理和 UI 状态更新。

final result = await OpenIM.iMManager.messageManager.sendMessage(
