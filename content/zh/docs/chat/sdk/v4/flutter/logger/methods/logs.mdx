## 功能介绍

主动写入一条客户端日志，适合业务层把关键 IM 操作写入 SDK 日志链路。

第三方能力至少要求 SDK 已初始化；日志和文件上传的进度通过对应 progress callback 返回。

## Core Binding 方法

| 项目     | 说明                                                                                                                        |
| -------- | --------------------------------------------------------------------------------------------------------------------------- |
| 方法签名 | `Logs(callback Base, operationID string, logLevel int, file string, line int, msgs string, err string, keyAndValue string)` |
| 所属能力 | 日志 |
| 源码位置 | `open_im_sdk/third.go`                                                                                                      |

## Flutter 调用参考

Flutter SDK v3.8.3 的 Dart 方法原型如下。Core binding 方法签名用于说明底层能力和参数映射。

源码位置：`lib/src/manager/im_manager.dart`。

```dart showLineNumbers
Future logs({ int logLevel = 5, String? file, int line = 0, String? msgs, String? err, List<dynamic>? keyAndValues, String? operationID, })
```

## 输入参数

| 参数名称     | 参数类型       | 是否必填 | 描述                          |
| ------------ | -------------- | -------- | ----------------------------- |
| logLevel     | int            | 否       | 按方法原型传入。              |
| file         | String?        | 否       | 按方法原型传入。              |
| line         | int            | 否       | 按方法原型传入。              |
| msgs         | String?        | 否       | 消息对象、消息内容或消息 ID。 |
| err          | String?        | 否       | 按方法原型传入。              |
| keyAndValues | List<dynamic>? | 否       | 按方法原型传入。              |

## 返回结果

返回 `Future`。调用失败时 Future 抛出异常，建议在业务层捕获并记录 operationID。

## 代码示例

以下示例展示 Flutter SDK v3.8.3 的调用骨架，请按业务上下文补充真实参数、错误处理和 UI 状态更新。

```dart showLineNumbers
final result = await OpenIM.iMManager.logs(
  logLevel: 0,
  file: 'file',
  line: 0,
  msgs: 'msgs',
  err: 'err',
  keyAndValues: [],
);
```
