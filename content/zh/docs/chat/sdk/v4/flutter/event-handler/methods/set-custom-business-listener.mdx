设置自定义业务消息监听器，用于接收业务自定义通道传入的消息。

## 功能介绍

设置自定义业务消息监听器，用于接收业务自定义通道传入的消息。

`Set*Listener` 会替换当前 SDK 保存的同类监听器；如果多个模块都要消费事件，应在业务层封装统一分发。

## Core Binding 方法

| 项目     | 说明                                                           |
| -------- | -------------------------------------------------------------- |
| 方法签名 | `SetCustomBusinessListener(listener OnCustomBusinessListener)` |
| 所属能力 | 监听器                                                         |
| 源码位置 | `open_im_sdk/listener.go`                                      |

## Flutter 调用参考

Flutter SDK v3.8.3 的 Dart 方法原型如下。Core binding 方法签名用于说明底层能力和参数映射。

源码位置：`lib/src/manager/im_message_manager.dart`。

```dart showLineNumbers
Future setCustomBusinessListener(OnCustomBusinessListener listener)
```

## 输入参数

| 参数名称 | 参数类型                 | 是否必填   | 描述           |
| -------- | ------------------------ | ---------- | -------------- |
| listener | OnCustomBusinessListener | 按方法原型 | 回调或监听器。 |

## 返回结果

返回 `Future`。调用失败时 Future 抛出异常，建议在业务层捕获并记录 operationID。

以下示例展示 Flutter SDK v3.8.3 的调用骨架，请按业务上下文补充真实参数、错误处理和 UI 状态更新。

final result = await OpenIM.iMManager.messageManager.setCustomBusinessListener(
